1. Executive Overview

This report details a cybersecurity assessment for Maple Leaf Logistics, conducted on November 22, 2025. The analysis correlates findings from a network scan, a security controls questionnaire, and a review of existing risks.

The assessment reveals several critical and high-risk vulnerabilities that require immediate attention. The most significant findings include a complete lack of Multi-Factor Authentication (MFA) across all critical systems, including email and sensitive data repositories. This represents a severe gap in access control.

Furthermore, technical scanning identified a public-facing web server running a significantly outdated and vulnerable version of nginx (1.18.0). This outdated software exposes the organization to numerous publicly known exploits. These issues, combined with gaps in the security training process for new employees, place the organization at a high risk of unauthorized access, data breach, and ransomware attacks.

Immediate remediation of the identified risks is strongly recommended to improve the organization’s security posture.

4. Technical Scan Results

An Nmap scan was performed against the target IP address 192.168.10.5 on November 22, 2025. The scan identified one open port with a service running an outdated software version.

Open Port and Service Information
Port State Service Product Version
443/tcp open https nginx 1.18.0

Analysis of Technical Findings

  • Outdated Nginx Server: The web server is running nginx version 1.18.0, which was released in April 2020. This version is no longer supported and has multiple publicly disclosed vulnerabilities (CVEs) that could be exploited by attackers to achieve remote code execution, denial of service, or information disclosure.

  • SSL Certificate Mismatch: The scan revealed the server’s SSL certificate was issued for the Common Name www.acme-corp.com, which does not match the organization’s domain. This misconfiguration can cause browser trust errors and may indicate a placeholder or improperly configured certificate.

The following table synthesizes findings from the security questionnaire and technical scan. As no pre-existing risks were provided, this assessment is based entirely on new discoveries. Each risk has been assigned a severity level based on its potential impact and likelihood of exploitation.

Identified Risks and Severity

6. Recommendations

The following actionable recommendations are provided to mitigate the identified risks. They are prioritized based on severity.

R-1: Implement Multi-Factor Authentication (MFA)

(Mitigates RISK-01, RISK-03)
Immediately prioritize the deployment of a robust MFA solution across the organization.

  • Phase 1 (Critical): Enforce MFA for all access to sensitive data systems, administrator accounts, and remote access solutions (e.g., VPN).

  • Phase 2 (High): Roll out MFA for all employee email accounts and endpoint logins.

R-2: Remediate Vulnerable Web Server

(Mitigates RISK-02)
Develop and execute a patch management plan to upgrade the nginx server at 192.168.10.5 to the latest stable version. All public-facing software should be inventoried and placed on a regular patch cycle.

R-3: Enhance Security Awareness Program

(Mitigates RISK-04)
Integrate mandatory security awareness training into the new employee onboarding process. This training must cover, at a minimum, phishing identification, acceptable use policies, and password hygiene.

R-4: Correct SSL Certificate Configuration

(Mitigates RISK-05)
Replace the existing SSL certificate on the web server with a valid certificate that correctly corresponds to the www.MapleLeafLogistics.com domain.
