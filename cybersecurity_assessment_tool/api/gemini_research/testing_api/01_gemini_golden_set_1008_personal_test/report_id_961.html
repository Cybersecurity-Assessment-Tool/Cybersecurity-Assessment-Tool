Executive Summary

This report provides a comprehensive analysis of the cybersecurity posture for Ember Glow Hospitality, based on technical network scans, a security controls questionnaire, and a review of known risks. The assessment was conducted on November 22, 2025.

The overall security posture is considered critically weak and requires immediate remediation. Several significant security gaps were identified that expose the organization to a high risk of unauthorized access, data breach, and service disruption.

Key findings include:

  • Critical - No Multi-Factor Authentication (MFA): The complete absence of MFA for email, computer logins, and sensitive data systems represents a severe vulnerability. This significantly increases the risk of account compromise via phishing or credential theft.

  • High - Vulnerable Web Server: The external-facing web server at 192.168.10.5 is running an outdated version of Nginx (1.18.0), which has multiple publicly known vulnerabilities.

  • High - Inadequate Employee Security Policies: The lack of an Acceptable Use Policy and mandatory security training for new employees creates a significant risk from insider threats, whether malicious or unintentional.

  • Medium - SSL Certificate Mismatch: The SSL certificate on the web server does not match the organization’s domain, which can erode user trust and indicates poor configuration management.

Immediate action is required to address these findings. Recommendations are detailed in Section 6.

Technical Findings and Analysis

  • Outdated Nginx Version: The web server is running Nginx version 1.18.0, which was released in 2020. This version is significantly outdated and is known to be vulnerable to multiple security threats, including but not limited to request smuggling and denial-of-service attacks. Running unsupported and unpatched software on an internet-facing server presents a high risk of compromise.

  • SSL Certificate Mismatch: The scan revealed that the SSL certificate’s Common Name is www.acme-corp.com. This does not match the organization’s domain (www.EmberGlowHospitality.com). This misconfiguration will cause browser trust warnings for users, can be a symptom of other configuration issues, and may be leveraged by attackers in certain scenarios.

Recommendations

The following actions are recommended to mitigate the identified risks. They are prioritized based on severity and potential impact.

  1. [Immediate Priority] Implement Multi-Factor Authentication (MFA):

    • Enforce MFA for all user accounts for accessing email, VPN, and any systems containing sensitive organizational data.

    • This is the single most effective control to prevent unauthorized access from compromised credentials.

  2. [High Priority] Patch Vulnerable Web Server:

    • Upgrade the Nginx server at 192.168.10.5 from version 1.18.0 to the latest stable version immediately.

    • Implement a formal patch management process to ensure all internet-facing systems are updated in a timely manner.

  3. [High Priority] Establish Foundational Employee Security Policies:

    • Develop and implement a mandatory security awareness training module for all new employees as part of their onboarding process.

    • Draft and enforce a formal Acceptable Use Policy (AUP) that all employees must read and sign.

  4. [Medium Priority] Correct SSL Certificate Configuration:

    • Replace the current SSL certificate on the web server with a valid one that matches the domain www.EmberGlowHospitality.com.

    • Ensure certificates are monitored for expiration and proper configuration.

Conclusion

The security posture of Ember Glow Hospitality contains multiple critical and high-risk vulnerabilities that require immediate attention. The combination of technical flaws, such as the outdated web server, and significant policy gaps, particularly the lack of MFA, creates a high probability of a successful cyberattack.

By implementing the prioritized recommendations in this report, the organization can significantly reduce its attack surface and build a stronger, more resilient security foundation.
