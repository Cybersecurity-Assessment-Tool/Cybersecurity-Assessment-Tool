Executive Summary

This report provides a comprehensive cybersecurity assessment for Terraform Global, based on a technical network scan, a review of organizational security controls, and an analysis of pre-existing risk data. The assessment was conducted on 2026-01-31.

The analysis reveals critical deficiencies in identity and access management controls, specifically a complete lack of Multi-Factor Authentication (MFA) across all key systems. Additionally, a foundational policy gap was identified due to the absence of an employee Acceptable Use Policy (AUP). These administrative gaps present a significantly higher risk to the organization than the technical findings.

On the technical side, the network scan of the target host 192.168.0.5 did not reveal any open ports or active vulnerabilities. This finding contradicts a pre-existing risk concerning an unencrypted web server on Port 80. This suggests the pre-existing risk may be outdated or has been remediated.

Immediate remediation should focus on implementing MFA and establishing a formal AUP to mitigate the most severe risks of account compromise and insider threat.

Recommendations

The following actionable recommendations are provided to address the identified risks. They are prioritized based on severity and potential impact.

Priority 1: Implement Multi-Factor Authentication (Critical)

  • Action: Deploy a robust MFA solution across the organization.

  • Details: Prioritize the rollout in the following order:

    1. All administrative and privileged accounts.

    2. All user accounts for email access (e.g., Office 365, Google Workspace).

    3. All remote access solutions (e.g., VPN).

    4. All systems containing sensitive data.

  • Justification: This is the single most effective control to prevent unauthorized account access and will drastically reduce the organization’s attack surface.

Priority 2: Develop and Enforce an Acceptable Use Policy (High)

  • Action: Draft, approve, and implement a formal Acceptable Use Policy (AUP).

  • Details: The policy should clearly define the rules for using company networks, devices, and data. All employees must be required to read and formally acknowledge the policy. This AUP should be integrated into the existing security awareness training program.

  • Justification: An AUP establishes a baseline for secure behavior, reduces legal liability, and empowers the organization to enforce its security standards.

Priority 3: Verify Risk Remediation (Informational)

  • Action: Conduct a formal review of the "Unencrypted Web Server" risk.

  • Details: Confirm that Port 80 is closed on all relevant production web servers and that all web traffic is encrypted using TLS/SSL (HTTPS). Update the central risk register to formally close this item once verified.

  • Justification: Maintaining an accurate risk register is crucial for an effective security program. This ensures that resources are focused on current, relevant threats.
