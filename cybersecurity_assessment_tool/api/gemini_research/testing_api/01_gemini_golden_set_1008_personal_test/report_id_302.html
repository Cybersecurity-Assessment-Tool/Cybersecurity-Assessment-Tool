Executive Summary

This report provides a comprehensive analysis of the cybersecurity posture of Paper Plane Publishing. The assessment combines a review of organizational security controls, an external network scan, and an evaluation of pre-existing risks.

The analysis revealed several critical and high-risk vulnerabilities that require immediate attention. A key finding is an externally exposed FTP server (10.0.0.15) running a critically outdated version of vsftpd (2.3.4), which is known to contain a backdoor vulnerability (CVE-2011-2523). This service also permits anonymous login, presenting a direct and immediate threat to the network.

Furthermore, significant gaps were identified in access control policies. The lack of Multi-Factor Authentication (MFA) for computer and sensitive data system access drastically increases the risk of unauthorized access should credentials be compromised. This is compounded by the absence of annual security awareness training for all staff, leaving the organization more susceptible to social engineering and phishing attacks.

Immediate remediation of the vulnerable FTP server is paramount. Following this, a phased rollout of MFA and the implementation of a recurring security training program are strongly recommended to mitigate the identified risks and improve the organizationâ€™s overall security resilience.

Recommendations

Based on the assessment, the following actions are recommended to mitigate the identified risks. Recommendations are prioritized by severity.

Immediate Priority (Critical Risk)

  • Remediate Vulnerable FTP Server:

    1. Immediately take the server at 10.0.0.15 offline or firewall port 21 to prevent exploitation.

    2. Investigate the server for signs of existing compromise.

    3. If FTP is a business requirement, upgrade to a modern, patched version of an FTP server and immediately disable anonymous access.

    4. If possible, migrate from FTP to a more secure protocol like SFTP (SSH File Transfer Protocol).

High Priority

  • Implement Multi-Factor Authentication (MFA):

    1. Develop a project plan to roll out MFA for all computer logins (e.g., via Windows Hello, Duo, or similar).

    2. Identify all systems storing sensitive data and enforce MFA for access.

  • Establish Annual Security Training:

    1. Procure and implement a security awareness training platform.

    2. Mandate that all employees complete security awareness training annually.

    3. Conduct regular phishing simulations to test and reinforce training.

Medium Priority

  • Address Outdated Windows Policy:

    1. Continue with the existing recommendation to upgrade all Windows 7 workstations to a supported operating system, such as Windows 10 or 11.

    2. Develop a hardware refresh cycle to ensure systems can support modern, secure operating systems.
