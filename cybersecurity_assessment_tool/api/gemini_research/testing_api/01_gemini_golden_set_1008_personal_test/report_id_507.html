Executive Overview

This report details the findings of a cybersecurity assessment for Harbor Light Foundation. The analysis is based on a network scan, a review of organizational security controls, and an evaluation of pre-existing risk data.

The assessment has identified several critical and high-risk vulnerabilities that require immediate attention. The most severe finding is the direct network exposure of a MySQL database service (Port 3306). This service is running on an End-of-Life (EOL) version (MySQL 5.7.33), which no longer receives security updates, making it a prime target for exploitation.

This technical vulnerability is severely compounded by significant gaps in organizational security controls. Specifically, the complete absence of Multi-Factor Authentication (MFA) for email, computer logins, and access to sensitive data systems creates a high-risk environment. An attacker who compromises a single set of credentials could gain direct access to the exposed database. Furthermore, the lack of security awareness training for new employees increases the organization’s susceptibility to social engineering and phishing attacks.

Immediate remediation should focus on isolating the exposed database via firewall rules, planning an urgent upgrade for the EOL software, and implementing MFA across all critical systems.

Analysis

The lack of Multi-Factor Authentication (MFA) across all critical access points (email, endpoints, and sensitive data) is a critical deficiency. MFA is a foundational security control that prevents the majority of attacks based on stolen credentials. Additionally, the absence of security training during the employee onboarding process leaves the organization vulnerable, as new staff may not be aware of security policies and common threats like phishing.

Analysis

The scan identified that port 3306 (MySQL) is open to the network. Direct exposure of a database service is highly discouraged as it significantly increases the attack surface.

Critical Finding: The detected version, MySQL 5.7.33, reached its official End of Life (EOL) in October 2023. EOL software no longer receives security patches from the vendor, meaning any newly discovered vulnerabilities will remain unpatched. This exposes the database to a high risk of compromise.

Recommendations

The following actions are recommended to mitigate the identified risks. They are prioritized based on severity and potential impact.

Immediate Priority (Critical)

  1. Restrict Access to Database Port: Immediately implement firewall rules to restrict all access to TCP port 3306. Access should only be permitted from specific, trusted IP addresses of application servers that require a direct connection. Public access must be disabled.

  2. Plan Database Upgrade: Initiate a project to upgrade the MySQL 5.7.33 instance to a currently supported version (e.g., MySQL 8.x). This is essential to ensure the system receives ongoing security patches.

High Priority

  1. Implement Multi-Factor Authentication (MFA):

    • Phase 1 (Email): Enforce MFA for all email accounts immediately.

    • Phase 2 (Sensitive Systems): Roll out MFA for all systems identified as containing sensitive data, including the database management interface.

    • Phase 3 (All Logins): Extend MFA to all employee computer logins.

  2. Enhance Security Awareness Training: Develop and implement a mandatory security awareness training module for all new employees as part of their onboarding process. This should cover the acceptable use policy, phishing identification, and password hygiene.
