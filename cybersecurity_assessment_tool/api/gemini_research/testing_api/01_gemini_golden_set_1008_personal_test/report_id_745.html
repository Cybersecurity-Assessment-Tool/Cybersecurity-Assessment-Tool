Executive Summary

This report provides a comprehensive analysis of the cybersecurity posture for Structure & Form, based on data collected on November 22, 2025. The assessment combines a review of organizational security controls, an external network scan, and an evaluation of known risks.

The analysis has identified several areas of significant concern that elevate the organization’s risk profile. Key findings include:

  • Critical Control Gap: Multi-Factor Authentication (MFA) is not enforced for email access. This represents a critical vulnerability, as email accounts are a primary target for account takeover and subsequent phishing attacks.

  • High-Risk Technical Finding: The external-facing web server is running an outdated version of Nginx (1.18.0), which is known to have multiple security vulnerabilities. This exposes the organization to potential compromise.

  • High-Risk Procedural Gap: The organization lacks a formal security awareness training program for both new and existing employees. This increases susceptibility to social engineering attacks like phishing.

Immediate remediation of these identified risks is strongly recommended to reduce the likelihood of a security incident. Detailed findings and actionable recommendations are provided in the subsequent sections of this report.

Analysis of Control Gaps

  • MFA for Email (Critical Gap): The absence of MFA on email is a severe weakness. Email is a central communication hub and often the key to password resets for other services. A compromised email account can lead to a widespread breach.

  • Security Awareness Training (High Risk): The lack of a training program for new and existing employees leaves the organization highly vulnerable to phishing, social engineering, and other human-targeted attacks.

Analysis of Technical Findings

The scan identified an Nginx web server, version 1.18.0, exposed to the internet. This version was released in April 2020 and is now significantly outdated. The current stable version of Nginx is much newer. Running outdated software poses a serious security risk, as numerous vulnerabilities have been discovered and patched in subsequent releases. This server is likely vulnerable to known exploits and should be updated as a matter of priority.

Recommendations

Based on the consolidated risk assessment, the following actions are recommended to improve the security posture of Structure & Form.

RISK-001: Lack of MFA on Email (Critical)

  • Immediate Action: Procure and implement an MFA solution for the organization’s email system.

  • Policy: Update IT policy to mandate the use of MFA for all email accounts, both for existing employees and during new employee onboarding.

  • Timeline: This should be treated as the highest priority and be fully implemented within 30 days.

RISK-002: Outdated Web Server Software (High)

  • Immediate Action: Develop a patch management and upgrade plan for the Nginx server.

  • Execution: Test the upgrade in a non-production environment before deploying to production. Upgrade the server to the latest stable version of Nginx to mitigate known vulnerabilities.

  • Ongoing: Incorporate this server into a regular patch management cycle to ensure it remains up-to-date.

  • Timeline: Plan and execute the upgrade within 60 days.

RISK-003: Inadequate Security Awareness Training (High)

  • Immediate Action: Select and implement a security awareness training platform or program.

  • Policy: Establish a mandatory training program that includes:

    • Initial training for all new hires as part of their onboarding process.

    • Annual refresher training for all employees.

    • Regular simulated phishing campaigns to test and reinforce learning.

  • Timeline: A program should be selected and initial training rolled out within 90 days.
