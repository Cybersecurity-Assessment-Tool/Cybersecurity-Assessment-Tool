Executive Summary

This report details the findings of a cybersecurity posture assessment for True North Travel. The analysis is based on a combination of an external network scan, a review of existing risks, and an organizational security controls questionnaire.

The assessment reveals a critical risk posture. Key findings indicate a complete absence of fundamental security controls, including Multi-Factor Authentication (MFA) across all systems and any form of employee security awareness training. These deficiencies create a high probability of security incidents, such as account compromise through phishing or credential stuffing attacks.

Furthermore, the technical scan identified an exposed Secure Shell (SSH) service on the network perimeter. When combined with the lack of MFA and user training, this exposed service presents an immediate and significant threat to the organization’s infrastructure.

Urgent and decisive action is required to remediate these critical vulnerabilities. Recommendations are provided in this report to establish a baseline of security and mitigate the most severe risks.

Recommendations

The following actions are recommended to address the identified risks. They are prioritized based on severity and potential impact.

Priority 1 (RISK-001): Implement MFA Immediately


Deploy Multi-Factor Authentication across all systems, with the highest priority on:

  • Email accounts (e.g., Office 365, Google Workspace).

  • All remote access systems, including VPNs and the exposed SSH service.

  • Access to systems containing sensitive or critical data.

Priority 2 (RISK-002): Establish a Security Awareness Program


Immediately develop and deploy a security awareness training program that includes:

  • Mandatory training for all new hires during onboarding.

  • Annual mandatory training for all employees covering topics like phishing, password security, and data handling.

  • Consider periodic phishing simulations to test and reinforce training.

Priority 3 (RISK-003): Secure the SSH Service


Reduce the attack surface of the exposed SSH service by implementing the following controls:

  • If possible, place the service behind a VPN and disable public access.

  • If public access is required, use a firewall to restrict access to only known, trusted IP addresses.

  • Disable password-based authentication and enforce the use of strong SSH keys.

Priority 4 (RISK-004): Develop an Acceptable Use Policy (AUP)


Create a formal AUP that clearly outlines the rules and responsibilities for all employees when using company IT assets. Ensure the policy is distributed to all staff and that they provide written acknowledgment of its receipt and understanding.
