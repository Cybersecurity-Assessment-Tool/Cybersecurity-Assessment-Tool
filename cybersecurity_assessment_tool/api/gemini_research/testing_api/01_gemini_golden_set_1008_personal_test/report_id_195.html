Executive Summary

This report provides a comprehensive analysis of the cybersecurity posture for Oasis Wellness, based on network scans, organizational data, and a review of existing risks. The assessment was conducted on 2026-01-31.

The analysis reveals several critical-risk findings that require immediate attention. The most significant concerns are the complete absence of Multi-Factor Authentication (MFA) across all critical systems (email, computers, and sensitive data access) and the public exposure of a database server running an End-of-Life (EOL) version of MySQL (5.7.33).

The lack of MFA creates a substantial risk of account compromise, which could lead to unauthorized access and data breaches. This risk is severely amplified by the exposed MySQL database. An attacker who compromises a single set of credentials could potentially gain direct access to sensitive data without any secondary security checks. The database software itself is no longer receiving security updates, leaving it vulnerable to known exploits.

Immediate remediation should focus on implementing strict firewall rules to isolate the database and beginning a phased rollout of MFA, starting with email and sensitive systems. A plan to upgrade the unsupported database software is also a high-priority recommendation.

Analysis of Findings

The primary finding is the exposure of a MySQL database on port 3306. Further investigation reveals a critical vulnerability:

  • End-of-Life Software: MySQL version 5.7 reached its official End of Life (EOL) in October 2023. This means it no longer receives security patches from the vendor, leaving it perpetually vulnerable to any newly discovered exploits.

  • Direct Exposure: Exposing a database port directly to the network is a highly discouraged practice. It significantly increases the attack surface and makes the database a prime target for brute-force attacks, credential stuffing, and exploitation of known vulnerabilities.

Recommendations

The following actionable recommendations are provided to mitigate the identified risks. They are prioritized based on severity and potential impact.

Risk: Exposed End-of-Life Database (Critical)

  • Immediate Action (Containment): Implement strict firewall rules to deny all access to TCP port 3306 from the public internet and internal network segments. Only allow connections from specific, authorized application server IP addresses. This immediately reduces the exposure.

  • Short-Term (Remediation): Develop and execute a plan to upgrade the MySQL 5.7 instance to a currently supported version (e.g., MySQL 8.x). This is essential to ensure the system receives security patches.

  • Long-Term (Strategic): Adopt a Zero Trust network model where databases are never directly exposed. Access should be brokered through a secure application layer or require a VPN with strict access controls for administrative purposes.

Risk: Lack of Multi-Factor Authentication (Critical)

  • Immediate Action (High-Impact): Procure and deploy an MFA solution for the email system (OasisWellness.org). As email is a primary target for phishing and account takeover, this is the most critical first step.

  • Short-Term (Expansion): Extend the MFA requirement to all remote access solutions (e.g., VPN), privileged user accounts, and logins to sensitive data systems.

  • Long-Term (Comprehensive): Establish a formal Identity and Access Management (IAM) policy that mandates MFA for all user accounts and services wherever technically feasible, including standard computer logins.
