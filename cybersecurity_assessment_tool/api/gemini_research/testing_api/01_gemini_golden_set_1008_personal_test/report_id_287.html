Executive Summary

This report provides a comprehensive cybersecurity assessment for Open Door. The analysis is based on a combination of network scanning, a review of existing risk documentation, and an evaluation of organizational security controls via a questionnaire.

The assessment identified several high-risk areas requiring immediate attention. Critical gaps were found in administrative controls, including the absence of a mandatory Multi-Factor Authentication (MFA) policy for sensitive data systems, a lack of a formal Acceptable Use Policy, and no security awareness training program for employees. These deficiencies significantly increase the organization’s susceptibility to social engineering, unauthorized access, and insider threats.

Technically, the scan confirmed a pre-existing critical risk: an exposed service on a local loopback interface (127.0.0.1). While this is typically a low-risk interface, its exposure in a production context can indicate severe misconfigurations, such as insecure port forwarding, which could allow an attacker to bypass perimeter defenses.

Urgent remediation is recommended, focusing on implementing foundational security controls (MFA, policies, training) and investigating the exposed service to mitigate the identified risks.

Recommendations

The following actions are recommended to mitigate the identified risks. Recommendations are prioritized based on severity and potential impact.

Immediate Priority (Remediate within 7 days)

  1. Investigate and Secure Exposed SSH Service:

    • Determine the business justification for the open SSH port on 127.0.0.1.

    • If the service is not required, disable it immediately.

    • If required, ensure it is configured securely: restrict access to specific IP addresses, enforce key-based authentication, and disable password-based logins.

  2. Implement MFA on Sensitive Systems:

    • Immediately begin a project to enforce MFA on all systems classified as containing sensitive or critical data.

    • Prioritize systems that are publicly accessible or hold personally identifiable information (PII).

High Priority (Remediate within 30-60 days)

  1. Establish a Security Awareness Training Program:

    • Procure and implement a training solution for all employees.

    • Ensure the program is mandatory for all new hires during onboarding.

    • Conduct annual refresher training for all staff, including phishing simulations.

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Draft a formal AUP that clearly outlines the rules for using company networks, devices, and data.

    • Require all employees to read and formally acknowledge the policy.

    • Integrate the AUP into the new hire onboarding process.
