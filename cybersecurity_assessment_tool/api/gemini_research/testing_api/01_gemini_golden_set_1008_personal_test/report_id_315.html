1. Executive Summary

This report details the findings of a cybersecurity assessment conducted for True North Travel. The analysis is based on a network vulnerability scan, a review of organizational security controls, and an evaluation of pre-existing risks.

The assessment identified a risk: a publicly accessible MySQL database server (172.16.50.20) running an outdated and End-of-Life (EOL) version (MySQL 5.7.33). This exposes the organization to significant threats, including data breach, ransomware, and service disruption.

Furthermore, critical gaps in security policies were identified. The lack of multi-factor authentication (MFA) for sensitive data systems and the absence of security awareness training for new employees represent organizational risks. These policy gaps directly increase the likelihood of a successful cyberattack against the technically vulnerable systems.

Immediate remediation is required to address the exposed database. Strategic improvements to security policies are necessary to build a more resilient and secure operational environment.

4.2 Technical Analysis

The scan revealed that port 3306, the default port for MySQL databases, is open to the network. This directly confirms the "Database Exposure" risk.

More critically, the detected version, MySQL 5.7.33, is part of the MySQL 5.7 series which reached its official End-of-Life (EOL) in October 2023. EOL software no longer receives security updates, patches, or support from the vendor. Any vulnerabilities discovered after this date will remain unpatched, making the system an easy and attractive target for attackers.

6. Recommendations

The following actionable recommendations are provided in order of priority to mitigate the identified risks.

Priority 1: Immediate Actions (To be completed within 72 hours)

  1. Restrict Database Access: Immediately implement firewall rules to block all public access to TCP port 3306 on host 172.16.50.20. Access should be restricted to a whitelist of trusted internal IP addresses only. This is the most critical step to mitigate the exposure risk.

Priority 2: High-Priority Actions (To be completed within 30 days)

  1. Plan and Execute Database Upgrade: Initiate a project to upgrade the MySQL 5.7 database to a currently supported version (e.g., MySQL 8.x). This is essential to ensure the system receives ongoing security patches.

  2. Enforce MFA on All Sensitive Systems: Procure and deploy an MFA solution for all sensitive data systems, with the database administration interface being the top priority. This directly addresses the critical control gap.

Priority 3: Medium-Priority Actions (To be completed within 90 days)

  1. Integrate Security into Onboarding: Develop and mandate a security awareness training module for all new employees as a standard part of the onboarding process. This training should cover phishing, password hygiene, and the acceptable use policy.
