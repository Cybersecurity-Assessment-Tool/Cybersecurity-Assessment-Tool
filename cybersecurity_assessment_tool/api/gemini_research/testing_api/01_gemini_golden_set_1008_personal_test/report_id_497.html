Executive Summary

This report details the findings of a cybersecurity posture assessment for Ember Glow Hospitality. The analysis combines a review of organizational security controls, technical network scanning, and a summary of pre-existing risks to provide a holistic view of the current security landscape.

The assessment identified several critical-risk findings that require immediate attention. The most severe issue is the systemic lack of Multi-Factor Authentication (MFA) across all key access points, including computer logins, email, and sensitive data systems. This administrative gap is dangerously compounded by technical findings, specifically the discovery of a new host (10.10.10.51) exposing Remote Desktop Protocol (RDP) to the network. This mirrors a pre-existing risk on another host, indicating a pattern of misconfiguration.

The combination of exposed RDP and the absence of MFA for computer logins creates a direct and high-probability pathway for ransomware attacks and unauthorized network access. An attacker who obtains a single user credential could gain significant control over internal systems.

Overall, the security posture of Ember Glow Hospitality is considered to be at a level. Immediate remediation of the identified MFA and RDP vulnerabilities is strongly recommended to mitigate the threat of a significant security incident.

Recommendations

Based on the consolidated risk assessment, the following remediation actions are recommended. They are prioritized to address the most critical threats first.

Immediate Priority (Remediate within 72 hours)

  • Contain RDP Exposure (RISK-001, RISK-002): Immediately close TCP port 3389 on the network firewall for hosts 10.10.10.51 and 10.10.10.50. If remote access is a business requirement, it must be placed behind a Virtual Private Network (VPN) that requires Multi-Factor Authentication.

  • Deploy MFA for Privileged Access: At a minimum, immediately enforce MFA for all administrator accounts and any remote access solutions (e.g., VPN).

High Priority (Remediate within 30 days)

  • Full MFA Rollout (RISK-003, RISK-004, RISK-005): Implement a project to roll out MFA across the entire organization. The priority should be:

    1. Email accounts (e.g., Microsoft 365, Google Workspace).

    2. All computer logins (e.g., via Windows Hello for Business, Duo).

    3. All applications hosting sensitive data.

  • Network Vulnerability Scan: Conduct a comprehensive, authenticated vulnerability scan of the entire internal network to identify other misconfigurations or unpatched systems.

Medium Priority (Remediate within 90 days)

  • Review Network Segmentation: Ensure that critical servers are on separate network segments from user workstations to limit an attacker’s ability to move laterally after an initial compromise.

  • Update Incident Response Plan: Review and update the corporate Incident Response plan to specifically address scenarios for ransomware and Business Email Compromise, incorporating the new MFA and VPN controls.
