Analysis of Technical Findings

The presence of an open HTTP port (80) is a significant concern. The HTTP protocol does not encrypt data in transit, meaning that any information, including potential login credentials or sensitive data exchanged with the web server, can be intercepted and read by an attacker on the same network. It is industry best practice to use HTTPS (HTTP over TLS/SSL) for all web communications, which encrypts the data channel.

Recommendations

The following actions are recommended to address the identified risks and improve the security posture of .

Immediate Actions (0-30 Days)

  1. Implement MFA (RISK-001): Prioritize the deployment of MFA across all critical systems. At a minimum, this must include:

    • Email (e.g., Office 365, Google Workspace).

    • VPN and remote access solutions.

    • Access to all systems storing or processing sensitive corporate or client data.

Short-Term Actions (30-90 Days)

  1. Establish Annual Security Training (RISK-002): Procure and deploy a security awareness training platform. Mandate that all employees complete a foundational training module annually, with regular phishing simulations to reinforce learning.

  2. Remediate Unencrypted Services (RISK-003): Investigate the service running on port 80.

    • If the service is necessary, configure it to use HTTPS with a valid TLS certificate and redirect all HTTP traffic to HTTPS.

    • If the service is not business-critical, disable it and block port 80 at the host or network firewall.

Long-Term Actions (90+ Days)

  1. Review Risk Register Integrity (RISK-004): Conduct a full audit of the internal risk register to identify and correct erroneous or invalid entries. Establish a formal process for risk identification, assessment, and documentation to ensure data accuracy.
