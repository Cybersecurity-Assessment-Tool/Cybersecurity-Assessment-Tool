Technical Analysis

  • Outdated Nginx Version: The web server is running Nginx version 1.18.0, which was released in April 2020. This version is significantly outdated and is known to be vulnerable to multiple security exploits (e.g., CVE-2021-23017). Running unsupported software poses a high risk of compromise.

  • SSL Certificate Mismatch: The SSL certificate presented by the server has a Common Name of www.acme-corp.com, which does not match the organization’s domain (www.AstraeusAerospace.org). This misconfiguration can lead to browser trust errors and may be indicative of a deployment issue.

Risk Assessment

The following table synthesizes findings from the security control review and the technical scan into a prioritized list of identified risks. Each risk is assigned a severity level based on its potential impact and likelihood of exploitation.

Risk ID Risk Title Description Severity
RISK-001 Lack of MFA on Sensitive Systems The absence of MFA on systems containing sensitive data allows an attacker with stolen credentials to gain direct access to critical assets, potentially leading to a major data breach. Critical
RISK-002 Outdated Web Server Software The public-facing web server is running an outdated version of Nginx with known vulnerabilities. This could allow an attacker to compromise the server, leading to service disruption or data exfiltration. High
RISK-003 No Security Training for New Hires New employees are not receiving security awareness training, making them highly susceptible to phishing and social engineering attacks. This gap increases the risk of initial compromise. High
RISK-004 SSL Certificate Mismatch The web server presents a certificate for a different domain, which erodes user trust and could be exploited in man-in-the-middle scenarios. It also indicates a lack of configuration management. Medium

Recommendations

Based on the risks identified in this assessment, the following actions are recommended to strengthen the security posture of Astraeus Aerospace.

  • (Critical) Implement MFA on All Sensitive Systems (RISK-001):

    • Immediately prioritize the deployment of a robust multi-factor authentication solution across all applications and systems that store, process, or transmit sensitive data.

    • This control is the single most effective measure to prevent unauthorized access resulting from credential compromise.

  • (High) Upgrade Web Server and Institute Patch Management (RISK-002):

    • Plan and execute an immediate upgrade of the Nginx web server from version 1.18.0 to the latest stable version.

    • Establish a formal patch management policy to ensure all public-facing systems are regularly updated to protect against newly discovered vulnerabilities.

  • (High) Implement Mandatory Onboarding Security Training (RISK-003):

    • Develop a security awareness training module and integrate it into the mandatory onboarding process for all new employees and contractors.

    • The training should cover, at a minimum, phishing identification, password hygiene, and the acceptable use policy.

  • (Medium) Correct SSL Certificate Configuration (RISK-004):

    • Investigate the SSL certificate misconfiguration on the web server at 192.168.10.5.

    • Procure and deploy a valid SSL certificate that matches the organization’s domain, www.AstraeusAerospace.org, to ensure encrypted connections are trusted by users and browsers.
