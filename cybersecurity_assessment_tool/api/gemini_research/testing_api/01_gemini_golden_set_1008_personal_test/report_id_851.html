Executive Summary

This report provides a comprehensive analysis of the cybersecurity posture for Moxie Marketing, conducted on 2026-01-31. The assessment combines a review of organizational security controls, an external network scan, and an analysis of pre-existing risk data.

The overall security posture is assessed as Poor due to the identification of several critical and high-risk vulnerabilities. These findings indicate significant exposure to common cyber threats such as Business Email Compromise (BEC), unauthorized access, and social engineering.

Key Critical Findings include:

  • Lack of Multi-Factor Authentication (MFA) on Email: The absence of MFA on the primary email system (MoxieMarketing.net) presents a critical vulnerability. This significantly increases the risk of account takeovers through phishing or credential stuffing attacks.

  • Exposed Localhost Service: A network service (SSH on port 22) was identified running on the localhost interface (127.0.0.1). This aligns with a pre-identified critical risk and suggests a severe misconfiguration that could allow unauthorized access if the service is improperly exposed to external networks.

Key High-Risk Findings include:

  • Inadequate Security Awareness Training: The organization does not provide mandatory annual security awareness training for all employees. This gap allows security knowledge to become outdated, making the organization more susceptible to evolving threats like sophisticated phishing attacks.

Immediate remediation of these issues is strongly recommended to reduce the organization’s attack surface and mitigate the risk of a significant security incident. Detailed analysis and actionable recommendations are provided in the subsequent sections of this report.

This finding indicates a service intended for local machine administration is running. If this host is a gateway, firewall, or multi-homed server, there is a critical risk that this internal service is improperly exposed to the internet. This could allow an attacker to attempt to brute-force credentials or exploit a vulnerability in the SSH service to gain unauthorized access to the system. The lack of version information from the scan prevents a detailed vulnerability assessment, but the exposure itself is a critical issue.

Recommendations

The following actionable recommendations are provided to address the identified risks. Remediation should be prioritized based on severity.

RISK-001: Lack of MFA on Email Accounts (Critical)

  • Immediate Action: Procure and enforce an MFA solution for all user accounts on the MoxieMarketing.net email platform immediately.

  • Guidance: Prioritize phishing-resistant MFA methods such as FIDO2 security keys or authenticator apps (TOTP). Avoid less secure methods like SMS-based MFA if possible.

  • Long-Term: Integrate MFA across all other critical cloud and on-premise applications to establish a consistent security baseline.

RISK-002: Exposed Localhost Service (SSH) (Critical)

  • Immediate Action: Urgently investigate the system at 57.21.58.23 to determine why a service is listening on its localhost interface (127.0.0.1) and how this was detected externally.

  • Guidance:

    1. Firewall/Configuration Review: Ensure that firewall rules block all inbound access to management ports (like SSH) from the internet. The SSH service should be reconfigured to bind only to a private management network interface, not a public-facing one.

    2. Access Control: If remote administration is required, access should be restricted to a trusted IP whitelist and ideally placed behind a Virtual Private Network (VPN).

RISK-003: Inadequate Annual Security Training (High)

  • Immediate Action: Schedule and mandate a security awareness training session for all current employees within the next quarter.

  • Guidance: Implement a formal, ongoing security awareness program that includes:

    • Mandatory annual training covering topics like phishing, password security, data handling, and acceptable use.

    • Regular (e.g., quarterly) simulated phishing campaigns to test and reinforce employee knowledge.

    • A clear process for employees to report suspected security incidents.
