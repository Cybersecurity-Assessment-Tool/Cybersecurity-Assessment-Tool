Executive Summary

This report details the findings of a cybersecurity assessment conducted for Nova Terra. The analysis synthesizes data from an external network scan, a security controls questionnaire, and a review of pre-existing risk documentation.

The assessment identified several high-impact risks that require immediate attention. Key findings include:

  • Confirmed Network Exposure: The external network scan confirmed that a MySQL database service (port 3306) is publicly accessible from the internet. This aligns with a known risk and presents a significant threat of data breach.

  • Outdated Software: The exposed MySQL service is running version 5.7.33, which reached its official End of Life (EOL) in October 2023. EOL software no longer receives security updates and is a prime target for attackers.

  • Critical Gaps in Identity and Access Management: The organization does not enforce Multi-Factor Authentication (MFA) for accessing email or for computer logins. This dramatically increases the risk of account compromise and unauthorized access.

  • Insufficient Security Awareness: The lack of a formal security awareness training program for new and existing employees creates a significant human-layer vulnerability, making the organization more susceptible to phishing and social engineering attacks.

The overall security posture is considered high-risk due to these fundamental control gaps. This report provides specific, actionable recommendations to mitigate these findings and improve the organization’s defensive capabilities.

Analysis of Findings

The scan confirms that a MySQL database server is directly exposed to the network.

  • Service Exposure: Port 3306 is the default port for MySQL. Exposing this service directly to the internet is highly discouraged as it allows attackers to perform brute-force attacks, exploit vulnerabilities, or attempt to exfiltrate data.

  • Outdated Version: The identified version, MySQL 5.7.33, has reached its End of Life (EOL). This means the vendor no longer provides security patches for newly discovered vulnerabilities, leaving the server permanently at risk.

Recommendations

The following actions are recommended to mitigate the identified risks and strengthen the organization’s security posture.

Immediate Priority (0-30 Days)

Remediate Database Exposure:

Immediately implement strict firewall rules to restrict all access to port 3306. Access should only be permitted from specific, trusted IP addresses. For remote administration, a Virtual Private Network (VPN) should be required.

High Priority (30-90 Days)

Implement Multi-Factor Authentication (MFA):

Procure and deploy an MFA solution across the organization. Prioritize enforcement on the following systems:

  • Email (e.g., Office 365, Google Workspace)

  • All remote access solutions (VPNs)

  • Workstation/computer logins

  • All administrative accounts

Upgrade End-of-Life Software:

Plan and execute the upgrade of the MySQL 5.7.33 database to a currently supported version (e.g., MySQL 8.x). This will ensure the system receives critical security patches. Test applications for compatibility before upgrading the production environment.

Establish Security Awareness Training:

Implement a formal security awareness training program. All new hires must complete the training as part of their onboarding. All existing employees must complete the training annually. This program should cover phishing, password security, and acceptable use policies.
