Executive Summary

This report details the findings of a cybersecurity assessment conducted for Atlas Mapping. The evaluation combined a review of organizational security controls, an automated network scan of key infrastructure, and an analysis of pre-existing risk documentation.

The assessment identified several high-priority risks that require immediate attention. Key findings include:

  • Critical Gaps in Access Control: Multi-Factor Authentication (MFA) is not enforced for employee email or computer access. This represents a significant vulnerability, as compromised credentials could lead to widespread system access and data breaches.

  • Exposed and Outdated Database: A MySQL database server on the internal network (172.16.50.20) is directly accessible via port 3306. Furthermore, the database is running MySQL version 5.7.33, which reached its official End-of-Life (EOL) in October 2023 and no longer receives security updates. This exposes the organization to numerous known vulnerabilities.

  • Positive Security Practices: The organization has established an acceptable use policy and conducts regular security awareness training, which are commendable foundational controls.

Immediate remediation should focus on implementing MFA across all critical systems and securing the exposed database by applying network access controls and planning an urgent software upgrade.

Recommendations

The following actions are recommended to mitigate the identified risks. Recommendations are prioritized based on severity.

R-01: Remediate End-of-Life Database Software (Critical)

The use of unsupported software is a critical risk. An attacker can exploit known vulnerabilities to compromise the database, leading to data theft, modification, or destruction.

  • Immediate Action: Develop a migration plan to upgrade the MySQL database from version 5.7 to a currently supported version (e.g., MySQL 8.x).

  • Long-Term Strategy: Implement a formal patch and vulnerability management program to ensure all software and systems are kept up-to-date.

R-02: Implement MFA for Email Access (Critical)

Email is a primary target for attackers. Securing it is paramount.

  • Immediate Action: Enforce mandatory MFA for all user accounts on the AtlasMapping.com email system.

  • Supporting Action: Conduct a follow-up awareness campaign to ensure all employees understand how to use the new MFA system.

R-03: Secure Exposed Database Service (High)

Direct access to a database from the general network is a severe security flaw.

  • Immediate Action (Easy Fix): Implement strict firewall rules to ensure that only authorized application servers and administrative hosts can connect to port 3306 on 172.16.50.20. All other traffic should be denied.

  • Long-Term Strategy (Best Practice): Require all database administration to be performed over a secure, authenticated channel, such as a Virtual Private Network (VPN) with MFA.

R-04: Implement MFA for Endpoint Access (High)

Endpoints are the gateway to the corporate network.

  • Immediate Action: Deploy an MFA solution for all Windows, macOS, and Linux computer logins, prioritizing users with access to sensitive data.
