Executive Overview

This report provides a comprehensive cybersecurity assessment for Structure & Form, based on network scan data, organizational security controls, and pre-existing risk information. The analysis was conducted to identify security gaps, evaluate the current risk posture, and provide actionable recommendations for improvement.

The assessment reveals a mixed security posture. The organization has implemented positive security controls, such as requiring Multi-Factor Authentication (MFA) for sensitive data systems and conducting regular security awareness training for all employees. These measures are commendable and form a solid foundation for a security program.

However, several critical and high-risk vulnerabilities were identified that require immediate attention. Key findings include:

  • Lack of MFA: MFA is not enforced for accessing email or for logging into employee computers. This represents a significant vulnerability to credential theft and account takeover attacks.

  • Governance Gap: The absence of an employee Acceptable Use Policy (AUP) creates ambiguity regarding the secure and appropriate use of company assets, increasing insider and operational risks.

  • Insecure Network Service: A technical scan of the internal host 172.16.0.1 discovered an open port 80 (HTTP), indicating that data is likely being transmitted in cleartext. This exposes the organization to eavesdropping and man-in-the-middle attacks.

This report details these findings and concludes with prioritized recommendations to mitigate the identified risks and strengthen the overall security posture of Structure & Form.

Recommendations

The following actions are recommended to mitigate the identified risks and improve the overall security posture of the organization. Recommendations are prioritized based on severity and impact.

Priority 1: Immediate Actions (0-30 Days)

  1. Enforce MFA on Email and Endpoints:

    • Action: Immediately enable and enforce MFA for all user accounts on the email system (e.g., Microsoft 365, Google Workspace) and for all computer logins (e.g., via Windows Hello for Business, Duo, or a similar solution).

    • Impact: Drastically reduces the risk of account takeovers from compromised credentials.

  2. Secure Internal Web Server:

    • Action: For the server at 172.16.0.1, obtain and install a TLS/SSL certificate. Reconfigure the web service to use HTTPS (port 443) exclusively and disable or redirect all traffic from HTTP (port 80).

    • Impact: Encrypts all data in transit, protecting it from eavesdropping and modification.

Priority 2: Short-Term Actions (30-90 Days)

  1. Develop and Implement an Acceptable Use Policy (AUP):

    • Action: Draft a formal AUP that clearly defines the rules for using company computers, networks, and data. The policy should be reviewed by legal counsel, communicated to all employees, and formally acknowledged via signature.

    • Impact: Establishes clear security expectations for all employees, reduces insider risk, and provides a basis for disciplinary action in case of violation.
