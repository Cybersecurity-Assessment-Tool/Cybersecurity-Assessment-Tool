Executive Summary

This report provides a comprehensive cybersecurity assessment for Moxie Marketing, synthesizing findings from a network vulnerability scan, a review of existing risks, and an analysis of organizational security controls.

The assessment identified a mixed security posture. On the positive side, the organization has implemented strong Multi-Factor Authentication (MFA) controls across email, computer logins, and sensitive data systems. Furthermore, a technical scan of the target host 192.168.0.5 did not reveal any open ports or immediate vulnerabilities, directly contradicting a previously documented risk concerning an unencrypted web server. This suggests the prior risk has been successfully remediated.

However, the analysis revealed two critical gaps in administrative and procedural controls. The absence of a formal Employee Acceptable Use Policy and the lack of mandatory security awareness training for new hires represent significant organizational risks. These gaps expose the organization to potential insider threats, social engineering attacks, and non-compliance issues.

Our primary recommendations focus on immediately addressing these policy and training deficiencies to build a more resilient security culture and formalize security expectations for all employees.

Recommendations

The following actionable recommendations are prioritized based on the risk and potential impact to the organization.

Priority 1: High-Impact Recommendations

  1. Develop and Implement an Acceptable Use Policy (AUP):

    • Action: Draft a formal AUP that clearly defines the rules and expectations for employees when using company computers, networks, email, and data. The policy should cover topics such as data handling, password security, internet usage, and incident reporting.

    • Justification: An AUP is a foundational administrative control that reduces insider risk, sets clear security expectations, and provides a basis for disciplinary action in case of policy violations.

  2. Establish a Mandatory New-Hire Security Training Program:

    • Action: Create a security awareness training module that is a mandatory part of the onboarding process for all new employees and contractors. This training should cover phishing, social engineering, password hygiene, and the new AUP.

    • Justification: New hires are a primary target for attackers. Immediate training significantly reduces the organization’s vulnerability to common cyber threats from day one of a new employee’s tenure.

Priority 2: Administrative and Validation Tasks

  1. Validate and Update the Risk Register:

    • Action: Formally review the "Unencrypted Web Server" risk. Based on the technical scan results showing Port 80 as closed, this risk should be marked as remediated or closed in the official risk register.

    • Justification: Maintaining an accurate risk register is crucial for focusing security efforts and resources on genuine, active threats.
