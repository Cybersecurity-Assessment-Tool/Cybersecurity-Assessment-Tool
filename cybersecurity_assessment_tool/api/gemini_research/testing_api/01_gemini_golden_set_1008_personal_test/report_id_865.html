Executive Summary

This report details the findings of a cybersecurity posture assessment conducted for Cinder & Ash. The assessment combined a review of organizational security controls, an external network vulnerability scan, and an analysis of pre-existing risks.

The overall security posture reveals significant gaps that expose the organization to a high level of risk. While foundational controls like Multi-Factor Authentication (MFA) for email are in place, critical systems and user endpoints lack this protection. Furthermore, the absence of an Acceptable Use Policy and a recurring security awareness training program weakens the human element of the security framework.

The external network scan identified a public-facing web server running an outdated version of Nginx (1.18.0), which is known to have multiple vulnerabilities. This, combined with the identified policy and procedure gaps, creates a substantial attack surface.

Immediate remediation is recommended, focusing on implementing comprehensive MFA, updating vulnerable software, and establishing formal security policies and training programs to mitigate the identified risks.

Analysis of Findings

  • Outdated Web Server: The scan identified Nginx version 1.18.0, which was released in April 2020. This version is significantly outdated and is no longer supported. It is known to be vulnerable to multiple security threats, including but not limited to CVE-2021-23017. Running unsupported software on a public-facing server presents a critical risk.

  • SSL Certificate Mismatch: The SSL certificate presented by the server has a Common Name of www.acme-corp.com, which does not match the organization’s domain (www.CinderAsh.org). This misconfiguration can cause trust errors for users and may indicate improper server setup.

Recommendations

Based on the identified risks, the following prioritized actions are recommended to improve the cybersecurity posture of Cinder & Ash.

  1. Implement Comprehensive MFA (Critical):

    • Immediately enforce MFA for all access to systems containing sensitive data.

    • Roll out mandatory MFA for all employee computer and remote access logins. This single control significantly reduces the risk of unauthorized access from compromised credentials.

  2. Upgrade Web Server Software (High):

    • Plan and execute an upgrade of the Nginx server from version 1.18.0 to the latest stable version.

    • Implement a patch management policy to ensure all public-facing systems are regularly updated.

  3. Develop and Implement Security Policies (High):

    • Create a formal Acceptable Use Policy (AUP) that defines the rules for using company assets and data.

    • Ensure all employees read and acknowledge the AUP upon hiring and annually thereafter.

  4. Establish Annual Security Training (High):

    • Institute a mandatory security awareness training program for all employees, to be completed at least once per year.

    • Training should cover topics such as phishing, password security, and the new Acceptable Use Policy.

  5. Correct SSL Certificate Configuration (Medium):

    • Replace the current SSL certificate on the web server with a valid certificate that matches the organization’s domain, www.CinderAsh.org.
