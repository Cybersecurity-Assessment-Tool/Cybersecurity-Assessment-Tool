Executive Summary

This report provides a comprehensive cybersecurity assessment for Opal Sky Media, synthesizing data from organizational questionnaires, technical network scans, and a review of pre-existing risks.

The analysis reveals a mixed security posture. While the organization has implemented some positive controls, such as requiring Multi-Factor Authentication (MFA) for computer access and conducting annual security training, several critical vulnerabilities require immediate attention.

The most severe finding is a publicly exposed MySQL database service on an internal host (172.16.50.20). This database is running MySQL version 5.7.33, which is an unsupported, End-of-Life (EOL) version with known vulnerabilities. This finding, correlated with the pre-existing risk of "Database Exposure," represents a direct and significant threat to data confidentiality and integrity.

Furthermore, critical procedural gaps were identified, including the lack of mandatory MFA for email access and the absence of security awareness training for new employees. These weaknesses substantially increase the organization’s susceptibility to phishing attacks, business email compromise, and insider threats.

This report outlines these findings in detail and provides a prioritized list of actionable recommendations to mitigate the identified risks and strengthen the overall security posture of Opal Sky Media.

Recommendations

The following actions are recommended to mitigate the identified risks. Recommendations are prioritized based on severity.

Critical Risk: Exposed & Outdated Database

  1. Immediate (Containment): Implement strict firewall rules to block all access to TCP port 3306 on host 172.16.50.20 from any untrusted network. Access should only be permitted from specific, authorized application servers or administrative workstations.

  2. Short-Term (Remediation): Develop a plan to upgrade the MySQL 5.7.33 instance to a currently supported version (e.g., MySQL 8.0 or higher) to ensure it receives security patches.

  3. Long-Term (Architectural): For any required remote database access, implement a secure access solution such as a Virtual Private Network (VPN) or a bastion host (jump box) instead of direct exposure.

Critical Risk: No MFA on Email Accounts

  1. Immediate (Remediation): Procure and enforce a mandatory MFA policy for all user accounts on the OpalSkyMedia.net email domain. This is the single most effective control to prevent unauthorized account access.

High Risk: Inadequate Employee Onboarding

  1. Short-Term (Remediation): Develop a mandatory security awareness training module and integrate it into the formal onboarding process for all new hires, to be completed within their first week of employment.

  2. Long-Term (Process Improvement): The training should cover key topics such as the acceptable use policy, phishing identification, password hygiene, and incident reporting procedures.
