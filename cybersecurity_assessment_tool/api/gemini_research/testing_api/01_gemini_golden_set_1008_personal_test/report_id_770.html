Executive Overview

This report details the findings of a cybersecurity assessment conducted for Sterling Silver. The assessment combined a technical network scan, a review of organizational security controls via a questionnaire, and an analysis of pre-existing risks.

The analysis revealed several critical and high-severity risks that expose the organization to significant threats. The most critical finding is an externally facing FTP server (10.0.0.15) running a dangerously outdated and vulnerable version of vsftpd (2.3.4), which is known to contain a backdoor. This service is further misconfigured to allow anonymous access, presenting an immediate and severe risk of system compromise and data breach.

Furthermore, significant gaps were identified in administrative controls. The lack of Multi-Factor Authentication (MFA) for computer and sensitive data access, coupled with the absence of an employee Acceptable Use Policy and security training for new hires, indicates a weak foundational security posture. These deficiencies dramatically increase the likelihood of a successful cyberattack through credential theft or insider threat.

Immediate remediation of the vulnerable FTP server is paramount. Concurrently, the organization must prioritize the implementation of MFA and the development of core security policies and training programs to mitigate these risks effectively.

Recommendations

The following actions are recommended to mitigate the identified risks.

RISK-001: Remediate Vulnerable FTP Server (Immediate)

  • Immediate Action: Disable the FTP service on 10.0.0.15 immediately to remove the threat.

  • Short-Term: Investigate the business need for this service. If it is not required, decommission the server permanently.

  • Long-Term: If a file transfer service is required, replace FTP with a secure alternative such as SFTP (SSH File Transfer Protocol) or FTPS (FTP over SSL/TLS). Ensure any new service is configured to require strong authentication.

RISK-002: Implement Multi-Factor Authentication (High Priority)

  • Action: Procure and deploy an MFA solution across the organization.

  • Priority 1: Enforce MFA for all logins to employee computers and workstations.

  • Priority 2: Enforce MFA for all access to systems containing sensitive or critical data.

RISK-003: Establish Foundational Policies & Training (High Priority)

  • Action 1: Develop and implement a formal Acceptable Use Policy (AUP) that all employees must read and acknowledge.

  • Action 2: Integrate mandatory cybersecurity awareness training into the new employee onboarding process. This training should cover topics such as phishing, password security, and the new AUP.

RISK-004: Upgrade Outdated Workstations (Medium Priority)

  • Action: Continue with the existing plan to upgrade all workstations from Windows 7 to a modern, supported operating system (e.g., Windows 10/11). This mitigates risks associated with unpatched vulnerabilities inherent in end-of-life software.
