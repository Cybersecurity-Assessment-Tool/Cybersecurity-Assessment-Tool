Executive Summary

This report details the findings of a cybersecurity assessment conducted for Great Lakes. The assessment combined a review of organizational security controls, an external network scan, and an analysis of known risks.

The overall security posture requires immediate attention. Several critical and high-risk vulnerabilities were identified that expose the organization to significant threats, including account compromise, data breaches, and service disruption.

Key findings include:

  • Critical Gaps in Multi-Factor Authentication (MFA): MFA is not enforced for accessing email or other sensitive data systems. This represents a critical vulnerability, as compromised credentials could grant attackers direct access to core business communications and confidential information.

  • Outdated Web Server Software: The external-facing web server at 192.168.10.5 is running an outdated version of Nginx (1.18.0). This version is several years old and is known to have multiple security vulnerabilities, making it a prime target for exploitation.

  • Deficient Security Policies and Training: The organization lacks a formal Acceptable Use Policy and does not conduct mandatory annual security awareness training for all staff. These gaps weaken the human element of security, increasing susceptibility to phishing and social engineering attacks.

Immediate remediation of these issues is strongly recommended to reduce the organization’s risk profile. Detailed recommendations are provided in Section 6.

Recommendations

The following actions are recommended to mitigate the identified risks.

RISK-001: Remediate Critical MFA Gaps

  • Immediate Action: Procure and implement an MFA solution for all user accounts.

  • Priority 1: Enforce MFA for access to all email accounts (e.g., Office 365, Google Workspace).

  • Priority 2: Enforce MFA for access to all systems containing sensitive or confidential data.

RISK-002: Upgrade Outdated Web Server Software

  • Immediate Action: Plan and execute an upgrade of the Nginx server on host 192.168.10.5 to the latest stable version.

  • Long-Term Action: Implement a formal patch management policy. This policy should mandate regular scanning for outdated software and a defined process for testing and deploying security patches in a timely manner.

RISK-003: Establish Security Policies & Training Program

  • Immediate Action: Develop and ratify a formal Acceptable Use Policy (AUP) that all employees must read and acknowledge. This policy should define the rules for using company IT assets.

  • Long-Term Action: Establish a mandatory, annual security awareness training program for all employees. This program should cover key topics such as phishing, password security, and data handling.
