Executive Summary

This report details the findings of a cybersecurity assessment conducted for Astraeus Aerospace on November 22, 2025. The assessment combined a review of organizational security controls, an external network scan, and an analysis of pre-existing risks.

The overall security posture is determined to be critically weak, presenting a high likelihood of compromise. The primary areas of concern are systemic failures in identity and access management, coupled with a vulnerable internet-facing system.

Key Findings:

  • Critical Control Gaps: The organization has not implemented Multi-Factor Authentication (MFA) for email, computer logins, or access to sensitive data. This represents a severe deficiency that significantly increases the risk of account takeover and unauthorized access.

  • Vulnerable External Service: The external web server is running an outdated and vulnerable version of Nginx (1.18.0). This service is a prime target for automated attacks and could serve as an initial entry point into the network.

  • Policy and Training Deficiencies: The absence of an Acceptable Use Policy and a mandatory annual security awareness training program indicates a lack of foundational cybersecurity governance. This creates an environment where employees are more susceptible to social engineering attacks.

Immediate remediation of these issues is strongly recommended to reduce the organization’s exposure to common and impactful cyber threats.

Analysis of Technical Findings

The scan identified a single open port, 443 (HTTPS), running an Nginx web server. The detected version, 1.18.0, was released in April 2020 and is now significantly outdated. This version is known to be vulnerable to multiple Common Vulnerabilities and Exposures (CVEs), including but not limited to request smuggling and denial-of-service attacks (e.g., CVE-2021-23017). Running unsupported and unpatched software on an internet-facing system constitutes a high-risk vulnerability that could be exploited by attackers to gain initial access to the network.

Recommendations

To mitigate the identified risks and improve the overall security posture, the following actions are recommended with urgency.

  1. Implement Mandatory MFA (RISK-001):

    • Action: Deploy a robust MFA solution across the organization.

    • Priority:

      1. Immediately enforce MFA for all email accounts (cloud and on-premise).

      2. Enforce MFA for all remote access (VPN) and administrative accounts.

      3. Phase in MFA for all computer logins and access to systems containing sensitive data.

  2. Remediate Vulnerable Web Server (RISK-002):

    • Action: Upgrade the Nginx server on host 192.168.10.5 to the latest stable version immediately.

    • Process Improvement: Establish a formal patch management program for all internet-facing systems. This program should include regular vulnerability scanning and a defined schedule for applying security patches.

  3. Establish Security Governance (RISK-003):

    • Action: Develop and ratify a formal Acceptable Use Policy (AUP) that all employees must read and acknowledge.

    • Training: Implement a mandatory, annual security awareness training program for all employees. This training should cover topics such as phishing, password hygiene, and social engineering.
