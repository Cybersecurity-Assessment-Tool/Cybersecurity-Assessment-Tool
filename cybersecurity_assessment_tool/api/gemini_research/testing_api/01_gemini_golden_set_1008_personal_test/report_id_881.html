Executive Overview

This report details the findings of a cybersecurity posture assessment conducted for Mainframe Managed. The assessment combined a review of organizational security controls, an external network scan, and an analysis of pre-existing risks.

The analysis reveals several and risks that require immediate attention. The most significant findings are:

  • Systemic Lack of Multi-Factor Authentication (MFA): The organization does not enforce MFA for email, computer logins, or access to sensitive data systems. This represents a critical security gap, dramatically increasing the risk of unauthorized access from compromised credentials.

  • Direct Exposure of Remote Desktop Protocol (RDP): The network scan identified an open RDP port (3389) on host 10.10.10.51. This finding, combined with a pre-existing risk for a similar exposure on another host, indicates a systemic vulnerability. Exposed RDP is a primary vector for ransomware attacks and unauthorized network intrusion.

  • Insufficient Security Awareness Program: The organization lacks a formal security awareness training program for new or existing employees. This elevates the risk of human error, making the organization highly susceptible to phishing and social engineering attacks.

The combination of exposed remote services and the absence of MFA creates a direct and easily exploitable pathway for threat actors. We strongly recommend prioritizing the remediation steps outlined in Section 6 to mitigate these immediate threats to the organization’s security and operational integrity.

Analysis

The scan confirms that port 3389 is open, exposing the Remote Desktop Protocol (RDP) service directly to the network. RDP is a high-value target for attackers who use techniques like brute-force password guessing or exploit known vulnerabilities (e.g., BlueKeep) to gain complete control over the target system. When combined with the lack of MFA, this exposure constitutes a vulnerability.

Recommendations

Based on the assessment, we recommend the following actions, prioritized by severity.

Immediate Actions (Critical Priority)

  1. Remediate All RDP Exposures (R-02, R-03):

    • Immediately close port 3389 on hosts 10.10.10.51 and 10.10.10.50 and any other systems where it is exposed to untrusted networks.

    • If remote access is required, implement a Virtual Private Network (VPN) solution with MFA enabled. Access to RDP should only be permitted through the VPN.

  2. Implement Multi-Factor Authentication (R-01):

    • Prioritize the deployment of MFA for all users on the email system (e.g., Office 365, G Suite).

    • Enforce MFA for all administrative accounts and any remote access solutions (e.g., VPN).

    • Develop a phased rollout plan to enable MFA for all computer and sensitive system logins.

High Priority Actions

  1. Establish a Security Awareness Program (R-04):

    • Implement mandatory security awareness training for all new hires during their onboarding process.

    • Conduct annual security awareness training for all employees. This training should cover, at a minimum: phishing identification, password security, and the organization’s acceptable use policy.

    • Consider periodic phishing simulation campaigns to test and reinforce employee training.
