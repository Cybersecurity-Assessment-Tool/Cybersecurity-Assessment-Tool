Executive Summary

This report presents a cybersecurity assessment for Midnight Oil Studios, synthesizing data from technical network scans, an organizational security questionnaire, and a review of pre-existing risk documentation.

The assessment identified several critical and high-risk findings that require immediate attention. The most severe issue is the discovery of a web service on the internal network at 10.5.5.5:8080 titled "TOP SECRET DB", suggesting a potentially exposed sensitive database. This finding directly contradicts the current risk register, which lists this port as a secure false positive.

Furthermore, significant procedural gaps were identified, including the lack of Multi-Factor Authentication (MFA) for email and computer access, and the absence of an employee Acceptable Use Policy. These weaknesses substantially increase the risk of account compromise and unauthorized access to sensitive information.

Immediate remediation should focus on securing the exposed database interface and implementing MFA across all critical systems. A comprehensive review of the risk management process is also strongly recommended.

The discovery of a web service on port 8080 with the title "TOP SECRET DB" is a critical finding. This suggests that an interface to a sensitive database is accessible on the network. This finding is particularly alarming as it directly contradicts the existing risk documentation (Input_3_Current_Risks_JSON), which incorrectly classifies this port as a secure false positive. Recommendations

Based on the analysis, the following actions are recommended to mitigate the identified risks and improve the overall security posture of Midnight Oil Studios.

Immediate Actions (To Be Completed within 72 Hours)

  • Investigate and Secure R-01: Immediately investigate the service running on 10.5.5.5:8080.

    • Determine the nature and sensitivity of the "TOP SECRET DB".

    • Restrict all access to the service to only authorized personnel via firewall rules or service configuration.

    • If the service is not essential, it should be disabled and removed.

  • Deploy MFA for Privileged Accounts: Begin the rollout of MFA for all administrative and executive accounts across all systems, especially email.

High-Priority Actions (To Be Completed within 30 Days)

  • Full MFA Implementation (R-02): Complete the deployment of MFA for all user email accounts and computer logins.

  • Develop and Implement AUP (R-03): Create a comprehensive Acceptable Use Policy that clearly defines the rules for using company IT resources. Ensure all employees read and acknowledge the policy.

  • Review and Update Risk Register (R-04): Conduct a full review of the existing risk register. Validate all entries against current technical data and ensure a formal process is established for regular review and updates.
