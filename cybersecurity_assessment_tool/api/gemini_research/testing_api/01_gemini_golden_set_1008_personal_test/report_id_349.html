Executive Summary

This report details the findings of a cybersecurity assessment conducted for Opal Sky Media. The assessment combined a technical network scan, a review of existing security controls via a questionnaire, and an analysis of previously identified risks. The objective is to provide a clear overview of the current security posture and offer actionable recommendations to mitigate identified vulnerabilities.

The assessment revealed several areas of significant concern requiring immediate attention. Key findings include:

  • Outdated Web Server Software: A public-facing web server was found to be running an outdated version of Nginx (1.18.0), which is susceptible to multiple known vulnerabilities.

  • Critical Gaps in Access Control: Multi-Factor Authentication (MFA) is not enforced for access to sensitive data systems, creating a critical risk of unauthorized data access and breach.

  • Foundational Policy Deficiencies: The organization lacks a formal Acceptable Use Policy (AUP) and does not provide security awareness training for new employees during onboarding. These gaps increase the risk of insider threats and susceptibility to social engineering attacks.

While some positive security controls are in place, such as MFA for email and computer logins, the identified vulnerabilities present a high level of risk to the organization’s data and operations. We strongly recommend prioritizing the remediation steps outlined in Section 6 of this report.

Analysis of Findings

The scan revealed that the web server is running Nginx version 1.18.0. This version was released in April 2020 and is now significantly outdated. It is known to be vulnerable to several publicly disclosed security flaws, including but not limited to:

  • CVE-2021-23017: A DNS resolver vulnerability that could allow an attacker to cause a denial of service or potentially execute arbitrary code.

  • Other vulnerabilities affecting modules and core components that have been patched in subsequent releases.

Running outdated software on a public-facing service constitutes a high-severity risk, as it provides a direct vector for attackers to compromise the system.

Risk Assessment

This section correlates the findings from the security control review and the technical scan into a prioritized list of risks. No pre-existing vulnerabilities were reported.

Summary of Identified Risks
Risk ID Severity Description
RISK-001 Critical Lack of Multi-Factor Authentication (MFA) on systems storing or processing sensitive data.
RISK-002 High Public-facing web server running outdated and vulnerable Nginx 1.18.0 software.
RISK-003 High No formal Acceptable Use Policy (AUP) for employees, leading to unclear security expectations.
RISK-004 High No mandatory security awareness training for new employees during the onboarding process.

Recommendations

The following actions are recommended to mitigate the identified risks and improve the overall security posture of Opal Sky Media.

For RISK-001 (Critical):

Implement MFA for Sensitive Systems.

  • Immediately prioritize the deployment and enforcement of MFA across all applications, databases, and administrative interfaces that contain sensitive organizational or customer data.

For RISK-002 (High):

Upgrade Web Server Software.

  • Plan and execute an upgrade of the Nginx server on host 192.168.10.5 to the latest stable version.

  • Implement a patch management policy to ensure all public-facing systems are updated in a timely manner.

For RISK-003 (High):

Develop an Acceptable Use Policy.

  • Create a comprehensive AUP that clearly defines the rules and responsibilities for all employees and contractors regarding the use of company assets, data handling, and internet access.

  • Ensure all current employees read and acknowledge the policy, and integrate it into the new-hire onboarding process.

For RISK-004 (High):

Establish New-Hire Security Training.

  • Develop a mandatory security awareness training module for all new employees to be completed within their first week of employment.

  • This training should cover key topics such as phishing, password security, data handling, and the new AUP.
