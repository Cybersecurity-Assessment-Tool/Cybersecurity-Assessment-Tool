1. Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Obsidian Operatives, based on network scans, a security controls questionnaire, and a review of known risks. The analysis reveals several critical and high-severity risks that require immediate attention to mitigate the potential for significant security incidents.

The most critical finding is the external exposure of a MySQL database service (172.16.50.20:3306). This service is running an outdated, End-of-Life version of MySQL (5.7.33), which no longer receives security updates. This exposure presents a direct path for attackers to compromise sensitive data.

Furthermore, significant gaps were identified in organizational security policies. The lack of mandatory Multi-Factor Authentication (MFA) for computer logins, the absence of an employee Acceptable Use Policy (AUP), and the failure to provide security awareness training to new hires create substantial vulnerabilities related to human error and credential compromise.

This report outlines these findings in detail and provides a prioritized list of actionable recommendations to strengthen the organization’s overall security posture.

6. Recommendations

The following actions are recommended to address the identified risks. They are prioritized to focus on the most critical vulnerabilities first.

  1. Immediately Restrict Database Access (Critical):

    • Apply firewall rules to block all public access to port 3306 on host 172.16.50.20.

    • Access should be restricted to a strict whitelist of internal application servers or through a secure VPN connection.

  2. Immediately Enforce Endpoint MFA (Critical):

    • Procure and deploy an MFA solution for all employee computer (desktop and laptop) logins.

    • This is a critical defense against credential theft and lateral movement within the network.

  3. Plan Database Upgrade (High):

    • Develop a migration plan to upgrade the MySQL 5.7.33 instance to a currently supported version (e.g., MySQL 8.x).

    • This will ensure the service receives timely security patches.

  4. Develop and Implement Security Policies (High):

    • Draft and ratify a formal Employee Acceptable Use Policy (AUP) that clearly outlines the rules for using company technology and data.

    • Integrate mandatory security awareness training into the standard onboarding process for all new employees.
