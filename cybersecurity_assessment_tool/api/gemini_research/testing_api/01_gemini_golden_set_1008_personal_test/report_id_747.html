Executive Summary

This report provides a comprehensive assessment of the cybersecurity posture for Catalyst Consulting, based on a combination of technical network scanning, a review of existing security risks, and an analysis of organizational security controls.

The assessment identified several critical and high-risk areas requiring immediate attention. While the organization has implemented Multi-Factor Authentication (MFA) for email and sensitive data systems, significant gaps exist in foundational security practices.

Key findings include:

  • Critical Database Vulnerability: A network scan confirmed the presence of a publicly accessible MySQL database. This system is running version 5.7.33, which is past its End-of-Life (EOL) and no longer receives security updates, posing a severe risk of data breach.

  • Endpoint Security Gaps: The absence of MFA for computer logins presents a high risk, as compromised credentials could lead to direct workstation and network access.

  • Policy and Training Deficiencies: The organization lacks a formal Acceptable Use Policy and does not conduct security awareness training for employees. These gaps significantly increase the susceptibility to human-centric attacks like phishing and social engineering.

This report outlines these findings in detail and provides a series of actionable recommendations to mitigate the identified risks and strengthen the overall security posture of Catalyst Consulting.

Analysis of Findings

The scan identified an open MySQL port (3306). The running version, MySQL 5.7.33, reached its official End-of-Life (EOL) in October 2023. EOL software no longer receives security patches from the vendor, making it an easy target for exploitation of known vulnerabilities. This finding confirms the "Database Exposure" risk provided in the initial data.

Recommendations

The following actions are recommended to mitigate the identified risks. Recommendations are categorized by the associated risk ID.

R-01: Publicly Exposed End-of-Life Database

  • Immediate (0-7 Days): Implement strict firewall rules to restrict all access to port 3306 on 172.16.50.20 to only trusted, internal IP addresses.

  • Short-Term (1-3 Months): Plan and execute a migration from MySQL 5.7 to a currently supported version (e.g., MySQL 8.x) to ensure security patches are available.

  • Long-Term (3-6 Months): Re-architect the network so that the database server resides in a private, non-internet-facing subnet, accessible only via a bastion host or VPN.

R-02: Lack of Endpoint MFA

  • Immediate (0-30 Days): Procure and pilot an MFA solution for workstation logins with the IT department and key executives.

  • Short-Term (1-3 Months): Develop a phased rollout plan to deploy MFA across the entire organization.

  • Long-Term (3-6 Months): Enforce MFA for all user logins to company workstations and laptops without exception.

R-03: Insufficient Security Awareness Training

  • Immediate (0-30 Days): Select and procure a security awareness training platform.

  • Short-Term (1-3 Months): Enroll all employees in foundational training covering phishing, password hygiene, and data handling. Make this training mandatory for all new hires during onboarding.

  • Long-Term (Ongoing): Establish a continuous security culture program, including mandatory annual refresher training and quarterly simulated phishing campaigns.

R-04: Missing Foundational Security Policies

  • Immediate (0-30 Days): Draft a formal Acceptable Use Policy (AUP) that clearly defines rules for using company assets, data handling, and internet access.

  • Short-Term (1-2 Months): Have the draft AUP reviewed by HR, legal, and senior management for approval.

  • Long-Term (Ongoing): Implement a process to have all new and existing employees formally read and acknowledge the AUP. Review and update the policy annually.
