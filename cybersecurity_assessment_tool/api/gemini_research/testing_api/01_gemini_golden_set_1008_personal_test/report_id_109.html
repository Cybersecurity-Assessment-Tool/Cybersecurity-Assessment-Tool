Executive Overview

This report details the findings of a cybersecurity assessment conducted for Vanguard Heritage on November 22, 2025. The assessment combined an external network scan, a review of organizational security controls via a questionnaire, and an analysis of pre-existing risks.

The analysis revealed several critical and high-risk security gaps that expose the organization to significant threats, including account compromise, data breaches, and service disruption. Key findings include:

  • Critical Lack of Multi-Factor Authentication (MFA): The complete absence of MFA for email, computer logins, and sensitive data systems represents a critical vulnerability. This is the most significant weakness identified and dramatically increases the risk of unauthorized access.

  • Outdated External Services: The public-facing web server is running an outdated version of nginx (1.18.0), which is known to have security vulnerabilities. This exposes the organization to potential exploitation from external attackers.

  • Insufficient Security Policies and Training: The organization lacks a formal Acceptable Use Policy and does not conduct annual security awareness training for all staff. These foundational gaps weaken the human element of the security posture.

  • Web Server Misconfiguration: An SSL certificate mismatch was identified on the primary web server, which can erode user trust and may indicate underlying configuration issues.

Immediate and decisive action is required to remediate these findings. Recommendations provided in this report are prioritized to address the most critical risks first.

Analysis of Findings

  • Outdated Nginx Version: The web server is running nginx version 1.18.0, which was released in April 2020. This version is significantly outdated and is missing numerous security patches for vulnerabilities discovered since its release. Running unsupported software on an internet-facing system presents a high risk of exploitation.

  • SSL Certificate Mismatch: The scan revealed that the SSL certificate presented by the server is for the common name . This does not match the organization’s domain, www.VanguardHeritage.org. This misconfiguration will cause browser trust errors and could be exploited in man-in-the-middle scenarios.

Recommendations

The following actions are recommended to mitigate the identified risks. They are prioritized based on severity.

  1. (RISK-001) Implement Multi-Factor Authentication:

    • Action: Immediately deploy a robust MFA solution across the organization.

    • Priority: Start with email (e.g., Office 365, Google Workspace), followed by remote access (VPN), and all systems containing sensitive data.

    • Impact: Drastically reduces the risk of account compromise.

  2. (RISK-002) Remediate Outdated Nginx Server:

    • Action: Plan and execute an upgrade of the nginx server at 192.168.10.5 to the latest stable version.

    • Priority: This should be done in a staging environment first to ensure compatibility, then deployed to production as soon as possible.

    • Impact: Mitigates known vulnerabilities and protects the server from exploitation.

  3. (RISK-003 & 004) Establish Security Policies and Training:

    • Action: Draft and ratify a formal Acceptable Use Policy (AUP). Concurrently, procure and schedule mandatory annual security awareness training for all employees.

    • Priority: These foundational controls should be implemented within the next quarter.

    • Impact: Strengthens the human firewall and establishes clear security expectations.

  4. (RISK-005) Correct SSL Certificate Configuration:

    • Action: Investigate the certificate mismatch. Procure and correctly install a valid SSL/TLS certificate for the domain www.VanguardHeritage.org.

    • Priority: Remediate to ensure secure connections and maintain user trust.

    • Impact: Resolves browser security warnings and secures web traffic.
