Executive Overview

This report provides a comprehensive cybersecurity assessment for Stone Arch Masonry. The analysis is based on a correlation of technical network scan data, a review of organizational security controls via a questionnaire, and an evaluation of pre-existing risks.

The assessment reveals a mixed security posture. While the organization has implemented Multi-Factor Authentication (MFA) for critical systems like email and sensitive data access, there are significant and high-risk gaps in foundational administrative and procedural controls.

Key findings include:

  • Critical Policy Gaps: The complete absence of an employee Acceptable Use Policy (AUP) and a security awareness training program presents a critical risk. These gaps leave the organization highly vulnerable to social engineering, phishing attacks, and insider threats.

  • Endpoint Security Weakness: The lack of MFA for computer logins is a high-risk vulnerability that could allow an attacker with compromised credentials to gain direct access to internal network resources.

  • External Service Exposure: A technical scan identified an open Secure Shell (SSH) port on an external IPv6 address. While necessary for remote administration, this service must be rigorously secured to prevent unauthorized access.

Immediate and decisive action is required to address the identified policy and training deficiencies. Strengthening these foundational controls will significantly improve the organization’s resilience against common cyber threats.

Recommendations

Based on the analysis, the following actions are recommended to mitigate the identified risks and improve the overall security posture of Stone Arch Masonry.

  • [Critical Priority] Implement a Security Awareness Program:

    • Immediately establish a mandatory security awareness training program for all employees.

    • Training should be conducted for all new hires during onboarding and annually for all staff.

    • Key topics must include phishing identification, password security, safe browsing, and data handling procedures.

  • [High Priority] Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a formal AUP that clearly defines the rules for using company networks, devices, and data.

    • Require all current and new employees to read and sign the policy to acknowledge their understanding and agreement.

  • [High Priority] Enforce MFA on All Endpoints:

    • Deploy a solution to require MFA for all computer and laptop logins. This significantly reduces the risk of unauthorized access from compromised credentials.

    • Solutions can include Windows Hello for Business, Duo Security, or other third-party MFA providers.

  • [Medium Priority] Secure the Exposed SSH Service:

    • Review the business justification for exposing SSH on .

    • If required, implement firewall rules to restrict access to only trusted source IP addresses.

    • Enforce public key authentication and disable password-based logins to prevent brute-force attacks.
