Executive Summary

This report provides a comprehensive cybersecurity assessment for Nebula Creative, synthesizing findings from technical network scans, a review of organizational security controls, and an analysis of pre-existing risks. The assessment was conducted to identify vulnerabilities, security gaps, and misconfigurations that could expose the organization to cyber threats.

The analysis revealed several critical and high-risk findings that require immediate attention. A key vulnerability was discovered on an internal server (10.0.0.15), which is running a critically outdated and backdoored version of the vsftpd FTP service. This is compounded by an insecure configuration allowing anonymous user access.

Furthermore, significant gaps were identified in organizational security policies. The absence of Multi-Factor Authentication (MFA) on email accounts presents a high risk of business email compromise. The lack of a formal Acceptable Use Policy for employees also introduces unnecessary risk.

This report details these findings and provides prioritized, actionable recommendations to mitigate the identified risks and strengthen the overall security posture of Nebula Creative.

Consolidated Risk Assessment

This section correlates findings from the security control review, technical scans, and pre-existing risk data to provide a unified view of the organization’s risk landscape. Risks are prioritized based on their potential impact and likelihood of exploitation.

Risk ID Description Severity Affected Systems
RISK-001 A public-facing FTP server is running vsftpd 2.3.4, which contains a known critical backdoor (CVE-2011-2523), allowing an attacker to gain remote command execution. Critical Server at 10.0.0.15
RISK-002 The FTP server is configured to allow anonymous logins, permitting unauthorized users to access, upload, or download files, potentially leading to data leakage or malware distribution. High Server at 10.0.0.15
RISK-003 Email accounts are not protected by Multi-Factor Authentication (MFA). This significantly increases the risk of account takeover via phishing or credential stuffing attacks. High All email accounts

Recommendations

The following actions are recommended to mitigate the identified risks. They are prioritized to address the most severe threats first.

Critical Priority

  • Remediate Vulnerable FTP Service (RISK-001): Immediately take the server at 10.0.0.15 offline. The vsftpd 2.3.4 software must be upgraded to the latest stable version or replaced with a secure file transfer solution (e.g., SFTP). If FTP is not a business requirement, the service should be disabled entirely.

High Priority

  • Disable Anonymous FTP (RISK-002): Once the FTP service is patched, reconfigure it to disable anonymous access. All access should require authentication with strong, unique credentials.

  • Implement MFA for Email (RISK-003): Enforce mandatory MFA for all user accounts on the NebulaCreative.net email domain. This is the single most effective control to prevent account compromise.

Medium Priority

  • Develop an Acceptable Use Policy (RISK-004): Create and implement a formal AUP that clearly defines rules for the use of company technology, data handling, and internet access. Ensure all employees read and acknowledge the policy.

  • Upgrade Outdated Operating Systems (RISK-005): Continue with the plan to upgrade or replace all workstations running Windows 7. Prioritize systems used by employees who handle sensitive data.
