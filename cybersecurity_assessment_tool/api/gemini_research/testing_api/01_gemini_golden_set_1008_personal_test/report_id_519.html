Executive Summary

This report details the findings of a cybersecurity assessment conducted for Symmetry Architecture. The analysis combines a review of organizational security controls, an external network scan, and a summary of pre-existing risks.

The assessment identified several critical and high-risk vulnerabilities that require immediate attention. Key findings include:

  • Critical Service Exposure: An externally accessible MySQL database was identified. This service is running an End-of-Life (EOL) version (5.7.33), which no longer receives security updates, posing a significant risk of compromise.

  • Insufficient Access Controls: Multi-Factor Authentication (MFA) is not enforced for employee access to email or computer systems. This represents a critical gap that severely weakens the organization’s defense against credential theft and phishing attacks.

  • Policy Gaps: The organization lacks a formal Acceptable Use Policy (AUP). This foundational document is essential for establishing clear security expectations and responsibilities for all employees.

The overall security posture is considered high-risk due to the combination of direct external threats and internal control weaknesses. We strongly recommend prioritizing the remediation actions outlined in Section 6 to mitigate these risks and improve the organization’s defensive capabilities.

Analysis

The lack of MFA for email and computer access is a critical vulnerability. Email accounts are a primary target for attackers, and a successful compromise could lead to business email compromise (BEC), data breaches, or further infiltration of the network. The absence of an Acceptable Use Policy indicates a gap in security governance.

Analysis

The scan confirms that a MySQL database server is directly exposed to the internet. This configuration is highly discouraged as it exposes the database to brute-force attacks, credential stuffing, and exploitation of known vulnerabilities.

Furthermore, the detected version, MySQL 5.7.33, reached its official End-of-Life (EOL) in October 2023. EOL software no longer receives security patches from the vendor, meaning any newly discovered vulnerabilities will remain unpatched, leaving the system perpetually at risk.

Recommendations

Based on the consolidated risk assessment, we provide the following prioritized recommendations to mitigate the identified vulnerabilities.

Priority 1: Critical Risks (Immediate Action Required)

  1. Restrict Database Access: Immediately implement firewall rules to block all public access to TCP port 3306. Access should be restricted to only trusted internal IP addresses, such as application servers. For remote administration, a Virtual Private Network (VPN) should be required.

  2. Upgrade End-of-Life Software: Plan and execute an urgent migration of the MySQL 5.7 database to a currently supported version (e.g., MySQL 8.x). This is essential to ensure the system receives ongoing security updates.

  3. Implement MFA: Deploy Multi-Factor Authentication across all user accounts for both email access (e.g., Office 365, Google Workspace) and computer logins (e.g., Windows Hello, Duo). This is the single most effective control to prevent account compromise.

Priority 2: High Risks (Action within 30-60 days)

  1. Develop and Implement an AUP: Draft a formal Acceptable Use Policy that clearly defines the rules and responsibilities for employees when using company technology and data. This policy should be reviewed by management, distributed to all employees, and acknowledged with a signature.

Priority 3: Foundational Improvements

  1. Enhance Security Training: While a training program exists, consider enhancing it with specific modules on the dangers of phishing and the importance of the new MFA and AUP policies. This will help reinforce secure behaviors and ensure policy adoption.
