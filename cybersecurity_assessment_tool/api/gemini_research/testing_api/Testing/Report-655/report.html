Cybersecurity Assessment Report

Overview

The organization's network scan identified at least one active host with an open SSH (port 22) service. While the organization has implemented Multi-Factor Authentication (MFA) for email and sensitive data systems, it lacks MFA for computer logins. An acceptable use policy is in place, and security awareness training is provided for new employees, but annual security awareness training for all employees is not conducted.

Identified Risks

Lack of Multi-Factor Authentication for Computer Login - High

Overview: Without MFA for computer logins, an attacker who compromises a user's credentials (e.g., via phishing or brute-force) can gain unauthorized access to internal systems and data without an additional layer of authentication. This significantly increases the risk of unauthorized access, data breaches, and lateral movement within the network, as a single compromised password could grant full system access.

Affected Elements:

  • User accounts
  • Workstations
  • Servers requiring login

Insufficient Annual Security Awareness Training - Medium

Overview: The absence of regular, annual security awareness training for all employees leaves the organization vulnerable to social engineering attacks (phishing, vishing, pretexting) and other human-factor risks. Employees may not be up-to-date on the latest threats, best practices for data handling, or how to identify and report suspicious activities, increasing the likelihood of successful attacks that bypass technical controls.

Affected Elements:

  • All employees
  • Organizational data
  • Network security

Exposed SSH Service - Medium

Overview: An open SSH (port 22) service, if not properly secured, can be a target for brute-force attacks, credential stuffing, or exploitation of SSH vulnerabilities. While SSH is a secure protocol, misconfigurations such as weak passwords, root login enabled, or outdated software can lead to unauthorized access to the host, potentially providing a direct entry point into the network.

Affected Elements:

  • Host(s) with open port 22

Recommendations

  • To mitigate the identified risks, it is strongly recommended to implement Multi-Factor Authentication for all computer logins to significantly reduce the impact of compromised credentials. Additionally, a comprehensive annual security awareness training program should be established for all employees to educate them on current threats and best security practices, thereby strengthening the human firewall. For the exposed SSH service, ensure it is configured securely by disabling password authentication in favor of key-based authentication, restricting access via firewall rules to only necessary IP addresses, and regularly patching the SSH server software to address known vulnerabilities.

Conclusion

The organization demonstrates a foundational understanding of cybersecurity by implementing MFA for sensitive systems and providing initial security training. However, critical gaps exist in access control for computer logins and ongoing employee security education, which present significant attack vectors. The presence of an open SSH port also warrants immediate attention to prevent unauthorized access. Addressing these vulnerabilities will substantially improve the organization's overall security posture and readiness against common cyber threats.
