Cybersecurity Assessment Report

Overview

The organization's network scan identified an Nginx web server running version 1.18.0 on port 443 (HTTPS). While HTTPS is in use, the Nginx version is older and may lack recent security updates. Organizational data indicates a mixed security posture: an acceptable use policy is in place, and new employees receive security awareness training. However, critical gaps exist, including the absence of Multi-Factor Authentication (MFA) for email access and computer logins, and a lack of annual security awareness training for all employees. MFA is only required for sensitive data systems.

Identified Risks

Lack of Multi-Factor Authentication (MFA) for Email Access - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for email access. This significantly increases the risk of account compromise through phishing, credential stuffing, or brute-force attacks. If an attacker gains access to an employee's email, they could potentially reset passwords for other services, access sensitive communications, or launch further attacks within the organization.

Affected Elements:

  • Email system
  • Employee accounts

Lack of Multi-Factor Authentication (MFA) for Computer Login - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for computer logins. This leaves employee workstations vulnerable to unauthorized access if credentials are stolen or guessed. An attacker gaining access to a workstation could potentially access internal networks, sensitive data, or deploy malware, leading to data breaches or operational disruption.

Affected Elements:

  • Employee workstations
  • Internal network access

Outdated Nginx Web Server Version - Medium

Overview: The Nginx web server is running version 1.18.0. While this is a stable release, it is an older version (released in 2020) and may not include the latest security patches and bug fixes available in newer stable branches (e.g., 1.20.x, 1.22.x, 1.24.x). Running outdated software increases the risk of exploitation by known vulnerabilities for which patches exist in newer versions, potentially leading to unauthorized access or service disruption.

Affected Elements:

  • Web server (nginx 1.18.0)
  • Port 443 (HTTPS)

Insufficient Annual Security Awareness Training - Medium

Overview: The organization conducts security awareness training for new employees but does not provide annual training for all employees. This means existing employees may not be regularly updated on new threats, evolving best practices, and organizational policies, increasing the risk of human-factor vulnerabilities such as phishing, social engineering, and improper data handling, which can lead to security incidents.

Affected Elements:

  • All employees
  • Organizational security posture

Recommendations

  • To mitigate the identified risks, it is strongly recommended to implement Multi-Factor Authentication (MFA) for all email accounts and computer logins across the organization. This is a critical step to significantly enhance protection against credential theft. Furthermore, the Nginx web server should be updated to the latest stable version to ensure all known security patches are applied and to benefit from the most recent security features. Finally, a comprehensive annual security awareness training program should be established for all employees, not just new hires, to ensure continuous education on current threats, best practices, and organizational security policies.

Conclusion

The organization demonstrates some foundational security practices, such as an acceptable use policy and initial security awareness training. However, significant vulnerabilities exist due to the lack of MFA for email and computer access, the use of an older web server version, and insufficient ongoing security awareness training. Addressing these critical and medium-severity risks is paramount to improving the organization's overall cybersecurity posture and readiness against common attack vectors.
