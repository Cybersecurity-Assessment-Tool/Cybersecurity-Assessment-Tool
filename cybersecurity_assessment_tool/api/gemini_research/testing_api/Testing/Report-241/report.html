Cybersecurity Assessment Report

Overview

The organization's network scan revealed an open HTTPS port (443) running Nginx version 1.18.0. While the network footprint appears minimal from the scan, an assessment of organizational policies indicates significant security gaps. Specifically, the organization does not enforce Multi-Factor Authentication (MFA) for email access, computer logins, or sensitive data systems. Furthermore, there is no established employee acceptable use policy. On a positive note, the organization does conduct security awareness training for both new and all employees annually.

Identified Risks

Lack of MFA for Email Access - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for accessing email accounts. This significantly increases the risk of unauthorized access to employee inboxes if credentials are compromised through phishing, brute-force attacks, or credential stuffing. A single compromised password could grant an attacker full access to sensitive communications and potentially lead to further internal network penetration.

Affected Elements:

  • Email system
  • User accounts
  • Employee communications

Lack of MFA for Computer Login - High

Overview: Multi-Factor Authentication is not enforced for logging into organizational computers. This leaves workstations vulnerable to unauthorized access if employee passwords are stolen or guessed. Attackers could gain direct access to internal systems, data, and network resources, potentially leading to data breaches, malware infections, or lateral movement within the network.

Affected Elements:

  • Workstations
  • Endpoints
  • User accounts
  • Internal network

Lack of MFA for Sensitive Data Systems - Critical

Overview: Access to sensitive data systems does not require Multi-Factor Authentication. This is a critical vulnerability as it means that a compromised single factor (e.g., a password) is sufficient to gain access to the organization's most valuable and confidential information. The impact of such a breach could be severe, including regulatory fines, reputational damage, and significant financial loss.

Affected Elements:

  • Sensitive data systems
  • Databases
  • Confidential information repositories

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization currently lacks an Employee Acceptable Use Policy (AUP). Without a formal AUP, employees may be unaware of the expected standards for using company IT resources, internet access, and data handling. This can lead to misuse of company assets, increased risk of malware introduction, data leakage, and potential legal or compliance issues due to undefined boundaries for employee conduct.

Affected Elements:

  • Organizational policy
  • Employee conduct
  • IT resources
  • Data handling practices

Outdated Nginx Version - Low

Overview: The web server identified is running Nginx version 1.18.0. While not immediately critical without specific CVEs, older software versions may contain known vulnerabilities that have been patched in newer releases. Running outdated software increases the attack surface and could expose the organization to exploits if unpatched vulnerabilities are discovered or targeted.

Affected Elements:

  • Web server (Nginx 1.18.0)
  • Public-facing services

Recommendations

  • To significantly enhance the organization's security posture, it is strongly recommended to implement Multi-Factor Authentication (MFA) across all critical access points. This includes enforcing MFA for email access, computer logins, and especially for all sensitive data systems. Furthermore, the organization should promptly develop and implement a comprehensive Employee Acceptable Use Policy (AUP) to clearly define expectations for IT resource usage and data handling, ensuring all employees are trained on its contents. Regular patching and updating of all software, including the Nginx web server, should be a standard practice to mitigate risks associated with known vulnerabilities. While security awareness training is already in place, it should be continuously reinforced with specific emphasis on phishing prevention and the importance of MFA.

Conclusion

The organization currently faces significant security risks primarily due to the absence of Multi-Factor Authentication across key access points and the lack of a formal Employee Acceptable Use Policy. These policy-level vulnerabilities create substantial opportunities for unauthorized access and data breaches, despite the positive practice of regular security awareness training. Addressing these fundamental gaps by implementing MFA and establishing clear usage policies is paramount to improving the organization's overall cybersecurity readiness and protecting its assets from potential threats.
