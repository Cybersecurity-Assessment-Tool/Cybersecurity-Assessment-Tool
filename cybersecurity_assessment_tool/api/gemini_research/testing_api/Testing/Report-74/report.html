Cybersecurity Assessment Report

Overview

The organization's network currently hosts an FTP server running vsftpd version 2.3.4, which was found to allow anonymous login. This version of vsftpd is significantly outdated and known to contain critical vulnerabilities. Furthermore, the organization's workstations are running Windows 7, an operating system that has reached its end-of-life and no longer receives security updates. From an organizational policy standpoint, there is a critical lack of Multi-Factor Authentication (MFA) for accessing email, logging into computers, and accessing sensitive data systems. The organization also lacks an employee acceptable use policy and does not provide security awareness training for new employees, although annual training is conducted for all existing employees. These factors collectively indicate a low security readiness posture with several significant risks.

Identified Risks

Anonymous FTP Login Allowed - High

Overview: The FTP server (vsftpd 2.3.4) is configured to allow anonymous login. This configuration can lead to unauthorized access to files, potential data leakage, or even server compromise if write permissions are misconfigured. This vulnerability was identified during an Nmap scan which detected anonymous FTP login was permitted.

Affected Elements:

  • FTP Server (vsftpd 2.3.4)
  • Port 21

Outdated FTP Software (vsftpd 2.3.4) - Critical

Overview: The FTP server is running vsftpd version 2.3.4, which is an outdated version known to have critical vulnerabilities, including a backdoor command execution vulnerability (CVE-2011-2523). Running unpatched and end-of-life software significantly increases the attack surface and risk of system compromise. This was identified via Nmap service detection.

Affected Elements:

  • FTP Server (vsftpd 2.3.4)
  • Port 21

Outdated Windows Operating System - High

Overview: Several workstations within the organization are running Windows 7. Windows 7 has reached its end-of-life, meaning it no longer receives security updates or patches from Microsoft. This leaves these systems highly vulnerable to new and existing exploits, malware, and other cyber threats, posing a significant risk to the entire network.

Affected Elements:

  • Workstations running Windows 7

Lack of Multi-Factor Authentication (MFA) - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for critical access points such as email, computer logins, or sensitive data systems. This significantly increases the risk of unauthorized access in the event of credential theft, phishing attacks, or brute-force attempts, as a single compromised password could grant full access.

Affected Elements:

  • Email systems
  • User accounts
  • Sensitive data systems
  • Computer logins

Absence of Employee Acceptable Use Policy (AUP) - Medium

Overview: The organization currently lacks a formal Employee Acceptable Use Policy. Without clear guidelines on the appropriate use of company IT resources, employees may inadvertently or intentionally engage in activities that introduce security risks, violate compliance requirements, or lead to data breaches.

Affected Elements:

  • Organizational IT resources
  • Employee behavior

Insufficient New Employee Security Training - Medium

Overview: New employees do not receive dedicated security awareness training upon joining the organization. This gap means new hires may be unaware of critical security policies, best practices, and common threats like phishing, making them more susceptible to social engineering attacks and potentially introducing vulnerabilities from the outset.

Affected Elements:

  • New employees
  • Organizational security posture

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately disable anonymous login on the FTP server and either upgrade vsftpd to a current, supported version or replace it with a more secure file transfer solution. All workstations running Windows 7 must be upgraded to a currently supported operating system, such as Windows 10 or 11, to ensure they receive critical security updates. Implementing Multi-Factor Authentication (MFA) across all critical access points, including email, computer logins, and sensitive data systems, is paramount to enhance credential security. Furthermore, the organization should develop and enforce a comprehensive Employee Acceptable Use Policy to guide employee behavior regarding IT resource usage. Finally, mandatory security awareness training should be introduced for all new employees, complementing the existing annual training for all staff, to foster a strong security culture from day one.

Conclusion

The current cybersecurity assessment reveals significant vulnerabilities and gaps in the organization's security posture. Critical technical risks include an outdated and insecure FTP server, end-of-life operating systems on workstations, and a pervasive lack of Multi-Factor Authentication. These technical issues are compounded by policy and training deficiencies, specifically the absence of an acceptable use policy and initial security training for new employees. While annual security training is a positive step, the overall readiness is low, exposing the organization to a high risk of data breaches, unauthorized access, and system compromise. Addressing these findings with urgency is crucial to improve the organization's security resilience.
