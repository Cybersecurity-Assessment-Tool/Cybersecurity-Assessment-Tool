Cybersecurity Assessment Report

Overview

The organization's network currently exposes an Nginx web server (version 1.18.0) on port 443 (HTTPS). While security awareness training is conducted for new and existing employees annually, several critical security controls are missing. Specifically, Multi-Factor Authentication (MFA) is not required for email access or computer logins, significantly increasing the risk of unauthorized access. Furthermore, the organization lacks an employee acceptable use policy, which can lead to inconsistent security practices and increased insider threats.

Identified Risks

Lack of MFA for Email Access - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for accessing email accounts. This leaves email accounts vulnerable to credential stuffing, phishing, and brute-force attacks. A compromised email account can serve as a gateway for business email compromise (BEC) scams, data exfiltration, and further penetration into the organization's network. This was identified from the organizational data indicating no MFA for email.

Affected Elements:

  • Email system
  • User accounts
  • Organizational data

Lack of MFA for Computer Login - High

Overview: Multi-Factor Authentication (MFA) is not enforced for logging into computers. This means that if an attacker obtains a user's password through phishing, malware, or other means, they can gain direct access to the employee's workstation. This significantly increases the risk of unauthorized access, lateral movement within the network, data theft, and the deployment of malicious software. This was identified from the organizational data indicating no MFA for computer login.

Affected Elements:

  • All employee workstations
  • Network access points
  • Internal systems

Absence of Employee Acceptable Use Policy (AUP) - Medium

Overview: The organization currently lacks an employee acceptable use policy. Without a clear AUP, employees may engage in activities that pose security risks, such as installing unauthorized software, visiting malicious websites, or mishandling company data, without clear guidelines or consequences. This can lead to increased exposure to malware, data breaches, and potential legal liabilities. This was identified from the organizational data indicating no acceptable use policy.

Affected Elements:

  • All employees
  • Organizational data
  • Network resources
  • IT infrastructure

Outdated Nginx Web Server - High

Overview: The Nginx web server running on port 443 is version 1.18.0. This version is End-of-Life (EOL) and no longer receives security updates or patches. Running EOL software exposes the web server to known and newly discovered vulnerabilities, which can be exploited for remote code execution, denial of service attacks, information disclosure, or defacement. This was identified during the network scan.

Affected Elements:

  • Web server (nginx 1.18.0)
  • Port 443
  • Hosted web applications
  • Public-facing services

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately implement Multi-Factor Authentication (MFA) for all email accounts and computer logins across the organization. This critical step will significantly enhance protection against credential-based attacks. Furthermore, develop and enforce a comprehensive Employee Acceptable Use Policy (AUP) to clearly define acceptable behavior regarding company IT resources and data, ensuring all employees are aware of their responsibilities. Finally, the Nginx web server should be updated to a currently supported and patched version to address known vulnerabilities and ensure ongoing security updates. Regular vulnerability scanning and patch management should be integrated into the security operations.

Conclusion

The organization demonstrates a foundational commitment to security through its regular security awareness training. However, significant vulnerabilities exist due to the absence of MFA for email and computer logins, the lack of an acceptable use policy, and the use of an outdated Nginx web server. Addressing these high-severity risks is paramount to improving the organization's overall cybersecurity posture and readiness against common attack vectors.
