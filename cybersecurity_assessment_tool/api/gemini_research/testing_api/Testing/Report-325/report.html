Cybersecurity Assessment Report

Overview

The organization operates a web server running Nginx 1.18.0, accessible via HTTPS on port 443. From an organizational perspective, strong multi-factor authentication (MFA) is implemented for email access, computer logins, and sensitive data systems, indicating a robust approach to identity and access management. However, there are notable deficiencies in foundational security policies and ongoing employee security education.

Identified Risks

Outdated Web Server Software - High

Overview: The Nginx web server identified on port 443 is running version 1.18.0. This version, released in April 2020, is an older stable release and is known to be susceptible to various vulnerabilities that have since been patched in newer versions. Examples include CVE-2021-23017 (resolver stack buffer overflow) and CVE-2021-23019 (integer overflow). Exploitation of such vulnerabilities could lead to denial of service, information disclosure, or potentially remote code execution, compromising the web server and its hosted content. This was identified through network scanning.

Affected Elements:

  • Web server (Nginx 1.18.0)
  • Port 443 (HTTPS service)

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization lacks a formal Employee Acceptable Use Policy (AUP). Without clear guidelines, employees may not fully understand their responsibilities concerning the use of company IT resources, data handling, and internet access. This increases the risk of misuse of company assets, accidental data breaches, introduction of malware, and non-compliance with regulatory requirements due to a lack of defined behavioral expectations. This was identified from organizational policy data.

Affected Elements:

  • Organizational policy
  • Employee behavior
  • IT assets and data

Infrequent Security Awareness Training - Medium

Overview: While new employees receive security awareness training, the organization does not conduct annual security awareness training for all employees. This gap means that existing staff may not be kept up-to-date on the latest cyber threats, evolving attack techniques (e.g., advanced phishing), and changes in organizational security policies and best practices. This significantly increases the likelihood of successful social engineering attacks and human error leading to security incidents. This was identified from organizational training data.

Affected Elements:

  • Employee knowledge and behavior
  • Organizational security posture
  • Data integrity and confidentiality

Recommendations

  • To enhance the organization's security posture, it is strongly recommended to immediately upgrade the Nginx web server to the latest stable version to address known vulnerabilities and ensure ongoing security. Furthermore, a comprehensive Employee Acceptable Use Policy should be developed and implemented to clearly define acceptable behavior regarding IT resource usage and data handling. Finally, establish a mandatory annual security awareness training program for all employees to ensure continuous education on current threats and best practices, thereby strengthening the human element of security.

Conclusion

The organization demonstrates a commendable commitment to identity and access management through widespread MFA implementation. However, critical vulnerabilities exist in the form of an outdated web server and significant gaps in foundational security policies and continuous employee training. Addressing these identified risks is paramount to reduce the overall attack surface, mitigate potential threats, and build a more resilient defense against the evolving landscape of cyber threats.
