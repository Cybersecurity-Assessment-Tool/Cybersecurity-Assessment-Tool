Cybersecurity Assessment Report

Overview

The organization's network scan identified an active FTP server running vsftpd 2.3.4 with anonymous login enabled. From an organizational standpoint, MFA is utilized for email and sensitive data systems, but not for computer logins. An acceptable use policy is in place, and annual security awareness training is conducted for all employees, though new employees do not receive specific training upon onboarding.

Identified Risks

Outdated Windows Operating System - Medium

Overview: Computers are running Windows 7, which has reached end-of-life and no longer receives security updates. This exposes the workstations to known vulnerabilities and potential exploits, significantly increasing the risk of compromise. Identified via internal assessment data.

Affected Elements:

  • Workstations

Anonymous FTP Access Enabled - High

Overview: The FTP server (vsftpd 2.3.4) allows anonymous login, which means unauthorized users can access and potentially modify files on the server without authentication. This poses a significant data breach risk and could be used for malware distribution or data exfiltration. Identified via Nmap scan script 'ftp-anon'.

Affected Elements:

  • FTP Server (vsftpd 2.3.4)
  • Port 21

No MFA for Computer Logins - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for computer logins. This significantly increases the risk of unauthorized access if credentials are stolen or guessed, as a single factor (password) is the only barrier. Identified via organizational policy review.

Affected Elements:

  • Employee Workstations
  • Login Systems

Inadequate New Employee Security Training - Medium

Overview: New employees do not receive specific security awareness training upon onboarding. This leaves them vulnerable to social engineering attacks, phishing, and other common threats during their initial period, potentially introducing risks to the organization before they understand security protocols. Identified via organizational policy review.

Affected Elements:

  • New Employees
  • Organizational Security Posture

Recommendations

  • Immediately disable anonymous FTP access; if FTP functionality is required, implement secure alternatives like SFTP or FTPS with strong authentication and ensure the server software is regularly patched. Upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or 11) to ensure ongoing security updates. Implement Multi-Factor Authentication (MFA) for all computer logins across the organization to enhance credential security. Establish a mandatory security awareness training program for all new employees as part of their onboarding process, covering essential security practices and common threats. Furthermore, regularly conduct vulnerability scans and penetration tests to proactively identify and address emerging risks, and ensure all software and operating systems are kept up-to-date with the latest security patches.

Conclusion

The organization demonstrates a mixed security posture. While MFA is utilized for email and sensitive data, critical vulnerabilities exist, including an exposed FTP server with anonymous access, outdated operating systems on workstations, and a lack of MFA for computer logins. Additionally, the absence of security awareness training for new employees presents a human factor risk. Addressing these identified risks is crucial to significantly improve the overall security readiness and reduce the attack surface.
