Cybersecurity Assessment Report

Overview

The organization's network currently exposes an Nginx web server (version 1.18.0) on port 443 (HTTPS). While the organization has implemented Multi-Factor Authentication (MFA) for email and computer logins, it lacks MFA for sensitive data systems. Furthermore, there is no established employee acceptable use policy. Security awareness training is conducted for new employees and annually for all staff, which is a positive aspect of their security posture.

Identified Risks

Outdated Nginx Web Server - High

Overview: The Nginx web server running on port 443 is identified as version 1.18.0. This version was released in April 2020 and is significantly outdated, no longer receiving active security updates or bug fixes. Running an End-of-Life (EOL) or substantially older software version exposes the organization to known vulnerabilities (CVEs) that have been patched in newer releases. This could lead to various security incidents, including remote code execution, denial of service, or information disclosure. The vulnerability was identified through an Nmap scan.

Affected Elements:

  • Nginx web server (version 1.18.0) on port 443

Lack of MFA for Sensitive Data Systems - High

Overview: Despite having MFA for email and computer logins, the organization does not require MFA to access sensitive data systems. This significantly increases the risk of unauthorized access to critical information. If an attacker compromises a user's credentials for a sensitive data system, they can gain full access without needing a second authentication factor, potentially leading to severe data breaches, data manipulation, or system compromise. This was identified from the organizational data provided.

Affected Elements:

  • All systems classified as sensitive data systems

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization currently lacks an employee acceptable use policy (AUP). Without a formal AUP, employees may not have clear guidelines regarding the appropriate use of company IT resources, internet access, email, and data handling. This can lead to unintentional security breaches, increased legal liabilities, and reduced productivity. Employees might engage in risky behaviors such as installing unauthorized software, visiting malicious websites, or mishandling sensitive information due to a lack of explicit rules. This was identified from the organizational data provided.

Affected Elements:

  • All employees
  • Organizational IT resources

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately upgrade the Nginx web server to the latest stable version to patch known vulnerabilities and ensure ongoing security support. Implement Multi-Factor Authentication (MFA) for all sensitive data systems to add an essential layer of security against credential compromise. Develop and enforce a comprehensive employee acceptable use policy (AUP) to clearly define expectations for IT resource usage and data handling, ensuring all employees are trained on its contents. Continue with regular security awareness training for all employees to reinforce best practices and educate them on emerging threats.

Conclusion

The organization demonstrates a foundational commitment to security through its existing MFA implementations for email and computer logins, and its consistent security awareness training program. However, critical vulnerabilities exist with the outdated Nginx web server and the absence of MFA for sensitive data systems, posing significant risks to data integrity and confidentiality. The lack of an acceptable use policy also introduces potential compliance and operational risks. Addressing these key areas will substantially enhance the organization's overall cybersecurity posture and readiness against potential threats.
