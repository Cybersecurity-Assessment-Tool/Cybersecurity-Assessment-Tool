Cybersecurity Assessment Report

Overview

The organization operates a network with an Nginx web server on port 443. While strong security practices are in place for email and sensitive data systems, including MFA and regular security awareness training, a critical gap exists in endpoint security.

Identified Risks

Outdated Web Server Software - High

Overview: The Nginx web server running on port 443 is identified as version 1.18.0. This version is significantly outdated, with the latest stable release being much newer. Older software versions often contain known security vulnerabilities that have been patched in subsequent releases. Exploitation of these vulnerabilities could lead to various impacts, including denial-of-service, information disclosure, or remote code execution, compromising the web server and potentially other connected systems. This was identified via an Nmap scan revealing the specific Nginx version.

Affected Elements:

  • Nginx web server (version 1.18.0) on port 443

Lack of Multi-Factor Authentication (MFA) for Computer Logins - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for logging into computers. This significantly increases the risk of unauthorized access to employee workstations and the internal network. If an attacker obtains an employee's username and password through phishing, malware, or other means, they can gain direct access to the computer without an additional authentication factor. This could lead to data breaches, lateral movement within the network, and system compromise. This was identified through the organizational data indicating "Do you require MFA to log into computers?":"No".

Affected Elements:

  • All employee computers/endpoints requiring login

Recommendations

  • To mitigate the identified risks, it is strongly recommended that the organization immediately update the Nginx web server to the latest stable version to patch known vulnerabilities and improve overall security posture. Furthermore, implementing Multi-Factor Authentication (MFA) for all computer logins is crucial. This will add an essential layer of security, significantly reducing the risk of unauthorized access even if primary credentials are compromised. Regular vulnerability scanning and patch management processes should also be established or reinforced for all network infrastructure.

Conclusion

The organization demonstrates a good foundation in security awareness and MFA for critical systems like email and sensitive data. However, the presence of an outdated Nginx web server and the absence of MFA for computer logins represent significant vulnerabilities that could be exploited by attackers. Addressing these high-severity risks by updating software and implementing MFA for endpoints is paramount to enhancing the organization's overall cybersecurity readiness and protecting against potential breaches.
