Cybersecurity Assessment Report

Overview

The organization demonstrates a foundational understanding of security by implementing Multi-Factor Authentication (MFA) for email access. However, significant gaps exist in critical areas, including the absence of MFA for computer logins and sensitive data systems, which exposes core assets to credential-based attacks. Furthermore, the lack of an employee acceptable use policy and annual security awareness training indicates a need for improved human-centric security controls. A network scan revealed an open SSH port (22), which, if not adequately secured, represents a direct entry point for potential attackers.

Identified Risks

Lack of Multi-Factor Authentication for Computer Login - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for employees to log into their computers. This significantly increases the risk of unauthorized access if employee credentials are compromised through phishing, malware, or other means. Without MFA, a single compromised password can grant an attacker full access to an employee's workstation and potentially facilitate lateral movement within the network. This vulnerability was identified through a review of organizational security policies.

Affected Elements:

  • Employee workstations
  • Internal network access points

Lack of Multi-Factor Authentication for Sensitive Data Systems - Critical

Overview: Sensitive data systems within the organization do not enforce Multi-Factor Authentication (MFA). This critical oversight means that systems containing valuable or regulated data are protected only by single-factor authentication. A breach of these systems could lead to severe data loss, regulatory non-compliance, financial penalties, and significant reputational damage. This vulnerability was identified through a review of organizational security policies.

Affected Elements:

  • Databases containing sensitive data
  • File servers with confidential information
  • CRM systems
  • HR systems
  • Financial applications

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization lacks a formal employee acceptable use policy. Without clear guidelines, employees may unknowingly engage in activities that introduce security risks, such as installing unauthorized software, visiting malicious websites, or mishandling sensitive data. This absence of policy increases the likelihood of human error leading to security incidents, data breaches, or malware infections. This vulnerability was identified through a review of organizational security policies.

Affected Elements:

  • All employees
  • Organizational data
  • Network resources
  • IT infrastructure

Insufficient Annual Security Awareness Training - Medium

Overview: The organization does not conduct security awareness training for all employees at least once per year. Regular training is essential to educate employees about evolving cyber threats, such as phishing, social engineering, and malware, and to reinforce secure computing practices. The lack of annual training leaves employees more susceptible to common attack vectors, increasing the risk of successful cyberattacks that exploit human vulnerabilities. This vulnerability was identified through a review of organizational security policies.

Affected Elements:

  • All employees
  • Organizational data
  • Network security
  • Email systems

Open SSH Port (22) - High

Overview: A network scan identified an open SSH port (22) on a host. While SSH is a legitimate service for secure remote access, an open port represents a potential attack surface if not properly secured. Without robust configurations (e.g., strong passwords, key-based authentication, restricted IP access, rate limiting), this port can be targeted by brute-force attacks, credential stuffing, or other exploits to gain unauthorized remote access to the system. This vulnerability was identified via an Nmap network scan.

Affected Elements:

  • Host with open port 22
  • Network perimeter

Recommendations

  • To significantly enhance the organization's security posture, it is strongly recommended to implement Multi-Factor Authentication (MFA) for all computer logins and access to sensitive data systems immediately. Develop and enforce a comprehensive employee acceptable use policy to guide appropriate technology usage and mitigate human-related risks. Establish a mandatory annual security awareness training program for all employees to keep them informed about current threats and best practices. For the identified open SSH port (22), ensure it is secured with strong, key-based authentication, disable password authentication, restrict access to trusted IP addresses only, and consider implementing a robust intrusion detection/prevention system to monitor and block malicious attempts.

Conclusion

The organization currently faces several significant cybersecurity risks, primarily stemming from insufficient implementation of Multi-Factor Authentication for critical access points and a lack of comprehensive human-centric security policies and training. While MFA for email is a positive step, the absence of it for computer logins and sensitive data systems, coupled with an open SSH port, creates substantial attack vectors. The lack of an acceptable use policy and annual security awareness training further exacerbates these risks by increasing the likelihood of human error and successful social engineering attacks. Addressing these vulnerabilities is crucial for improving the organization's overall security readiness and protecting its assets from potential cyber threats.
