Cybersecurity Assessment Report

Overview

The organization's network currently operates an FTP server (vsftpd 2.3.4) that permits anonymous login, posing a significant risk for unauthorized data access or potential malicious uploads. While Multi-Factor Authentication (MFA) is utilized for email access, it is notably absent for computer logins and sensitive data systems, leaving critical access points vulnerable to credential compromise. Furthermore, the organization lacks comprehensive security awareness training for both new and existing employees, increasing susceptibility to social engineering and human error. An acceptable use policy is in place, but its effectiveness is diminished without accompanying training. A critical finding is the continued use of Windows 7 on workstations, an end-of-life operating system, which exposes the network to numerous unpatched vulnerabilities.

Identified Risks

Anonymous FTP Access - High

Overview: The FTP server, identified as vsftpd version 2.3.4, is configured to allow anonymous login. This configuration permits unauthorized users to access, and potentially modify or upload, files without authentication. This can lead to data leakage, unauthorized data manipulation, or the introduction of malicious content onto the server. The vulnerability was identified via an Nmap scan using the 'ftp-anon' script, which confirmed anonymous login is allowed.

Affected Elements:

  • FTP Server (vsftpd 2.3.4) on port 21

Lack of MFA for Computer Login - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for employees to log into their computers. This significantly increases the risk of unauthorized access to internal systems and data if a single set of credentials (username and password) is compromised through phishing, brute-force attacks, or other means. Without MFA, a compromised password provides direct access to the workstation.

Affected Elements:

  • All employee computers/workstations

Lack of MFA for Sensitive Data Systems - Critical

Overview: Multi-Factor Authentication (MFA) is not enforced for accessing systems containing sensitive organizational data. This critical oversight means that if credentials for these systems are compromised, attackers can gain direct, unauthorized access to sensitive information without an additional layer of security. The impact could include severe data breaches, regulatory non-compliance, and significant reputational damage.

Affected Elements:

  • Sensitive data systems

Lack of Security Awareness Training - High

Overview: The organization does not conduct security awareness training for new employees, nor does it provide annual training for all employees. This absence leaves employees vulnerable to social engineering attacks (e.g., phishing, pretexting), increases the likelihood of human error leading to security incidents (e.g., clicking malicious links, mishandling sensitive data), and reduces overall organizational resilience against cyber threats.

Affected Elements:

  • All employees
  • Organizational data
  • Network infrastructure

Outdated Windows Policy - Critical

Overview: A significant number of workstations are currently running Windows 7. Windows 7 reached its end-of-life on January 14, 2020, meaning it no longer receives security updates, patches, or technical support from Microsoft. This exposes these workstations to known and newly discovered vulnerabilities, making them highly susceptible to exploitation by malware, ransomware, and other cyberattacks. Continued use of this operating system poses a severe risk to the entire network.

Affected Elements:

  • Workstations running Windows 7

Recommendations

  • Immediate action is required to address the identified vulnerabilities. Firstly, the anonymous access on the FTP server (vsftpd 2.3.4) must be disabled, and the server should be upgraded to a supported version or replaced with a more secure file transfer solution like SFTP or FTPS. Secondly, Multi-Factor Authentication (MFA) must be implemented across the board for all computer logins and, critically, for access to all sensitive data systems to prevent unauthorized access from compromised credentials. Thirdly, a mandatory and recurring security awareness training program should be established for all employees, including new hires and annual refreshers, to educate them on common cyber threats and best security practices. Finally, all workstations running Windows 7 must be upgraded to a currently supported operating system, such as Windows 10 or 11, to ensure they receive essential security updates and patches, thereby mitigating the risk of exploitation from known vulnerabilities.

Conclusion

The organization currently faces several critical and high-severity cybersecurity risks, primarily stemming from outdated software, inadequate access controls, and a lack of comprehensive security awareness. The presence of an anonymous FTP server, the absence of MFA for critical systems and computer logins, and the continued use of end-of-life operating systems like Windows 7 create significant attack vectors that could lead to data breaches, system compromise, and operational disruption. While an acceptable use policy is in place, the overall security posture indicates a low readiness level against sophisticated cyber threats. Addressing these vulnerabilities immediately through system upgrades, MFA implementation, and robust security training is paramount to safeguarding organizational assets and data.
