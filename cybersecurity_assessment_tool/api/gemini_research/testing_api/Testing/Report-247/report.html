Cybersecurity Assessment Report

Overview

The network scan identified an open SSH port (22). Organizational data indicates that Multi-Factor Authentication (MFA) is not required for computer logins. While MFA is used for email and sensitive data systems, security awareness training is not conducted for new employees or annually for all employees. An employee acceptable use policy is in place.

Identified Risks

Unsecured Remote Access - Medium

Overview: The Nmap scan identified an open SSH port (22). While SSH is a secure protocol, an open port presents a potential attack vector if not properly configured with strong authentication (e.g., key-based authentication, disabled password authentication) and access controls (e.g., IP whitelisting). Unauthorized access could lead to system compromise, data exfiltration, or further network penetration.

Affected Elements:

  • Port 22
  • Network perimeter

Weak Authentication for Endpoints - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for computer logins. This significantly increases the risk of unauthorized access to employee workstations and potentially the internal network if credentials are compromised through phishing, brute-force attacks, or malware. A single compromised password could grant an attacker access.

Affected Elements:

  • All employee computers
  • Internal network access

Increased Human Error Vulnerability - Medium

Overview: The organization does not conduct security awareness training for new employees. New hires may not be familiar with the organization's security policies, common threat vectors (e.g., phishing, social engineering), or best practices for data handling. This lack of initial training makes them more susceptible to falling victim to attacks, potentially compromising organizational security.

Affected Elements:

  • New employees
  • Organizational data
  • Network security

Stagnant Security Knowledge & Practices - Medium

Overview: The organization does not conduct security awareness training for all employees at least once per year. Without regular refreshers, employees' knowledge of evolving threats and security best practices can become outdated. This increases the likelihood of employees making security errors, falling for sophisticated social engineering attacks, or mishandling sensitive information, leading to potential breaches.

Affected Elements:

  • All employees
  • Organizational data
  • Network security

Recommendations

  • Implement strict access controls for the open SSH port (22), including IP whitelisting, disabling password authentication in favor of strong key-based authentication, and ensuring regular patching and monitoring of the SSH server. Consider changing the default SSH port. Mandate Multi-Factor Authentication (MFA) for all computer logins across the organization to significantly enhance endpoint security. Establish a comprehensive security awareness training program that includes mandatory training for all new employees during onboarding and annual refresher training for all existing employees, covering topics such as phishing, social engineering, password hygiene, data handling, and incident reporting. Conduct regular network vulnerability scans and penetration tests to identify and remediate new vulnerabilities proactively.

Conclusion

The organization demonstrates some foundational security practices, such as requiring MFA for email and sensitive data systems and having an acceptable use policy. However, significant vulnerabilities exist, particularly concerning endpoint authentication (lack of MFA for computer logins) and human factors (absence of comprehensive security awareness training for new and existing employees). The open SSH port also presents a potential entry point if not rigorously secured. Addressing these critical areas will substantially improve the organization's overall security posture and readiness against common cyber threats.
