Cybersecurity Assessment Report

Overview

The organization's network scan revealed an open SSH port (22), indicating potential remote access capabilities. From an organizational policy standpoint, the company has implemented security awareness training for all employees, including new hires, which is a positive step. However, critical gaps exist in multi-factor authentication (MFA) implementation, as it is not required for computer logins or access to sensitive data systems. Furthermore, the absence of an employee acceptable use policy leaves the organization vulnerable to various internal security risks.

Identified Risks

Open SSH Port (22) - High

Overview: The Nmap scan identified an open SSH port (22) on at least one host. While SSH is a legitimate service for secure remote access, an open port presents a potential attack surface. If not properly secured with strong authentication, rate limiting, and regular patching, it can be exploited for brute-force attacks, credential stuffing, or unauthorized access, leading to system compromise.

Affected Elements:

  • Host(s) with open port 22

Lack of MFA for Computer Login - Critical

Overview: The organization does not require multi-factor authentication (MFA) for employees to log into their computers. This significantly increases the risk of unauthorized access if credentials are stolen, phished, or brute-forced. A single compromised password could grant an attacker full access to an employee's workstation and potentially the internal network, bypassing other security controls.

Affected Elements:

  • All employee workstations and associated login systems

Lack of MFA for Sensitive Data Systems - Critical

Overview: Multi-factor authentication is not enforced for accessing sensitive data systems. This poses a critical risk to data confidentiality and integrity. If an attacker gains access to a single set of credentials, they can potentially access, exfiltrate, or tamper with sensitive organizational data without an additional layer of security. This could lead to severe data breaches, regulatory non-compliance, and reputational damage.

Affected Elements:

  • All sensitive data systems and their access mechanisms

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization lacks an employee acceptable use policy (AUP). An AUP defines the proper and improper use of company IT resources, including computers, networks, and data. Without clear guidelines, employees may inadvertently engage in risky behaviors (e.g., installing unauthorized software, visiting malicious websites, sharing sensitive information inappropriately) that could introduce malware, facilitate data leaks, or violate compliance requirements. This lack of policy creates ambiguity and increases the likelihood of human error leading to security incidents.

Affected Elements:

  • All employees
  • IT systems
  • Data

Recommendations

  • To mitigate the identified risks, the following recommendations are provided: 1. Secure SSH Access: Implement robust security measures for all SSH services. This includes using strong, complex passwords or, preferably, SSH key-based authentication, disabling root login, configuring rate limiting for failed login attempts, and ensuring SSH servers are regularly patched and updated. Consider restricting SSH access to specific IP ranges or internal networks where possible. 2. Implement MFA for Computer Logins: Mandate multi-factor authentication for all computer logins across the organization. This significantly enhances security by requiring a second form of verification beyond a password, making it much harder for attackers to gain unauthorized access even if they compromise credentials. 3. Implement MFA for Sensitive Data Systems: Enforce multi-factor authentication for all access points to systems containing sensitive data. This is a critical control to protect against data breaches and ensure that only authorized individuals with multiple forms of authentication can access high-value assets. 4. Develop and Enforce an Employee Acceptable Use Policy: Create a comprehensive acceptable use policy that clearly outlines employee responsibilities and expectations regarding the use of company IT resources. This policy should be communicated to all employees, acknowledged, and regularly reinforced through security awareness training. 5. Regular Vulnerability Scanning and Patch Management: Continue regular network scans to identify open ports and services, and establish a robust patch management program to ensure all systems and software are up-to-date with the latest security patches.

Conclusion

The organization demonstrates a foundational commitment to security through its consistent security awareness training programs. However, significant vulnerabilities exist, primarily due to the lack of multi-factor authentication for computer logins and sensitive data systems, which are critical controls for preventing unauthorized access and data breaches. The presence of an open SSH port, if not adequately secured, also presents a high-risk entry point. Furthermore, the absence of an acceptable use policy leaves the organization susceptible to human error and internal misuse of resources. Addressing these critical gaps by implementing MFA across key access points, securing remote access services, and establishing clear employee usage policies is paramount to enhancing the organization's overall cybersecurity posture and readiness against evolving threats.
