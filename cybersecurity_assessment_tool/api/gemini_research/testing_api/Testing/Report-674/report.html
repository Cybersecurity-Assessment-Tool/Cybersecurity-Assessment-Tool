Cybersecurity Assessment Report

Overview

The organization operates a network that includes an FTP server with an outdated version (vsftpd 2.3.4) allowing anonymous access. Several workstations are running Windows 7, an end-of-life operating system. Furthermore, the organization exhibits significant gaps in its security policies and practices, notably the complete absence of Multi-Factor Authentication (MFA) for email, computer logins, and sensitive data systems. There is also a lack of an employee acceptable use policy and no security awareness training for new or existing employees. These factors collectively indicate a low level of security readiness and a high exposure to various cyber threats.

Identified Risks

Outdated Operating System - High

Overview: Computers are running Windows 7, which is an end-of-life operating system. This means it no longer receives security updates, making systems highly vulnerable to known and newly discovered exploits. The impact includes potential data breaches, system compromise, and non-compliance with security standards.

Affected Elements:

  • Workstations

Anonymous FTP Access Allowed - High

Overview: The FTP server (vsftpd 2.3.4) allows anonymous login, which means anyone can connect to the server without credentials. This can lead to unauthorized access to files, potential data exfiltration, or even the uploading of malicious content if write permissions are misconfigured. The service version (vsftpd 2.3.4) is also old and known to have vulnerabilities.

Affected Elements:

  • FTP Server (port 21)
  • vsftpd 2.3.4

No MFA for Critical Systems - Critical

Overview: The organization does not require Multi-Factor Authentication (MFA) for accessing email, logging into computers, or accessing sensitive data systems. This significantly increases the risk of unauthorized access if credentials are stolen or compromised through phishing or other attacks, as a single factor (password) is the only barrier.

Affected Elements:

  • Email systems
  • User login systems
  • Sensitive data systems

Absence of Acceptable Use Policy - Medium

Overview: The organization lacks an employee acceptable use policy. This means there are no clear guidelines for employees on the proper and secure use of company IT resources, data, and internet access. This can lead to misuse of resources, increased risk of malware infections, data leakage, and legal liabilities.

Affected Elements:

  • All employees
  • IT resources
  • Data

Inadequate Security Awareness Training - High

Overview: The organization does not conduct security awareness training for new employees, nor does it provide annual training for all employees. This leaves employees unprepared to identify and respond to common cyber threats like phishing, social engineering, and malware. A lack of training significantly increases the human element risk, making the organization more susceptible to breaches.

Affected Elements:

  • All employees
  • Organizational data
  • Network infrastructure

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately disable anonymous FTP access and update the vsftpd server to a current, supported version, or replace it with a more secure file transfer solution. All workstations running Windows 7 must be upgraded to a supported operating system like Windows 10 or 11; a long-term strategy should include a hardware refresh cycle to support modern OS requirements. Implementing Multi-Factor Authentication (MFA) across all critical access points—email, computer logins, and sensitive data systems—is paramount to significantly reduce the risk of unauthorized access. The organization must develop and enforce a comprehensive employee acceptable use policy to guide secure behavior. Crucially, a robust security awareness training program should be established, including mandatory training for all new hires and annual refresher training for all employees, to educate them on identifying and responding to cyber threats.

Conclusion

The organization currently faces a high level of cybersecurity risk due to a combination of outdated software, critical configuration weaknesses (anonymous FTP), and significant deficiencies in fundamental security policies and employee training. The absence of MFA for critical systems, coupled with an end-of-life operating system on workstations, creates easily exploitable entry points for attackers. Without immediate and comprehensive remediation efforts, the organization remains highly vulnerable to data breaches, system compromises, and operational disruptions. Improving security readiness requires a multi-faceted approach addressing technical vulnerabilities, policy gaps, and human factors.
