Cybersecurity Assessment Report

Overview

The organization's network scan revealed an open SSH port (22) on at least one host, indicating a potential entry point for attackers if not adequately secured. Furthermore, a review of organizational policies indicates a critical lack of Multi-Factor Authentication (MFA) across key access points, including email, computer logins, and sensitive data systems. While an acceptable use policy is in place and annual security awareness training is conducted for all employees, there is a notable absence of security awareness training for new employees, leaving them vulnerable during their initial period. These findings suggest significant vulnerabilities in access control and initial employee security posture.

Identified Risks

Open SSH Port - Critical

Overview: An Nmap scan identified an open SSH port (22) on a network host. This poses a critical risk as it can be a target for brute-force attacks, credential stuffing, or exploitation of SSH vulnerabilities, potentially leading to unauthorized remote access to the system. The severity is critical due to the direct remote access potential.

Affected Elements:

  • Port 22 (SSH)
  • Network hosts

Lack of Multi-Factor Authentication (MFA) - High

Overview: The organization currently does not require MFA for accessing email, logging into computers, or accessing sensitive data systems. This significantly increases the risk of successful credential compromise, as a single stolen password could grant an attacker full access to critical organizational resources. This vulnerability was identified through a review of organizational security policies.

Affected Elements:

  • Email systems
  • Employee workstations
  • Sensitive data systems

Inadequate New Employee Security Training - Medium

Overview: New employees do not receive security awareness training as part of their onboarding process. While annual training is conducted for all employees, this initial gap leaves new hires susceptible to social engineering attacks, phishing, and unintentional security policy violations during their critical initial period. This was identified through a review of HR and security training policies.

Affected Elements:

  • New employees
  • Organizational security posture

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately implement robust security measures for all open SSH ports, including enforcing strong, unique passwords, utilizing key-based authentication, disabling root login, and restricting access to trusted IP addresses. Furthermore, the organization must prioritize the implementation of Multi-Factor Authentication (MFA) across all critical access points, specifically for email, computer logins, and sensitive data systems, to significantly enhance credential security. Finally, a mandatory security awareness training program should be integrated into the onboarding process for all new employees to ensure they are equipped with essential security knowledge from day one. The existing acceptable use policy and annual security awareness training should be leveraged and reinforced to support these new initiatives.

Conclusion

The current cybersecurity posture of the organization presents several critical and high-severity risks, primarily stemming from an exposed SSH port and a pervasive lack of Multi-Factor Authentication across key systems. While foundational elements like an acceptable use policy and annual security training are present, the absence of initial security training for new employees further exacerbates the human element risk. Addressing these vulnerabilities, particularly the implementation of MFA and securing remote access points, is paramount to significantly improving the organization's readiness against common cyber threats and protecting sensitive assets.
