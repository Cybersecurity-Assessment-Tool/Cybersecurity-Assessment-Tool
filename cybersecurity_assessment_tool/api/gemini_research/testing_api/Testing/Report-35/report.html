Cybersecurity Assessment Report

Overview

The organization currently employs Multi-Factor Authentication (MFA) for email and sensitive data systems, which is a positive security measure. They also conduct security awareness training for both new hires and all employees annually. However, a significant gap exists as MFA is not required for computer logins. Furthermore, the organization lacks a formal employee acceptable use policy. Network scanning revealed an open SSH port (22) on a host, presenting a potential attack vector. A critical vulnerability related to localhost exposure was also identified, indicating a severe internal access risk.

Identified Risks

Localhost Exposed - Critical

Overview: The system's localhost interface is exposed, potentially allowing unauthorized access to services running on the local machine. This is a critical vulnerability as it can bypass network firewalls and directly access internal services, leading to data compromise or system control.

Affected Elements:

  • IP

No MFA for Computer Login - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for logging into computers. This significantly increases the risk of unauthorized access if credentials are stolen or compromised, as a single factor (password) is sufficient for authentication, making endpoints vulnerable to various attacks.

Affected Elements:

  • Employee Workstations
  • Servers (if applicable)
  • Login Portals

Absence of Employee Acceptable Use Policy - Medium

Overview: The organization lacks an employee acceptable use policy. This means there are no formal guidelines for employees regarding the appropriate use of company IT resources, data handling, and security practices. This absence can lead to misuse, data breaches, and compliance issues due to a lack of clear expectations and accountability.

Affected Elements:

  • Organizational Policies
  • Employee Conduct
  • Data Handling Procedures

Open SSH Port (22) - Medium

Overview: An open SSH (port 22) was detected on a host. While SSH is a legitimate service for secure remote access, an open port presents a potential attack surface for brute-force attacks, credential stuffing, or exploitation of SSH vulnerabilities if not properly secured. Without strong authentication and restricted access, this port could be an entry point for attackers.

Affected Elements:

  • Host with open port 22
  • SSH Service

Recommendations

  • To significantly enhance the organization's security posture, it is strongly recommended to implement Multi-Factor Authentication (MFA) for all computer logins, thereby protecting endpoints from credential theft. A comprehensive employee acceptable use policy must be developed and enforced to establish clear guidelines for IT resource usage, data handling, and security best practices, mitigating human-factor risks. For the identified open SSH port (22), ensure it is secured with strong, key-based authentication, disable password authentication, and restrict access to trusted IP addresses only. Regular patching and updating of SSH services are also crucial. The critical localhost exposure must be addressed by implementing proper network segmentation and access controls to prevent unauthorized access to internal services and sensitive data.

Conclusion

The organization demonstrates a foundational commitment to cybersecurity through the use of MFA for critical systems and regular security awareness training. However, several significant vulnerabilities persist, including the absence of MFA for computer logins, a lack of a formal acceptable use policy, an open SSH port, and a critical localhost exposure. Addressing these identified risks is paramount to reducing the overall attack surface, strengthening the organization's defenses, and achieving a more robust security posture against evolving cyber threats.
