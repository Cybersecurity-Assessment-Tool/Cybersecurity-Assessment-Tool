Cybersecurity Assessment Report

Overview

The organization's network currently presents several critical security vulnerabilities. A network scan identified an open FTP service allowing anonymous login, which poses a significant risk for unauthorized data access and manipulation. Furthermore, the organization's internal security posture is weakened by the use of outdated operating systems (Windows 7 workstations) and a lack of multi-factor authentication (MFA) for computer logins and access to sensitive data systems. While annual security awareness training is conducted for all employees, there is no specific training for new hires, and a formal acceptable use policy is absent. These gaps indicate a need for immediate attention to both technical and policy-driven security controls.

Identified Risks

Anonymous FTP Access - High

Overview: The FTP service (vsftpd 2.3.4) on port 21 allows anonymous login, meaning unauthorized users can access and potentially modify files on the FTP server without credentials. This vulnerability can lead to data breaches, unauthorized data modification, or the hosting of malicious content. It was identified via an Nmap scan using the 'ftp-anon' script.

Affected Elements:

  • FTP Server (vsftpd 2.3.4 on port 21)

Outdated Operating System - High

Overview: Computers are running Windows 7, which has reached its end-of-life and no longer receives security updates from Microsoft. This leaves systems vulnerable to known exploits and new threats, significantly increasing the risk of compromise and making them a prime target for attackers.

Affected Elements:

  • Workstations

No MFA for Computer Login - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for computer logins. This means that if an attacker obtains a user's password, they can gain full access to the computer without an additional layer of security, increasing the risk of unauthorized access and lateral movement within the network.

Affected Elements:

  • All employee computers/workstations

No MFA for Sensitive Data Systems - Critical

Overview: The organization does not require Multi-Factor Authentication (MFA) to access sensitive data systems. This significantly increases the risk of unauthorized access to critical and confidential information if credentials are compromised, potentially leading to severe data breaches, regulatory non-compliance, and reputational damage.

Affected Elements:

  • Sensitive data systems

Missing Acceptable Use Policy - Medium

Overview: The organization lacks an employee acceptable use policy. Without clear guidelines on appropriate use of company IT resources, employees may inadvertently engage in risky behaviors (e.g., visiting malicious websites, installing unauthorized software) that could compromise network security. This also makes it difficult to enforce security standards and hold employees accountable.

Affected Elements:

  • All employees
  • IT systems

Inadequate New Employee Security Training - Medium

Overview: The organization does not conduct security awareness training for new employees. This means new hires may not be aware of common cyber threats (phishing, social engineering) or organizational security policies, making them more susceptible to attacks and potentially introducing vulnerabilities into the network from day one.

Affected Elements:

  • New employees
  • Overall organizational security posture

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately disable anonymous FTP access or secure the FTP service with strong authentication and access controls. All workstations running Windows 7 should be upgraded to a currently supported operating system like Windows 10 or 11, with a long-term plan for hardware refit if necessary. Implement Multi-Factor Authentication (MFA) for all computer logins and, critically, for access to all sensitive data systems to significantly enhance access security. Develop and enforce a comprehensive employee acceptable use policy to guide appropriate IT resource usage. Additionally, introduce mandatory security awareness training for all new employees to ensure they are equipped with essential cybersecurity knowledge from the start, complementing the existing annual training.

Conclusion

The organization faces a combination of critical technical vulnerabilities and significant policy gaps. The presence of anonymous FTP access, outdated operating systems, and the absence of MFA for critical access points expose the organization to a high risk of data breaches and unauthorized system access. While some security awareness is present, the lack of a formal acceptable use policy and specific training for new hires further exacerbates the human element risk. Addressing these issues promptly through technical remediation and policy implementation is crucial for improving the organization's overall cybersecurity posture and readiness against evolving threats.
