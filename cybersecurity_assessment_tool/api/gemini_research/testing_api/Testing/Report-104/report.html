Cybersecurity Assessment Report

Overview

The organization demonstrates a foundational level of security with Multi-Factor Authentication (MFA) implemented for email and computer logins, and an acceptable use policy in place. However, significant gaps exist in critical areas, particularly regarding the protection of sensitive data systems, employee security awareness, and network hygiene. An open port 80 was identified, indicating potential unencrypted traffic or an exposed service.

Identified Risks

Lack of MFA for Sensitive Data Systems - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for accessing sensitive data systems. This significantly increases the risk of unauthorized access to critical information if credentials are compromised. An attacker gaining access to a single set of credentials could potentially access sensitive data without an additional layer of security, leading to data breaches and compliance violations.

Affected Elements:

  • Sensitive data systems
  • User accounts accessing sensitive data

No Security Awareness Training for New Employees - Medium

Overview: New employees are not provided with security awareness training. This leaves them vulnerable to social engineering attacks, phishing, malware, and other common cybersecurity threats from the outset. Untrained employees can inadvertently introduce new attack vectors into the organization, compromising security through human error.

Affected Elements:

  • New employees
  • Organizational security posture

No Annual Security Awareness Training - Medium

Overview: The organization does not conduct annual security awareness training for all employees. Regular and ongoing training is crucial to keep employees informed about evolving threats, best practices, and the organization's security policies. Without it, employee knowledge degrades over time, increasing susceptibility to sophisticated attacks and weakening the overall security posture.

Affected Elements:

  • All employees
  • Organizational security posture

Open Port 80 (HTTP) - Medium

Overview: A network scan identified an open port 80, typically used for unencrypted HTTP traffic. This could indicate that web services are running without encryption, exposing sensitive information transmitted over the network to eavesdropping, or that an outdated/unsecured web server is publicly accessible. This poses a risk for data interception and potential exploitation of web server vulnerabilities.

Affected Elements:

  • Network perimeter
  • Web servers
  • Data transmitted over HTTP

Recommendations

  • To mitigate the identified risks, it is strongly recommended to immediately implement Multi-Factor Authentication (MFA) for all sensitive data systems to prevent unauthorized access. Furthermore, establish a comprehensive security awareness training program that includes mandatory onboarding training for all new employees and annual refresher training for all existing staff. This will significantly improve the human firewall against social engineering and phishing attacks. Finally, investigate the service running on port 80; if it is a web server, ensure HTTPS is enforced for all traffic, and HTTP is either redirected or blocked. If it is another service, ensure it is necessary, properly secured, and not exposing unnecessary information.

Conclusion

While the organization has implemented some basic security controls like MFA for email and computer logins, and an acceptable use policy, its overall cybersecurity readiness is compromised by critical vulnerabilities. The absence of MFA for sensitive data systems, coupled with a lack of comprehensive and ongoing security awareness training, exposes the organization to significant risks from both external threats and internal human error. The presence of an open port 80 further highlights potential network security weaknesses. Addressing these areas is paramount to enhancing the organization's defense against cyber threats and protecting its sensitive assets.
