Cybersecurity Assessment Report

Overview

The organization's network currently presents several significant security vulnerabilities. A network scan identified an FTP server running an extremely outdated version of vsftpd (2.3.4) with anonymous login enabled, posing a critical risk for unauthorized access and data compromise. Internally, the organization does not enforce Multi-Factor Authentication (MFA) for computer logins, which significantly weakens workstation security. Furthermore, there is a lack of fundamental security policies, specifically an employee acceptable use policy, and security awareness training is not conducted annually for all employees, increasing human-factor risks. Several workstations are also running the end-of-life Windows 7 operating system, leaving them exposed to known and unpatched vulnerabilities. While MFA is used for email and sensitive data systems, these other critical gaps undermine the overall security posture.

Identified Risks

Outdated FTP Service (vsftpd 2.3.4) - Critical

Overview: The Nmap scan identified an FTP server running vsftpd version 2.3.4 on port 21. This version is extremely old (released in 2011) and is known to contain multiple critical vulnerabilities, including a severe backdoor (CVE-2011-2523). Operating such an outdated service exposes the organization to remote code execution, denial of service, and unauthorized access, making it a prime target for attackers.

Affected Elements:

  • FTP Server (vsftpd 2.3.4)
  • Port 21

Anonymous FTP Access Enabled - High

Overview: The Nmap scan revealed that the FTP server on port 21 allows anonymous login. This configuration permits unauthorized users to access, and potentially modify or upload, files without authentication. This can lead to data leakage, unauthorized content distribution, and the potential for attackers to stage malicious files or compromise the server's integrity.

Affected Elements:

  • FTP Server (port 21)
  • Network perimeter

Insufficient Authentication Controls for Workstations - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for logging into computers. This significantly increases the risk of unauthorized access to employee workstations if credentials are stolen, phished, or brute-forced. Without MFA, a single compromised password can grant an attacker full access to an employee's local system and potentially facilitate lateral movement within the network.

Affected Elements:

  • Employee workstations
  • Login systems

Outdated Operating System (Windows 7) - Critical

Overview: Several workstations are running Windows 7, an operating system that has reached its end-of-life for mainstream support. This means these systems no longer receive critical security updates from Microsoft, leaving them vulnerable to newly discovered exploits, malware, and zero-day attacks. This significantly increases the risk of system compromise, data loss, and network infection.

Affected Elements:

  • Workstations running Windows 7

Lack of Employee Acceptable Use Policy - Medium

Overview: The organization currently lacks an employee acceptable use policy. Without formal guidelines, employees may unknowingly engage in activities that introduce security risks, such as visiting malicious websites, downloading unauthorized software, or mishandling sensitive data. This absence can lead to increased insider threats, accidental data breaches, malware infections, and potential non-compliance with regulatory requirements.

Affected Elements:

  • All employees
  • IT systems
  • Organizational data

Inadequate Security Awareness Training - Medium

Overview: While new employees receive security awareness training, the organization does not conduct annual training for all employees. This infrequent training schedule means employees may not be up-to-date on the latest cybersecurity threats, best practices, and organizational policies. This increases the susceptibility to social engineering attacks (e.g., phishing), malware infections, and human error leading to security incidents.

Affected Elements:

  • All employees
  • Organizational data
  • Network

Recommendations

  • To mitigate the identified risks, it is critically important to immediately address the outdated FTP service by either upgrading to a current, supported version or decommissioning it if no longer needed. If an FTP service is required, ensure anonymous access is disabled and strong authentication is enforced. Implement Multi-Factor Authentication (MFA) for all computer logins to significantly enhance workstation security against credential theft. All workstations running Windows 7 must be upgraded to a currently supported operating system, such as Windows 10 or 11, to ensure they receive essential security updates. Furthermore, develop and implement a comprehensive employee acceptable use policy to guide secure behavior and conduct mandatory, annual security awareness training for all employees to keep them informed about evolving threats and best practices.

Conclusion

The organization faces a significant array of cybersecurity risks, ranging from critical infrastructure vulnerabilities like an outdated and insecure FTP server and end-of-life operating systems, to fundamental weaknesses in authentication and employee security practices. While MFA is in place for email and sensitive data, the absence of MFA for computer logins, lack of an acceptable use policy, and infrequent security training create substantial attack vectors. The current readiness level is low, and immediate action is required to address these critical and high-severity vulnerabilities to prevent potential data breaches, system compromises, and operational disruptions.
