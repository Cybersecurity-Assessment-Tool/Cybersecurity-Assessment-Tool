Cybersecurity Assessment Report

Overview

The organization's network currently operates a web server running Nginx version 1.18.0 on port 443 (HTTPS). An assessment of organizational data reveals critical security gaps, including the absence of Multi-Factor Authentication (MFA) for email and computer logins, a lack of an employee acceptable use policy, and insufficient recurring security awareness training for all employees.

Identified Risks

Outdated Nginx Server - High

Overview: The Nginx web server identified is running version 1.18.0, which is an older release. Older software versions often contain known vulnerabilities that have been patched in newer releases. For example, Nginx 1.18.0 is susceptible to CVE-2021-23017 (resolver stack buffer overflow) and other potential security flaws. Running outdated software increases the attack surface and makes the server vulnerable to exploits that could lead to data breaches, denial of service, or unauthorized access. This was identified through the network scan data.

Affected Elements:

  • Nginx 1.18.0 web server
  • Port 443

Lack of MFA for Email Access - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for accessing email accounts. This significantly increases the risk of account compromise through phishing, brute-force attacks, or credential stuffing. If an attacker gains access to an employee's email, they could potentially pivot to other systems, impersonate the employee, or access sensitive information. This was identified from the organizational data.

Affected Elements:

  • Employee email accounts
  • Email system

Lack of MFA for Computer Login - High

Overview: Multi-Factor Authentication (MFA) is not required for employees to log into their computers. This leaves workstations vulnerable to unauthorized access if primary credentials (username/password) are compromised. An attacker gaining access to a computer could potentially access internal networks, sensitive data, or install malware. This was identified from the organizational data.

Affected Elements:

  • Employee workstations
  • Internal network access

Absence of Employee Acceptable Use Policy (AUP) - Medium

Overview: The organization lacks a formal Employee Acceptable Use Policy (AUP). Without an AUP, employees may be unaware of the proper and secure use of company resources, including IT systems, internet access, and data. This can lead to increased risks such as data leakage, malware infections from unauthorized software installations or website visits, and legal liabilities. This was identified from the organizational data.

Affected Elements:

  • All employees
  • Company IT resources
  • Data handling practices

Insufficient Recurring Security Awareness Training - Medium

Overview: While new employees receive security awareness training, the organization does not conduct annual or regular security awareness training for all employees. This means that employees may not be kept up-to-date on the latest threats, best practices, and company policies, leading to a degradation of security posture over time. Human error remains a leading cause of security incidents, and consistent training is crucial to mitigate this risk. This was identified from the organizational data.

Affected Elements:

  • All employees
  • Organizational security posture

Recommendations

  • To mitigate the identified risks, the organization should prioritize updating its Nginx web server to the latest stable version to patch known vulnerabilities. Implementing Multi-Factor Authentication (MFA) for all email accounts and computer logins is critical to significantly reduce the risk of unauthorized access and account compromise. Furthermore, the organization must develop and implement a comprehensive Employee Acceptable Use Policy (AUP) to clearly define expectations for secure behavior and resource usage. Finally, establishing a program for mandatory annual security awareness training for all employees will ensure that the workforce remains vigilant against evolving cyber threats and adheres to security best practices.

Conclusion

The organization currently faces several significant cybersecurity risks, primarily stemming from an outdated public-facing web server and critical gaps in its identity and access management policies, specifically the lack of Multi-Factor Authentication for email and computer logins. Additionally, the absence of an Acceptable Use Policy and insufficient recurring security awareness training expose the organization to increased risks from human error and misuse of resources. Addressing these vulnerabilities is crucial for enhancing the overall security posture and reducing the likelihood of a successful cyberattack.
