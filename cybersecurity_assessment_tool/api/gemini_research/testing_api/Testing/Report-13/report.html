Cybersecurity Assessment Report

Overview

The organization's network scan revealed an Nginx web server (version 1.18.0) running on port 443 (HTTPS). Organizational data indicates a lack of multi-factor authentication (MFA) for email and computer logins, although it is used for sensitive data systems. Furthermore, there is no employee acceptable use policy in place, and security awareness training is only conducted for new employees, not annually for all staff.

Identified Risks

Lack of Multi-Factor Authentication (MFA) for Email and Computer Logins - High

Overview: The organization does not enforce Multi-Factor Authentication (MFA) for accessing email or logging into computers. This significantly increases the risk of unauthorized access if credentials are compromised through phishing, brute-force attacks, or other means. Without MFA, a single compromised password can grant an attacker full access to an employee's email account or workstation, potentially leading to data breaches, lateral movement within the network, and further compromise. This was identified through organizational policy review.

Affected Elements:

  • Email systems
  • Employee workstations/laptops
  • User accounts

Outdated Nginx Web Server - High

Overview: The Nginx web server identified on port 443 is running version 1.18.0. This version is significantly outdated and is known to contain several security vulnerabilities that have been patched in newer releases. Running outdated software exposes the organization to known exploits, which attackers can leverage to gain unauthorized access, disrupt services, or compromise the server. For example, CVE-2021-23017 (resolver stack buffer overflow) affected versions up to 1.20.0. This was identified via network port scanning.

Affected Elements:

  • Web server (nginx 1.18.0)
  • Port 443

Absence of Employee Acceptable Use Policy (AUP) - Medium

Overview: The organization lacks a formal Employee Acceptable Use Policy. An AUP defines the proper and acceptable use of organizational IT resources, including computers, networks, and data. Without such a policy, employees may unknowingly engage in activities that pose security risks, such as installing unauthorized software, visiting malicious websites, or mishandling sensitive information. This absence creates ambiguity regarding expected behavior and makes it difficult to enforce security best practices. This was identified through organizational policy review.

Affected Elements:

  • All employees
  • Organizational IT resources
  • Data handling procedures

Insufficient Security Awareness Training - Medium

Overview: While new employees receive security awareness training, the organization does not conduct annual training for all employees. Cybersecurity threats evolve rapidly, and regular training is crucial to keep employees informed about new attack vectors (e.g., advanced phishing techniques, social engineering) and reinforce secure behaviors. Without ongoing training, employees are more susceptible to falling victim to attacks, which can lead to credential compromise, malware infections, and data breaches. This was identified through organizational policy review.

Affected Elements:

  • All employees
  • Organizational security posture
  • Phishing susceptibility

Recommendations

  • To significantly enhance the organization's security posture, it is strongly recommended to implement Multi-Factor Authentication (MFA) for all email accounts and computer logins immediately. Furthermore, the Nginx web server should be updated to the latest stable version to patch known vulnerabilities and ensure optimal security. The organization must develop and enforce a comprehensive Employee Acceptable Use Policy to guide employee behavior regarding IT resource usage. Finally, a program for mandatory annual security awareness training for all employees should be established to ensure continuous education on evolving cyber threats and best practices.

Conclusion

The current assessment reveals several critical and high-severity vulnerabilities that significantly impact the organization's cybersecurity readiness. Key areas of concern include the lack of comprehensive Multi-Factor Authentication, an outdated web server, and deficiencies in foundational security policies and ongoing employee training. Addressing these issues promptly through the recommended actions is crucial to mitigate the risk of unauthorized access, data breaches, and operational disruptions, thereby strengthening the overall security posture.
