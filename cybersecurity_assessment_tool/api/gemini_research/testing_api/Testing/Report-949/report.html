Cybersecurity Assessment Report

Overview

The organization operates a web server running Nginx version 1.18.0 on port 443 with HTTPS. Security controls include Multi-Factor Authentication (MFA) for email and computer logins, and an employee acceptable use policy is in place. The organization conducts annual security awareness training for all employees. However, MFA is not required for sensitive data systems, and security awareness training is not provided to new employees.

Identified Risks

Outdated Web Server Software - Critical

Overview: The Nginx web server (version 1.18.0) detected on port 443 is end-of-life (EOL) since May 2022. Running EOL software means it no longer receives security updates, leaving it vulnerable to known and newly discovered exploits. This significantly increases the risk of compromise, data breaches, and service disruption.

Affected Elements:

  • Nginx 1.18.0 on port 443

Inadequate Authentication for Sensitive Data - High

Overview: The organization does not require Multi-Factor Authentication (MFA) for access to sensitive data systems. This significantly increases the risk of unauthorized access to critical information if primary credentials (username/password) are compromised through phishing, brute-force attacks, or credential stuffing.

Affected Elements:

  • Sensitive data systems

Insufficient New Employee Security Training - Medium

Overview: The organization does not conduct security awareness training for new employees. This leaves new hires potentially unaware of critical security policies, common attack vectors (e.g., phishing, social engineering), and best practices from the start of their employment, increasing the likelihood of human error leading to security incidents.

Affected Elements:

  • New employees
  • Organizational security posture

Recommendations

  • It is strongly recommended to immediately upgrade the Nginx web server to a currently supported and stable version to mitigate critical vulnerabilities associated with end-of-life software. Furthermore, implement mandatory Multi-Factor Authentication (MFA) for all sensitive data systems to significantly enhance protection against unauthorized access. Finally, introduce comprehensive security awareness training as part of the onboarding process for all new employees to ensure they are equipped with essential security knowledge from day one, complementing the existing annual training program.

Conclusion

The organization currently faces critical security risks primarily due to an outdated web server and a significant gap in authentication for sensitive data systems. While some security measures like MFA for email and computer logins are in place, the absence of MFA for sensitive data and initial security training for new employees leaves the organization vulnerable. Addressing these identified vulnerabilities is crucial for improving the overall security posture and readiness against cyber threats.
