Cybersecurity Assessment Report

Overview

The organization's network currently exposes an Nginx web server (version 1.18.0) on port 443 (HTTPS). Internally, the organization lacks multi-factor authentication (MFA) for critical access points including email, computer logins, and sensitive data systems. While an acceptable use policy is in place and annual security awareness training is conducted for all employees, there is no specific security awareness training for new employees.

Identified Risks

Lack of Multi-Factor Authentication (MFA) - Critical

Overview: The absence of MFA across email, computer login, and sensitive data systems significantly increases the risk of unauthorized access. If an attacker compromises a single set of credentials, they gain full access to these critical resources without an additional layer of security. This vulnerability was identified through organizational policy review.

Affected Elements:

  • Email access
  • Computer login
  • Sensitive data systems

Outdated Nginx Web Server - High

Overview: The Nginx web server running on version 1.18.0 is not the latest stable release. Older software versions often contain known vulnerabilities that have been patched in newer releases. Running an outdated version exposes the organization to potential exploits that target these known weaknesses, which could lead to data breaches, denial of service, or unauthorized access. This was identified via network scan data.

Affected Elements:

  • Nginx 1.18.0 web server (port 443)

No Security Awareness Training for New Employees - Medium

Overview: While annual security awareness training is provided, the lack of dedicated training for new employees means that individuals joining the organization may not be immediately aware of current security policies, best practices, and common threats (e.g., phishing, social engineering). This oversight can lead to new employees inadvertently introducing vulnerabilities or falling victim to attacks during their initial employment period. This was identified through organizational policy review.

Affected Elements:

  • New employees
  • Overall organizational security posture

Recommendations

  • To significantly enhance the organization's security posture, immediate implementation of multi-factor authentication (MFA) is crucial for all email accounts, computer logins, and access to sensitive data systems. This will provide a vital additional layer of security against credential compromise. Furthermore, the Nginx web server should be updated to the latest stable version to patch known vulnerabilities and ensure ongoing security. Finally, a comprehensive security awareness training program specifically tailored for new employees should be established to ensure all personnel are equipped with the necessary knowledge to identify and mitigate cybersecurity risks from day one.

Conclusion

The organization currently faces significant cybersecurity risks primarily due to the absence of multi-factor authentication across critical systems, the use of outdated web server software, and a gap in security awareness training for new hires. While an acceptable use policy and annual training are positive steps, these vulnerabilities leave the organization susceptible to unauthorized access and potential data breaches. Addressing these issues promptly is essential to improve the overall security readiness and protect organizational assets.
