Executive Summary

This report provides a comprehensive assessment of Falcon Heavy’s current cybersecurity posture, synthesizing findings from a network scan, organizational security questionnaire, and pre-existing risk data. The analysis reveals a mixed security landscape with commendable practices in some areas, such as maintaining an acceptable use policy and conducting annual security awareness training. However, critical gaps exist concerning Multi-Factor Authentication (MFA) implementation for computer logins and sensitive data access, as well as a lack of security awareness training for new employees. Furthermore, a critical database service (MySQL 5.7.33) is openly exposed, posing a significant risk of unauthorized access and data compromise. Addressing these identified risks and implementing the recommended controls is crucial to enhancing the organization’s overall security resilience.

Recommendations

To mitigate the identified risks and improve Falcon Heavy’s cybersecurity posture, the following recommendations are provided:

  1. Database Exposure Mitigation:

    • Immediate Fix: Implement strict firewall rules to restrict access to port 3306 (MySQL) only from necessary internal IP addresses or trusted applications. Access from the general network or external sources should be blocked.

    • Long-Term Fix: Upgrade MySQL to version 8.x, which is actively supported and receives security updates. If external access is required, implement a VPN or bastion host to secure the connection, rather than direct exposure. Regularly patch and update all database software.

  2. Implement MFA for Computer Logins:

    • Mandate and enforce Multi-Factor Authentication (MFA) for all computer logins across the organization. This significantly reduces the risk of unauthorized access even if primary credentials are compromised.

  3. Implement MFA for Sensitive Data Systems:

    • Require Multi-Factor Authentication (MFA) for accessing all systems that store, process, or transmit sensitive data. This is a critical control for protecting confidential information.

  4. Develop New Employee Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new employees as part of their onboarding process. This training should cover topics such as phishing, social engineering, password best practices, and data handling policies.
