Executive Summary

This report provides a comprehensive cybersecurity assessment for Obsidian Operatives, synthesizing findings from network scans, organizational security posture questionnaires, and pre-existing risk documentation. The overall security posture indicates a mixed state with commendable strengths in Multi-Factor Authentication (MFA) implementation across critical systems, yet significant gaps exist in foundational security controls and specific technical vulnerabilities.

Key strengths include the widespread adoption of MFA for email, computer logins, and sensitive data access, which significantly enhances protection against unauthorized access. However, critical deficiencies were identified, including the absence of an Employee Acceptable Use Policy (AUP) and a lack of security awareness training for new employees. These gaps present substantial risks related to insider threats, compliance, and human error. Technically, a critical risk (CVSS 10.0) associated with "Localhost Exposed" was identified, requiring urgent investigation and remediation, especially considering an open SSH port (22) on the internal target 127.0.0.1.

Urgent recommendations include developing and implementing an AUP, establishing mandatory security awareness training for all new hires, and immediately addressing the "Localhost Exposed" vulnerability by hardening SSH and restricting access.

Recommendations

Based on the assessment findings, the following recommendations are provided to enhance the cybersecurity posture of Obsidian Operatives:

Governance and Policy

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Action: Create a comprehensive AUP that clearly outlines permissible and prohibited activities, data handling guidelines, and the responsible use of company assets (hardware, software, network, internet).

    • Benefit: Reduces the risk of misuse, improves compliance, and sets clear expectations for employee conduct.

    • Priority: High

Security Awareness and Training

  • Mandate Security Awareness Training for New Employees:

    • Action: Establish a mandatory security awareness training program as part of the onboarding process for all new hires. This should cover common threats (phishing, malware), data handling, password best practices, and reporting security incidents.

    • Benefit: Reduces the "human factor" risk by equipping new employees with essential security knowledge from day one.

    • Priority: High

  • Continue Annual Security Awareness Training:

    • Action: Continue the existing annual security awareness training for all employees to reinforce best practices and update on new threats.

    • Benefit: Maintains a vigilant security culture and keeps employees informed.

    • Priority: Ongoing

Technical Security

  • Investigate and Remediate "Localhost Exposed" Vulnerability (Port 22):

    • Action:

      1. Conduct Root Cause Analysis: Determine precisely how the localhost service (SSH on port 22) is exposed or accessible in a critical manner (e.g., exposed to other internal subnets, misconfigured firewall rules, or accessible via specific routes).

      2. Harden SSH Configuration: Ensure SSH is configured with strong security practices, including:

        • Disabling root login.

        • Using key-based authentication instead of passwords (or strong passwords with MFA).

        • Restricting user access.

        • Limiting listening interfaces if possible (e.g., ListenAddress 127.0.0.1 if it shouldn’t be exposed).

      3. Restrict Network Access: Implement strict firewall rules to limit access to port 22 on 127.0.0.1 only from necessary and authorized internal systems. If it’s truly only meant for local access, ensure it cannot be reached from other hosts.

      4. Patching and Updates: Verify that the SSH server and underlying operating system are fully patched and up-to-date to mitigate known vulnerabilities.

    • Benefit: Eliminates a critical exposure risk, preventing potential unauthorized access, data breaches, or further lateral movement within the network.

    • Priority: Critical / Urgent
