Executive Summary

This report details the cybersecurity posture of Nebula Creative based on a review of organizational data, existing risk assessments, and a targeted network scan. Key findings indicate significant vulnerabilities related to outdated software, critical gaps in multi-factor authentication (MFA) implementation across various access points, and inadequate recurring security awareness training. The most critical risks involve an outdated FTP server with anonymous access, the widespread lack of MFA, and the continued use of end-of-life operating systems like Windows 7. Immediate and strategic recommendations are provided to mitigate these identified risks and enhance the overall security posture.

Technical Scan Results

A targeted network scan was performed on 10.0.0.15. The scan identified the following open port and associated services:

  • Target IP: 10.0.0.15

  • Port 21/TCP (FTP):

    • Service: FTP

    • Product: vsftpd

    • Version: 2.3.4

    • Details: Anonymous FTP login allowed.

    • Analysis: vsftpd version 2.3.4 is significantly outdated (released in 2011) and contains known vulnerabilities, including CVE-2011-2523 (backdoor command execution). The allowance of anonymous FTP logins further exacerbates this risk by enabling unauthenticated access and potential data exfiltration or unauthorized file modification.

Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture of Nebula Creative, the following recommendations are provided:

  1. Upgrade and Secure FTP Server:

    • Immediately upgrade vsftpd to the latest stable version or replace it with a more secure file transfer protocol (e.g., SFTP, SCP) if FTP functionality is still required.

    • Disable anonymous FTP access entirely. If external file sharing is necessary, implement secure, authenticated methods.

    • Ensure strong access controls and monitoring are in place for any file transfer services.

  2. Implement Multi-Factor Authentication (MFA) Universally:

    • Mandate MFA for all email accounts to protect against credential stuffing and phishing attacks.

    • Implement MFA for all computer logins, especially for administrative accounts and systems processing sensitive data.

    • Extend MFA requirements to all sensitive data systems and applications.

  3. Enhance Security Awareness Training Program:

    • Establish and enforce a mandatory annual security awareness training program for all employees, not just new hires.

    • Include current threat landscapes, phishing simulations, and best practices for data handling in the training curriculum.

  4. Migrate from End-of-Life Operating Systems:

    • Prioritize and execute the upgrade of all Windows 7 machines to a currently supported operating system (e.g., Windows 10 or 11).

    • Develop a hardware refresh strategy if existing hardware cannot support newer OS versions.

    • Implement robust patch management for all active operating systems and applications.

Conclusion

Nebula Creative faces several critical and high-severity cybersecurity risks, primarily stemming from outdated software, a lack of comprehensive MFA implementation, and the use of end-of-life operating systems. Addressing these vulnerabilities proactively will significantly enhance the organization’s defense against cyber threats, protect sensitive data, and improve overall operational resilience.
