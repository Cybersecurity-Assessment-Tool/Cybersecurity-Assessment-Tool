Overview

This report provides a comprehensive assessment of Oasis Wellness’s cybersecurity posture, integrating findings from a network scan, organizational security questionnaire responses, and pre-existing risk data. The analysis reveals a mix of strong security practices and critical vulnerabilities that require immediate attention.

Key findings include:

  • Positive Controls: The organization effectively utilizes Multi-Factor Authentication (MFA) for email and sensitive data systems, and maintains an acceptable use policy. Annual security awareness training for all employees is also in place.

  • Critical Gaps: Significant vulnerabilities were identified, including the use of End-of-Life (EOL) software (MySQL 5.7.33), lack of MFA for computer logins, and insufficient security awareness training for new employees.

  • Network Exposure: A critical database service (MySQL) is exposed on port 3306, further escalating the risk associated with the EOL software.

The recommendations section outlines actionable steps to mitigate identified risks and enhance the overall security posture of Oasis Wellness.

Technical Analysis: The most significant technical finding is the active MySQL database service running version 5.7.33 on port 3306. MySQL 5.7 reached its End-of-Life (EOL) in October 2023. This means that the software no longer receives security updates, critical bug fixes, or technical support, leaving it highly vulnerable to known and future exploits. The fact that this port is open to the network further exacerbates the risk, making the database a prime target for attackers.

Recommendations

To mitigate the identified risks and enhance Oasis Wellness’s cybersecurity posture, the following recommendations are provided. These are prioritized based on severity and potential impact.

High Priority Recommendations

  1. Upgrade/Migrate End-of-Life (EOL) MySQL Server:

    • Action: Immediately plan and execute an upgrade of the MySQL server from version 5.7.33 to a currently supported and patched version (e.g., MySQL 8.0). Alternatively, consider migrating the database to a modern, secure, and fully supported database solution.

    • Rationale: EOL software is a critical vulnerability due to the absence of security updates, making it a prime target for attackers.

  2. Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Action: Mandate MFA for all user accounts logging into organizational computers (workstations and servers). Evaluate and deploy suitable MFA solutions (e.g., hardware tokens, authenticator apps, biometrics).

    • Rationale: MFA significantly reduces the risk of unauthorized access even if primary credentials are stolen or guessed.

  3. Establish Comprehensive Onboarding Security Awareness Training:

    • Action: Develop and implement mandatory security awareness training for all new employees during their onboarding process. This training should cover topics such as phishing, social engineering, password hygiene, data handling policies, and the organization’s acceptable use policy.

    • Rationale: New employees are a common entry point for attacks if not adequately trained on security best practices from day one.

  4. Restrict Database Access with Firewall Rules:

    • Action: Implement strict firewall rules to restrict access to port 3306 (MySQL). Only allow connections from authorized internal hosts or applications that explicitly require database access. Block all external access.

    • Rationale: Limiting network exposure of critical services reduces the attack surface and minimizes the risk of exploitation.

Long-Term and Sustained Security Recommendations

  1. Enhance Existing Security Awareness Program:

    • Action: Regularly review and update the annual security awareness training to address new threats and vulnerabilities. Consider incorporating regular phishing simulation exercises to test employee vigilance.

    • Rationale: Continuous training reinforces security best practices and helps employees recognize evolving threats.

  2. Implement VPN for Remote Database Access:

    • Action: For any legitimate remote access requirements to the database, enforce the use of a Virtual Private Network (VPN) to ensure encrypted and authenticated connections, rather than direct exposure.

    • Rationale: VPNs provide a secure tunnel for remote connections, protecting data in transit and authenticating users before granting network access.

  3. Regular Vulnerability Scanning and Penetration Testing:

    • Action: Conduct regular vulnerability scans and periodic penetration tests to proactively identify and address security weaknesses across the IT infrastructure.

    • Rationale: Proactive testing helps discover vulnerabilities before malicious actors can exploit them.
