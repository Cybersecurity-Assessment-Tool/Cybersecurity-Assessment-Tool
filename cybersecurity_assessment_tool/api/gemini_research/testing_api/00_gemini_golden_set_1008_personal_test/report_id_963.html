Executive Summary

This report details the cybersecurity posture of Calyx Botany, based on a review of organizational data, security questionnaire responses, and a targeted network scan. Our analysis reveals critical gaps in multi-factor authentication (MFA) implementation and the presence of an outdated, exposed database server. These findings present significant risks related to unauthorized access, data breaches, and potential compliance failures. Immediate recommendations focus on enhancing identity and access management, patching critical systems, and implementing robust network segmentation.

Analysis of Security Controls

The organization demonstrates good practice in security awareness training and policy development. However, the lack of Multi-Factor Authentication (MFA) for email access and computer logins presents a severe vulnerability. Without MFA, compromised credentials can lead to unauthorized access to user accounts, internal systems, and potential lateral movement within the network. While MFA is used for sensitive data systems, the initial access points (email and general computer login) remain unprotected by this crucial control.

Analysis of Technical Findings

The presence of an open MySQL database port (3306) running version 5.7.33 is a critical concern. MySQL 5.7 reached its End-of-Life (EOL) in October 2023. This means that Oracle no longer provides public security updates, bug fixes, or technical support for this version. Any vulnerabilities discovered after this date will remain unpatched, exposing the database to known and future exploits. Coupled with the port being open to the network, this presents a significant risk for data exposure, unauthorized access, and potential compromise of the database contents.

Recommendations

Based on the identified risks, the following recommendations are provided to enhance the cybersecurity posture of Calyx Botany:

Immediate Actions

  • Implement MFA for Email Access: Immediately enforce Multi-Factor Authentication for all email accounts. This is a critical control against credential theft and phishing.

  • Implement MFA for Computer Logins: Deploy and enforce Multi-Factor Authentication for all computer logins, especially for administrative accounts and systems holding sensitive data.

  • Isolate and Patch/Upgrade Database Server:

    • **Firewall Rule:** Immediately restrict access to port 3306 (MySQL) to only trusted internal IP addresses or specific applications that require connectivity. Block all external access.

    • **Upgrade/Migrate:** Plan and execute an upgrade of the MySQL database server from version 5.7.33 to a currently supported, stable version (e.g., MySQL 8.0). Prioritize this upgrade as 5.7 is EOL.

    • **Network Segmentation:** Implement network segmentation to isolate the database server within a dedicated subnet, limiting its network exposure.

Long-Term Strategy

  • Regular Vulnerability Scanning: Implement a schedule for regular internal and external vulnerability scanning to proactively identify and address security weaknesses.

  • Patch Management Program: Establish a comprehensive patch management program for all operating systems, applications, and network devices to ensure all software is kept up-to-date.

  • Security Awareness Program Enhancement: While good, continuously update and expand security awareness training to include advanced phishing techniques, social engineering, and the importance of MFA.

  • Incident Response Plan: Develop or review an incident response plan to ensure the organization can effectively detect, respond to, and recover from security incidents.
