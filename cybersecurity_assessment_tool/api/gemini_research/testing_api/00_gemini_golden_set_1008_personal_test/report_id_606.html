Executive Summary

This report provides a comprehensive cybersecurity assessment for True North Travel, based on an organizational security questionnaire, external network scan data, and pre-existing risk information. The assessment reveals several critical areas for improvement, primarily concerning the widespread absence of Multi-Factor Authentication (MFA) across email, computer logins, and sensitive data systems, coupled with a lack of an Employee Acceptable Use Policy. While the recent network scan did not identify any open ports on the target IP, indicating a good security posture from an external exposure perspective, the identified policy and authentication gaps present significant internal and external risks that require immediate attention. The organization demonstrates strength in regular security awareness training. Implementing the recommended controls will significantly enhance the organization’s overall security posture.

5. Recommendations

To address the identified risks and improve the overall cybersecurity posture of True North Travel, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) Systematically:

    • Email Access: Immediately deploy and enforce MFA for all employee email accounts. This is a critical first step to protect against phishing and credential theft.

    • Computer Logins: Implement MFA for all computer logins within the organization, particularly for systems accessing sensitive data or internal networks.

    • Sensitive Data Systems: Extend MFA requirements to all systems that store, process, or access sensitive organizational data.

  2. Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining appropriate use of company IT resources, internet, email, and data handling procedures.

    • Ensure all employees read, understand, and formally acknowledge the AUP, ideally during onboarding and annually thereafter.

    • Integrate AUP review and related best practices into existing security awareness training programs.

  3. Review Network Configurations and Prioritize Secure Defaults:

    • Investigate the historical risk of an "Unencrypted Web Server" on Port 80. While the current scan shows it closed, understand why it was previously identified as a risk. Ensure that if any web services are needed, they are configured with HTTPS/TLS encryption.

    • Conduct regular internal and external network scans to ensure no unintended services are exposed or misconfigured.

    • Implement a robust patch management program to ensure all systems and software are kept up-to-date, reducing the risk of vulnerabilities.
