Overview

This document presents the findings of a cybersecurity assessment conducted for Ember Glow Hospitality. The assessment involved a comprehensive review of organizational security controls through a structured questionnaire, a targeted technical network scan, and an analysis of pre-existing risk data. The objective is to provide a holistic view of the current security posture and offer actionable recommendations for enhancement.

The analysis identified critical gaps in foundational security controls, specifically concerning multi-factor authentication for sensitive systems, the absence of key security policies, and insufficient new employee security awareness training. While the technical scan on the specified target revealed no immediately exploitable open ports, a significant discrepancy was noted with a pre-existing risk related to an unencrypted web server. These findings collectively indicate several areas requiring immediate attention to significantly enhance the overall security posture of Ember Glow Hospitality.

Analysis of Questionnaire Gaps

The questionnaire revealed several significant security control deficiencies that pose immediate risks:

  • MFA for Sensitive Data Systems: The absence of Multi-Factor Authentication (MFA) for accessing sensitive data systems is a critical vulnerability. This significantly increases the risk of unauthorized access to critical information in the event of compromised credentials (e.g., via phishing).

  • Employee Acceptable Use Policy (AUP): The lack of an AUP means there are no formal, documented guidelines for employees regarding the proper and secure use of organizational IT assets and resources. This can lead to misuse, increased risk of data breaches, and potential non-compliance with industry regulations.

  • New Employee Security Awareness Training: New employees do not receive dedicated security awareness training. This leaves new hires vulnerable to common social engineering attacks (e.g., phishing, pretexting) and unaware of organizational security policies from the outset, posing an immediate and preventable risk.

Recommendations

Based on the assessment, the following actionable recommendations are provided to enhance Ember Glow Hospitality’s cybersecurity posture, categorized by area of impact.

5.1. Identity and Access Management

  • Implement MFA for Sensitive Data Systems: Immediately deploy and enforce Multi-Factor Authentication for all systems that contain or provide access to sensitive organizational data. Prioritize systems based on data classification and potential impact of compromise. This is a critical control to prevent unauthorized access even if primary credentials are breached.

5.2. Policy and Training

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines employee responsibilities for using company IT resources, data handling, internet usage, and communication. Ensure all employees formally review and acknowledge the policy annually, and integrate it into onboarding.

  • Introduce Mandatory New Employee Security Awareness Training: Establish mandatory security awareness training for all new hires during their onboarding process. This training should cover common cybersecurity threats (e.g., phishing, malware), organizational security policies, best practices for secure computing, and incident reporting procedures.

  • Review and Enhance Annual Security Awareness Training: While annual training is currently conducted, review its content to ensure it is current, engaging, and effectively covers relevant emerging threats, recent policy updates, and organization-specific security challenges. Consider varied formats and regular refreshers.

5.3. Technical Controls & Vulnerability Management

  • Reconcile Discrepancy for Unencrypted Web Server Risk:

    • Conduct an immediate investigation to understand why the pre-existing risk states “Port 80 is open” when the recent scan shows it as closed on 192.168.0.5.

    • If the risk pertains to a different asset, update the risk register to correctly associate the risk with the relevant asset(s).

    • If port 80 on 192.168.0.5 was recently mitigated (e.g., closed), ensure the risk register is updated to reflect its current status and remediation.

    • If port 80 is intended to be open for specific purposes, ensure that all traffic is securely redirected to HTTPS (Port 443) with a valid, trusted SSL/TLS certificate, enforcing encrypted communication.

  • Implement Regular Network Vulnerability Scanning: Establish a schedule for regular, automated network vulnerability scanning (e.g., monthly or quarterly) to proactively identify newly exposed services, insecure configurations, outdated software versions, and other technical weaknesses across the IT infrastructure.
