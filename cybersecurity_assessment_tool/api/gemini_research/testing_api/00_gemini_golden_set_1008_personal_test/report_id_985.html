Executive Summary

This report details the findings of a cybersecurity posture assessment for Ember Glow Hospitality, conducted on 2025-11-22. The assessment involved a review of organizational security controls via a questionnaire and a technical network scan of an internal host.

Overall, Ember Glow Hospitality demonstrates several critical security gaps that significantly elevate its risk profile. Key issues include a lack of multi-factor authentication (MFA) for critical systems like email and sensitive data access, the absence of fundamental security policies (e.g., Acceptable Use Policy), and a complete lack of security awareness training for employees. Technically, an exposed Nginx web server was identified running an End-of-Life (EOL) version (1.18.0), posing a significant vulnerability risk.

Immediate action is recommended to implement foundational security controls, address software vulnerabilities, and establish a robust security awareness program. Failing to address these identified risks could lead to data breaches, operational disruptions, and reputational damage.

Key Findings from Security Control Review

The review reveals significant gaps in fundamental security practices:

  • Multi-Factor Authentication (MFA) Deficiencies: MFA is not uniformly applied across critical access points, specifically for email and sensitive data systems. This significantly increases the risk of account compromise through stolen credentials.

  • Lack of Security Policies: The absence of an Employee Acceptable Use Policy leaves the organization vulnerable to employee misconduct, misuse of resources, and unclear guidelines regarding security responsibilities.

  • Absence of Security Awareness Training: Critical for mitigating human-factor risks, the lack of both initial and recurrent security awareness training exposes the organization to social engineering attacks, phishing, and general lack of security hygiene among employees.

Recommendations

To address the identified risks and improve the overall cybersecurity posture of Ember Glow Hospitality, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) System-Wide:

    • Immediately enable and enforce MFA for all email accounts.

    • Implement MFA for access to all sensitive data systems and critical business applications.

    • Evaluate existing MFA for computer logins for robustness and coverage.

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Draft a comprehensive AUP that clearly outlines expected employee behavior regarding company IT resources, internet usage, and data handling.

    • Ensure all employees review and acknowledge the AUP, preferably as part of onboarding and annually thereafter.

  3. Establish a Comprehensive Security Awareness Training Program:

    • Introduce mandatory security awareness training for all new hires during their onboarding process.

    • Implement annual mandatory security awareness training for all employees, covering topics such as phishing, social engineering, password hygiene, and data protection.

    • Conduct regular simulated phishing exercises to gauge employee susceptibility and reinforce training.

  4. Upgrade and Secure Web Servers:

    • Immediately upgrade the Nginx web server (192.168.10.5) from version 1.18.0 to a currently supported and patched stable version (e.g., 1.24.x or 1.26.x depending on current releases).

    • Implement a regular patch management schedule for all publicly exposed services and internal infrastructure.

    • Review the SSL certificate configuration for the Nginx server to ensure the common name (www.acme-corp.com) is appropriate for the service provided by Ember Glow Hospitality.

  5. Continuous Monitoring and Vulnerability Management:

    • Implement a vulnerability management program to regularly scan systems, identify new vulnerabilities, and track remediation efforts.

    • Establish a process for logging and monitoring security events to detect and respond to threats in a timely manner.
