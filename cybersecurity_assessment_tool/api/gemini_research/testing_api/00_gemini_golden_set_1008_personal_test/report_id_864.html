3.1. Analysis of Questionnaire Findings

The questionnaire responses highlight the following areas of concern:

  • Lack of MFA for Email Access: Not requiring MFA for email access is a critical vulnerability. Email accounts are frequently targeted in phishing attacks and, if compromised, can provide attackers with access to sensitive information, impersonation capabilities, and a foothold into the network.

  • Lack of MFA for Computer Login: Similar to email, not enforcing MFA for computer logins significantly increases the risk of unauthorized access to endpoints. Credential stuffing, brute-force attacks, and stolen credentials become far more effective without MFA.

  • Inconsistent Security Awareness Training: While new employees receive training, the lack of annual security awareness training for all employees means that knowledge can become outdated, new threats may not be understood, and good security practices might degrade over time. This leaves the organization vulnerable to social engineering and phishing attacks.

6. Recommendations

Based on the analysis of available data, the following recommendations are provided to improve Vivid Vision’s cybersecurity posture:

  • Implement MFA for Email Access: Immediately implement and enforce Multi-Factor Authentication (MFA) for all email accounts. This is a fundamental security control that drastically reduces the risk of email account compromise.

  • Implement MFA for Computer Logins: Deploy and enforce MFA for all computer logins across the organization. This should apply to both local machine access and network/domain logins to protect endpoints.

  • Establish Annual Security Awareness Training: Develop and implement a mandatory annual security awareness training program for all employees. This training should cover current threats, phishing recognition, data handling best practices, and organizational security policies.

  • Resolve Data Input Issues: Address the root cause of the malformed JSON data for network scans and existing vulnerabilities. Ensuring accurate and complete data inputs is crucial for future comprehensive security assessments.

  • Conduct a Full Technical Vulnerability Assessment: Once data input issues are resolved, perform a comprehensive external and internal vulnerability scan and penetration test to identify and remediate technical weaknesses.

  • Review and Update Existing Risk Register: With accurate data, ensure the organization maintains an up-to-date risk register that includes all identified vulnerabilities, their severity, and corresponding mitigation plans.
