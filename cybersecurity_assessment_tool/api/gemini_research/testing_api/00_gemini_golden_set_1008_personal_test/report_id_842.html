Recommendations

To mitigate the identified risks and strengthen the overall security posture, the following actionable recommendations are provided:

  1. Immediately Address Anonymous FTP Access:

    • Disable anonymous FTP access on without delay.

    • If the FTP service is not essential, consider completely removing or disabling it.

    • If FTP functionality is required, migrate to a more secure file transfer protocol (e.g., SFTP, SCP) or update to a current, supported, and patched version, configuring it securely with strong authentication.

  2. Implement Multi-Factor Authentication (MFA):

    • Deploy MFA for all critical access points, including but not limited to email accounts, computer logins, and all sensitive data systems. This significantly reduces the risk of credential compromise.

  3. Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly defines the acceptable and prohibited use of organizational IT resources, data, and internet access.

    • Ensure all employees are trained on the AUP and acknowledge their understanding and compliance.

  4. Upgrade End-of-Life Operating Systems:

    • Plan and execute an upgrade path for all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10/11) to ensure continued security updates and vulnerability patching.

  5. Conduct Regular Security Awareness Training:

    • While new and annual training is conducted, reinforce the importance of secure practices, especially concerning phishing, password hygiene, and data handling, given the lack of MFA and AUP.

  6. Regular Security Audits and Vulnerability Management:

    • Establish a routine schedule for network vulnerability scanning and penetration testing to proactively identify and remediate security weaknesses before they can be exploited.
