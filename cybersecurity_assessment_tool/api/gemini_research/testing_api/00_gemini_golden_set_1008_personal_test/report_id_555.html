Executive Summary

This report details the findings of a cybersecurity posture assessment for Blue Marble. The assessment encompassed a review of organizational security controls via a questionnaire, a technical network scan, and an analysis of pre-existing risk documentation.

Overall, Blue Marble demonstrates some foundational security practices, such as requiring Multi-Factor Authentication (MFA) for email and sensitive data systems, and having an acceptable use policy. However, critical gaps were identified, particularly concerning the lack of MFA for computer logins and insufficient security awareness training programs for both new and existing employees. The technical scan confirmed the exposure of a MySQL database service (port 3306) to the network, which aligns with a documented pre-existing risk. This exposure represents a significant vulnerability that could lead to unauthorized data access or system compromise if not adequately protected.

Key areas requiring immediate attention include the implementation of MFA for computer logins, establishing comprehensive security awareness training, and securing the exposed MySQL database. Adhering to the recommendations outlined in this report will significantly enhance the organization’s security posture and reduce its overall cyber risk.

Analysis of Technical Findings

The scan revealed an open MySQL database service (port 3306) running version 5.7.33. While MySQL 5.7.33 is not End-of-Life (EOL), exposing a database port directly to the network without proper segmentation or access controls is a significant security risk. This aligns directly with a pre-existing risk documented by the organization. Unauthorized access to this port could lead to data exfiltration, tampering, or service disruption.

Recommendations

Based on the assessment, the following recommendations are provided to enhance the security posture of Blue Marble:

Immediate Actions (High Priority)

  1. Implement MFA for Computer Logins:

    • Action: Deploy and enforce Multi-Factor Authentication (MFA) for all user accounts accessing organizational computers and internal systems.

    • Impact: Significantly reduces the risk of unauthorized access through compromised credentials.

  2. Secure Exposed MySQL Database (Port 3306):

    • Action: Implement strict firewall rules to restrict access to port 3306 (MySQL) to only trusted internal IP addresses or specific application servers.

    • Action: Consider placing the database server in a separate, isolated network segment or requiring VPN access for any remote management.

    • Action: Ensure the MySQL service is configured with strong authentication, least privilege access, and regular patching.

    • Impact: Prevents unauthorized external access to sensitive database information.

Strategic Actions (Medium Priority)

  1. Establish Comprehensive Security Awareness Training:

    • Action (New Employees): Implement mandatory security awareness training as part of the onboarding process for all new hires. This should cover common threats like phishing, social engineering, and safe data handling practices.

    • Action (Annual Training): Establish and enforce a program for annual security awareness training for all employees. This training should be updated regularly to reflect current threat landscapes and organizational policies.

    • Impact: Empowers employees to recognize and report threats, reducing the human element as a common attack vector.

Conclusion

This report highlights several critical areas for improvement in Blue Marble’s cybersecurity posture. By addressing the identified gaps and implementing the recommended actions, especially regarding MFA for computer logins and securing the exposed database, the organization can significantly strengthen its defenses against potential cyber threats and improve its overall resilience. Continuous monitoring, regular security assessments, and ongoing employee training are crucial for maintaining a robust security posture in the long term.
