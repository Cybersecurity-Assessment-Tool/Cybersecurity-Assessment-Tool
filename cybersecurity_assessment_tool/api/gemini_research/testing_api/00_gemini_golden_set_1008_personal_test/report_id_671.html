Executive Summary

This report details the cybersecurity posture of Hearth & Home based on a comprehensive review of organizational data, a technical network scan, and an assessment of pre-existing risks. The organization demonstrates commendable adherence to Multi-Factor Authentication (MFA) best practices across email, computer logins, and sensitive data systems. However, critical gaps were identified in employee policies, specifically the absence of an Acceptable Use Policy (AUP), and in security awareness training for new employees. A significant technical finding is the exposure of localhost (127.0.0.1) with an open SSH port, which directly correlates with a pre-existing high-severity risk. Addressing these identified areas is paramount for enhancing the overall security posture and significantly reducing the organization’s attack surface against potential threats.

Recommendations

To mitigate the identified risks and strengthen the organization’s cybersecurity posture, the following actionable recommendations are provided:

  1. Address Localhost Exposure (127.0.0.1 - Port 22 SSH):

    • Review the necessity of having SSH (Port 22) open on the localhost interface. If not strictly required for administrative or specific application functions, disable the service immediately.

    • If SSH access is essential, implement stringent security controls:

      • Restrict access to specific trusted IP addresses or internal networks.

      • Disable password authentication; enforce key-based authentication only.

      • Ensure strong, unique SSH keys are used and properly secured.

      • Implement a robust logging and monitoring solution for all SSH access attempts.

      • Regularly update the SSH daemon to patch known vulnerabilities.

  2. Develop and Implement a Comprehensive Acceptable Use Policy (AUP):

    • Draft and finalize a clear, concise AUP that outlines acceptable and unacceptable use of all company IT resources (networks, devices, software, data, internet).

    • The AUP should explicitly cover data handling, privacy expectations, intellectual property, social media usage, and consequences for non-compliance.

    • Ensure all employees read, understand, and formally acknowledge the AUP as part of their onboarding and on an annual basis.

  3. Establish Mandatory Security Awareness Training for New Employees:

    • Integrate a foundational security awareness training program into the onboarding process for every new employee.

    • This training should cover essential topics such as identifying phishing attempts, safe browsing habits, strong password practices, data classification, and incident reporting procedures.

    • Emphasize the importance of the AUP and other organizational security policies from the outset.

  4. Review and Enhance Existing Annual Security Awareness Training:

    • While annual training is conducted, review its content to ensure it is up-to-date with current threat landscapes and relevant to the organization’s specific risks.

    • Incorporate interactive elements, such as simulated phishing exercises, to improve engagement and test employee vigilance.

    • Tailor training modules to different employee roles (e.g., IT, finance, general staff) where specific risks apply.
