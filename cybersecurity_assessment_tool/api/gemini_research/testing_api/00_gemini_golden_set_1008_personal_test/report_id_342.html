Executive Summary

This report provides a comprehensive cybersecurity posture assessment for , synthesizing information from a network scan, organizational questionnaire responses, and pre-existing risk data.

The organization demonstrates some foundational security practices, such as an acceptable use policy and annual security awareness training for all employees. However, significant vulnerabilities were identified. Additionally, new employees do not receive security awareness training, leaving them susceptible to initial attacks.

Technically, a recent network scan on the target identified no open ports, which is a positive indicator for direct network exposure on that specific host. This finding, however, creates a discrepancy with a pre-existing risk concerning an "Unencrypted Web Server" on port 80, which the scan shows as closed. This contradiction requires immediate investigation.

Overall, ’s security posture is fair, but several critical gaps, particularly around MFA and initial employee training, expose the organization to significant cyber threats that require urgent attention and remediation.

Recommendations

Based on the assessment, the following recommendations are provided to improve ’s cybersecurity posture:

High Priority Recommendations

  1. Implement Multi-Factor Authentication (MFA):

    • Immediately implement and enforce MFA for all email access.

    • Implement MFA for all user logins to company computers and endpoints.

    • Deploy MFA for access to all sensitive data systems and applications.

  2. Establish New Employee Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new employees, to be completed prior to receiving access to organizational systems.

    • Cover topics such as phishing, social engineering, password hygiene, and data handling best practices.

Medium Priority Recommendations

  1. Investigate and Remediate "Unencrypted Web Server" Discrepancy:

    • Conduct an immediate investigation to clarify the "Unencrypted Web Server" risk. Determine if the risk refers to an asset other than or if the current scan data is incomplete/outdated.

    • If an unencrypted web server on Port 80 is confirmed on any asset, prioritize securing it by implementing SSL/TLS encryption, migrating to HTTPS (Port 443), and/or closing Port 80 if the service is not strictly required.

    • Update the internal risk register to reflect current findings accurately.

General Best Practices

  • Regularly review and update the employee acceptable use policy.

  • Continuously monitor for new vulnerabilities and apply patches promptly.

  • Consider periodic external penetration testing to identify overlooked vulnerabilities.
