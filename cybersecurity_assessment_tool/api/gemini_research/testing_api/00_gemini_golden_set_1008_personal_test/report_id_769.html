Executive Summary

This report details the cybersecurity posture of Aeon Pharmaceuticals based on a review of organizational data, security control questionnaires, and a targeted network scan. Our analysis identifies several critical vulnerabilities and significant security gaps that require immediate attention.

Key findings include:

  • Lack of Multi-Factor Authentication (MFA) for email access, posing a severe risk of account compromise.

  • An SSL certificate common name mismatch (‘www.acme-corp.com‘ instead of ‘www.AeonPharmaceuticals.com‘) on the observed web server, potentially leading to trust issues and user warnings.

  • Use of outdated Nginx web server software (version 1.18.0), leaving the system susceptible to known exploits.

  • Absence of an Employee Acceptable Use Policy, which can lead to inconsistent security practices and legal liabilities.

Addressing these findings through the provided recommendations is crucial for enhancing the organization’s security posture and protecting sensitive assets.

Detailed Observations

  • Outdated Web Server Software: The Nginx web server detected on port 443 is running version 1.18.0. This version was released in 2020 and is considered outdated. It no longer receives security updates, making it vulnerable to numerous publicly disclosed exploits. This poses a risk to the confidentiality, integrity, and availability of the web server and hosted applications.

  • SSL Certificate Common Name Mismatch: The SSL certificate presented by the server has a Common Name (CN) of www.acme-corp.com. This does not match the organization’s website domain www.AeonPharmaceuticals.com. This mismatch is a vulnerability as it will cause browser warnings for users, undermine trust, and could potentially indicate a misconfiguration, a non-valid certificate, or even a Man-in-the-Middle (MITM) attack attempt if not properly managed.

Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Email Access: Immediately deploy and enforce Multi-Factor Authentication (MFA) for all email accounts across the organization. This significantly reduces the risk of unauthorized access through compromised credentials.

  2. Resolve SSL Certificate Mismatch: Obtain and install a valid SSL/TLS certificate for the web server that correctly matches the organization’s primary domain (www.AeonPharmaceuticals.com). Ensure certificate renewal processes are robust to prevent future expirations or mismatches.

  3. Upgrade Nginx Web Server: Upgrade Nginx to the latest stable and supported version. Implement a regular patch management process for all server software to ensure known vulnerabilities are addressed promptly.

  4. Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines employee responsibilities, acceptable use of company IT resources, and security best practices. Distribute the AUP to all employees and require acknowledgment of understanding.

  5. Conduct Regular Vulnerability Scanning and Penetration Testing: Implement a routine schedule for vulnerability scans to proactively identify and address technical weaknesses. Consider periodic penetration testing to simulate real-world attacks.

  6. Review and Enhance Security Awareness Training: While training is conducted, ensure that security awareness training modules specifically cover the importance of MFA, phishing recognition, and adherence to company policies (including the new AUP).
