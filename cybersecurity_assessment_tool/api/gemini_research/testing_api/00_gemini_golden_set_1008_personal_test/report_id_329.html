Executive Summary

This report provides a comprehensive assessment of the cybersecurity posture for Blue Marble, based on a review of organizational security controls, an internal network scan, and existing risk data. The overall security posture indicates several areas requiring immediate attention to mitigate potential risks.

Key findings include critical gaps in Multi-Factor Authentication (MFA) implementation for sensitive data systems, the absence of a formal Acceptable Use Policy (AUP), and inadequate annual security awareness training for all employees. Furthermore, network scans revealed open Remote Desktop Protocol (RDP) services on internal hosts, correlating with existing known RDP exposure risks. Addressing these findings is crucial for enhancing the organization’s resilience against cyber threats.

Analysis of Questionnaire Findings

  • MFA for Sensitive Data Systems: The lack of MFA for sensitive data systems represents a significant vulnerability. Without this control, an attacker who compromises user credentials could gain unrestricted access to critical data.

  • Employee Acceptable Use Policy (AUP): The absence of an AUP leaves employees without clear guidelines on acceptable technology use, increasing the risk of misuse, data breaches, and non-compliance.

  • Annual Security Awareness Training: While new employees receive training, the lack of annual refresher training for all employees means that threat awareness may become outdated, leaving the workforce susceptible to evolving social engineering tactics and other cyber threats.

Analysis of Technical Findings

The discovery of an open RDP service on an internal host (10.10.10.51) is a significant finding. While potentially intended for legitimate administrative access, RDP, especially if not adequately secured with strong passwords, network-level authentication (NLA), or restricted to specific source IPs/VPNs, can be a vector for:

  • Brute-force Attacks: Attackers can attempt to guess credentials.

  • Lateral Movement: If an attacker gains a foothold elsewhere in the network, an unsecured internal RDP port provides an easy path for lateral movement to other systems.

  • Exploitation: Historically, RDP services have been targets for various vulnerabilities (e.g., BlueKeep).

This finding correlates with existing known RDP exposure risks within the organization.

Recommendations

To address the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Sensitive Data Systems:

    • Deploy and enforce Multi-Factor Authentication for all systems that handle or store sensitive organizational data. This significantly reduces the risk of unauthorized access even if credentials are compromised.

  2. Address RDP Exposures:

    • For 10.10.10.50 (Existing): Review the necessity of RDP exposure. If not strictly required externally, block port 3389 at the firewall. If remote access is critical, implement a Virtual Private Network (VPN) solution as the primary access method, requiring users to connect to the VPN before attempting RDP.

    • For 10.10.10.51 (Internal): Restrict RDP access to a whitelist of necessary administrative IP addresses and personnel. Ensure strong password policies are enforced, and Network Level Authentication (NLA) is enabled. Consider disabling RDP entirely if not required, or replacing it with more secure remote access methods where possible.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities and acceptable conduct when using company resources. Ensure all employees review and acknowledge the policy annually.

  4. Establish Annual Security Awareness Training:

    • Implement mandatory, recurring (at least annual) security awareness training for all employees, not just new hires. This training should cover current threat landscapes, phishing, social engineering, data handling best practices, and the importance of the AUP.
