5. Recommendations

To address the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

5.1. Implement Multi-Factor Authentication (MFA) for Computer Logins (High Priority)

  • Action: Mandate the use of Multi-Factor Authentication for all user accounts logging into company workstations and critical systems.

  • Benefit: Significantly reduces the risk of unauthorized access due to compromised credentials, as an attacker would need both the password and a second authentication factor.

  • Considerations: Evaluate various MFA solutions (e.g., hardware tokens, authenticator apps, biometrics) to find one that best fits the organizational environment and user experience needs. Implement a phased rollout with clear user training and support.

5.2. Harden SSH Service (Medium Priority)

  • Action: Review and strengthen the configuration of the SSH service running on .

  • Specific Steps:

    • Disable Root Login: Configure SSH to disallow direct root login.

    • Use SSH Keys: Prefer strong, unique SSH keys for authentication over passwords. If passwords are required, enforce complex password policies and regular rotations.

    • Rate Limiting: Implement firewall rules or intrusion prevention systems to rate-limit connection attempts to SSH, mitigating brute-force attacks.

    • Regular Updates: Ensure the SSH software is kept up-to-date with the latest stable versions to patch known vulnerabilities.

    • Restrict Access: Limit SSH access to only necessary administrative IP addresses or ranges.

    • Monitoring: Implement logging and monitoring for SSH access attempts and activity to detect suspicious behavior.

  • Benefit: Reduces the likelihood of successful attacks targeting the SSH service, safeguarding remote access.

5.3. Conduct Regular Vulnerability Scans (General Practice)

  • Action: Establish a routine schedule for vulnerability scanning of all internet-facing assets and internal systems.

  • Benefit: Proactively identifies new exposures, outdated software versions, and misconfigurations before they can be exploited by attackers.

  • Considerations: Implement both authenticated and unauthenticated scans. Regularly review scan reports and prioritize remediation efforts based on risk.
