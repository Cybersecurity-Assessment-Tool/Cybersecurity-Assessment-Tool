Executive Summary

This assessment reveals a mixed security posture for Structure & Form. Positive findings include robust implementation of Multi-Factor Authentication (MFA) across critical access points and consistent security awareness training for employees. However, two significant areas of concern have been identified: the lack of an Employee Acceptable Use Policy (AUP) and the deployment of an End-of-Life (EOL) Nginx web server (version 1.18.0). These issues introduce unnecessary risk, potentially leading to misuse of company resources, compliance issues, and exposure to known vulnerabilities. Remediation efforts should prioritize the development and enforcement of an AUP and the urgent upgrade or patching of the web server to a currently supported version.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Action: Immediately develop a comprehensive AUP that clearly outlines employee responsibilities and acceptable conduct when using company IT assets, networks, and data.

    • Details: The AUP should cover topics such as internet usage, email etiquette, software installation, data handling, use of personal devices (BYOD), and incident reporting.

    • Timeline: Prioritize within 1-2 months.

    • Benefit: Reduces insider threat risks, improves data security, and ensures compliance with organizational policies and legal requirements.

  2. Upgrade Nginx Web Server to a Supported Version:

    • Action: Upgrade the Nginx web server from version 1.18.0 to a currently supported stable version (e.g., Nginx 1.24.x or the latest mainline version).

    • Details: Before upgrading, ensure proper backup procedures are in place. Test the upgrade in a staging environment to confirm compatibility with existing applications and configurations. Apply security patches promptly upon release.

    • Timeline: Urgent, within 1-2 weeks.

    • Benefit: Addresses known vulnerabilities, ensures ongoing security updates, and improves overall server stability and performance.

  3. Regular Software and System Audits:

    • Action: Implement a schedule for regular audits of all critical software and systems to identify outdated versions or misconfigurations.

    • Details: Utilize automated vulnerability scanning tools and maintain an inventory of all deployed software versions.

    • Timeline: Ongoing.

    • Benefit: Proactively identifies and remediates potential security risks before they can be exploited.
