Analysis of Security Controls

The questionnaire results indicate a mixed security posture. While Multi-Factor Authentication (MFA) is commendably implemented for email and sensitive data systems, its absence for computer logins (Q2) represents a critical vulnerability, significantly increasing the risk of unauthorized access via compromised credentials. Detailed Risk Overview

  • RA-01: Lack of MFA for Computer Logins

    • Overview: Bluebird Bio does not require Multi-Factor Authentication (MFA) for users logging into their computers. This significantly increases the risk of successful credential stuffing, brute-force attacks, or phishing attacks leading to unauthorized access to internal systems, as a single compromised password could grant an attacker entry.

    • Impact: Potential for unauthorized access to endpoints, lateral movement within the network, data breaches, system compromise, and disruption of operations.

    • Severity: High

  • RA-02: Inadequate Annual Security Training

    • Overview: While new employees receive initial security awareness training, annual recurring training for all employees is not consistently conducted. This practice can lead to a degradation of security awareness over time, making employees more susceptible to evolving social engineering attacks such as phishing, whaling, and pretexting, which remain primary initial access vectors for many cyberattacks.

    • Impact: Increased susceptibility to social engineering attacks, higher likelihood of malware infections, and a greater risk of human error leading to security incidents and data loss.

    • Severity: High

  • RA-03: Remote Desktop Protocol (RDP) Exposure

    • Overview: The Nmap scan confirmed that RDP (Port 3389) is open on 10.10.10.51. This finding directly correlates with an existing high-severity vulnerability previously noted as "RDP Exposure" (with a CVSS score of 9.0), affecting similar internal network assets (e.g., ‘10.10.10.50‘). Exposing RDP, particularly without robust compensating controls such as a VPN gateway, strong network segmentation, strict account lockout policies, and MFA, is a critical attack vector frequently exploited by threat actors, including ransomware groups and state-sponsored attackers, for initial access and persistence.

    • Impact: Potential for unauthorized remote access, full system compromise, data exfiltration, deployment of ransomware or other malicious payloads, and significant disruption of business operations.

    • Severity: Critical (CVSS 9.0)

Recommendations

To effectively mitigate the identified risks and substantially improve Bluebird Bio’s overall cybersecurity posture, the following actionable recommendations are provided:

  • Implement MFA for Computer Logins (Addressing RA-01):

    • Mandate Multi-Factor Authentication for all computer logins across the organization. This should encompass both domain and local accounts where applicable. Implementing MFA significantly increases the difficulty for attackers to gain unauthorized access even if user credentials are compromised.

    • Evaluate solutions such as Windows Hello for Business, third-party MFA providers integrated with Active Directory, or hardware security keys.

  • Establish a Comprehensive Annual Security Awareness Training Program (Addressing RA-02):

    • Develop and implement a mandatory, engaging, and up-to-date annual security awareness training program for all employees.

    • Training content should cover current threat landscapes (e.g., advanced phishing techniques, ransomware, insider threats), best practices for password hygiene, data handling, and incident reporting procedures.

    • Supplement training with regular simulated phishing exercises to test employee vigilance and reinforce learned behaviors.

  • Secure Remote Desktop Protocol (RDP) Exposure (Addressing RA-03):

    • Conduct Comprehensive RDP Audit: Perform an immediate audit of all systems with RDP enabled to identify all instances and their exposure levels.

    • Immediate Mitigation: For any RDP exposed directly to the internet, implement immediate firewall rules to block or strictly restrict access to only known, trusted, and necessary IP addresses. For internal systems, ensure all RDP accounts utilize strong, unique passwords and aggressive account lockout policies.

    • Long-Term Secure Access Solution: Implement a Virtual Private Network (VPN) or a secure gateway solution (e.g., Azure Bastion, AWS Session Manager, JumpCloud) for all external RDP access. All RDP access, whether internal or external, should be conditional and ideally require MFA.

    • Network Segmentation: Isolate systems that require RDP access into appropriate network segments, limiting their ability to interact with critical assets or other parts of the network unnecessarily.

    • Regular Patching: Ensure all systems running RDP are continuously patched and updated against known vulnerabilities to prevent exploitation.

    • Disable Unused RDP: Disable RDP on any systems where it is not strictly required for operational purposes.
