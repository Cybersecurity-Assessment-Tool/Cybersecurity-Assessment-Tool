Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

Immediate Actions (High Priority)

  • Implement MFA for Computer Logins: Immediately enforce Multi-Factor Authentication (MFA) for all user logins to computers across the organization. This significantly reduces the risk of unauthorized access from stolen or phished credentials.

  • Implement MFA for Sensitive Data Systems: Deploy and enforce MFA for access to all systems containing sensitive organizational or customer data. Prioritize systems based on data criticality and exposure.

  • Conduct Comprehensive Security Awareness Training: Develop and implement an annual mandatory security awareness training program for all employees. This training should cover topics such as phishing, social engineering, password hygiene, data handling, and incident reporting.

Mid-Term Actions (Medium Priority)

  • Deep Dive Network Scan on Port 22: Conduct a more comprehensive Nmap scan (e.g., using -sV option) on the identified open port 22 () to determine the exact service, product, and version information.

  • Vulnerability Assessment for Identified Services: Once service/version information is available for port 22, perform a vulnerability scan against the identified service to check for known Common Vulnerabilities and Exposures (CVEs). If SSH is running, ensure it is configured securely, using strong ciphers and disabling password authentication in favor of key-based authentication where possible.

  • Regular Security Policy Review: Establish a schedule for annual review and update of the employee acceptable use policy and other security policies to ensure they remain current and address evolving threats.

General Best Practices

  • Regular Vulnerability Scanning: Implement a routine schedule for internal and external vulnerability scanning of all network assets to proactively identify and remediate weaknesses.

  • Incident Response Plan: Develop and regularly test an incident response plan to ensure the organization can effectively detect, respond to, and recover from cyber incidents.

  • Least Privilege Principle: Ensure that employees and systems are granted only the minimum necessary permissions to perform their tasks.

Conclusion

While Verve & Vigor has some foundational security controls in place, particularly regarding initial security awareness training and an acceptable use policy, there are critical deficiencies in MFA implementation and ongoing security education. Addressing these areas, alongside further investigation into exposed services, will significantly bolster the organization’s defense against common cyber threats. Continuous monitoring, regular assessments, and a proactive security culture are essential for maintaining a strong cybersecurity posture.
