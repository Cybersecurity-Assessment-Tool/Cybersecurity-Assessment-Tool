Executive Summary

This report provides a comprehensive assessment of Iron Bridge Legal’s current cybersecurity posture, synthesizing findings from an organizational security questionnaire, a targeted network scan, and an analysis of pre-existing risk data.

Key findings highlight several critical vulnerabilities and significant security gaps:

  • Exposed Remote Desktop Protocol (RDP): An internal host (10.10.10.51) was found to have RDP open, mirroring a pre-existing high-severity risk on 10.10.10.50. This indicates a systemic exposure to a common attack vector.

  • Lack of Multi-Factor Authentication (MFA) for Computer Logins: MFA is not required for accessing computers, significantly increasing the risk of unauthorized access through compromised credentials.

  • Absence of Fundamental Security Policies: The organization lacks an Employee Acceptable Use Policy, which is crucial for defining expected employee conduct regarding IT assets.

  • Insufficient Security Awareness Training: There is no security awareness training for either new or existing employees, leaving the organization vulnerable to social engineering and phishing attacks.

The overall cybersecurity posture requires immediate and substantial improvements across technical controls, policy implementation, and human factor training to effectively mitigate identified high-impact risks.

Technical Scan Results

A network scan was performed to identify open ports and services on a specified internal host.

Scan Metadata

  • Scanner: Nmap

  • Target Host: 10.10.10.51

Findings

The scan revealed the following open port and associated service on the target host:

  • Host: 10.10.10.51

  • Port 3389/tcp: Open

  • Service Name: ms-wbt-server (Microsoft Remote Desktop Protocol - RDP)

The presence of an open RDP port on an internal system can present a significant security risk if not adequately protected. RDP is a common target for brute-force attacks and exploits, potentially allowing unauthorized access to the internal network.

Recommendations

Based on the identified risks, the following recommendations are provided to improve Iron Bridge Legal’s cybersecurity posture. These steps are prioritized to address the most critical and high-impact vulnerabilities first.

  1. Harden Remote Desktop Protocol (RDP) Access:

    • Immediate Action: Implement strong Multi-Factor Authentication (MFA) for all RDP connections. Restrict RDP access to only necessary personnel and IP addresses through firewall rules. If RDP is not strictly necessary for 10.10.10.51, close port 3389.

    • Long-Term Solution: Consider placing RDP services behind a Virtual Private Network (VPN) or a dedicated bastion host, removing direct exposure to internal or external networks. Ensure all RDP servers are regularly patched and configured with robust security settings (e.g., Network Level Authentication, strong password policies).

  2. Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Action: Mandate MFA for all user logins to company computers and endpoints. This significantly reduces the risk of unauthorized access even if primary credentials are compromised. Evaluate and deploy an appropriate MFA solution (e.g., hardware tokens, authenticator apps).

  3. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Action: Draft, communicate, and enforce a comprehensive AUP outlining the acceptable use of company IT resources, data, and internet access. Ensure all employees formally acknowledge and understand this policy during onboarding and periodically thereafter.

  4. Establish a Robust Security Awareness Training Program:

    • Action for New Employees: Implement mandatory security awareness training as part of the onboarding process for all new employees.

    • Action for All Employees: Conduct annual security awareness training for all employees, covering topics such as identifying phishing and social engineering attacks, strong password hygiene, data handling best practices, and incident reporting procedures. Consider regular simulated phishing exercises.

  5. Implement Regular Vulnerability Scanning and Patch Management:

    • Action: Establish a routine vulnerability scanning program to proactively identify and address security weaknesses across the network and endpoints. Ensure all operating systems, applications, and network devices are kept up-to-date with the latest security patches and configurations.
