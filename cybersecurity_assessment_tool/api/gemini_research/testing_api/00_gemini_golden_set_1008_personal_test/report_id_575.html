Executive Summary

This report presents a comprehensive cybersecurity posture assessment for Open Door, integrating organizational data, technical network scan results, and pre-existing risk information. The analysis reveals several critical security gaps and vulnerabilities that warrant immediate attention. Key findings include the absence of Multi-Factor Authentication (MFA) for email access, a lack of an Employee Acceptable Use Policy (AUP), and insufficient annual security awareness training for all employees. Furthermore, a technical scan identified an open SSH port (22) on ‘127.0.0.1‘, which correlates with a pre-existing critical risk concerning "Localhost Exposed". The report provides actionable recommendations designed to mitigate these identified risks and significantly enhance the organization’s overall security posture.

Recommendations

The following recommendations are provided to address the identified risks and enhance the overall cybersecurity posture of Open Door. These actions are prioritized to mitigate the most significant vulnerabilities.

  1. Implement Multi-Factor Authentication (MFA) for Email Access:

    • Action: Mandate and enforce MFA for all user accounts accessing email services across the organization.

    • Benefit: Provides a robust additional layer of security, significantly reducing the risk of unauthorized access to email, even if primary credentials are stolen.

  2. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Action: Create a comprehensive AUP that clearly outlines acceptable use of all company IT assets, data handling procedures, internet usage, and security responsibilities. Distribute the policy to all employees and require signed acknowledgment of understanding and compliance.

    • Benefit: Establishes clear expectations for employee behavior, minimizes misuse of corporate resources, reduces legal and compliance risks, and strengthens the overall security culture.

  3. Conduct Regular and Comprehensive Security Awareness Training:

    • Action: Implement a mandatory annual security awareness training program for all employees, supplementing initial new-hire training. Training should cover current threats such as phishing, ransomware, social engineering, secure password practices, and data protection guidelines.

    • Benefit: Increases employee vigilance and knowledge, transforming them into a more effective human firewall against sophisticated cyber threats.

  4. Review and Harden Localhost Services (Port 22/SSH):

    • Action: Investigate the presence of an open SSH port on ‘127.0.0.1‘. If SSH access is necessary, ensure it is configured with strong security practices:

      • Disable root login.

      • Enforce strong password policies or, preferably, key-based authentication.

      • Limit access to authorized users and IP addresses.

      • Change the default SSH port (22) to a non-standard port.

      • Implement a firewall to restrict access to only necessary sources.

      • Monitor SSH access logs for suspicious activity.

    • Benefit: Reduces the attack surface, prevents unauthorized remote access, and mitigates the risk associated with an exposed SSH service.
