Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Mainframe Managed, based on a review of organizational security controls, an internal network scan, and pre-existing risk documentation. Key findings highlight critical gaps in Multi-Factor Authentication (MFA) implementation for email access and significant deficiencies in security awareness training across the organization. The internal network scan did not identify any open ports on the target host, specifically finding port 80 to be closed. This contrasts with an existing documented risk concerning an "Unencrypted Web Server" with port 80 open, indicating a potential discrepancy requiring further investigation. Recommendations prioritize addressing MFA and training shortcomings, alongside clarifying the status of the web server vulnerability.

Recommendations

To significantly improve Mainframe Managed’s cybersecurity posture, the following actionable recommendations are provided, prioritized by their potential impact and urgency.

Immediate Actions (High Priority)

  1. Implement MFA for Email Access: Immediately deploy and enforce Multi-Factor Authentication (MFA) for all organizational email accounts. This should be a mandatory requirement for all users.

    • Details: Utilize modern MFA methods such as authenticator apps, FIDO2 security keys, or push notifications rather than SMS-based MFA where possible.

Short-Term Actions (Medium Priority)

  1. Establish New Employee Security Awareness Training: Develop and implement a mandatory security awareness training module as part of the onboarding process for all new hires.

    • Details: Training should cover topics such as phishing, malware recognition, password hygiene, acceptable use policies, and incident reporting procedures.

  2. Initiate Annual Security Awareness Training: Implement a recurring, mandatory annual security awareness training program for all employees.

    • Details: This training should be updated regularly to reflect current threat landscapes and reinforce security best practices, including simulated phishing exercises.

  3. Investigate Uncorroborated Web Server Risk: Conduct an immediate follow-up assessment to clarify the status of the "Unencrypted Web Server" risk.

    • Details: Determine if the risk refers to an external-facing asset, a different internal server, or if the documentation is outdated. If the vulnerability is confirmed to exist on any production system, implement SSL/TLS encryption, reconfigure the server to exclusively serve traffic over HTTPS (port 443), and close or redirect all traffic from port 80.
