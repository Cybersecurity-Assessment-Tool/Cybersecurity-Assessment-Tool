1. Executive Summary

This report details the cybersecurity posture of Moxie Marketing based on a review of organizational data, a technical network scan, and an assessment of existing risks. The analysis reveals several critical vulnerabilities and significant control gaps, primarily related to outdated software, weak authentication practices, and insufficient policy enforcement.

Key findings include the presence of an outdated and vulnerable FTP server (vsftpd 2.3.4) allowing anonymous access, a lack of Multi-Factor Authentication (MFA) for computer logins and sensitive data systems, and a complete absence of an employee acceptable use policy and security awareness training for new employees. Additionally, an existing risk highlights the use of End-of-Life (EOL) Windows 7 operating systems. These issues collectively represent a high-risk profile, potentially exposing the organization to unauthorized access, data breaches, and compliance violations. Immediate action is required to remediate these critical and high-severity findings.

6. Recommendations

Based on the identified risks, the following recommendations are provided to enhance Moxie Marketing’s cybersecurity posture. Recommendations are prioritized by severity.

6.1. Critical Recommendations

  • Decommission or Secure FTP Server (vsftpd 2.3.4): Immediately patch or upgrade vsftpd to a current, supported version, or, preferably, decommission the service if it is not essential. If FTP is required, implement SFTP or FTPS with strong authentication, disable anonymous access, and restrict user directories to prevent unauthorized data access.

6.2. High-Severity Recommendations

  • Implement MFA for All System Logins: Deploy and enforce Multi-Factor Authentication (MFA) for all computer logins across the organization. This significantly reduces the risk of unauthorized access even if primary credentials are stolen.

  • Implement MFA for Sensitive Data Systems: Ensure all systems containing or accessing sensitive data require MFA for login. This is a critical control to protect sensitive information from unauthorized access.

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that outlines acceptable and unacceptable use of company IT resources. Ensure all employees read, understand, and sign the AUP annually.

  • Mandatory Security Awareness Training for New Hires: Institute a mandatory security awareness training program for all new employees during their onboarding process. This should cover common threats (e.g., phishing, malware) and organizational security policies.

6.3. Medium-Severity Recommendations

  • Upgrade All Windows 7 Workstations: Plan and execute an immediate upgrade path for all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or 11). This is crucial for receiving vital security updates and patches.

6.4. General Best Practices

  • Regular Vulnerability Scanning and Penetration Testing: Implement a routine schedule for vulnerability scanning and periodic penetration testing to identify new weaknesses before malicious actors can exploit them.

  • Patch Management Program: Establish and strictly adhere to a comprehensive patch management program for all operating systems, applications, and network devices.

  • Security Policy Review and Updates: Regularly review and update all cybersecurity policies and procedures to ensure they remain relevant and effective against evolving threats.
