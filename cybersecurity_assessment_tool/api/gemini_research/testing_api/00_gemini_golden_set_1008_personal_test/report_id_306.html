4. Risk Assessment

This section synthesizes findings from the security control review and pre-existing vulnerabilities. Risks are categorized by severity, with "High" indicating immediate and significant impact, and "Medium" indicating potential impact.

Risk Name Overview Severity
Lack of MFA for Computer Logins Without MFA, user credentials (passwords) are a single point of failure. Compromised credentials can lead to unauthorized access to internal systems and data. High
Absence of Acceptable Use Policy (AUP) Lack of an AUP means employees may not be aware of expected behavior regarding company IT resources, increasing risks of misuse, data breaches, and non-compliance. High
No Security Awareness Training for New Employees New employees are often targets for social engineering attacks. Without initial security training, they may be more susceptible to phishing, malware, and other threats, leading to potential breaches. High

5. Recommendations

The following recommendations address the identified risks and aim to enhance the overall cybersecurity posture of True Grit.

  • Implement MFA for Computer Logins:

    • Deploy a robust Multi-Factor Authentication (MFA) solution across all corporate computer logins.

    • Enforce mandatory MFA for all employees, especially those with access to sensitive systems, to significantly reduce the risk of credential theft.

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Create a clear and comprehensive AUP outlining the appropriate use of company IT assets, data, and internet access.

    • Ensure all employees review and acknowledge the AUP annually to reinforce proper conduct and security expectations.

  • Initiate Security Awareness Training for New Employees:

    • Implement mandatory security awareness training as part of the onboarding process for all new hires.

    • Cover critical topics such as phishing identification, strong password hygiene, social engineering tactics, and the procedure for reporting suspicious activities or incidents.

  • Review and Address Previous Web Server Risk:

    • Although Port 80 was found closed in the current scan, investigate the historical context of the "Unencrypted Web Server" risk from Input 3.

    • If a web server is planned for deployment or exists on another IP, ensure it exclusively uses HTTPS (Port 443) with valid, up-to-date SSL/TLS certificates for all communications to protect data in transit.
