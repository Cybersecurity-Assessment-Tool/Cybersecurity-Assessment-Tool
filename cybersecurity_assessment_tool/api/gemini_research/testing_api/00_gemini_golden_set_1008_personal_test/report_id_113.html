1. Executive Summary

The assessment of Structure & Form’s cybersecurity posture highlights significant vulnerabilities that demand immediate attention. Critical findings include multiple instances of RDP exposure, compounded by a general lack of Multi-Factor Authentication (MFA) for computer logins and sensitive data systems. Furthermore, foundational security elements such as an Acceptable Use Policy (AUP) and comprehensive security awareness training are entirely absent. These gaps collectively present a high-risk environment, making the organization susceptible to various cyber threats including unauthorized access, data breaches, and social engineering attacks. Addressing these issues through strategic implementation of MFA, RDP hardening, and robust security training and policies is paramount.

6. Recommendations

Based on the identified risks, the following actionable recommendations are provided to enhance Structure & Form’s cybersecurity posture:

6.1. Immediate Actions - RDP Security

  • Secure RDP Access on Exposed Systems:

    • For 10.10.10.50 and 10.10.10.51, restrict RDP access to only trusted IP addresses or via a Virtual Private Network (VPN) gateway. External exposure should be minimized or eliminated.

    • Implement Network Level Authentication (NLA) for all RDP connections to add an initial layer of authentication before a full session is established.

    • Strongly Recommend MFA for RDP: Mandate Multi-Factor Authentication for all RDP logins to prevent unauthorized access even if primary credentials are compromised.

    • Ensure strong, unique passwords for all accounts with RDP access and enforce regular password changes.

    • Disable RDP entirely on systems where remote access is not absolutely necessary.

6.2. Multi-Factor Authentication (MFA) Implementation

  • Implement MFA for Computer Logins: Deploy and enforce MFA for all employee computer logins across the organization. This should integrate with existing identity providers (e.g., Active Directory) where possible.

  • Implement MFA for Sensitive Data Systems: Extend MFA requirements to all systems storing, processing, or providing access to sensitive organizational data, including databases, file servers, and critical applications.

6.3. Policy and Training Development

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines the proper and secure use of all company resources, data handling procedures, and expected security behaviors for all employees. Ensure all employees formally acknowledge and understand this policy.

  • Establish Mandatory Security Awareness Training for New Employees: Integrate a robust security awareness training program into the onboarding process for all new hires. This training should cover essential security concepts, common threats (e.g., phishing, social engineering), and organizational policies.

  • Implement Annual Security Awareness Training for All Employees: Establish a recurring, mandatory annual security awareness training program for all existing employees. This training should be updated regularly to address emerging threats, reinforce best practices, and remind employees of their security responsibilities.

6.4. General Security Best Practices

  • Regular Vulnerability Management: Conduct regular vulnerability scans of all network assets and implement a robust patch management program to ensure operating systems, applications, and firmware are kept up-to-date with the latest security patches.

  • Incident Response Plan: Develop and regularly test a comprehensive incident response plan to ensure the organization can effectively detect, respond to, and recover from cybersecurity incidents.
