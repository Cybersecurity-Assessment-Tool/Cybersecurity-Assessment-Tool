Executive Summary

This report details the findings of a cybersecurity posture assessment for Verve & Vigor. The assessment synthesized information from network scans, an organizational security questionnaire, and pre-existing risk documentation. Key findings indicate critical vulnerabilities related to an exposed database service and significant gaps in foundational security controls, including Multi-Factor Authentication (MFA) for sensitive data systems, the absence of an Employee Acceptable Use Policy, and insufficient annual security awareness training. Recommendations are provided to mitigate identified risks and enhance the overall security posture.

Technical Scan Results

A network scan was conducted on the target system 172.16.50.20. The scan identified an open port with a running service, which presents a significant exposure.

Target: 172.16.50.20

  • Open Port(s) and Services:

    • Port: 3306/tcp

    • State: Open

    • Service: MySQL

    • Product: MySQL

    • Version: 5.7.33

The presence of an open MySQL database port (3306) is a critical finding. MySQL version 5.7.33 is an older release (initial release 2016, End-of-Life for general availability support was October 2023). While specific vulnerabilities for this exact patch level are not confirmed by the scan alone, running an exposed database service on an older, potentially unpatched version significantly increases the attack surface and risk of data breach or unauthorized access.

Recommendations

To address the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Restrict Access to MySQL Database (Port 3306):

    • Implement strict firewall rules to restrict access to port 3306 (MySQL) to only trusted internal IP addresses or specific applications that require it.

    • Consider placing the database server within a segmented network or behind a Virtual Private Network (VPN) to prevent direct exposure to the broader network or internet.

    • Evaluate the necessity of external access to this database. If required, implement a secure gateway or reverse proxy.

  2. Upgrade and Patch MySQL Database:

    • Migrate the MySQL database from version 5.7.33 to a currently supported and actively maintained version (e.g., MySQL 8.0 or newer).

    • Ensure all critical security patches are applied promptly to the database server and operating system.

  3. Implement MFA for Sensitive Data Systems:

    • Mandate Multi-Factor Authentication (MFA) for all access to systems containing sensitive organizational or customer data. This significantly reduces the risk of credential theft leading to data breaches.

  4. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive Employee Acceptable Use Policy that clearly defines the rules and expectations for employee conduct when using company IT resources.

    • Ensure all employees review, understand, and formally acknowledge the AUP, ideally during onboarding and annually thereafter.

  5. Establish Mandatory Annual Security Awareness Training:

    • Implement a recurring, mandatory security awareness training program for all employees at least once per year.

    • The training should cover current threats such as phishing, social engineering, malware, data handling best practices, and the importance of strong passwords and MFA.

    • Regularly update training content to reflect evolving threat landscapes.
