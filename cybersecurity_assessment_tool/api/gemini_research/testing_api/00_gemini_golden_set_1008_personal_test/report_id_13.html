1. Executive Summary

This report details the findings from a cybersecurity posture assessment conducted for Catalyst Consulting. The assessment involved a review of the organization’s self-reported security controls via a questionnaire, an external network scan, and an evaluation of existing known risks.

Key findings indicate critical gaps in foundational security practices, specifically the absence of an Employee Acceptable Use Policy and a lack of initial security awareness training for new employees. Technically, an older version of Nginx was identified on a public-facing service, and an SSL certificate common name mismatch was noted, raising concerns about patching cadence and proper certificate provisioning. While no pre-existing vulnerabilities were provided, the identified gaps and technical findings pose significant risks to the organization’s security posture. Recommendations are provided to address these issues and strengthen overall defenses.

5. Risk Assessment

Based on the analysis of the questionnaire and technical scan results, the following risks have been identified. No pre-existing vulnerabilities were provided in the input data.

Risk Name Severity ID Overview
Lack of Acceptable Use Policy (AUP) High QA-001 The absence of an AUP leaves employees without clear guidelines on appropriate use of company resources, data handling, and expected cybersecurity conduct. This increases the risk of accidental data breaches, malware infections, and misuse of company assets.
No Security Awareness Training for New Employees High QA-002 New hires are often the most susceptible to social engineering attacks (e.g., phishing) if not immediately equipped with essential security awareness training. This creates a significant entry point for attackers into the organization.
Outdated Web Server Version (Nginx) Medium TS-001 Running an older stable version of Nginx (1.18.0, released April 2020) suggests that the web server may not be benefiting from the latest performance improvements, features, or security fixes available in more recent stable branches (e.g., 1.24.x, 1.26.x). While not immediately critical, this increases the risk of eventually encountering known vulnerabilities or missing proactive patches.
SSL Certificate Common Name Mismatch Medium TS-002 The SSL certificate presented by for HTTPS traffic has a common name of www.acme-corp.com. This does not match the reported organization’s website domain (). This mismatch can lead to browser warnings for users, undermine trust, and could indicate a misconfigured server or an incorrectly provisioned certificate.

6. Recommendations

To mitigate the identified risks and enhance Catalyst Consulting’s overall cybersecurity posture, the following recommendations are provided:

  1. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP document that clearly outlines employee responsibilities regarding the use of company assets, internet, email, and data handling.

    • Ensure all employees review and acknowledge the AUP, preferably annually or upon significant updates.

    • Integrate the AUP into new employee onboarding processes.

  2. Implement Mandatory Security Awareness Training for New Employees:

    • Develop a mandatory, introductory security awareness training module for all new hires.

    • Cover fundamental topics such as phishing identification, password best practices, physical security, and reporting suspicious activities.

    • Ensure this training is completed before new employees gain full access to sensitive systems.

  3. Update and Maintain Web Server Software (Nginx):

    • Plan and execute an upgrade of the Nginx web server to a currently supported and more recent stable version (e.g., 1.24.x or 1.26.x).

    • Establish a regular patching and update schedule for all public-facing infrastructure.

    • Test updates thoroughly in a staging environment before deploying to production.

  4. Correct SSL Certificate Mismatch:

    • Investigate why the SSL certificate for presents www.acme-corp.com as its common name.

    • Replace the certificate with one that correctly reflects the organization’s domain, .

    • Ensure all public-facing services use valid, correctly configured, and up-to-date SSL/TLS certificates.

7. Conclusion

Catalyst Consulting demonstrates strengths in enforcing MFA across critical systems and maintaining annual security awareness training. However, the lack of an Acceptable Use Policy and initial security training for new employees, coupled with technical observations like an older Nginx version and an SSL certificate mismatch, introduce significant risks. Addressing these identified areas through the recommended actions will substantially enhance the organization’s cybersecurity posture and reduce its attack surface.
