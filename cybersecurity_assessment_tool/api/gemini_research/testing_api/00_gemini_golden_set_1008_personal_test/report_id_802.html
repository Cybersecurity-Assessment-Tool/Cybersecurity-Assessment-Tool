3.1 Analysis of Questionnaire Responses

The review of the questionnaire responses reveals critical gaps in foundational security practices:

  • MFA Implementation: MFA is notably absent for email and computer logins, which are primary vectors for credential compromise. While MFA is used for sensitive data systems, the lack of it for initial access points leaves a significant vulnerability.

  • Security Awareness Training: There is a complete absence of security awareness training, both for new employees and on an annual basis for all staff. This poses a substantial human factor risk, as employees may not be equipped to identify and respond to common threats like phishing or social engineering.

  • Acceptable Use Policy (AUP): The presence of an AUP is positive, indicating some foundational policy is in place.

5. Risk Assessment

Based on the provided information, the primary risks identified stem from the deficiencies in organizational security controls.

Identified Risks Summary

RA1 Lack of MFA for Email Access Without MFA, email accounts are highly susceptible to credential stuffing, phishing, and brute-force attacks, leading to potential data breaches, business email compromise (BEC), and unauthorized access to other systems. High

6. Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  • Implement Multi-Factor Authentication (MFA):

    • Immediately implement MFA for all email accounts.

    • Implement MFA for all computer logins across the organization.

    • Review and ensure MFA is consistently enforced for all sensitive data systems.

  • Establish Comprehensive Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new employees during their onboarding process.

    • Conduct annual, mandatory security awareness training for all employees, ensuring content is updated to reflect current threat landscapes (e.g., phishing, ransomware, social engineering).

  • Conduct Regular Vulnerability Scanning and Penetration Testing:

    • Engage in regular external vulnerability scans (at least quarterly) of internet-facing assets (e.g., ) to identify and remediate technical weaknesses.

    • Consider periodic internal vulnerability scans and penetration tests to uncover potential entry points and validate security controls.

  • Develop a Robust Incident Response Plan:

    • Ensure a well-defined and tested incident response plan is in place to effectively detect, respond to, and recover from security incidents.

  • Review and Enhance Security Policies:

    • Regularly review the Acceptable Use Policy and develop additional policies (e.g., Data Handling Policy, Password Policy, Remote Work Policy) to address current security requirements.
