1. Executive Summary

This report details the findings of a cybersecurity posture assessment for Vertex Solutions, based on provided organizational data, network scan results, and existing risk information. The analysis reveals a commendable commitment to foundational security controls, particularly in the widespread implementation of Multi-Factor Authentication (MFA) across critical systems and the establishment of an Acceptable Use Policy (AUP).

However, several significant vulnerabilities and gaps were identified. Specifically, a critical deficiency exists in providing security awareness training for new employees, which leaves the organization vulnerable to human-factor exploits. Furthermore, a publicly exposed MySQL database, running an End-of-Life (EOL) version (5.7.33), represents a severe technical risk due to the absence of ongoing security updates and its potential for unauthorized access. Immediate action is recommended to mitigate these high-priority risks.

6. Recommendations

To address the identified risks and enhance Vertex Solutions’ cybersecurity posture, the following recommendations are provided:

6.1. Process & Policy Enhancements

  • Implement Mandatory New Employee Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new hires before they are granted system access.

    • Training should cover essential topics such as phishing, social engineering, strong password practices, data handling best practices, and incident reporting procedures. This will significantly reduce the risk of human-factor exploits.

6.2. Technical Mitigations

  • Restrict Access to MySQL Port 3306 (Database Exposure):

    • Immediately implement firewall rules to restrict inbound access to to only necessary and explicitly authorized internal systems.

    • For any legitimate external access requirements, enforce secure connectivity methods such as a Virtual Private Network (VPN) or SSH tunneling, rather than direct exposure.

    • Consider implementing a Web Application Firewall (WAF) or dedicated database firewall if the database serves web applications.

  • Upgrade or Decommission End-of-Life MySQL Server:

    • Prioritize planning and execution of an upgrade of the MySQL server on to a currently supported version (e.g., MySQL 8.0). This is a critical action due to the high vulnerability associated with EOL software.

    • Thoroughly test the upgrade in a staging environment to ensure compatibility and functionality before deploying to production.

    • If the database is no longer actively used, decommission the server entirely to eliminate the risk.

    • Ensure all critical data is backed up before any upgrade or decommissioning process.
