1. Executive Summary

This assessment evaluates the cybersecurity posture of Stone Arch Masonry by correlating network scan data, organizational security practices, and a list of existing vulnerabilities. The organization demonstrates good practices in security awareness training for all employees. However, significant control gaps were identified, primarily concerning the limited deployment of Multi-Factor Authentication (MFA) across critical access points (computer logins and sensitive data systems) and the absence of an Employee Acceptable Use Policy (AUP). The network scan of target did not reveal any open ports, including port 80, which appears to contradict an existing risk entry regarding an unencrypted web server. This discrepancy suggests either a recent remediation or a need to re-validate the persistent risk. Remediation efforts should focus on strengthening access controls, formalizing security policies, and investigating the status of the previously identified web server risk.

3.1 Control Gap Analysis

Based on the questionnaire, the following critical control gaps were identified, which represent departures from security best practices:

  • Limited MFA Deployment: Multi-Factor Authentication (MFA) is not required for logging into computers or accessing sensitive data systems. This significantly increases the risk of unauthorized access due to compromised credentials, as a single stolen or guessed password could grant an attacker entry.

  • Absence of Acceptable Use Policy (AUP): The lack of a formal AUP means employees may not be fully aware of their responsibilities regarding the proper use of organizational IT resources and data. This can lead to increased risks of misuse, unintentional data breaches, and challenges in enforcing security standards.

5.1 Analysis of Discrepancies

A notable discrepancy exists between the pre-existing risk "Unencrypted Web Server," which states "Port 80 is open," and the recent network scan finding that port 80 on target is closed. This suggests one of the following scenarios:

  • The pre-existing risk has been remediated since it was identified, and the web server is no longer accessible via an unencrypted channel on port 80 on this specific asset.

  • The scan target is not the system to which the "Unencrypted Web Server" risk pertains, and another asset within the environment still exposes an unencrypted web server.

Further investigation is required to confirm the current status of this specific risk and its applicable asset(s). For the purpose of this report, it remains listed as an existing risk due to its inclusion in the provided risk data.

6. Recommendations

Based on the assessment, the following actionable recommendations are provided to enhance Stone Arch Masonry’s cybersecurity posture and mitigate identified risks.

  1. Implement Multi-Factor Authentication (MFA) Broadly:

    • Action: Mandate MFA for all user logins to computers and workstations across the organization.

    • Action: Enforce MFA for access to all sensitive data systems and applications, ensuring critical information is well-protected.

    • Benefit: Significantly reduces the risk of account compromise and unauthorized access, even if passwords are stolen or phished. This is a foundational control for modern cybersecurity.

  2. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Action: Create a formal AUP that clearly defines expectations for employee conduct when using company IT assets, networks, and data.

    • Action: Distribute the AUP to all employees and require acknowledgment of receipt and understanding. Incorporate it into new employee onboarding and annual security awareness training.

    • Benefit: Sets clear boundaries for appropriate technology use, reduces misuse, enhances data security, and provides a basis for disciplinary action if policies are violated.

  3. Verify Status of "Unencrypted Web Server" Risk:

    • Action: Investigate the discrepancy between the existing risk stating "Port 80 is open" and the scan results showing it as closed on .

    • Action: If the risk pertains to a different asset, assess that asset for an open, unencrypted port 80. If it truly is no longer present on any asset, mark the risk as remediated in your risk register.

    • Benefit: Ensures the risk register accurately reflects the current environment, preventing either the oversight of actual exposures or unnecessary remediation efforts. If an active unencrypted web server is found, secure it immediately with HTTPS (port 443) and strong TLS configurations.

  4. Maintain Strong Security Awareness Training:

    • Action: Continue robust security awareness training programs for new hires and annually for all employees.

    • Benefit: Reinforces good security practices and keeps employees informed about current threats, making them a stronger line of defense against social engineering and other attacks.
