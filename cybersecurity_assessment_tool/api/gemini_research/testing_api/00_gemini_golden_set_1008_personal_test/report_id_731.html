Executive Summary

Overview

This report details the findings of a cybersecurity posture assessment for Stone Arch Masonry. The assessment involved analyzing organizational security control practices, reviewing network scan results, and correlating these with pre-existing identified risks.

Overall, the organization exhibits significant vulnerabilities primarily due to a lack of fundamental security controls, particularly in Multi-Factor Authentication (MFA) implementation, security awareness training, and formal security policies. While MFA is utilized for sensitive data systems, its absence for email and computer logins presents critical attack vectors. Network scanning identified an open SSH port on the localhost interface, which correlates with an existing "Localhost Exposed" critical risk. These gaps collectively indicate a heightened risk of unauthorized access, data breaches, and system compromise. Immediate action is required to address the identified deficiencies and improve the overall security posture.

Risk Assessment

Based on the analysis of security control gaps, technical scan results, and pre-existing vulnerabilities, the following risks have been identified.

Identified Cybersecurity Risks
Risk ID Risk Name Category Severity Overview / Details
R1 Absence of MFA for Email Access Control High Lack of multi-factor authentication for email access significantly increases the risk of account compromise via phishing or brute-force attacks, potentially leading to business email compromise.
R2 Absence of MFA for Computer Login Access Control High Without MFA for computer logins, compromised user credentials can grant direct access to internal systems, leading to unauthorized access, data breaches, or system takeovers.
R3 No Employee Acceptable Use Policy (AUP) Governance & Policy High The absence of an AUP leads to unclear expectations for employee behavior regarding the use of company assets and data, increasing risks of misuse, intellectual property theft, and non-compliance.
R4 Inadequate Security Awareness Training (New Employees) Human Factor High New employees are a common target for social engineering attacks (e.g., phishing). Lack of initial security training increases their susceptibility to such threats.
R5 Inadequate Security Awareness Training (Annual) Human Factor High Regular, mandatory security awareness training is crucial to keep all employees updated on evolving cyber threats, best practices, and the organization’s security policies.
R6 Localhost Exposed / Open SSH Network Exposure Critical Port 22 (SSH) is open on the internal loopback interface (127.0.0.1), directly correlating with the pre-existing "Localhost Exposed" risk (CVSS 10.0). While localhost is typically internal, this indicates a potentially misconfigured or unmanaged service that could be abused if other local vulnerabilities exist, or if this refers to a broader exposure beyond the loopback.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided. These should be prioritized based on their severity and potential impact.

Strategic Recommendations

  1. Implement Multi-Factor Authentication (MFA):

    • Deploy and enforce MFA for all email accounts to significantly reduce the risk of phishing and credential stuffing attacks.

    • Implement MFA for all computer logins to prevent unauthorized access even if passwords are compromised.

  2. Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities and acceptable conduct regarding company IT resources, data, and internet usage.

    • Communicate the AUP to all employees and obtain signed acknowledgments.

  3. Establish a Robust Security Awareness Training Program:

    • Implement mandatory security awareness training for all new employees during their onboarding process.

    • Conduct mandatory annual security awareness training for all employees to keep them informed about current threats (e.g., phishing, social engineering) and best practices.

  4. Secure SSH Access on Localhost (127.0.0.1):

    • Review Necessity: Determine if the SSH service on 127.0.0.1 is truly required. If not, disable it immediately.

    • Strengthen Authentication: If required, ensure that SSH uses strong, unique passwords or, preferably, key-based authentication. Disable root login directly.

    • Update and Patch: Ensure the SSH server software is kept up-to-date with the latest security patches to prevent exploitation of known vulnerabilities.

    • Clarify Risk Scope: Investigate the scope of the "Localhost Exposed" risk. If it implies potential external exposure of an internal service, immediate remediation is critical.

  5. Prioritize Existing Critical Risks:

    • Address the "Localhost Exposed" risk with the highest priority as it has been assigned a CVSS score of 10.0, indicating critical severity. Follow recommended mitigation steps outlined in any existing risk register.
