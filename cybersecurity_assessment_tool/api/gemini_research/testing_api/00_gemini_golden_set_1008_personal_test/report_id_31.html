Executive Summary

This report provides an analysis of the cybersecurity posture for Urban Jungle Planning. The assessment is based on a review of organizational security controls via a questionnaire and a limited network scan. Key findings indicate significant vulnerabilities, primarily stemming from a pervasive lack of Multi-Factor Authentication (MFA) across critical systems (email, computer logon, sensitive data systems) and the absence of a formal Employee Acceptable Use Policy (AUP). Additionally, an open SSH port was identified on a scanned target, presenting a potential entry point if not properly secured. On a positive note, the organization demonstrates commitment to security awareness training for both new and existing employees. Immediate action is recommended to implement MFA, establish a comprehensive AUP, and secure exposed services.

Recommendations

To mitigate the identified risks and enhance Urban Jungle Planningâ€™s overall cybersecurity posture, the following actionable recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) Across All Critical Systems:

    • Email Access: Immediately implement and enforce MFA for all employee email accounts to significantly reduce the risk of account takeover.

    • Computer Logon: Deploy and require MFA for all computer logon events, especially for administrative accounts and systems handling sensitive data.

    • Sensitive Data Systems: Mandate MFA for access to all systems that store, process, or transmit sensitive organizational data to protect against unauthorized disclosure.

  2. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines acceptable use of all company IT resources, including networks, internet, email, software, and hardware.

    • Ensure all employees review, understand, and formally acknowledge the AUP during onboarding and annually thereafter, with regular training to reinforce its principles.

  3. Secure Exposed SSH Service on :

    • Conduct a thorough audit of the SSH server configuration on .

    • Enforce strong, key-based authentication and disable password authentication entirely.

    • Disable direct root login via SSH.

    • Implement rate limiting and IP-based access restrictions (e.g., allow SSH connections only from trusted management networks or specific client IPs).

    • Ensure the SSH daemon is regularly updated and patched to the latest stable version to mitigate known vulnerabilities.

    • Consider moving the SSH service to a non-standard port if direct internet exposure cannot be avoided, as a basic obscurity measure.
