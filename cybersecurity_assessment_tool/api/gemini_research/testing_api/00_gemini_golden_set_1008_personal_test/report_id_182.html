Executive Summary

This report provides a comprehensive assessment of Verve & Vigor’s current cybersecurity posture, based on network scan data, organizational security questionnaire responses, and a review of existing vulnerabilities.

Key findings indicate several critical and high-severity risks, including a severely outdated FTP server with anonymous access enabled, and significant gaps in security controls such as the absence of Multi-Factor Authentication (MFA) for computer logins and essential security policies/training. Furthermore, a pre-existing risk of outdated operating systems (Windows 7) on workstations compounds the overall risk exposure.

Addressing these issues is paramount to enhancing the organization’s defensive capabilities against prevalent cyber threats and ensuring data integrity and confidentiality. Prompt implementation of the provided recommendations is strongly advised.

Risk Assessment

Based on the comprehensive analysis of organizational data, technical scan results, and existing vulnerability information, the following risks have been identified:

Risk Name Overview Severity
Anonymous FTP Access Enabled An FTP server (vsftpd 2.3.4) on internal IP allows anonymous login, which enables unauthorized file access and potential data exfiltration or malware injection.
Extremely Outdated FTP Server Software The vsftpd server version 2.3.4 running on internal IP is severely outdated (End-of-Life since 2011) and contains critical known vulnerabilities, including a backdoor. This poses a severe risk of system compromise.
Lack of Multi-Factor Authentication (MFA) for Computer Logins The organization does not require MFA for logging into computers, significantly increasing the risk of unauthorized access through compromised credentials.
Outdated Windows Policy Computers are running Windows 7, which is an End-of-Life OS and no longer receives security updates.
Absence of Employee Acceptable Use Policy (AUP) There is no formal Acceptable Use Policy in place for employees, leading to potential misuse of company resources and unclear expectations for employee conduct.
Infrequent Security Awareness Training for Employees Security awareness training is not conducted annually for all employees, which can lead to a decline in security posture as employees become less aware of evolving threats.

Recommendations

  • Technical System Hardening

    • Immediately disable anonymous FTP access on the server at .

  • Multi-Factor Authentication (MFA) Implementation

    • Implement and enforce Multi-Factor Authentication (MFA) for all computer logins across the organization.

    • Develop, disseminate, and enforce a clear Employee Acceptable Use Policy (AUP) to guide employee conduct regarding IT assets.

    • Establish a mandatory annual security awareness training program for all employees to keep them informed about current threats and best practices.
