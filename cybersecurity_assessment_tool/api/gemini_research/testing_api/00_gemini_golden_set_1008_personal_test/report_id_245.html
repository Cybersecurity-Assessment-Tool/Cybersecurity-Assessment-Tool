Executive Summary

This report details the findings of a cybersecurity posture assessment for Falcon Heavy, conducted by the Cybersecurity Analyst Team. The assessment integrated organizational self-reported security controls, an external network scan, and an analysis of existing vulnerability records.

Key findings indicate a moderate-to-high risk posture, primarily driven by critical gaps in Multi-Factor Authentication (MFA) implementation for sensitive data systems and the direct exposure of Remote Desktop Protocol (RDP) services on an internal network host. These issues, combined with a lack of annual security awareness training for all employees, highlight areas requiring immediate attention to mitigate potential unauthorized access and data breach risks. While MFA is utilized for general email and computer logins, its absence for sensitive data access presents a significant vulnerability. The observed RDP exposure mirrors a previously identified critical risk, suggesting a systemic issue in network perimeter control or internal segmentation.

Recommendations provided herein aim to offer actionable steps to address these vulnerabilities, enhance the overall security posture, and improve organizational resilience against cyber threats.

Recommendations

The following recommendations are provided to address the identified risks and enhance the overall cybersecurity posture of Falcon Heavy.

Strategic and Policy Recommendations

  • Implement MFA for Sensitive Data Systems:

    • Action: Mandate Multi-Factor Authentication for all access attempts to systems, applications, and databases containing sensitive organizational or customer data.

    • Benefit: Significantly reduces the risk of unauthorized access even if primary credentials are stolen or compromised.

    • Priority: High

  • Conduct Annual Security Awareness Training:

    • Action: Establish and enforce a mandatory annual security awareness training program for all employees.

    • Benefit: Educates staff on current cyber threats (e.g., phishing, malware, social engineering) and best practices, transforming employees into a strong line of defense.

    • Priority: Medium

Technical Recommendations

  • Secure RDP Access on and :

    • Action (Immediate): Restrict direct RDP access from untrusted networks (e.g., the internet) via firewall rules.

    • Action (Preferred Long-Term): Implement a secure remote access solution, such as a Virtual Private Network (VPN) or an RDP Gateway, which requires strong authentication (preferably MFA) before allowing RDP sessions.

    • Action (Internal): Ensure strong, unique passwords and account lockout policies are enforced for all RDP accounts. Limit RDP access to only necessary users and IP addresses.

    • Benefit: Eliminates a common attack vector for initial system compromise and unauthorized access.

    • Priority: Critical

  • Review All Internet-Facing Services:

    • Action: Conduct a comprehensive review of all services exposed to the internet or less trusted internal networks to identify and secure other potentially vulnerable points.

    • Benefit: Proactively identifies and mitigates similar exposures that could lead to system compromises.

    • Priority: High

  • Implement Regular Patch Management:

    • Action: Ensure all operating systems, applications, and network devices are kept up-to-date with the latest security patches.

    • Benefit: Prevents exploitation of known vulnerabilities that attackers frequently target.

    • Priority: High
