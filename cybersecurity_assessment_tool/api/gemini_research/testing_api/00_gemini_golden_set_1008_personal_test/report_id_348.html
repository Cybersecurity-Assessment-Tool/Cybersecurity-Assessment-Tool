Recommendations

To address the identified gaps and enhance Symmetry Architecture’s cybersecurity posture, the following actionable recommendations are provided. These are prioritized to address the most significant risks first.

Immediate Actions

  • Implement MFA for Email Access: Mandate Multi-Factor Authentication for all email accounts across the organization. This is one of the most effective controls against credential theft and phishing for email systems and should be prioritized.

  • Implement MFA for Computer Login: Require MFA for all computer login processes, especially for systems that access sensitive data or network resources. This adds a critical layer of security against unauthorized access from stolen or guessed credentials.

  • Develop and Implement an Employee Acceptable Use Policy (AUP): Create a comprehensive AUP outlining appropriate use of company IT resources, data, and internet access. Ensure all employees read, understand, and formally acknowledge the policy.

Ongoing Improvements

  • Conduct Annual Security Awareness Training for All Employees: Supplement new employee training with mandatory annual refresher courses for all staff. Training should cover current threats, phishing recognition, safe browsing, data handling, and company policies.

  • Perform Regular External Network Scans: Engage a qualified cybersecurity professional to conduct regular (e.g., quarterly or semi-annual) external network vulnerability scans. These scans are essential to identify open ports, services, and software versions that may expose the organization to attack.

  • Establish a Vulnerability Management Program: Implement a systematic process for regularly identifying, assessing, and remediating known vulnerabilities across all IT assets. This should include leveraging vulnerability databases, patching schedules, and asset inventory.

  • Review and Update Security Policies Regularly: Ensure that all security policies, including the AUP, data handling procedures, and incident response plans, are reviewed and updated at least annually or when significant changes occur in the IT environment or threat landscape.
