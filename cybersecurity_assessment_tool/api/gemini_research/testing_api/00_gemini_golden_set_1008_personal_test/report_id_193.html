Overview

This report details the cybersecurity posture of Terraform Global based on a review of organizational security controls, a targeted network scan, and an assessment of existing risks. The analysis reveals several critical areas requiring immediate attention, particularly concerning multi-factor authentication for computer access, new employee security awareness training, and network service configurations. While some foundational controls are in place, identified gaps pose significant risks to data integrity, confidentiality, and availability.

Analysis of Security Control Gaps:

Two critical gaps were identified through the security questionnaire:

  • Lack of MFA for Computer Logins: Not requiring multi-factor authentication for computer logins significantly increases the risk of unauthorized access, especially through compromised credentials. This is a primary target for attackers seeking initial access or lateral movement.

  • No Security Awareness Training for New Employees: While annual training is conducted, the absence of initial security awareness training for new employees leaves them vulnerable to social engineering, phishing, and other human-centric attacks from the start of their tenure, potentially exposing the organization prematurely.

Technical Analysis:

  • Nginx Version 1.18.0: While Nginx 1.18.0 was an official stable release, it was initially released in 2020. Newer stable branches (e.g., 1.24.x, 1.26.x) have since been released, providing updated features and critical security fixes. Depending on the underlying operating system’s patching cycle, this version may be receiving security updates, but it is not the latest mainline or stable version from the Nginx project. Running older software versions can expose systems to known vulnerabilities if not properly patched.

  • SSL Certificate Common Name Mismatch: The SSL certificate found on port 443 has a Common Name (www.acme-corp.com) that does not match the organization’s provided website domain (www.TerraformGlobal.org) or email domain (TerraformGlobal.org). This is a significant finding that could indicate:

    • A misconfiguration where a generic or placeholder certificate is being used.

    • The server is not intended for public access under the organization’s primary domain.

    • A potential trust issue if users access this service directly, leading to browser warnings.

    • An indicator of a test environment inadvertently exposed or repurposed without proper certificate update.

    This mismatch impacts trust, security, and the professional appearance of any public-facing service.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Action: Mandate MFA for all user logins to company computers and endpoints.

    • Benefit: Significantly reduces the risk of unauthorized access even if primary credentials are compromised.

    • Priority: Critical

  2. Establish Mandatory New Employee Security Awareness Training:

    • Action: Develop and implement a comprehensive security awareness training program that all new employees must complete during their onboarding process.

    • Benefit: Equips new hires with the knowledge to identify and report potential threats from day one, reducing the human attack surface.

    • Priority: High

  3. Address SSL Certificate Common Name Mismatch:

    • Action: Investigate the server at 192.168.10.5. If it is an internal service, ensure a proper internal CA-issued certificate is used. If it is intended for external access, ensure a publicly trusted certificate matching www.TerraformGlobal.org is installed.

    • Benefit: Restores trust, prevents browser warnings, and ensures secure, authenticated communication.

    • Priority: High

  4. Review and Update Nginx Server Software:

    • Action: Assess the Nginx server (1.18.0) on 192.168.10.5. Determine if it’s receiving current security patches from its operating system vendor. Plan for an upgrade to a currently supported and patched stable or mainline version of Nginx.

    • Benefit: Ensures the server benefits from the latest security fixes, performance improvements, and features, reducing exposure to known vulnerabilities.

    • Priority: Medium
