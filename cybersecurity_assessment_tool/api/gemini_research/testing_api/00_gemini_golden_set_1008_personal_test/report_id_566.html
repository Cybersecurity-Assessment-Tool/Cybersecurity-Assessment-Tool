Executive Summary

This report details the current cybersecurity posture of North Star Education, based on a network scan, a security questionnaire review, and a list of pre-existing organizational risks. The analysis reveals several critical vulnerabilities and significant gaps in security controls that collectively present a high risk to the organization’s information assets. Key findings include an exposed and highly outdated FTP service with anonymous access, a pervasive lack of Multi-Factor Authentication (MFA) across critical systems, absence of foundational security policies, and unaddressed legacy operating systems. Immediate and decisive action is required to mitigate these severe risks and establish a more robust security framework.

Recommendations

The following actionable recommendations are provided to address the identified risks and improve the overall cybersecurity posture of North Star Education.

  1. Upgrade/Replace FTP Service: The current FTP server (vsftpd 2.3.4) is severely outdated and has known critical vulnerabilities. It must be upgraded to the latest stable version or replaced with a more secure file transfer solution (e.g., SFTP, FTPS with explicit TLS, or a managed file transfer service) that supports modern encryption and authentication protocols.

  2. Disable Anonymous FTP: Immediately reconfigure the FTP server to disallow anonymous login. All access should require authenticated credentials.

  3. Restrict Network Access to FTP: Limit network access to the FTP server (on :port 21) to only necessary internal IPs or specific VPN tunnels. Consider placing it behind a firewall that restricts traffic to only trusted sources.

  4. Implement MFA for Email: Mandate Multi-Factor Authentication (MFA) for all email accounts across the organization, prioritizing administrative accounts and key personnel.

  5. Implement MFA for Computer Logins: Implement MFA for all workstation and server logins to enhance protection against unauthorized access.

  6. Implement MFA for Sensitive Data Systems: Enforce MFA for access to all systems containing sensitive or critical organizational data.

  7. Develop and Enforce an Acceptable Use Policy (AUP): Create and enforce a comprehensive AUP for all employees, clearly outlining proper use of organizational IT resources, internet, email, and security responsibilities.

  8. Integrate Security Training for New Hires: Integrate mandatory security awareness training into the onboarding process for all new employees to establish a strong security foundation from day one.

  9. Address Outdated Operating Systems (Outdated Windows Policy): . Plan and execute an upgrade project to migrate all Windows 7 workstations to a currently supported operating system (e.g., Windows 10/11) or replace outdated hardware. Implement a robust patch management system.

  10. Regular Security Audits: Conduct periodic internal and external vulnerability scans and penetration tests to proactively identify and address new weaknesses and ensure ongoing compliance.

  11. Continuous Security Awareness Training: Beyond initial onboarding, ensure all employees receive regular (at least annual) security awareness training, including phishing simulations, to reinforce best practices and keep them informed of evolving threats.

Conclusion

The current cybersecurity posture of North Star Education is critical. The presence of highly vulnerable services, coupled with significant gaps in security policies and the absence of Multi-Factor Authentication across key systems, creates a substantial attack surface and elevates the risk of data breaches, unauthorized access, and operational disruption. It is imperative that the recommendations outlined in this report are prioritized and implemented immediately to enhance the organization’s security defenses and protect its valuable assets.
