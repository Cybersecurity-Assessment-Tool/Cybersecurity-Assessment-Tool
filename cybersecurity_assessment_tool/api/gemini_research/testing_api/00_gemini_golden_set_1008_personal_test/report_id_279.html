Executive Summary

This report details the findings from a cybersecurity posture assessment conducted for Apex Legends Group. The assessment involved a review of organizational security controls, an external network scan, and an analysis of existing risk data. Key findings indicate critical gaps in Multi-Factor Authentication (MFA) implementation for computer logins and sensitive data systems, as well as a lack of security awareness training for new employees. Furthermore, an open MySQL port (3306) was identified, corroborating an existing "Database Exposure" risk with a high CVSS score of 7.5. These issues collectively represent a significant attack surface and potential for data compromise. Immediate recommendations focus on implementing MFA, enhancing employee training, and securing database access.

Technical Analysis

The presence of an open MySQL port (3306) running version 5.7.33 is a significant finding. While MySQL 5.7.33 is not critically outdated (version 8.x is the latest major release), the fact that it is open to the network, especially if exposed publicly or to a wide internal network segment, presents a considerable risk. Databases are prime targets for attackers due to the sensitive information they often contain. This finding directly correlates with the pre-existing risk of "Database Exposure" mentioned in the current risk register.

Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture of Apex Legends Group, the following recommendations are provided:

Immediate Recommendations

  • Implement Multi-Factor Authentication (MFA):

    • Enable and enforce MFA for all computer logins across the organization.

    • Implement and enforce MFA for access to all sensitive data systems.

  • Enhance Security Awareness Training:

    • Develop and implement mandatory security awareness training for all new employees during their onboarding process. This should cover phishing, social engineering, password hygiene, and data handling best practices.

  • Secure Database Access (Port 3306):

    • Immediately review and restrict network access to MySQL port 3306 (172.16.50.20) using firewall rules. Ensure it is only accessible from trusted internal IP addresses or specific management hosts, not from general network segments or the internet.

    • If internet access is required for specific applications, implement a secure gateway or VPN for access, avoiding direct exposure.

Long-Term Recommendations

  • Regular Software Updates:

    • Establish a regular patch management program for all software, including operating systems, applications, and database management systems (like MySQL). Consider upgrading MySQL 5.7.33 to the latest stable version (MySQL 8.x) to benefit from enhanced security features and continued support.

  • Comprehensive Security Audits:

    • Conduct periodic vulnerability assessments and penetration tests (VA/PT) to proactively identify and address security weaknesses.

    • Implement regular internal security audits to ensure compliance with security policies and best practices.

  • Security Policy Enforcement:

    • Review and update the employee acceptable use policy regularly. Ensure all employees acknowledge and adhere to it.

    • Reinforce annual security awareness training with more frequent, targeted micro-training modules or simulated phishing campaigns.
