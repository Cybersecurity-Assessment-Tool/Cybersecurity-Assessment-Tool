Analysis of Questionnaire Findings

The questionnaire responses reveal several significant gaps in the organization’s security posture:

  • Positive Security Awareness Training: The organization does conduct security awareness training for new and existing employees, which is a positive indicator for promoting a security-conscious culture.

Recommendations

To address the identified risks and enhance the overall cybersecurity posture of Falcon Heavy, the following recommendations are provided, prioritized by severity:

  1. Implement Multi-Factor Authentication (MFA) Across All Critical Systems (Critical Priority):

    • Email Access: Immediately deploy and enforce MFA for all email accounts.

    • Computer Login: Implement MFA for all user logins to company computers and endpoints.

    • Sensitive Data Systems: Mandate MFA for accessing all systems that store, process, or transmit sensitive organizational or customer data.

  2. Develop and Implement an Employee Acceptable Use Policy (High Priority):

    • Create a comprehensive AUP that clearly outlines expectations for employee behavior regarding the use of company IT resources, data handling, internet usage, and security protocols.

    • Distribute the AUP to all employees and require acknowledgment of understanding and compliance.

    • Integrate AUP review into annual security awareness training.

  3. Harden SSH Service Configuration (Medium Priority):

    • Implement Key-Based Authentication: Disable password-based authentication for SSH and enforce the use of SSH keys.

    • Disable Root Login: Prevent direct root login via SSH. Use Strong Passphrases: For SSH keys, enforce strong passphrases.

    • Rate Limiting/Firewall Rules: Implement firewall rules and/or a host-based intrusion detection/prevention system (HIDS/HIPS) to limit connection attempts to port 22.

    • Change Default Port: Consider changing the default SSH port (22) to a non-standard port to reduce automated scan noise.

    • Keep Software Updated: Ensure the SSH daemon and underlying operating system are kept up-to-date with the latest security patches.

  4. Expand Network Visibility (Low Priority):

    • Conduct more comprehensive network scans regularly to discover all open ports, services, and associated versions. This will allow for the identification of potential vulnerabilities in specific software versions.

    • Implement an asset inventory system to keep track of all network-connected devices.

Conclusion

The assessment highlights that while Falcon Heavy has a good foundation in security awareness training, critical vulnerabilities exist due to the widespread absence of Multi-Factor Authentication and a formal Acceptable Use Policy. Addressing these foundational issues, alongside hardening public-facing services like SSH, should be the immediate focus to significantly improve the organization’s cybersecurity posture and reduce the risk of compromise. Regular security audits and continuous monitoring are essential to maintain a robust defense against evolving threats.
