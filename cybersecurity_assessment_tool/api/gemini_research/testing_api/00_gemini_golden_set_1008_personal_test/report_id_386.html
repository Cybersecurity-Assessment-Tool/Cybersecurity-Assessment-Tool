Executive Summary

This report provides a comprehensive cybersecurity assessment for Falcon Heavy, integrating insights from a recent network scan, organizational security posture questionnaire, and a review of existing vulnerabilities. The analysis reveals several critical security gaps and high-risk areas that require immediate attention. Key findings include a severely outdated and insecure FTP service allowing anonymous access, significant multi-factor authentication (MFA) deficiencies for email and sensitive data systems, and the absence of foundational security policies and training for new employees. While some controls are in place (MFA for computer login, annual security awareness), the identified critical risks pose a significant threat to the organization’s data integrity, confidentiality, and availability. Detailed recommendations are provided to mitigate these risks and enhance the overall security posture.

Technical Analysis

The discovery of vsftpd version 2.3.4 on port 21 with anonymous FTP login enabled is a critical security vulnerability. vsftpd 2.3.4, released in 2011, is severely outdated and contains a well-known backdoor vulnerability (CVE-2011-2523) that allows remote code execution. Furthermore, permitting anonymous FTP access presents an immediate and severe risk of unauthorized data access, modification, and potential data exfiltration without any authentication. This combination creates an extremely exploitable entry point into the network.

Recommendations

To address the identified risks and enhance Falcon Heavy’s cybersecurity posture, the following recommendations are provided:

  1. Immediately Address FTP Vulnerabilities:

    • Disable Anonymous FTP: Configure the vsftpd service on to disallow anonymous login immediately.

    • Upgrade/Replace FTP Service: Upgrade vsftpd to the latest stable and patched version, or, preferably, replace the insecure FTP service with a more secure file transfer solution such as SFTP (SSH File Transfer Protocol) or SCP (Secure Copy Protocol) to ensure data confidentiality and integrity during transfer.

    • Firewall Rules: Implement strict firewall rules to limit access to file transfer services only from trusted IP addresses or internal networks, if external access is not required.

  2. Implement Multi-Factor Authentication (MFA):

    • MFA for Email: Mandate and deploy MFA for all employee email accounts to significantly reduce the risk of phishing and credential theft leading to account compromise.

    • MFA for Sensitive Data Systems: Implement and enforce MFA for all systems that handle or store sensitive organizational data, ensuring an additional layer of security beyond passwords.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities for using organizational IT resources, internet access, email, and data handling. This policy should be formally communicated to all employees and acknowledged.

  4. Enhance Security Awareness Training Program:

    • Onboarding Training: Institute mandatory security awareness training for all new employees as part of their onboarding process to educate them on common cyber threats, company policies, and best practices from day one.

    • Continuous Training: Continue with annual security awareness training for all employees, ensuring content is updated to reflect current threat landscapes and organizational changes.

  5. Upgrade Outdated Operating Systems:

    • Develop and execute a plan to upgrade all machines currently running Windows 7 to a supported operating system (e.g., Windows 10 or 11). Prioritize machines that handle sensitive data or are exposed to the internet.
