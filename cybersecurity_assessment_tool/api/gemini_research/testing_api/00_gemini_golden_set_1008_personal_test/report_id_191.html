Executive Summary

This report details the cybersecurity posture of Vanguard Heritage, based on a review of organizational data, a simulated network scan, and existing risk information. The assessment reveals several strengths, particularly in the implementation of Multi-Factor Authentication (MFA) for email and computer logins, and the presence of an acceptable use policy. However, critical gaps were identified concerning MFA for sensitive data systems and the complete absence of regular security awareness training for employees. A network scan against the localhost revealed an open SSH port, which, while on a loopback address, represents a potential service exposure that warrants review, especially when correlated with an existing risk of "Localhost Exposed". Immediate recommendations focus on addressing these critical gaps and enhancing overall security awareness and controls.

Recommendations

Based on the analysis, the following recommendations are provided to enhance Vanguard Heritageâ€™s cybersecurity posture:

Immediate Actions (High Priority)

  1. Implement MFA for Sensitive Data Systems: Immediately deploy and enforce Multi-Factor Authentication (MFA) for all systems that handle or store sensitive organizational or customer data. This is a critical control to prevent unauthorized access even if credentials are compromised.

  2. Establish New Employee Security Awareness Training: Develop and implement a mandatory security awareness training program for all new employees before they gain access to company systems. This should cover common threats, organizational policies, and best practices.

  3. Initiate Annual Security Awareness Training: Develop and roll out an annual security awareness training program for all employees. Regular training helps reinforce security best practices, educates staff on evolving threats, and maintains a strong human firewall.

Technical Recommendations

  1. Investigate "Localhost Exposed" Risk: Thoroughly investigate the existing "Localhost Exposed" risk. Understand its origin, the specific nature of the exposure, and the implications of a CVSS score of 10.0. Prioritize remediation based on this detailed understanding.

  2. Review SSH on Localhost: For the open SSH port (22/tcp) on , assess if this service is genuinely required. If not, disable it. If it is required, ensure it is configured securely:

    • Disable password authentication, enforcing key-based authentication.

    • Disable root login.

    • Configure strong ciphers and MACs.

    • Ensure regular patching of the SSH server software.

General Best Practices

  1. Regular Vulnerability Scanning and Penetration Testing: Implement a schedule for regular internal and external vulnerability scanning and periodic penetration testing to proactively identify and address security weaknesses.

  2. Incident Response Plan Review: Ensure an up-to-date and tested incident response plan is in place to effectively handle and recover from security incidents.

  3. Data Backup and Recovery: Verify that robust data backup and recovery procedures are in place and regularly tested to minimize data loss in the event of a cyberattack or system failure.
