Control Gaps Identified

Based on the questionnaire, two critical control gaps were identified:

  • Lack of MFA for Email Access: Not requiring Multi-Factor Authentication (MFA) for email significantly increases the risk of account compromise through phishing or credential stuffing attacks. This can lead to business email compromise (BEC), data breaches, and unauthorized access to sensitive communications.

  • Absence of an Employee Acceptable Use Policy (AUP): The lack of a formal AUP leaves employees without clear guidelines on the appropriate use of organizational IT resources, potentially leading to misuse, data leakage, and non-compliance with industry regulations.

Technical Findings Analysis

The scan revealed one open port:

  • Port 3306 (MySQL): A MySQL database server, version 5.7.33, was identified as running and accessible. MySQL 5.7 reached its End-of-Life (EOL) in October 2023. This means it no longer receives security updates or bug fixes, making it highly susceptible to known vulnerabilities and exploits. Exposing an EOL database directly to the network without adequate protection is a severe security risk.

Recommendations

Based on the assessment, the following recommendations are provided to improve Deep Root Ecology’s cybersecurity posture. Recommendations are categorized by the primary area of concern.

Database Security

  • Immediate Action (Firewall): Implement strict firewall rules to restrict access to port 3306 (MySQL) to only necessary, trusted internal IPs or systems. Database ports should never be directly exposed to the internet or an untrusted network segment.

  • Medium-term Action (Upgrade): Plan and execute an upgrade of the MySQL database to a currently supported version (e.g., MySQL 8.0). Ensure the upgrade process includes comprehensive testing and a rollback plan.

  • Long-term Action (Segmentation & VPN): Implement network segmentation to isolate the database server within a secure network zone. For any necessary remote administration, mandate the use of a Virtual Private Network (VPN) or secure jump host.

Identity and Access Management (IAM)

  • Immediate Action (MFA for Email): Implement and enforce mandatory Multi-Factor Authentication (MFA) for all email accounts across the organization. Prioritize robust MFA methods like authenticator apps or hardware tokens over SMS-based MFA.

  • Medium-term Action (MFA Expansion): Review all critical internal applications and services and extend MFA requirements to these systems where feasible.

Policy and Governance

  • Immediate Action (AUP Development): Develop a comprehensive Employee Acceptable Use Policy (AUP) that clearly outlines guidelines for the use of company IT resources, data handling, and expected employee behavior regarding cybersecurity.

  • Medium-term Action (AUP Implementation): Communicate the newly developed AUP to all employees. Obtain signed acknowledgements to ensure understanding and compliance. Integrate the AUP into new employee onboarding processes and periodic security awareness training.

Conclusion

The cybersecurity assessment highlights several areas where Deep Root Ecology can significantly enhance its security posture. Addressing the critical gaps related to MFA for email, the absence of an AUP, and the exposed End-of-Life MySQL database should be prioritized. By implementing the recommended actions, the organization can substantially reduce its attack surface, mitigate identified risks, and foster a more secure operating environment.
