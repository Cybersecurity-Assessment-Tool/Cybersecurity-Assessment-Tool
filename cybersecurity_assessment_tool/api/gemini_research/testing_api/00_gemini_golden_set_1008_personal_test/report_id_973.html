Technical Analysis

The Nginx web server running on port 443 with version 1.18.0 is significantly outdated. This version was released in 2020 and reached its End-of-Life (EOL) in 2023, meaning it no longer receives security updates or bug fixes. Running EOL software exposes the organization to numerous known vulnerabilities that could be exploited by attackers.

Additionally, the SSL certificate’s common name (‘www.acme-corp.com‘) does not match the organization’s website domain (‘www.IronOakFurniture.net‘). This discrepancy suggests a potential misconfiguration, an expired certificate, or the use of a certificate not intended for this specific service, which could lead to browser warnings and trust issues for users.

Risk Assessment

Based on the questionnaire responses, technical scan findings, and absence of pre-existing vulnerabilities (Input 3 was empty), the following risks have been identified:

Lack of MFA for Computer Logins Employees can access company computers without multi-factor authentication, making these systems highly susceptible to credential-based attacks. High

Absence of Employee Acceptable Use Policy Without a formal AUP, employees may not be aware of appropriate conduct regarding IT resource usage, increasing operational and security risks. Medium

Recommendations

To mitigate the identified risks and enhance the organization’s cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) Universally:

    • Mandate MFA for all employee computer logins.

    • Implement MFA for access to all systems containing sensitive data.

  2. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining proper use of company IT resources.

    • Distribute the AUP to all employees and obtain signed acknowledgments.

    • Incorporate AUP review into new hire onboarding and annual training.

  3. Establish Mandatory Annual Security Awareness Training:

    • Implement a recurring program for all employees covering current cyber threats, best practices, and organizational policies.

    • Track completion rates and reinforce training as needed.

  4. Upgrade and Maintain Web Server Software:

    • Immediately upgrade Nginx to the latest stable and supported version.

    • Establish a regular patching schedule for all web server software and underlying operating systems.

  5. Correct SSL Certificate Configuration:

    • Investigate the SSL certificate common name mismatch on ‘192.168.10.5‘.

    • Install a valid SSL certificate issued for the correct domain (‘IronOakFurniture.net‘) and ensure it is properly configured.

Conclusion

The assessment reveals several areas requiring immediate attention to bolster the security posture of Iron Oak Furniture. Addressing the critical gaps in MFA, employee awareness, policy, and particularly the outdated web server software, will significantly reduce the organization’s attack surface and overall risk exposure. Proactive implementation of the provided recommendations is crucial for protecting organizational assets and maintaining data integrity.
