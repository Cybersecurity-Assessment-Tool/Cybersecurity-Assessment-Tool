Executive Summary

This report details the cybersecurity posture of Nebula Creative based on an analysis of organizational data, a targeted network scan, and a review of existing risks. The assessment identified several critical areas requiring immediate attention, including significant gaps in Multi-Factor Authentication (MFA) implementation for sensitive data systems, the absence of an acceptable use policy, and a lack of consistent security awareness training. Furthermore, the network scan revealed Remote Desktop Protocol (RDP) exposure on a critical internal asset, complementing a pre-existing RDP exposure risk. Addressing these findings with the provided recommendations is crucial for enhancing Nebula Creative’s security stance and mitigating potential threats.

3.2. Findings

The scan identified the following open port and service:

  • Port: 3389

  • Service: ms-wbt-server (Microsoft Windows Remote Desktop Protocol - RDP)

  • Status: Open

Remote Desktop Protocol (RDP) is a proprietary protocol developed by Microsoft, which allows a user to graphically control a remote computer. While essential for remote administration, exposing RDP directly to the network without proper security controls (e.g., VPN, strong MFA, network-level authentication, IP restrictions) poses a significant security risk. It is a frequent target for brute-force attacks and exploitation, potentially leading to unauthorized access and system compromise.

5. Recommendations

Based on the assessment findings, the following recommendations are provided to enhance Nebula Creative’s cybersecurity posture:

5.1. Strengthen Access Controls

  • Implement MFA for Sensitive Data Systems: Immediately deploy and enforce Multi-Factor Authentication for all systems storing or processing sensitive data. This adds a crucial layer of security against credential theft.

5.2. Enhance Security Policies and Training

  • Develop an Employee Acceptable Use Policy: Create and implement a clear, comprehensive Acceptable Use Policy (AUP) that outlines employee responsibilities for using company IT resources. Ensure all employees read and acknowledge the policy annually.

  • Conduct Annual Security Awareness Training: Establish and maintain a mandatory annual security awareness training program for all employees. Training should cover topics such as phishing, social engineering, data handling, and policy adherence.

5.3. Secure Remote Access Services

  • Mitigate RDP Exposure on 10.10.10.51 and 10.10.10.50:

    • Immediate Fix: Restrict direct internet access to RDP. Implement network-level access control lists (ACLs) to limit RDP access to only trusted administrative workstations or VPN-connected users.

    • Long-term Solution: Deploy a Virtual Private Network (VPN) solution for all remote access to internal resources, including RDP. Ensure the VPN itself utilizes strong authentication mechanisms like MFA.

    • Hardening Measures: If RDP must be exposed, implement Network Level Authentication (NLA), enforce strong password policies for RDP users, and regularly patch the operating system to protect against known vulnerabilities. Consider changing the default RDP port.
