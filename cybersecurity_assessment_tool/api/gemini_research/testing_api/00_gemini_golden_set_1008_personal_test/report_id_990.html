Executive Summary

The assessment for Orchid Isle reveals a mixed security posture with notable weaknesses in fundamental security controls. While a recent network scan showed no immediately open ports on the specific target host 192.168.0.5, significant gaps exist in areas such as Multi-Factor Authentication (MFA) for email and computer logins, the absence of an Acceptable Use Policy (AUP), and a lack of regular annual security awareness training. A pre-existing organizational risk of an unencrypted web server on Port 80 was also identified, which requires further investigation given the current scan results showing Port 80 as closed. Addressing these policy and control weaknesses is paramount to improving overall security and reducing potential attack vectors.

Recommendations

Based on the assessment, the following recommendations are provided to enhance Orchid Isle’s cybersecurity posture:

  • Implement Multi-Factor Authentication (MFA):

    • Immediately implement MFA for all email accounts to significantly reduce the risk of phishing and credential compromise.

    • Roll out MFA for all computer logins, especially for administrative accounts and systems processing sensitive data, to mitigate unauthorized system access.

  • Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a clear and comprehensive AUP outlining acceptable and unacceptable use of company IT resources.

    • Ensure all employees read, understand, and formally acknowledge the AUP, and provide regular refreshers.

  • Enhance Security Awareness Training:

    • Establish a mandatory annual security awareness training program for all employees, covering topics such as phishing, social engineering, password best practices, and secure data handling.

    • Supplement annual training with regular security updates and simulated phishing campaigns to keep employees informed of evolving threats.

  • Investigate and Remediate Pre-existing Risks:

    • Thoroughly investigate the status of the "Unencrypted Web Server" risk (PR-001). Confirm if Port 80 is indeed closed as per the current scan, or if the risk pertains to another system or context not covered by this specific assessment.

    • If any web servers are found to be running unencrypted services, ensure they are configured to use HTTPS with valid SSL/TLS certificates to protect data in transit.

  • Conduct Regular Security Audits:

    • Implement a schedule for regular vulnerability scanning and penetration testing across the organization’s network to proactively identify and address security weaknesses.

    • Perform periodic reviews of security policies, configurations, and incident response plans to ensure they remain effective and aligned with current best practices and threat landscapes.
