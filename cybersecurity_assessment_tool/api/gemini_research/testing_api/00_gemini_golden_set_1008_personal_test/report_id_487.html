Executive Summary

This report provides a comprehensive assessment of Midnight Oil Studios’s cybersecurity posture, based on a security questionnaire, a network scan, and a review of existing risks. The analysis reveals a mixed security landscape. While the organization demonstrates strengths in areas such as multi-factor authentication (MFA) for sensitive data systems and computers, and has an acceptable use policy, critical gaps persist. Specifically, the absence of MFA for email access and a lack of dedicated security awareness training for new employees present significant attack vectors. Furthermore, an open SSH service was identified on a target system, necessitating further investigation and hardening. This report details these findings and provides actionable recommendations to enhance the overall security posture.

Recommendations

The following recommendations are provided to mitigate the identified risks and enhance Midnight Oil Studios’s overall cybersecurity posture.

Implement MFA for Email Access

  • Action: Enable and enforce multi-factor authentication (MFA) for all organizational email accounts.

  • Details: This is a critical step that significantly reduces the risk of email account compromise, even if an attacker obtains a user’s password. Consider using hardware tokens or authenticator applications as preferred MFA methods. Communicate the importance and benefits of MFA to all users during implementation.

Establish Comprehensive New Employee Security Awareness Training

  • Action: Develop and implement mandatory security awareness training for all new hires. This training should be completed before new employees gain access to company systems or sensitive data.

  • Details: This initial training should cover essential security topics such as phishing identification, social engineering tactics, strong password practices, acceptable use policy, data handling guidelines, and incident reporting procedures. Tailor the content to be relevant to the organization’s specific threat landscape and policies.

Secure SSH Service

  • Action: Conduct a thorough review of the SSH configuration on the host .

  • Details:

    • Update SSH Daemon: Ensure the SSH server is running the latest stable and patched version to mitigate known vulnerabilities.

    • Disable Password Authentication: Prioritize and enforce key-based authentication, which is generally more secure than passwords.

    • Disable Root Login: Configure SSH to prevent direct root logins. Instead, users should log in with a non-root account and use ‘sudo‘ for elevated privileges.

    • Implement Rate Limiting/Intrusion Detection: Use tools like ‘fail2ban‘ or similar intrusion prevention systems to block IP addresses attempting brute-force attacks.

    • Restrict Access: If feasible, limit SSH access to specific trusted IP addresses or networks using firewall rules.

    • Monitor Logs: Regularly review SSH access logs for suspicious activity or failed login attempts.

Proactive Vulnerability Management

  • Action: Implement regular, scheduled vulnerability scanning and, if appropriate, penetration testing across the entire network infrastructure.

  • Details: This proactive approach will help identify and address potential weaknesses, including outdated software versions, misconfigurations, and new vulnerabilities, before they can be exploited by malicious actors. Ensure that scans include detailed version detection for all services, providing actionable data for remediation.
