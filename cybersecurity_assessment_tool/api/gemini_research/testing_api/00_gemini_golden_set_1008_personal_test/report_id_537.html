Executive Summary

This report details the cybersecurity posture of Harbor Light Foundation based on a review of organizational data, a recent network scan, and existing risk information. The organization demonstrates good practices in certain areas, such as requiring Multi-Factor Authentication (MFA) for computer logins and sensitive data systems, and maintaining an Acceptable Use Policy.

However, several critical gaps and high-severity risks have been identified. Specifically, the lack of MFA for email access and inconsistent annual security awareness training present significant vulnerabilities. Most critically, the network scan revealed an open port 8080 with an HTTP title indicating "TOP SECRET DB", directly contradicting a prior assessment that labeled this port as "secured". This discrepancy demands immediate and thorough investigation as it suggests a highly sensitive system might be unintentionally exposed. Overall, the organization’s security posture is a mix of strengths and severe weaknesses that require urgent attention, particularly regarding the exposed database and the consistency of fundamental security controls.

Recommendations

The following actionable recommendations are provided to address the identified risks and improve the overall cybersecurity posture of Harbor Light Foundation:

  1. Implement MFA for Email Access (High Priority):

    • Immediately enable and enforce Multi-Factor Authentication (MFA) for all employee email accounts. This significantly reduces the risk of account compromise due to stolen or weak credentials.

    • Provide clear instructions and support for employees to enroll in MFA.

  2. Establish Annual Security Awareness Training (High Priority):

    • Develop and implement a mandatory, comprehensive annual security awareness training program for all employees.

    • Ensure training covers current threats such as phishing, ransomware, social engineering, and best practices for data handling and strong password hygiene.

    • Track completion rates and reinforce key messages throughout the year.

  3. Immediate Investigation of Port 8080 (Critical Priority):

    • Verify Exposure: Immediately investigate the system or application running on port 8080 on 10.5.5.5. Determine if "TOP SECRET DB" genuinely reflects sensitive data being processed or stored.

    • Assess Necessity: If the service is not intended for public or network-wide access, implement strict access controls. This may include firewall rules to restrict access to specific IP addresses or subnets, or moving the service behind an internal-only network segment.

    • Secure Configuration: If the service must be accessible, ensure it is configured securely with strong authentication, authorization, and encryption (e.g., HTTPS). Change or remove any information that explicitly reveals sensitive content in HTTP titles or banners.

    • Vulnerability Scan: Conduct a thorough vulnerability assessment and penetration test specifically targeting the application/service on port 8080.

  4. Reconcile Conflicting Risk Information for Port 8080 (Critical Priority):

    • Audit Previous Assessment: Review the prior assessment that classified port 8080 as "secured" (severity 0.0). Determine the methodology, scope, and date of that assessment.

    • Update Risk Register: Based on the current scan findings, update the organization’s risk register to accurately reflect the critical severity of the exposed port 8080.

    • Process Improvement: Implement processes to ensure that risk assessments are regularly updated and validated against current technical findings to prevent such critical discrepancies from arising.
