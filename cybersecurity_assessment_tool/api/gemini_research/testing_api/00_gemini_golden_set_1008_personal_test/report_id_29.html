5. Recommendations

The following recommendations are provided to address the identified security gaps and vulnerabilities:

  1. Implement Multi-Factor Authentication (MFA) for Email Access: Enable MFA for all user accounts accessing email. This significantly reduces the risk of unauthorized access even if credentials are compromised.

  2. Implement Multi-Factor Authentication (MFA) for Computer Login: Require MFA for logging into all company computers, especially those accessing sensitive data or internal networks. This should include both local and domain accounts.

  3. Establish Comprehensive Security Awareness Training:

    • Implement mandatory security awareness training for all new employees during onboarding.

    • Conduct annual refresher security awareness training for all employees, covering topics like phishing, social engineering, password hygiene, and data handling best practices.

  4. Secure Remote Desktop Protocol (RDP) Access: The presence of an open RDP port () on (and potentially as per existing risks) represents a critical attack vector.

    • Immediate Action (Close Port): Restrict RDP access to only trusted internal networks via firewall rules. If external access is absolutely necessary, it should be done through a Virtual Private Network (VPN).

    • Enhance Authentication: If RDP must be exposed, enforce strong, unique passwords for all RDP accounts, implement account lockout policies, and consider network-level authentication (NLA). Ideally, use MFA for RDP access.

    • Patch Management: Ensure the operating system and RDP client/server are fully patched against known vulnerabilities.

    • Least Privilege: Ensure users connecting via RDP have only the minimum necessary privileges.
