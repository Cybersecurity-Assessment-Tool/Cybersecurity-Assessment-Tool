Executive Summary

This report provides a comprehensive assessment of Harbor Light Foundation’s current cybersecurity posture, synthesizing insights from a network scan, organizational security questionnaire responses, and pre-existing risk data.

The assessment reveals an acceptable posture with significant areas for improvement. While the organization demonstrates strong controls around Multi-Factor Authentication (MFA) adoption, critical gaps exist concerning foundational security practices. Key findings include:

  • A high-severity Database Exposure due to an open MySQL port, already identified in the risk register and confirmed by the network scan.

  • A critical gap in security policy, specifically the absence of an Employee Acceptable Use Policy (AUP).

  • A significant weakness in security awareness training, lacking an annual program for all employees.

Addressing these high-priority items is crucial for strengthening the overall security posture and reducing the attack surface against common threats. Recommendations provided herein offer actionable steps for immediate and long-term improvements.

Analysis:

The discovery of an open MySQL database server (version 5.7.33) on port 3306 is a critical finding. MySQL 5.7 is an older major version (though 5.7.33 itself is not critically old, reaching EOL in October 2023), and the exposure of a database port, especially to the wider network (if this target is publicly accessible or insufficiently segmented internally), presents a significant attack vector. An attacker could potentially exploit vulnerabilities in the MySQL service, attempt brute-force attacks on database credentials, or gain unauthorized access to sensitive data stored within the database. This finding directly correlates with the "Database Exposure" risk already present in the organization’s risk register.

Recommendations

The following actionable recommendations are provided to address the identified risks and enhance the organization’s cybersecurity posture.

Database Exposure (High Severity)

  • Immediate Fix (Firewall Rules): Implement strict firewall rules to restrict access to port 3306 (MySQL) from unauthorized IP addresses or networks. Only allow connections from trusted application servers or specific administration hosts.

  • Long-Term Fix (Secure Access & Segmentation):

    • If remote access is required, implement a Virtual Private Network (VPN) solution to ensure all connections to the database are encrypted and authenticated.

    • Consider network segmentation to isolate the database server within a secure network zone, separate from user-facing networks.

    • Review the necessity of MySQL 5.7.33. While not immediately End-of-Life, planning an upgrade to a currently supported major version (e.g., MySQL 8.0) is advisable for long-term security and access to the latest features and patches.

    • Ensure strong authentication mechanisms are in place for database users and review all existing user accounts for least privilege principles.

Lack of Employee Acceptable Use Policy (AUP) (High Severity)

  • Develop and Implement AUP: Immediately develop and formally implement a comprehensive Acceptable Use Policy. This policy should clearly outline expectations for employee behavior regarding company IT resources, data handling, internet usage, email, and software installations.

  • Employee Acknowledgment: Ensure all employees read, understand, and formally acknowledge the AUP, preferably annually or upon significant updates.

  • Integration: Integrate the AUP into new employee onboarding processes and existing HR policies.

Inadequate Annual Security Awareness Training (SAT) (Medium Severity)

  • Establish Annual Training Program: Develop and implement a mandatory annual security awareness training program for all employees. This training should cover current threats (e.g., phishing, ransomware), data protection best practices, password hygiene, and the importance of reporting suspicious activities.

  • Continuous Education: Beyond annual training, consider implementing ongoing security awareness initiatives, such as monthly security tips, phishing simulation exercises, and micro-learning modules to keep security top-of-mind.

  • Tailored Content: Ensure training content is relevant to the organization’s specific risks and the roles of its employees.
