Executive Summary

This report provides a comprehensive assessment of Terraform Global’s cybersecurity posture, integrating insights from a network scan, organizational security questionnaire responses, and a review of existing risks. The analysis reveals several critical areas requiring immediate attention alongside commendable existing controls. While strong multi-factor authentication (MFA) is implemented for computer logins and sensitive data systems, significant vulnerabilities exist concerning email MFA and a complete lack of employee security awareness training. Technically, the presence of an outdated Nginx web server version and, critically, an SSL certificate domain mismatch pose immediate risks to the organization’s web presence and trust. Addressing these identified gaps is crucial for enhancing Terraform Global’s overall security resilience against modern cyber threats.

443/tcp open https Nginx 1.18.0 SSL Cert: commonName=www.acme-corp.com

Key Technical Findings:

  • Outdated Nginx Version: The web server running on port 443 is Nginx version 1.18.0. This version was released in April 2020 and is not the latest stable branch (currently 1.24.x/1.26.x). Running outdated software can expose the organization to known vulnerabilities that have been patched in newer releases.

  • SSL Certificate Domain Mismatch: The SSL certificate presented by the Nginx server is for www.acme-corp.com, which does not match the organization’s primary domain, TerraformGlobal.com. This is a critical misconfiguration that will trigger browser warnings, erode user trust, and could potentially facilitate Man-in-the-Middle (MITM) attacks if not promptly corrected.

Recommendations

The following recommendations are provided to address the identified risks and enhance Terraform Global’s overall cybersecurity posture. Recommendations are prioritized by severity.

Critical Recommendations

  • R-004: Immediately Correct SSL Certificate Domain Mismatch:

    • Acquire and deploy a valid SSL/TLS certificate issued for TerraformGlobal.com (or relevant subdomains) on the web server at 192.168.10.5.

    • Verify that the certificate chain is correctly configured and all intermediate certificates are installed.

    • Implement automated certificate renewal processes to prevent expiration.

High Priority Recommendations

  • R-001: Implement Multi-Factor Authentication (MFA) for Email:

    • Enable and enforce MFA for all employee email accounts. This is a fundamental control against credential theft and phishing.

    • Educate employees on the importance and process of using MFA for email.

    • Consider adaptive MFA policies based on access location or device.

  • R-002: Establish Comprehensive Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new employees during their onboarding process.

    • Conduct annual refresher security awareness training for all existing employees.

    • Training should cover topics such as phishing, social engineering, password hygiene, data handling, and incident reporting.

    • Consider regular simulated phishing exercises to reinforce training effectiveness.

Medium Priority Recommendations

  • R-003: Update Nginx Web Server to Latest Stable Version:

    • Plan and execute an upgrade of the Nginx web server on 192.168.10.5 to the latest stable version (e.g., 1.24.x or 1.26.x at the time of this report).

    • Ensure proper testing of the web application after the upgrade to prevent service disruption.

    • Establish a regular patch management schedule for all operating systems and applications, especially internet-facing ones.

Conclusion

Terraform Global demonstrates a foundational commitment to security by implementing MFA for sensitive systems. However, critical vulnerabilities in email security, human element training, and web server configuration necessitate immediate action. By prioritizing and implementing the recommendations outlined in this report, Terraform Global can significantly strengthen its security posture, mitigate identified risks, and better protect its assets against evolving cyber threats. Ongoing monitoring, regular assessments, and continuous improvement are key to maintaining a robust cybersecurity defense.
