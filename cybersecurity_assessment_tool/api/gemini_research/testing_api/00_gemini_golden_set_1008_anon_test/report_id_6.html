6. Recommendations

Based on the analysis of the provided data, the following recommendations are provided to improve the cybersecurity posture:

6.1 Immediate Actions (Critical & High Risks)

  1. Implement Multi-Factor Authentication (MFA):

    • Prioritize the immediate implementation of MFA for all email access.

    • Implement MFA for all access to sensitive data systems. This is a critical control to prevent unauthorized access even if primary credentials are compromised.

  2. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines acceptable and unacceptable use of company IT resources, data, and systems.

    • Ensure all employees review, understand, and formally acknowledge the AUP. Integrate it into onboarding processes.

  3. Establish Mandatory Annual Security Awareness Training:

    • Implement a program for mandatory, annual security awareness training for all employees.

    • Training should cover topics such as phishing, social engineering, data handling, password hygiene, and incident reporting.

6.2 Technical Verifications & Best Practices

  1. Verify and Retire "Unencrypted Web Server" Risk:

    • Investigate the "Unencrypted Web Server" risk in the client’s risk register. Given the current scan shows port 80 as closed, verify if the underlying vulnerability has been resolved and if the port is intentionally closed or moved.

    • If confirmed resolved and no longer applicable, formally retire this risk from the organization’s risk register. If port 80 is intended to be open, ensure it redirects to HTTPS and only serves encrypted traffic.

  2. Regular Network Scanning and Vulnerability Management:

    • Implement a schedule for regular internal and external network scans to proactively identify open ports, services, and potential vulnerabilities.

    • Establish a robust vulnerability management program to continuously identify, assess, prioritize, and remediate technical vulnerabilities.
