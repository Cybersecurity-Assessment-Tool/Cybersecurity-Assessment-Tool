Technical Findings

The Nmap scan reveals an Nginx web server running on port 443 (HTTPS) with version 1.18.0. Nginx 1.18.0 was released in April 2020 and has since reached End-of-Life (EOL). This version is significantly outdated and is highly likely to contain known vulnerabilities that have been patched in newer releases. Running EOL software exposes the organization to known exploits and significantly increases the risk of compromise for the web server and potentially the underlying system.

Recommendations

To mitigate the identified risks and significantly enhance the overall cybersecurity posture, the following actionable recommendations are provided:

  • Implement MFA for Email Access: Mandate Multi-Factor Authentication (MFA) for all employee email accounts. This provides a crucial additional layer of security, making it significantly harder for attackers to gain unauthorized access even if primary credentials are stolen.

  • Implement MFA for Computer Logins: Deploy MFA for all user logins to company computers, laptops, and internal systems. This strengthens endpoint security and protects against credential-based attacks targeting internal resources.

  • Develop and Enforce an Acceptable Use Policy (AUP): Create a comprehensive Acceptable Use Policy (AUP) that clearly defines employee responsibilities, acceptable use of company IT resources, internet usage guidelines, and data handling procedures. Ensure all employees are trained on and formally acknowledge this policy.

  • Update Nginx Web Server: Immediately upgrade the Nginx web server (currently version 1.18.0) to the latest stable version (e.g., 1.26.x) or the latest Long Term Support (LTS) release (e.g., 1.24.x). Establish a robust patch management process for all internet-facing services to ensure software remains up-to-date and protected against known vulnerabilities.

  • Regular Vulnerability Scanning and Penetration Testing: Implement a schedule for regular external and internal vulnerability scanning, complemented by periodic penetration testing, to proactively identify and address security weaknesses before they can be exploited.

  • Ongoing Security Awareness Training: Continue and expand security awareness training to cover topics like advanced phishing tactics, social engineering, secure remote work practices, and the importance of reporting suspicious activities.

Conclusion
