6. Recommendations

Based on the comprehensive assessment, the following actionable recommendations are provided to enhance the organization’s cybersecurity posture and mitigate identified risks:

6.1. Multi-Factor Authentication (MFA)

  • Implement MFA for Computer Logins: Deploy a robust MFA solution across all workstations and systems that employees use to log into company resources. This should be mandatory for all users.

  • Implement MFA for Sensitive Data Systems: Enforce MFA for access to all systems that store, process, or transmit sensitive organizational data. Prioritize systems with the highest impact on confidentiality, integrity, and availability.

6.2. Policies and Governance

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly defines employee responsibilities, acceptable use of company IT resources (hardware, software, network, internet), and data handling guidelines. Ensure all employees acknowledge, understand, and comply with this policy through mandatory sign-offs.

6.3. Security Awareness Training (SAT)

  • Initiate Mandatory Security Awareness Training for New Employees: Integrate a comprehensive security awareness program into the onboarding process for all new hires. This training should cover essential topics such as phishing, social engineering, password best practices, and data protection.

  • Establish Annual Mandatory Security Awareness Training: Conduct regular (at least annual) mandatory security awareness training for all employees. The training content should be updated to reflect current threat landscapes and organizational policies. Consider phishing simulation exercises to test and improve employee vigilance.

6.4. Technical Security

  • Investigate and Address "Unencrypted Web Server" Risk: Conduct a thorough investigation into the "Unencrypted Web Server" vulnerability identified in the pre-existing risk register.

    • If the server is indeed active and reachable (perhaps on a different IP or port not covered by this specific scan): Ensure all web traffic is encrypted using HTTPS/TLS. Implement strict redirects from HTTP (port 80) to HTTPS (port 443) and consider blocking direct access to port 80 if not strictly required for redirection.

    • If the risk is outdated or pertains to a different asset: Update the organizational vulnerability register to accurately reflect the current state and asset information.

  • Regular Network Scanning and Vulnerability Management: Implement a schedule for regular, comprehensive network scans and vulnerability assessments across the entire IT infrastructure to proactively identify exposed services, outdated software versions, and configuration weaknesses.
