Risk Assessment

This section synthesizes findings from the security control review, technical scan results, and pre-existing risk data to present a comprehensive list of identified risks.

Identified Risks

Risk Name Overview Severity
Lack of MFA for Sensitive Data Systems The organization does not require Multi-Factor Authentication for access to sensitive data systems. This significantly increases the risk of unauthorized access in the event of compromised credentials. High
Absence of Employee Acceptable Use Policy There is no formal Acceptable Use Policy (AUP) for employees. This lack of clear guidelines increases legal, security, and reputational risks due to potential misuse of company assets or data. High
Insufficient Annual Security Awareness Training While new employees receive training, existing employees do not receive annual security awareness training. This leads to diminished awareness of evolving threats and best practices over time. Medium
Unencrypted Web Server (Discrepancy) While the recent network scan indicated Port 80 is closed, the organization’s existing risk register explicitly notes an "Unencrypted Web Server" where "Port 80 is open." Recommendations

To address the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided.

Immediate Actions

  • Implement MFA for Sensitive Data Systems: Mandate Multi-Factor Authentication for all systems and applications handling sensitive organizational data. Prioritize systems with the highest impact or most critical data.

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP outlining the proper use of company resources, data, and systems. Ensure all employees are required to read, understand, and acknowledge this policy. Integrate it into onboarding and annual compliance programs.

  • Verify and Address Port 80 Discrepancy: Immediately investigate the discrepancy regarding Port 80.

    • If Port 80 is indeed closed as per the recent scan, update the risk register to reflect this and confirm no web services are inadvertently exposed.

    • If Port 80 is actually open (contradicting the scan), ensure any web service running on it is protected with HTTPS/TLS encryption. If it’s not needed, close the port.

Long-Term Strategy

  • Establish Annual Security Awareness Training Program: Implement a mandatory annual security awareness training program for all employees. This training should cover current threats (e.g., phishing, social engineering), data handling best practices, and organizational security policies.

  • Regular Review of Security Policies: Periodically review and update all security policies, including the AUP and MFA requirements, to adapt to evolving threats and organizational changes.

  • Continuous Vulnerability Management: Implement a continuous vulnerability scanning and management program to regularly assess network and application security. This helps in proactive identification and remediation of new vulnerabilities.

  • Incident Response Planning: Develop and regularly test an incident response plan to ensure the organization can effectively detect, respond to, and recover from security incidents.
