3.1. Analysis of Security Control Gaps

The questionnaire revealed several high-priority gaps in fundamental security practices:

  • Lack of MFA for Email (Q1): Not requiring Multi-Factor Authentication for email access significantly increases the risk of account compromise through phishing and credential stuffing attacks. Email accounts are often a primary vector for initial access to corporate networks and data.

  • Missing Acceptable Use Policy (Q4): The absence of an Acceptable Use Policy (AUP) leaves employees without clear guidelines for acceptable behavior regarding company assets and data. This can lead to inconsistent practices, potential misuse, increased data breach risks, and compliance violations.

  • No Security Awareness Training for New Employees (Q5): New employees are often prime targets for social engineering attacks due to their unfamiliarity with organizational security protocols. Without initial security awareness training, they are more susceptible to phishing, malware, and other attacks, posing an immediate risk to the organization.

6. Recommendations

To address the identified risks and significantly improve the overall cybersecurity posture, the following actionable recommendations are provided.

6.1. Critical Recommendations

  • Immediately disable Anonymous FTP access: Review the FTP server configuration to disable anonymous login. If any form of public access is strictly required, implement stringent read-only permissions and ensure no sensitive data is accessible. Consider alternative, more secure methods for public file distribution.

  • Upgrade or replace vsftpd: Urgently upgrade the FTP server to a current, supported version of vsftpd or replace it with a more secure file transfer solution (e.g., SFTP, SCP, or a cloud-based file sharing service). Ensure all services are patched regularly. If the FTP server is not essential, consider decommissioning it entirely.

6.2. High-Priority Recommendations

  • Implement MFA for Email: Mandate Multi-Factor Authentication for all organizational email accounts. This is a fundamental security control that significantly reduces the risk of account compromise due to stolen credentials and phishing attacks.

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that outlines proper use of organizational IT assets, data handling procedures, internet usage guidelines, and expectations for employee conduct. Ensure all employees review, understand, and formally acknowledge this policy.

  • Initiate Security Awareness Training for New Employees: Implement mandatory security awareness training for all new hires as part of their onboarding process. This training should educate them on common threats like phishing, social engineering, malware, and safe data handling practices to mitigate immediate risks.

6.3. Moderate-Priority Recommendations

  • Upgrade End-of-Life Operating Systems: Plan and execute an upgrade path for all workstations and servers running Windows 7 to a currently supported operating system (e.g., Windows 10/11 or a modern server OS). Prioritize systems handling sensitive data or those with direct internet exposure. Implement a robust patch management program for all systems.

  • Enhance Regular Security Awareness Training: While existing training for all employees is commendable, ensure its content is routinely updated to cover evolving threats, new attack techniques, and current best practices. Incorporate phishing simulations and interactive modules.
