3.1. Analysis of Questionnaire Gaps

The questionnaire highlights three significant areas of concern that represent high risks to the organization:

  • Lack of MFA for Email Access: Without multi-factor authentication, email accounts are highly susceptible to compromise through phishing, brute-force attacks, or credential stuffing. Compromised email accounts can lead to business email compromise (BEC), data exfiltration, and further network intrusion.

  • Lack of MFA for Sensitive Data Systems: Protecting access to systems containing sensitive data is paramount. The absence of MFA for these critical systems significantly increases the risk of unauthorized access, data breaches, and regulatory non-compliance.

  • Absence of an Employee Acceptable Use Policy (AUP): An AUP defines expected employee behavior regarding company IT assets, data handling, and internet usage. Its absence can lead to inconsistent security practices, increased insider risk, and difficulties in enforcing security standards.

6. Recommendations

Based on the assessment, the following actionable recommendations are provided to improve the cybersecurity posture of [Organization Name].

  1. Implement MFA for Email Access:

    • Action: Mandate multi-factor authentication for all email accounts across the organization, including administrative accounts, using a robust MFA solution.

    • Benefit: Significantly reduces the risk of email account compromise, a leading cause of data breaches and business email compromise incidents.

  2. Implement MFA for Sensitive Data Systems:

    • Action: Enforce multi-factor authentication for all access attempts to systems containing sensitive organizational or client data, including databases, file shares, and critical applications.

    • Benefit: Provides an essential additional layer of security, making it substantially harder for unauthorized users to gain access to critical information assets.

  3. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Action: Create a comprehensive AUP that clearly outlines acceptable use of all company IT resources (hardware, software, networks, data), data handling procedures, and security expectations. This policy should be formally communicated to all employees, and signed acknowledgments should be obtained.

    • Benefit: Establishes clear boundaries and expectations for employee behavior, reduces misuse of IT assets, minimizes insider risk, and fosters a stronger security culture.

  4. Review and Verify Unencrypted Web Server Risk:

    • Action: Investigate the status of any web servers previously identified as unencrypted on Port 80 (as per the pre-existing risk register). While the recent network scan indicated Port 80 is currently closed, it is crucial to confirm if this server has been properly decommissioned, secured with encryption (e.g., HTTPS), or if the service is merely offline.

    • Benefit: Ensures that potential historical vulnerabilities are properly addressed and do not re-emerge, protecting against unencrypted data transmission should the service become active in the future.
