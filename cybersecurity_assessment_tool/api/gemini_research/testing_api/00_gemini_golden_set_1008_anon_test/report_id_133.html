Analysis of Findings

The presence of Nginx version 1.18.0 on an internet-facing port is a significant concern. Nginx 1.18.0 was released in April 2020, and its stable branch (1.18.x) reached end-of-life status on April 26, 2022. This means it no longer receives security updates or bug fixes, making it highly vulnerable to known exploits and misconfigurations. Running end-of-life software on a publicly accessible service significantly increases the attack surface and risk of compromise.

Recommendations

Based on the assessment, the following recommendations are provided to enhance the organization’s cybersecurity posture:

  1. Upgrade/Patch Nginx Server:

    • Immediately upgrade the Nginx web server on port 443 from version 1.18.0 to the latest stable release (e.g., 1.24.x or newer) to ensure all known security vulnerabilities are patched.

    • Implement a regular patching schedule for all internet-facing systems and critical infrastructure.

  2. Implement MFA for Computer Logins:

    • Mandate and enforce Multi-Factor Authentication for all computer logins across the organization. This significantly reduces the risk of unauthorized access even if primary credentials are compromised.

    • Evaluate suitable MFA solutions compatible with existing infrastructure and user experience needs.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities and acceptable behaviors regarding the use of organizational IT assets, data, and networks.

    • Ensure the AUP is communicated to all employees, understood, and formally acknowledged upon hiring and annually thereafter.

  4. Establish Security Awareness Training for New Employees:

    • Implement mandatory security awareness training as part of the onboarding process for all new hires.

    • This training should cover essential security topics such as phishing, strong password practices, data handling, clean desk policy, and the procedure for reporting security incidents.

  5. Conduct Regular Security Audits and Vulnerability Assessments:

    • Perform periodic external and internal vulnerability scans and, where appropriate, penetration tests to proactively identify and address security weaknesses.

    • Regularly review and update security policies and procedures to adapt to evolving threat landscapes and technological changes.
