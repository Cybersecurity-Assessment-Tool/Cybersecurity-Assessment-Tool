5. Risk Assessment

Based on the analysis of the questionnaire responses, technical scan data, and pre-existing risk information, the following risks have been identified:

Identified Cybersecurity Risks
Risk Name Overview Severity
Lack of Multi-Factor Authentication (MFA) for Computer Logins The organization does not enforce MFA for computer logins. This significantly increases the risk of unauthorized access if credentials are compromised, as a single factor (password) is sufficient for authentication. High
Absence of Employee Acceptable Use Policy (AUP) There is no formal Acceptable Use Policy for employees. This lack of clear guidelines can lead to inconsistent or insecure usage of company IT resources, increasing legal, compliance, and security risks. Medium
Open SSH Service (Port 22) An SSH service is exposed externally. While SSH is a secure protocol, misconfigurations, weak credentials, or unpatched software can lead to unauthorized access, remote code execution, or data exfiltration. Low / Informational

6. Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of [Organization Name], the following recommendations are provided:

  • Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Action: Enforce MFA for all user accounts logging into organizational computers and systems.

    • Benefit: Dramatically reduces the risk of credential compromise leading to unauthorized access.

    • Priority: Critical

  • Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Action: Create a comprehensive AUP outlining acceptable and unacceptable use of company IT resources, data, and communication systems. Ensure all employees are trained on it and acknowledge understanding.

    • Benefit: Provides clear guidelines, reduces misuse of resources, enhances legal protection, and fosters a more secure work environment.

    • Priority: High

  • Secure Open SSH Service (Port 22):

    • Action:

      1. Ensure SSH daemon (e.g., OpenSSH) is running the latest stable version with all security patches applied.

      2. Configure SSH to use strong, key-based authentication exclusively, disabling password authentication.

      3. Disable root login via SSH.

      4. Restrict SSH access to specific trusted IP addresses or networks if possible.

      5. Implement a firewall rule to rate-limit connection attempts to port 22.

      6. Consider changing the default SSH port (22) to a non-standard port, although this offers security through obscurity, not true security.

    • Benefit: Reduces the attack surface and risk of unauthorized access or brute-force attacks against the SSH service.

    • Priority: Medium

Implementing these recommendations will significantly strengthen the organization’s defenses and improve its overall resilience against cyber threats.
