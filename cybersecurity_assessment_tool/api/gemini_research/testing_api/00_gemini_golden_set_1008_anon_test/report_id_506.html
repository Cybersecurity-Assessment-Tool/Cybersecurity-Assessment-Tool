Key Observations

The questionnaire review highlights a significant gap in the organization’s security posture: the absence of a formal Employee Acceptable Use Policy (AUP). This lack of policy can lead to inconsistent employee behavior regarding IT resource usage, increasing the risk of data loss, unauthorized access, and compliance violations. While other controls like MFA and security awareness training appear to be in place, a foundational policy like an AUP is crucial for defining expected conduct and mitigating internal risks.

Technical Findings Analysis

The network scan revealed an open FTP service on port 21/tcp. Specifically, the service identified is . This version of vsftpd was released in 2011 and is End-of-Life, known to have several critical vulnerabilities, including a severe backdoor vulnerability (CVE-2011-2523) that allows arbitrary command execution. Furthermore, the FTP service is configured to allow anonymous login, which means any user can connect without authentication and potentially transfer files, leading to data exposure, unauthorized uploads, or even further system compromise if exploitable write permissions are present. This represents a critical and immediate security risk.

Recommendations

The following recommendations are provided to address the identified risks, prioritized by severity.

Critical Priority Actions

  • Address Outdated FTP Service with Anonymous Access:

    • Immediately disable anonymous FTP access to prevent unauthorized file transfers.

    • Evaluate the necessity of running an FTP service. If FTP is not required, disable or uninstall the service to remove the attack surface.

    • If FTP functionality is essential, upgrade to a currently supported and patched version of vsftpd or, preferably, migrate to a more secure file transfer protocol such as SFTP (SSH File Transfer Protocol) or FTPS (FTP Secure), which encrypt data in transit and offer stronger authentication mechanisms.

High Priority Actions

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly defines the appropriate use of organizational IT assets, data handling procedures, internet usage guidelines, and acceptable software installation policies.

    • Ensure all employees review, understand, and formally acknowledge the AUP, ideally during onboarding and annually thereafter.

    • Integrate AUP awareness into the regular security awareness training program to reinforce its importance and implications.

Medium Priority Actions

  • Upgrade Outdated Operating Systems (Windows 7):

    • Conduct a comprehensive inventory to identify all workstations and servers currently running End-of-Life operating systems, specifically Windows 7.

    • Plan and execute a phased upgrade strategy to migrate these systems to a currently supported operating system (e.g., Windows 10 or 11) within a defined timeframe.

    • For systems that cannot be immediately upgraded, implement stringent compensating controls. This includes isolating them on a separate network segment, applying strict firewall rules, implementing advanced endpoint detection and response (EDR) solutions, and monitoring them for suspicious activity until they can be decommissioned or upgraded.
