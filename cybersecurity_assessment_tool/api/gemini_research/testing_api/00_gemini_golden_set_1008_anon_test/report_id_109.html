Analysis of Questionnaire Findings

The questionnaire responses highlight significant weaknesses in key security areas:

  • MFA Gaps: While MFA is correctly implemented for email access, its absence for computer logins and sensitive data systems poses a severe risk. This leaves internal systems and critical data highly vulnerable to credential theft attacks (e.g., phishing, brute-force, malware-based credential extraction).

  • Inconsistent Security Awareness Training: The lack of mandatory annual security awareness training for all employees means that personnel may not be regularly updated on current threats, best practices, and organizational policies. This significantly increases susceptibility to social engineering attacks (e.g., phishing, pretexting) and human error, which are leading causes of security incidents.

Analysis of Technical Findings

The most significant finding from the network scan is the presence of an Nginx web server running version 1.18.0. Nginx 1.18.0 was released in April 2020. As of the assessment date (November 2025), this version is severely outdated and no longer receives security updates or bug fixes. Running outdated software, especially on public-facing services like a web server, exposes the organization to numerous known vulnerabilities that threat actors can easily exploit. These vulnerabilities could lead to unauthorized access, website defacement, data exfiltration, or the establishment of a foothold for further attacks into the internal network.

Recommendations

To address the identified risks and significantly improve the overall cybersecurity posture, the following actionable recommendations are provided:

  1. Implement MFA for Computer Logins:

    • Action: Mandate and implement Multi-Factor Authentication for all user accounts logging into organizational computers, workstations, and internal network resources.

    • Priority: High

    • Rationale: MFA significantly reduces the risk of unauthorized access even if primary credentials are stolen, forming a critical layer of defense.

  2. Implement MFA for Sensitive Data Systems:

    • Action: Deploy and enforce Multi-Factor Authentication for access to all systems and applications containing or processing sensitive organizational and customer data.

    • Priority: Critical

    • Rationale: Protecting sensitive data with MFA is paramount to prevent data breaches, comply with regulations, and maintain trust.

  3. Establish Mandatory Annual Security Awareness Training:

    • Action: Develop and implement a comprehensive, mandatory annual security awareness training program for all employees. This training should cover current threats, phishing recognition, data handling best practices, password hygiene, and organizational security policies.

    • Priority: High

    • Rationale: A well-informed workforce is the first line of defense against social engineering and human-error related security incidents.

  4. Upgrade/Patch Nginx Web Server:

    • Action: Immediately upgrade the Nginx web server from version 1.18.0 to the latest stable and supported version (e.g., a current stable branch like 1.24.x or newer). Prioritize this upgrade as the service is public-facing and highly vulnerable.

    • Priority: High

    • Rationale: Upgrading ensures that known vulnerabilities in older versions are mitigated and the server benefits from ongoing security patches and performance improvements.
