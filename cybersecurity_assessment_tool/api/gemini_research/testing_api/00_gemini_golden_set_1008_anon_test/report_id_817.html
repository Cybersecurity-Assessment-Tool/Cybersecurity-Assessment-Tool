Analysis of Technical Findings

The detection of Nginx version 1.18.0 on an open HTTPS port is a significant concern. Nginx 1.18.0 reached its End-of-Life (EoL) in April 2021. Running EoL software means it no longer receives crucial security updates, patches for newly discovered vulnerabilities, or bug fixes. This state leaves the web server highly susceptible to various known exploits, potentially allowing attackers to gain unauthorized access, deface the website, or compromise the underlying system.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Email Access:

    • Immediately enable and enforce Multi-Factor Authentication for all organizational email accounts.

    • Prioritize the adoption of phishing-resistant MFA methods (e.g., FIDO2 security keys) where feasible.

  2. Implement MFA for Sensitive Data Systems:

    • Deploy and enforce MFA for all access points to systems that handle or store sensitive organizational or client data.

    • Review access controls to ensure the principle of least privilege is applied alongside MFA.

  3. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Create a clear, comprehensive AUP that outlines the appropriate use of company IT resources, data handling, and internet usage.

    • Ensure all employees review, understand, and formally acknowledge the AUP, ideally on an annual basis or upon significant policy changes.

    • Incorporate AUP training into new employee onboarding and regular security awareness programs.

  4. Upgrade or Replace End-of-Life Nginx Server:

    • Prioritize upgrading the Nginx server to the latest stable, supported version (e.g., Nginx Open Source 1.25.x or Nginx Plus).

    • If an upgrade is not immediately feasible, consider temporarily isolating the server and implementing a robust Web Application Firewall (WAF) as an interim protective measure, understanding this does not eliminate the underlying risk.

    • Establish a regular patching and update management process for all internet-facing systems.
