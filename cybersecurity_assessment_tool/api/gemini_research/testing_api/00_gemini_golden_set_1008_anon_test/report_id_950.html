Key Gaps Identified

The most significant gaps identified from the questionnaire are the complete lack of Multi-Factor Authentication (MFA) for:

  • Email access

  • Computer login

  • Sensitive data systems access

The absence of MFA significantly increases the risk of credential compromise and unauthorized access, even if user passwords are weak or stolen.

Recommendations

The following recommendations are provided to address the identified risks and improve the overall security posture. Prioritization is given to critical findings.

Prioritized Recommendations

  • Implement Multi-Factor Authentication (MFA) (Critical Priority):

    • Email Access: Immediately implement MFA for all email accounts. Consider solutions like Microsoft 365 MFA, Google Workspace MFA, or third-party identity providers.

    • Computer Login: Deploy MFA for all corporate computer logins. Solutions like Windows Hello for Business, Duo Security, or Okta can be integrated.

    • Sensitive Data Systems: Enforce MFA for access to all systems containing sensitive organizational or client data.

  • Address Outdated FTP Server and Anonymous Access (Critical Priority):

    • Decommission or Upgrade FTP Server: Immediately decommission the vsftpd 2.3.4 server if itâ€™s not essential. If FTP functionality is required, migrate to a modern, patched, and securely configured FTP/SFTP/FTPS server (e.g., a recent version of ProFTPD, Pure-FTPd, or an SFTP server).

    • Disable Anonymous Access: If the FTP server cannot be immediately replaced, disable anonymous login to prevent unauthorized access. Ensure all accounts have strong passwords and implement access controls based on the principle of least privilege.

    • Review Data on FTP Server: Conduct an immediate audit of data stored on the FTP server to identify and remove any sensitive information.

  • Upgrade Outdated Operating Systems (High Priority):

    • Upgrade Windows 7 Workstations: Plan and execute a migration for all workstations running Windows 7 to a currently supported operating system, such as Windows 10 or 11. Ensure that these systems receive regular security updates and patches.

    • Hardware Refit (Long-Term): For hardware that does not support newer operating systems, plan for hardware refresh cycles to accommodate modern, secure platforms.

  • Maintain Security Awareness Training:

    • Continue regular security awareness training for all employees, emphasizing the importance of strong passwords, MFA, phishing detection, and safe computing practices.

  • Review and Update Acceptable Use Policy:

    • Ensure the employee acceptable use policy is current and effectively communicated to all staff, reflecting best practices for security.
