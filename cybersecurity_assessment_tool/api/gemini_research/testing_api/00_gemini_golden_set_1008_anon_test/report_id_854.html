Recommendations

The following recommendations are provided to address the identified risks and improve the organization’s cybersecurity posture. Recommendations are prioritized based on their potential impact and urgency.

  1. Immediately Address Vulnerable FTP Service (Critical Priority)

    • Action: Urgently upgrade vsftpd to the latest stable version or, preferably, replace it with a more secure file transfer solution (e.g., SFTP, SCP) that uses encryption and strong authentication.

    • Justification: vsftpd 2.3.4 contains a critical backdoor vulnerability (CVE-2011-2523) allowing remote command execution.

  2. Disable Anonymous FTP Access (High Priority)

    • Action: Configure the FTP server to disable anonymous login. Ensure that only authenticated users with strong, unique credentials can access the service, and limit access privileges to the absolute minimum required.

    • Justification: Anonymous access allows unauthenticated users to interact with the FTP server, posing risks of data exposure, unauthorized file transfers, and potential leverage for further attacks.

  3. Implement Multi-Factor Authentication (MFA) Across All Critical Systems (Critical Priority)

    • Action: Deploy and rigorously enforce MFA for all user accounts accessing email, logging into computers, and accessing sensitive data systems. This significantly reduces the risk of successful attacks stemming from compromised credentials.

    • Justification: The current lack of MFA for these critical access points leaves the organization highly susceptible to credential stuffing, phishing, and brute-force attacks.

  4. Develop and Implement an Employee Acceptable Use Policy (AUP) (High Priority)

    • Action: Create a comprehensive AUP that clearly defines acceptable and unacceptable use of organizational IT resources, data handling procedures, and employees’ security responsibilities. Ensure all employees formally acknowledge and adhere to this policy.

    • Justification: A clear AUP provides essential guidelines for employees, reducing human error, preventing misuse of resources, and supporting compliance efforts.

  5. Establish Mandatory Annual Security Awareness Training for All Employees (High Priority)

    • Action: Implement a program for mandatory annual security awareness training for all employees, covering current threat landscapes (e.g., phishing, ransomware, social engineering) and secure computing best practices.

    • Justification: Continuous training is vital to keep employees informed and vigilant against evolving cyber threats, transforming them into a strong line of defense rather than a vulnerability. Initial new hire training is a good start but insufficient for ongoing threats.

  6. Upgrade Outdated Operating Systems (Medium Priority - Immediate Planning)

    • Action: Initiate a project to identify all workstations running Windows 7 and prioritize their upgrade to a currently supported operating system (e.g., Windows 10 or 11). For systems that cannot be immediately upgraded, implement compensating controls like network segmentation and enhanced monitoring.

    • Justification: Windows 7 is an end-of-life operating system that no longer receives security updates, leaving systems highly vulnerable to known exploits and malware.
