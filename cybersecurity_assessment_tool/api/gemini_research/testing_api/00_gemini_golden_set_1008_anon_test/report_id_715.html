Recommendations

To address the identified risks and enhance ’s cybersecurity posture, the following recommendations are provided:

Immediate Actions (Critical & High Risks)

  1. Implement Multi-Factor Authentication (MFA):

    • Enable and enforce MFA for all email accounts (R1).

    • Implement MFA for all computer logins, especially for administrative accounts and systems processing sensitive data (R2).

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines acceptable and unacceptable use of company IT resources, data, and communication systems (R3).

    • Ensure all employees read, understand, and acknowledge the AUP, ideally during onboarding and annually thereafter.

  3. Enhance Security Awareness Training:

    • Implement mandatory annual security awareness training for all employees, covering current threats like phishing, social engineering, and best practices for data handling (R4).

    • Supplement annual training with regular security bulletins or simulated phishing exercises.

  4. Secure Exposed SSH Service:

    • Conduct an in-depth review of the SSH server configuration on (R5).

    • Enforce strong authentication methods, preferably SSH key-based authentication with passphrases, and disable password-based logins.

    • Implement MFA for SSH access if possible.

    • Restrict SSH access to specific trusted IP addresses or networks using firewall rules.

    • Ensure the SSH server software is up-to-date with the latest security patches.

    • Implement logging and monitoring for all SSH connection attempts and activity.

    • Consider changing the default SSH port (22) to a non-standard port to reduce automated scan noise, though this is not a strong security control on its own.

Ongoing Measures

  • Regular Vulnerability Scanning & Penetration Testing: Conduct periodic vulnerability scans, including authenticated scans, and penetration tests to uncover new technical weaknesses.

  • Incident Response Plan: Develop, document, and regularly test an Incident Response Plan (IRP) to ensure the organization can effectively detect, respond to, and recover from security incidents.

  • Policy Review: Regularly review and update all security policies and procedures to reflect changes in technology, threats, and business operations.
