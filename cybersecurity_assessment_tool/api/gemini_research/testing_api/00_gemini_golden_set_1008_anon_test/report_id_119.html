6. Recommendations

The following recommendations are provided to address the identified risks and improve the organization’s cybersecurity posture:

6.1 Multi-Factor Authentication (MFA)

  • Implement MFA for Computer Logins: Immediately enable and enforce MFA for all user accounts logging into organizational computers and networks.

  • Implement MFA for Sensitive Data Systems: Extend MFA enforcement to all systems containing or providing access to sensitive organizational data.

6.2 Security Awareness Training

  • New Employee Training Program: Develop and implement a mandatory security awareness training program for all new employees during their onboarding process.

  • Annual All-Employee Training: Establish and enforce a policy for mandatory annual security awareness training for all existing employees. Training should cover topics such as phishing, social engineering, password best practices, and incident reporting.

6.3 Network Security

  • Harden SSH Service:

    • Ensure strong, key-based authentication is enforced, and password authentication is disabled for SSH on .

    • Implement IP whitelisting to restrict SSH access only from known, necessary administrative IP addresses.

    • Consider moving SSH to a non-standard port to reduce automated scanning attempts.

    • Ensure the SSH server software is always up-to-date with the latest security patches.

  • Firewall Review: If external SSH access is not strictly required, block port 22 at the perimeter firewall.

6.4 Address Existing Vulnerabilities

  • Remediate “Localhost Exposed”: Conduct an immediate investigation into the “Localhost Exposed” vulnerability. This likely involves reviewing internal network configurations, firewall rules, and application bindings to ensure that internal-only services are not inadvertently exposed or accessible, especially given its critical CVSS score of 10.0.
