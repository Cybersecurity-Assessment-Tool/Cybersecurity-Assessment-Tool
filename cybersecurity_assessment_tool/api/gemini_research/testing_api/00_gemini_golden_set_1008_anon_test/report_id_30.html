Recommendations

Prioritized Action Items

Based on the assessment findings, the following recommendations are provided to enhance the organization’s cybersecurity posture:

  1. Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Action: Immediately implement and enforce Multi-Factor Authentication (MFA) for all user logins to company computers and endpoints. This should be a mandatory control across the organization.

    • Rationale: MFA is a fundamental security control that drastically reduces the risk of unauthorized access due to compromised or stolen credentials. Without it, a single password breach could lead to full system compromise.

    • Priority: High

  2. Review and Secure Web Server Exposure (Port 80):

    • Action: Conduct an internal audit to verify if any web services are intended to run on Port 80. If so, ensure they are either strictly confined to internal networks or secured with HTTPS (Port 443) using valid SSL/TLS certificates before public exposure. If no web services are intended for Port 80, confirm it remains closed and filtered at the perimeter.

    • Rationale: While the recent scan showed Port 80 as closed, the existence of a prior "Unencrypted Web Server" risk indicates this could be a recurring or planned vulnerability. Unencrypted web traffic exposes sensitive data to eavesdropping and interception, and can harm data integrity.

    • Priority: Medium (If a service exists or is planned for this port; otherwise, maintain its closed state)

  3. Regular Security Audits and Vulnerability Assessments:

    • Action: Establish a schedule for regular internal and external network scans, vulnerability assessments, and penetration testing.

    • Rationale: Continuous monitoring ensures that the organization’s security posture remains robust against evolving threats and helps identify new or re-emerging vulnerabilities before they can be exploited.

    • Priority: Ongoing
