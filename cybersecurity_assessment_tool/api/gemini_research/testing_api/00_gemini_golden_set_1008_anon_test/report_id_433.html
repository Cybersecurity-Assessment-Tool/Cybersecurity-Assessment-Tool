R-001 Lack of Security Awareness Training for New Employees High New employees are not receiving dedicated security awareness training during their onboarding process. This significantly increases the risk of them falling victim to phishing, social engineering attacks, or inadvertently violating security policies due to a lack of understanding.
R-002 Outdated Nginx Web Server Critical The Nginx web server running on port 443/HTTPS is version 1.18.0, which is End-of-Life (EOL). Recommendations

To mitigate the identified risks and enhance the organization’s cybersecurity posture, the following recommendations are provided:

R-001: Implement Mandatory Security Awareness Training for New Employees

  • Develop and implement a mandatory security awareness training program specifically for all new hires, to be completed during their initial onboarding period.

  • Topics should include phishing detection, social engineering tactics, password best practices, data handling policies, and incident reporting procedures.

  • Regularly update training materials to reflect current threat landscapes and provide refresher courses as needed.

R-002: Upgrade or Replace Outdated Nginx Web Server

  • Immediately plan and execute an upgrade of the Nginx web server to a currently supported, stable, and patched version (e.g., Nginx 1.24.x or 1.25.x mainline).

  • Prioritize this upgrade due to the critical nature of the vulnerability associated with End-of-Life software, especially on a public-facing server.

  • Ensure proper testing is performed in a staging environment before deploying any updates to production.

  • Alternatively, if an upgrade is not feasible or desired, plan for the replacement of the EOL software with a modern, secure, and supported web server solution.

  • Implement a regular patching and update schedule for all public-facing servers and critical infrastructure components.
