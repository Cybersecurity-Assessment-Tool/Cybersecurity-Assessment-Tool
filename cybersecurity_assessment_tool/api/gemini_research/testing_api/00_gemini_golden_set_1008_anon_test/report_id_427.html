Recommendations

To address the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  • Implement Multi-Factor Authentication (MFA):

    • Email Access: Immediately implement MFA for all email accounts to prevent unauthorized access even if passwords are compromised.

    • Computer Logins: Roll out MFA for employee computer logins, especially for devices accessing sensitive internal networks or data.

  • Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities for using organizational IT resources, data, and communication systems.

    • Ensure all employees read, understand, and acknowledge the AUP.

  • Establish Annual Security Awareness Training:

    • Implement a mandatory annual security awareness training program for all employees.

    • Training should cover current threats, phishing identification, social engineering tactics, password best practices, and data handling procedures.

  • Secure Open Port 22 (SSH):

    • Investigation: Determine the exact service running on port 22 and its purpose.

    • Hardening: Implement SSH best practices, including:

      • Disabling password authentication; use strong, key-based authentication only.

      • Disabling root login directly via SSH.

      • Implementing strong password policies for any remaining password-based access.

      • Restricting SSH access to known, trusted IP addresses using firewall rules.

      • Ensuring the SSH daemon is up-to-date to patch known vulnerabilities.

      • Implementing logging and monitoring for SSH access attempts.

  • Conduct Comprehensive Network Scans:

    • Perform regular, authenticated network vulnerability scans (both internal and external) to identify all open ports, running services, and specific software versions.

    • Utilize tools capable of identifying vulnerabilities associated with specific service versions.
