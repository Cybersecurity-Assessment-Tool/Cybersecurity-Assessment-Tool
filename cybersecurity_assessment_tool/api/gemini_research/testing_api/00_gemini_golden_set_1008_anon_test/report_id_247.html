Recommendations

The following recommendations are provided to address the identified risks and improve the organization’s cybersecurity posture.

  1. Recommendation for R-001: Implement MFA for Computer Login

    • Priority: Critical

    • Actions:

      • Deploy and enforce Multi-Factor Authentication (MFA) for all computer logins across the organization. This could involve using smart cards, biometrics, or mobile authenticator apps.

      • Integrate MFA with existing identity management systems where feasible.

    • Mitigation: Significantly reduces the risk of unauthorized access even if primary credentials are stolen.

  2. Recommendation for R-002: Establish Initial Security Awareness Training for New Employees

    • Priority: High

    • Actions:

      • Develop and implement a mandatory security awareness training program for all new employees as part of their onboarding process.

      • Cover topics such as phishing, social engineering, password hygiene, and data handling policies.

    • Mitigation: Educates new employees, making them the first line of defense against social engineering and other human-factor attacks from the outset.

  3. Recommendation for R-003: Implement Regular (Annual) Security Awareness Training for All Employees

    • Priority: High

    • Actions:

      • Implement mandatory annual security awareness training for all existing employees to keep them informed about evolving threats (e.g., phishing, ransomware) and best practices.

      • Regularly update training content to reflect current threat landscapes and organizational policies.

    • Mitigation: Maintains and enhances employee awareness, preventing degradation of the overall security posture and reducing susceptibility to evolving threats.

  4. Recommendation for R-004: Harden Exposed SSH Service

    • Priority: Medium

    • Actions:

      • Implement key-based authentication for SSH, disabling password authentication.

      • Ensure only necessary IP addresses or networks can access SSH (e.g., via firewall rules, VPNs).

      • Implement rate limiting on SSH login attempts to prevent brute-force attacks.

      • Regularly audit SSH configurations and ensure the SSH server software is kept up-to-date with the latest security patches.

      • Consider changing the default SSH port if external exposure is unavoidable.

    • Mitigation: Reduces the attack surface and makes brute-force attacks significantly harder, protecting against unauthorized access.
