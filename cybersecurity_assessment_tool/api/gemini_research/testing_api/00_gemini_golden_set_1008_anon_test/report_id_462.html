5. Risk Assessment

This section synthesizes findings from the security control review, technical scans, and pre-existing risk data. Risks are categorized by severity.

5.1. High Severity Risks

  • Lack of Multi-Factor Authentication (MFA) for Computer Login

    • Overview: The organization does not require MFA for users logging into computers. This significantly increases the risk of unauthorized access if credentials are stolen or compromised through phishing or other means.

    • Severity: High

  • Lack of Annual Security Awareness Training

    • Overview: While new employees receive security awareness training, existing employees do not receive annual refresher training. This leads to a degradation of security knowledge over time, making employees more susceptible to evolving social engineering tactics and phishing campaigns.

    • Severity: High

5.2. Medium Severity Risks

  • Absence of Employee Acceptable Use Policy (AUP)

    • Overview: The lack of an AUP leaves employees without clear guidelines regarding appropriate use of company IT resources, data handling, and internet browsing. This can lead to increased risks from malware, data leakage, and compliance issues.

    • Severity: Medium

  • Unencrypted Web Server (Historical Risk / Potentially Mitigated)

    • Overview: A pre-existing vulnerability noted an unencrypted web server on Port 80. However, the current network scan shows Port 80 as closed. This suggests the vulnerability is either remediated, or the asset is no longer active, or the risk refers to a different system not scanned. While the immediate technical risk on the scanned target appears mitigated, it’s crucial to confirm if this risk applies to other assets or if the remediation is complete and verified.

    • Severity (Historical): Medium (CVSS 5.0)

6. Recommendations

To address the identified risks and enhance the organization’s cybersecurity posture, the following recommendations are provided:

6.1. Immediate Actions (High Priority)

  • Implement MFA for Computer Logins: Immediately deploy and enforce Multi-Factor Authentication for all user accounts accessing company computers and systems. This is a critical control against credential theft.

  • Establish and Enforce Annual Security Awareness Training: Develop and implement a mandatory annual security awareness training program for all employees. This training should cover current threats, phishing detection, data handling best practices, and organizational policies.

6.2. Short-Term Actions (Medium Priority)

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that outlines employee responsibilities, acceptable use of company IT resources, data handling guidelines, and consequences for non-compliance. Ensure all employees acknowledge and understand the policy.

  • Verify Remediation of Historical Risks: Confirm the status of the "Unencrypted Web Server" risk. If Port 80 was indeed previously open, ensure a secure alternative (e.g., HTTPS with valid SSL/TLS certificates) is used for any web services, and verify that all affected systems are properly secured or decommissioned.

6.3. Long-Term Strategy (Continuous Improvement)

  • Regular Policy Review: Periodically review and update all security policies (including AUP, MFA policies, training policies) to ensure they remain current with evolving threats and organizational needs.

  • Proactive Threat Hunting and Monitoring: Implement advanced threat detection and monitoring solutions to identify and respond to suspicious activities promptly.

  • Regular Vulnerability Assessments and Penetration Testing: Conduct periodic vulnerability scans and penetration tests against all organizational assets to identify and remediate new security weaknesses before they can be exploited.
