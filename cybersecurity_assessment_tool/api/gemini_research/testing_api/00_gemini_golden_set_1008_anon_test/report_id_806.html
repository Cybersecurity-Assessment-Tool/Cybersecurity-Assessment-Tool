5. Recommendations

Based on the comprehensive assessment, the following recommendations are provided to address the identified risks and enhance the overall cybersecurity posture of [Organization Name].

5.1. Critical and High Priority Recommendations

  • Anonymous FTP Login Enabled:

    • Disable anonymous FTP login immediately.

    • If FTP access is required, configure it to use strong authentication (e.g., user accounts with strong passwords or client certificates).

    • Restrict FTP access to necessary users and IP addresses.

    • Implement logging and monitoring for all FTP activity.

  • Outdated FTP Service (vsftpd 2.3.4):

    • Upgrade vsftpd to the latest stable and supported version, or replace it with a more secure file transfer mechanism (e.g., SFTP, SCP).

    • Ensure all server software is regularly patched and updated.

  • Lack of MFA for email access:

    • Implement Multi-Factor Authentication (MFA) for email access.

  • Lack of MFA for log into computers:

    • Implement Multi-Factor Authentication (MFA) for log into computers.

  • Lack of MFA for access sensitive data systems:

    • Implement Multi-Factor Authentication (MFA) for access sensitive data systems.

5.2. Medium Priority Recommendations

  • Outdated Windows Policy:

    • Upgrade to Win 10

    • Refit hardware

  • Missing Employee acceptable use policy:

    • Develop and implement a comprehensive employee acceptable use policy.

5.3. General Best Practices and Long-Term Strategy

  • Implement a robust vulnerability management program to regularly identify, assess, and remediate security flaws.

  • Establish a comprehensive incident response plan and conduct regular drills.

  • Continuously review and update security policies and procedures to align with evolving threat landscapes.

  • Invest in ongoing security awareness training for all employees, beyond just annual refreshers.

  • Adopt a ’defense-in-depth’ strategy, employing multiple layers of security controls.
