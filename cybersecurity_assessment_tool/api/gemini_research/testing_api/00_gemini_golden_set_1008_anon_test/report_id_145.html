4.2 Technical Analysis

The primary technical finding is the presence of an Nginx web server running version 1.18.0. Nginx 1.18.0 was released in April 2020 and has reached its End-of-Life (EOL) status. EOL software no longer receives security updates, making it highly susceptible to known vulnerabilities and exploits. This poses a critical risk to the availability, integrity, and confidentiality of services hosted on this server.

6. Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Upgrade Nginx Web Server:

    • Immediately upgrade the Nginx web server from version 1.18.0 to a currently supported stable release (e.g., 1.24.x or newer) to ensure all known security vulnerabilities are patched.

    • Implement a regular patching and update schedule for all server software.

  2. Implement MFA for Email and Computer Logins:

    • Deploy and enforce Multi-Factor Authentication (MFA) for all employee email accounts.

    • Implement MFA for all corporate computer logins to protect against unauthorized access, especially for remote access.

  3. Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining acceptable use of company IT resources, data handling, and security expectations for all employees.

    • Ensure all employees review and acknowledge the AUP annually.

  4. Establish Comprehensive Security Awareness Training Programs:

    • Implement mandatory security awareness training for all new employees during their onboarding process.

    • Conduct annual, mandatory security awareness training for all employees, covering topics such as phishing, social engineering, password hygiene, and data handling best practices.

    • Consider implementing regular simulated phishing exercises to test employee vigilance.
