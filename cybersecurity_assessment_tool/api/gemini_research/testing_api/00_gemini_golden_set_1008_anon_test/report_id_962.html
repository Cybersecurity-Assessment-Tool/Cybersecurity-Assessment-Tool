Recommendations

The following recommendations are provided to address the identified vulnerabilities and improve the overall cybersecurity posture. These actions are prioritized by the severity of the associated risk.

Immediate Actions (Critical Risks)

  • Deactivate Anonymous FTP and Upgrade/Remove Outdated FTP Service:

    • Action: Immediately disable anonymous FTP access on Port 21/TCP. Restrict FTP access to authenticated users only, if the service must remain active.

    • Action: Assess the necessity of the FTP service. If it is not business-critical, remove the service entirely to eliminate the risk.

    • Action: If FTP is required, upgrade to a current, supported, and securely configured version (e.g., latest stable version, SFTP, or FTPS with strong authentication). Implement proper user authentication, authorization, and encryption.

    • Action: Ensure regular security patching and configuration reviews for any file transfer services.

Short-Term Actions (High Risks)

  • Implement Comprehensive Security Awareness Training:

    • Action: Develop and implement a mandatory security awareness training program for all new employees as part of their onboarding process, covering common threats like phishing, social engineering, and malware.

    • Action: Establish a recurring annual security awareness training program for all existing employees. This training should be interactive and updated regularly to reflect current threat landscapes.

    • Action: Conduct regular simulated phishing campaigns to test employee vigilance and reinforce the effectiveness of security awareness training. Provide immediate feedback and supplementary training for those who fall victim.

Long-Term Actions (Medium Risks)

  • Upgrade Outdated Operating Systems:

    • Action: Develop a strategic plan to migrate all workstations to a currently supported operating system (e.g., or ). Prioritize the upgrade of machines handling sensitive data.

    • Action: Conduct a hardware compatibility assessment. For hardware incompatible with newer OS versions, plan for timely replacements.

    • Action: Implement a robust patch and update management system to ensure all endpoints receive regular security updates and patches post-migration.
