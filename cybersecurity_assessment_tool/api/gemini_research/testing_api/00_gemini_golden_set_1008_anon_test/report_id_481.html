Technical Findings Analysis

The scan identified an Nginx web server running version 1.18.0 on port 443 (HTTPS). Nginx 1.18.0 is an older stable branch release from April 2020. As of late 2025, this version is significantly outdated and likely contains known vulnerabilities that have been patched in newer releases. Running outdated software increases the risk of successful exploitation by attackers.

Risk Assessment

Based on the security control review and technical scan, the following risks have been identified:

Identified Risks and Severities
Risk Name Overview Severity
Missing MFA for Email Access The organization does not require Multi-Factor Authentication (MFA) for email access, making accounts highly susceptible to credential stuffing, phishing, and brute-force attacks. High
Missing MFA for Sensitive Data Systems MFA is not enforced for access to sensitive data systems, increasing the risk of unauthorized access to critical information should credentials be compromised. High
Lack of Employee Acceptable Use Policy (AUP) The absence of an AUP means employees may not have clear guidelines on acceptable use of organizational IT resources, leading to potential misuse or security incidents. Medium
Lack of New Employee Security Awareness Training New employees are not receiving security awareness training, leaving them unprepared for common cyber threats (e.g., phishing) from the start of their employment. Medium
Outdated Nginx Web Server The Nginx web server (v1.18.0) is outdated and likely contains known security vulnerabilities, making it a target for exploitation and potentially leading to denial-of-service, data breaches, or defacement. High

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) for Email Access:

    • Enable and enforce MFA for all user accounts accessing organizational email. This significantly reduces the risk of unauthorized access even if passwords are stolen.

  2. Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems:

    • Extend MFA enforcement to all systems containing or providing access to sensitive organizational data.

  3. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP clearly defining the rules and expectations for employee use of IT resources. Ensure all employees review and acknowledge this policy.

    • Conduct regular reviews and updates of the AUP.

  4. Initiate Security Awareness Training for New Employees:

    • Establish a mandatory security awareness training program for all new hires as part of their onboarding process. This should cover common threats, organizational policies, and best practices.

  5. Upgrade Nginx Web Server to the Latest Stable Version:

    • Plan and execute an upgrade of the Nginx web server to the latest stable release (e.g., Nginx 1.24.x or newer mainline 1.26.x).

    • Ensure proper testing is conducted in a staging environment before deploying to production.

    • Implement a regular patch management schedule for all public-facing services.
