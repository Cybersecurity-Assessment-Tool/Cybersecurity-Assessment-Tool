Recommendations

Based on the assessment, the following recommendations are provided to address the identified risks and improve the overall cybersecurity posture.

  1. Implement MFA for Sensitive Data Systems: Prioritize the implementation of Multi-Factor Authentication for all systems that handle or store sensitive organizational data. This will significantly reduce the risk of unauthorized access from stolen or phished credentials.

  2. Secure Database Exposure:

    • Apply strict firewall rules to restrict access to MySQL port 3306 (and any other database ports) only to authorized internal hosts and specific services.

    • For any required remote access, ensure a secure method like a Virtual Private Network (VPN) is mandated, and direct internet exposure of database services is eliminated.

    • Regularly review and apply security updates and patches for database software.

    • Ensure strong, unique credentials are used for all database accounts and implement least privilege access.

  3. Implement MFA for Computer Login: Extend Multi-Factor Authentication requirements to all employee computer logins. This significantly enhances workstation security and prevents unauthorized access in case of credential theft or brute-force attacks.

  4. Institute New Employee Security Training: Develop and implement a mandatory security awareness training program for all new hires as part of their onboarding process. This ensures employees are educated on security policies, best practices, and potential threats from their first day.

  5. Regularly Review and Update Policies: While an acceptable use policy is in place, ensure it is regularly reviewed and updated (at least annually) to reflect current threats, technologies, and organizational changes.

  6. Continue Annual Security Awareness Training: Maintain and enhance the annual security awareness training for all employees to keep them informed about evolving cyber threats and reinforce best practices. Consider more frequent, bite-sized training modules.
