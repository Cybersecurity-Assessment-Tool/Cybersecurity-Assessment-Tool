Executive Summary

This report provides an assessment of the organization’s current cybersecurity posture, based on a security questionnaire and a limited network scan. The findings indicate several critical gaps in foundational security controls and employee awareness, posing significant risks to the organization’s information assets. A pervasive lack of multi-factor authentication (MFA) for critical access points, the absence of essential security policies, and inadequate security awareness training are identified as key areas of concern. Furthermore, a technical scan revealed an outdated public-facing web server component (Nginx 1.18.0) which has reached its End-of-Life (EOL) and is likely vulnerable to known exploits. Addressing these high-severity risks requires immediate attention and comprehensive remediation efforts to improve the overall security stance.

Recommendations

Based on the assessment, the following recommendations are provided to enhance the organization’s cybersecurity posture:

  1. Implement Multi-Factor Authentication (MFA):

    • MFA for Computers: Mandate MFA for all user logins to company computers and workstations to prevent unauthorized access via compromised credentials.

    • MFA for Sensitive Data Systems: Implement MFA for accessing all systems containing sensitive organizational or client data.

  2. Develop and Enforce Security Policies:

    • Acceptable Use Policy (AUP): Create a comprehensive AUP outlining the appropriate use of company IT resources, data, and internet access. Ensure all employees review and acknowledge the policy annually.

  3. Establish Robust Security Awareness Training Program:

    • New Employee Training: Implement mandatory security awareness training as part of the onboarding process for all new hires.

    • Annual Training: Conduct mandatory annual security awareness training for all employees, covering current threats, phishing prevention, data handling best practices, and policy updates.

  4. Address Technical Vulnerabilities:

    • Upgrade Nginx: Immediately upgrade the Nginx web server (currently version 1.18.0) to the latest stable and supported version. This will ensure the system receives crucial security patches and fixes, mitigating the risk of exploitation from known vulnerabilities.

    • Regular Patching: Establish a regular patching and update schedule for all software, operating systems, and network devices, especially those exposed to the internet.

Conclusion

The current cybersecurity posture of the organization presents several high-severity risks stemming from control deficiencies in foundational security practices and the use of outdated software. Proactive and swift implementation of the recommended actions is crucial to significantly improve the security posture, reduce the attack surface, and protect organizational assets from potential threats.
