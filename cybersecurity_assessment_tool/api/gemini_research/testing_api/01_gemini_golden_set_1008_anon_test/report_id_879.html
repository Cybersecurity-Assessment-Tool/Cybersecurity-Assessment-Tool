Executive Summary

This report details the findings of a cybersecurity assessment conducted for [Organization Name]. The assessment combined a review of organizational security controls, an external network scan, and an analysis of pre-existing risk data.

A risk was identified: an externally exposed MySQL database (5.7.33) on port 3306. This version is past its End-of-Life (EOL) as of October 2023, meaning it no longer receives security updates and is highly susceptible to known exploits. Direct exposure of a database to the public internet is a severe security risk that can lead to data breaches, ransomware, or complete system compromise.

Additionally, a administrative gap was noted: the absence of a formal employee acceptable use policy. This exposes the organization to insider threats and inconsistent security practices.

On a positive note, the organization has demonstrated a strong commitment to identity security by implementing Multi-Factor Authentication (MFA) across email, computer logins, and sensitive systems. A robust security awareness training program is also in place.

Immediate remediation should focus on isolating the exposed database from the internet and developing a plan to upgrade the outdated software.

Analysis and Observations

The technical scan confirms the pre-existing risk "Database Exposure". The finding is more severe than initially documented due to two key factors:

  1. Direct Public Exposure: Port 3306 (MySQL) is open to the public internet. This allows any attacker on the internet to attempt to connect, brute-force credentials, or exploit vulnerabilities in the database service.

  2. End-of-Life Software: The detected version, MySQL 5.7.33, reached its official End-of-Life (EOL) in October 2023. EOL software no longer receives security patches from the vendor, making it a prime target for attackers who can leverage publicly known vulnerabilities without fear of them being patched.

This combination represents a critical and immediate threat to the confidentiality, integrity, and availability of the data stored within this database.

Recommendations

The following actions are recommended to mitigate the identified risks.

RISK-001: Exposed & Outdated Database Service ()

  • Immediate (Containment):

    • Apply firewall rules to immediately block all public inbound access to TCP port 3306 on . Access should only be permitted from trusted internal IP addresses.

  • Short-Term (Remediation):

    • Develop and execute a plan to upgrade the MySQL 5.7 database to a currently supported version (e.g., MySQL 8.x).

    • Perform a vulnerability scan on the database server after the upgrade to ensure no other critical issues exist.

  • Long-Term (Strategic):

    • Implement a secure access solution, such as a Virtual Private Network (VPN) or a bastion host, for all future database administration and access. Avoid exposing database ports directly to the internet.

RISK-002: Lack of Acceptable Use Policy ()

  • Short-Term (Remediation):

    • Draft, approve, and implement a comprehensive Employee Acceptable Use Policy (AUP).

    • Communicate the new policy to all existing employees and incorporate it into the onboarding process for new hires.

    • Require all employees to formally acknowledge they have read and understood the policy.
