RISK-002: Lack of Security Policy & Training (High)

Action Required within 30-60 days.

  • Develop Policy: Create and implement a formal Acceptable Use Policy (AUP) that all employees must read and sign. This policy should clearly define rules for using company networks, devices, and data.

  • Implement Training: Institute a mandatory security awareness training program for all new hires.

  • Annual Refreshers: Conduct annual security awareness training for all staff to keep them updated on the latest threats and best practices.

RISK-003: Exposed SSH Service (High)

Action Required within 7-14 days.

  • Disable Password Authentication: Harden the SSH configuration to disable password-based logins and enforce the use of public-key cryptography only.

  • Disable Root Login: Prohibit direct root login via SSH by setting ‘PermitRootLogin no‘ in the ‘sshd_config‘ file. Administrators should log in with a standard user account and elevate privileges as needed.

  • Implement Fail2Ban: Deploy a tool like Fail2Ban to automatically block IP addresses that exhibit malicious behavior, such as repeated failed login attempts.
