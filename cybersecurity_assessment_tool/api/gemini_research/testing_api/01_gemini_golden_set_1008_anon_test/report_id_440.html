6. Recommendations

Based on the consolidated risk assessment, the following prioritized actions are recommended to improve the organization’s security posture.

  • [Critical] Remediate RISK-001: Implement MFA for Email.

    • Immediately enforce a policy requiring MFA for all users accessing the email system, whether via webmail or a client application. This is the single most effective control to prevent email account takeovers.

  • [High] Remediate RISK-002: Implement MFA for Endpoints.

    • Deploy and enforce an MFA solution for all employee computer logins (Windows, macOS, etc.). This adds a critical layer of defense against the use of stolen credentials to access the internal network.

  • [Medium] Remediate RISK-003: Secure Exposed Web Service.

    • Identify the business purpose of the web server running on [Target IP]:80.

    • If the service is necessary, obtain and install a valid TLS/SSL certificate and configure the server to use HTTPS (Port 443).

    • Disable or redirect all traffic from HTTP (Port 80) to HTTPS to ensure all communication is encrypted.

    • If the service is not necessary, it should be firewalled from external access.

  • [Informational] Investigate RISK-004: Review Risk Register Integrity.

    • Conduct an internal review of the risk management system or database to understand the origin of this unusual entry.

    • Ensure that access controls are in place to maintain the integrity and accuracy of risk assessment data.
