Technical Analysis

The presence of an open Port 80 (HTTP) is a significant security risk. The HTTP protocol transmits data, including potential login credentials or sensitive information, in cleartext. This makes the communication susceptible to eavesdropping and man-in-the-middle (MitM) attacks. All web traffic should be encrypted using HTTPS (Port 443) to ensure confidentiality and integrity.

Recommendations

To address the identified risks, the following actions are recommended. These are prioritized based on severity and potential impact.

  1. Remediate Unencrypted Web Traffic (High Priority):

    • Immediately investigate the service running on port 80.

    • If it is a web server, obtain and install a valid TLS/SSL certificate.

    • Configure the service to enforce HTTPS for all connections.

    • Implement a permanent redirect from HTTP to HTTPS.

    • Close port 80 on the firewall if it is not required for the redirect.

  2. Establish Annual Security Awareness Training (High Priority):

    • Procure or develop a security awareness training program that covers current threats such as phishing, ransomware, and social engineering.

    • Make this training mandatory for all employees on an annual basis.

    • Track completion to ensure 100% compliance.

  3. Develop and Implement an Acceptable Use Policy (Medium Priority):

    • Draft a formal AUP that clearly defines the rules and expectations for using company technology and data.

    • The policy should be reviewed by management and legal counsel.

    • Require all employees to read and formally acknowledge the policy upon implementation and during onboarding.
