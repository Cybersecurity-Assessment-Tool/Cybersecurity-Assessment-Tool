4. Technical Scan Results

An external network scan was performed using Nmap. The scan identified the following open ports and services on the target system.

Open Ports and Services

Port State Service Product & Version
21/tcp open ftp vsftpd 2.3.4

Analysis of Technical Findings

The scan revealed a critical vulnerability.

  • Vulnerable FTP Service: The server is running vsftpd version 2.3.4. This specific version is widely known to contain a critical backdoor vulnerability (CVE-2011-2523). If exploited, this vulnerability allows an attacker to execute arbitrary commands on the server with root-level privileges.

  • Anonymous FTP Enabled: The scan confirmed that "Anonymous FTP login allowed". This is a severe misconfiguration that permits any external user to connect to the FTP server without authentication, potentially allowing them to upload malicious files or download sensitive information.

The following table synthesizes findings from the security control review, technical scan, and pre-existing risk data into a prioritized list.

Severity Risk / Vulnerability Source

Lack of MFA Multi-Factor Authentication is not enforced for email or access to sensitive data systems, creating a high risk of account compromise and data breach.
No Security Awareness Training The absence of a training program for new or existing employees increases susceptibility to phishing, social engineering, and other human-targeted attacks.

6. Recommendations

Based on the analysis, we recommend the following actions, prioritized by severity.

Immediate Actions (Priority 1)

  1. Remediate FTP Server:

    • Immediately take the FTP server offline.

    • If the service is business-critical, upgrade vsftpd to the latest stable version and place it behind a firewall with strict access control lists.

    • If the service is not required, decommission it permanently.

    • Disable anonymous FTP login under all circumstances.

  2. Deploy MFA:

    • Implement and enforce MFA for all email accounts (e.g., Office 365, Google Workspace) immediately.

    • Prioritize the rollout of MFA for all systems identified as containing sensitive data.

High-Priority Actions (Priority 2)

  1. Establish Security Awareness Training:

    • Implement a mandatory security awareness training module for all new employees as part of their onboarding process.

    • Schedule and conduct annual security awareness training for all staff, covering topics like phishing, password hygiene, and acceptable use.

Medium-Priority Actions (Priority 3)

  1. Address Outdated Systems:

    • Continue with the existing plan to upgrade all Windows 7 workstations to a supported operating system like Windows 10 or 11.

    • Develop a formal patch and vulnerability management policy to prevent systems from becoming unsupported in the future.
