Analysis of Security Control Gaps:

The questionnaire reveals two significant gaps that pose a high risk to the organization:

  1. Lack of MFA for Computer Logins: Not requiring Multi-Factor Authentication (MFA) for computer logins significantly increases the risk of unauthorized access through compromised credentials, potentially leading to widespread system compromise.

Technical Scan Results

A network scan was performed against the designated target. The scan results provide insights into open ports and services, revealing potential entry points for attackers.

Nmap Scan Findings:

The Nmap scan against [Target IP] identified the following critical service:

  • Port 3389/tcp (Open):

    • Service: ms-wbt-server (Microsoft Remote Desktop Protocol - RDP)

    • Description: RDP is a proprietary protocol developed by Microsoft, which provides a user with a graphical interface to connect to another computer over a network connection.

    • Significance: An open RDP port, especially when exposed to the internet, is a frequent target for cyberattacks, including brute-force attempts, credential stuffing, and exploitation of known RDP vulnerabilities (e.g., BlueKeep). Successful exploitation can grant an attacker full remote control over the target system.

The presence of an open RDP port on [Target IP] aligns directly with the "RDP Exposure" risk documented in the organization’s pre-existing risk register.

Recommendations

To address the identified risks and strengthen the cybersecurity posture of [Organization Name], the following actionable recommendations are provided:

Recommendations for RDP Exposure (Critical Risk):

  1. Immediate Access Restriction:

    • Immediately restrict external access to RDP (port 3389) on [Target IP] and all other internet-facing systems. This can be achieved by configuring firewall rules to only permit RDP connections from trusted internal networks or specific, authorized jump hosts.

  2. Secure Remote Access Implementation:

    • Implement a Virtual Private Network (VPN) solution as the primary method for remote access to internal resources, including RDP services. All remote connections should terminate on the VPN, which should then allow access to internal systems.

    • Alternatively, consider a secure bastion host or a Remote Desktop Gateway (RD Gateway) to serve as a single, hardened entry point for RDP connections, requiring strong authentication and authorization.

  3. RDP Hardening:

    • Ensure all RDP users have strong, unique passwords, and enforce strict account lockout policies to thwart brute-force attacks.

    • Apply Network Level Authentication (NLA) for RDP connections, which requires users to authenticate before a full RDP session is established.

    • Regularly patch and update RDP servers to protect against known vulnerabilities.

Recommendations for Lack of MFA for Computer Logins (High Risk):

  1. Implement MFA for All Computer Logins:

    • Deploy and enforce Multi-Factor Authentication (MFA) for all user logins to organizational computers, including workstations and servers. This should apply to both local and domain accounts where technically feasible.

    • Evaluate MFA solutions compatible with the organization’s existing identity providers (e.g., Active Directory, Azure AD).

  2. Pilot Program and Rollout:

    • Start with a pilot program for a small group of users to iron out any technical or user experience issues.

    • Develop a phased rollout plan with clear communication and support for all employees.

Recommendations for Absence of Security Awareness Training for New Employees (High Risk):

  1. Develop Mandatory Onboarding Security Training:

    • Create and implement a mandatory cybersecurity awareness training program specifically for all new employees during their onboarding process.

    • The training should cover essential topics such as phishing identification, malware prevention, password hygiene, social engineering tactics, data handling policies, and incident reporting procedures.

  2. Regular Refresher Training:

    • While annual training is conducted, ensure that the content is current and addresses emerging threats. Integrate lessons learned from recent incidents if applicable.
