Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following actionable recommendations are provided:

  • Implement MFA for Computer Logins:

    • Deploy and enforce Multi-Factor Authentication (MFA) for all user logins to organizational computers and workstations. This significantly reduces the risk of credential compromise.

    • Consider robust MFA solutions such as FIDO2 security keys, authenticator applications (e.g., TOTP), or smart cards.

  • Implement MFA for Sensitive Data Systems:

    • Mandate MFA for all access to systems containing sensitive data, irrespective of the system’s location (on-premise or cloud).

    • Ensure strong authentication methods are in place for databases, file shares, and application backends holding critical information to protect against unauthorized access.

  • Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities, permitted and prohibited activities regarding IT assets, data handling, and internet usage.

    • Ensure all employees review, understand, and formally acknowledge the AUP annually or upon significant policy changes.

  • Implement Security Awareness Training for New Employees:

    • Incorporate mandatory, role-specific security awareness training as a critical component of the onboarding process for all new hires.

    • Training should cover essential topics such as identifying phishing and social engineering attempts, password best practices, safe browsing, and organizational data handling policies.

  • Secure SSH Service on :

    • Review and harden the SSH configuration on the target host. Recommended best practices include:

      • Disabling root login directly via SSH.

      • Enforcing strong, key-based authentication and disabling password authentication where feasible.

      • Implementing rate limiting (e.g., using ‘fail2ban‘) and intrusion detection systems to mitigate brute-force attacks.

      • Ensuring the SSH server software is consistently updated with the latest security patches to protect against known vulnerabilities.

      • Changing the default SSH port from 22 to a non-standard port (though this should be considered a secondary measure, not a primary security control).

  • Conduct Regular Vulnerability Scanning and Penetration Testing:

    • Establish a schedule for regular internal and external vulnerability scans to proactively identify and remediate technical weaknesses across the IT infrastructure.

    • Periodically engage reputable third-party penetration testers to simulate real-world attacks and assess the effectiveness of implemented security controls.
