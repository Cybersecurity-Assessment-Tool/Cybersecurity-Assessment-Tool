3.1. Analysis of Questionnaire Findings

The organization has successfully implemented Multi-Factor Authentication (MFA) across key access points (email, computer login, sensitive data systems), which is a critical security control. However, significant deficiencies were identified in the areas of security policies and employee training:

  • Lack of Acceptable Use Policy (AUP): The absence of an AUP means employees may not have clear guidelines on acceptable technology use, increasing risks of misuse, data breaches, and non-compliance.

  • No Security Awareness Training: The lack of both new hire and annual security awareness training leaves employees vulnerable to social engineering attacks (e.g., phishing, pretexting) and unaware of common cybersecurity threats and best practices. This is a critical gap, as employees are often considered the weakest link in the security chain without proper training.

6. Recommendations

To address the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Create a clear, concise AUP that outlines acceptable and unacceptable use of company IT resources, data, and internet access.

    • Ensure the AUP is communicated to all employees, acknowledged, and enforced consistently.

    • Integrate the AUP into new employee onboarding.

  • Implement Mandatory Security Awareness Training for New Employees:

    • Develop an onboarding security awareness program covering basic cybersecurity principles, phishing recognition, password best practices, and incident reporting procedures.

    • Make this training mandatory for all new hires before granting full system access.

  • Establish Annual Mandatory Security Awareness Training for All Employees:

    • Design and deliver an annual security awareness training program for all employees.

    • The training should be engaging, up-to-date with current threats, and include simulated phishing exercises to test and reinforce learning.

    • Track completion rates and comprehension.

  • Expand Technical Security Assessments:

    • While the external scan showed no open ports, consider deeper external penetration testing, web application security testing, and internal network vulnerability assessments to gain a more complete picture of technical risks.

    • Ensure future network scans include ‘scan_date‘ for proper reporting and historical tracking.
