Recommendations

To address the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA):

    • For Computer Logins: Immediately implement MFA for all employee computer logins, particularly for systems that access the internal network or sensitive data.

    • For Sensitive Data Systems: Mandate MFA for all access to systems containing or processing sensitive organizational data.

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that outlines acceptable and unacceptable use of organizational IT resources, data, and internet access.

    • Ensure all employees review and acknowledge the AUP, and integrate it into new employee onboarding and annual training.

  3. Secure SSH Service (Port 22):

    • Strong Authentication: Disable password-based authentication and enforce key-based authentication. Consider implementing MFA for SSH access.

    • Least Privilege: Restrict SSH access to only necessary users and IP addresses.

    • Update/Patch: Ensure the SSH server software is always running the latest patched version to mitigate known vulnerabilities.

    • Monitoring: Implement logging and monitoring for all SSH connection attempts, especially failed ones.

  4. Conduct Advanced Technical Scans:

    • Perform regular, authenticated vulnerability scans that include service version detection to identify specific software versions and known vulnerabilities on all open ports.
