6. Recommendations

The following recommendations are provided to mitigate the identified risks and enhance the organizationâ€™s cybersecurity posture.

  1. Implement MFA for Sensitive Data Systems:

    • Action: Immediately implement and enforce Multi-Factor Authentication (MFA) for all systems, applications, and services that handle or store sensitive organizational data.

    • Rationale: This significantly reduces the risk of unauthorized access, even if primary credentials are compromised, aligning with industry best practices.

  2. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Action: Create a comprehensive AUP that clearly defines the acceptable use of company IT resources (networks, devices, software, data, internet).

    • Action: Distribute the AUP to all employees, ensure they understand its contents, and obtain signed acknowledgments.

    • Action: Incorporate AUP review into annual security awareness training.

    • Rationale: A clear AUP sets expectations, reduces misuse, enhances legal protection, and supports a strong security culture.

  3. Upgrade Outdated Nginx Web Server:

    • Action: Plan and execute an upgrade of the Nginx web server on port 443 from version 1.18.0 to the latest stable or mainline version (e.g., Nginx 1.26.x or newer).

    • Action: Before upgrading, ensure proper backups are in place and test the new version in a staging environment to prevent service disruption.

    • Rationale: Upgrading to a supported version will patch known vulnerabilities, ensure ongoing security updates, and provide access to new features and performance improvements. Consider migrating away from EOL software if an upgrade path is not feasible.
