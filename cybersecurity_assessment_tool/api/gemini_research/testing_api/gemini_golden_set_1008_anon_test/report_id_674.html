3.2 Analysis of Control Gaps

The questionnaire results indicate severe deficiencies across critical security domains. The complete lack of Multi-Factor Authentication (MFA) across all listed systems (email, computer login, sensitive data) presents a Critical risk, making the organization highly susceptible to credential-based attacks, phishing, and account compromise. Furthermore, the absence of an Employee Acceptable Use Policy (AUP) and any form of security awareness training (for new or existing employees) introduces significant High risks related to insider threats, social engineering, human error, and general misuse of organizational assets. These gaps leave employees unprepared for common cyber threats and lack clear guidelines for secure behavior.

4.3 Technical Vulnerability Analysis

The presence of an open FTP service running vsftpd 2.3.4 with anonymous login enabled is a Critical vulnerability.

  • Anonymous FTP Access: Allowing anonymous FTP login permits unauthorized users to access, potentially upload to, or download from the FTP server without credentials. This can lead to data exposure, defacement, or malware distribution.

  • Outdated Software: vsftpd version 2.3.4 was released in 2011 and has numerous publicly known vulnerabilities, including backdoors (e.g., CVE-2011-2523) and other exploits that could lead to remote code execution or complete system compromise. Running such an outdated and insecure version poses an immediate and severe risk.

6. Recommendations

Based on the identified vulnerabilities and control gaps, the following actionable recommendations are provided to enhance ’s cybersecurity posture.

6.1 Immediate Remediation (Critical/High)

  • Disable Anonymous FTP & Update FTP Service:

    • Immediately disable anonymous login on the FTP service.

    • If FTP is required, migrate to a secure protocol like SFTP or FTPS with strong authentication (e.g., SSH keys or strong passwords).

    • Decommission the outdated vsftpd 2.3.4 service or upgrade to a currently supported version with all security patches applied. Consider if FTP is truly necessary; often, alternatives are more secure.

  • Implement Multi-Factor Authentication (MFA):

    • Deploy and enforce MFA for all organizational email accounts.

    • Implement MFA for all user logins to computers and network resources.

    • Extend MFA to all systems handling sensitive organizational data.

  • Develop and Implement Security Policies:

    • Create and formally adopt an Employee Acceptable Use Policy (AUP) that clearly defines appropriate use of organizational IT resources.

    • Communicate the AUP to all employees and obtain signed acknowledgments.

  • Initiate Security Awareness Training Program:

    • Implement mandatory security awareness training for all new employees during their onboarding process.

    • Establish a recurring annual security awareness training program for all existing employees, covering topics like phishing, social engineering, password hygiene, and data handling.

6.2 Mid-Term Remediation (Medium)

  • Upgrade Outdated Operating Systems:

    • Develop a plan to upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or 11).

    • For systems that cannot be immediately upgraded, implement compensating controls such as network segmentation, strict firewall rules, and robust endpoint detection and response (EDR) solutions to minimize their attack surface.
