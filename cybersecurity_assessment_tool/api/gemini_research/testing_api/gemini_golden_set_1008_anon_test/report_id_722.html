Analysis of Questionnaire Results

The questionnaire highlights critical deficiencies in multi-factor authentication coverage and security awareness training. While MFA is used for email and computer logins, its absence for sensitive data systems represents a high risk, as compromised credentials could lead directly to data exfiltration. The complete lack of security awareness training for both new and existing employees significantly increases the organization’s susceptibility to human-factor attacks such as phishing, social engineering, and malware delivery.

Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided. These are categorized for immediate action, strategic planning, and policy refinement.

Immediate Actions (Critical/High Risks)

  • Disable Anonymous FTP and Upgrade/Remove vsftpd:

    • Immediately disable anonymous FTP access on any systems running vsftpd or similar FTP services.

    • Patch vsftpd to a current, secure version, or preferably, decommission FTP if not essential. Replace with a secure file transfer protocol (e.g., SFTP, SCP, FTPS) with strong authentication and encryption.

    • Implement strict access controls and ensure that only authenticated, authorized users can access necessary resources.

  • Implement MFA for Sensitive Data Systems:

    • Deploy and enforce Multi-Factor Authentication for all systems that process or store sensitive organizational data. This should be a top priority to protect against credential theft and unauthorized access.

  • Establish Comprehensive Security Awareness Training Program:

    • Develop and implement a mandatory security awareness training program for all new employees during their onboarding process.

    • Institute annual refresher security awareness training for all employees to keep them informed about current threats and best practices (e.g., phishing recognition, password hygiene, incident reporting).

Strategic Actions (Medium/Long-term)

  • Operating System Upgrade Plan:

    • Develop and execute a detailed plan to upgrade all Windows 7 workstations to a currently supported and more secure operating system (e.g., Windows 10/11 or a supported Linux distribution). Prioritize systems that handle sensitive data or have network exposure.

    • Implement a robust patch management system to ensure all operating systems and applications are kept up-to-date with the latest security patches.

  • Regular Software Audits:

    • Implement a process for regularly auditing all deployed software and services to identify outdated versions, unpatched vulnerabilities, and insecure configurations.

    • Utilize vulnerability scanners and asset management tools to maintain an up-to-date inventory of all software and their respective versions.

Policy & Process Improvements

  • Review and Update Acceptable Use Policy (AUP):

    • Ensure the AUP is current and comprehensively covers secure system usage, data handling, password policies, and procedures for reporting security incidents.

    • Ensure all employees acknowledge and understand the AUP.

  • Incident Response Planning:

    • Develop or refine the organization’s incident response plan to ensure clear procedures are in place for detecting, containing, eradicating, and recovering from security incidents, especially those related to unauthorized access or data breaches.

    • Conduct regular tabletop exercises to test the effectiveness of the incident response plan.
