2.1 Analysis of Security Control Gaps

The questionnaire highlights significant deficiencies in critical security controls:

  • Lack of Multi-Factor Authentication (MFA): MFA is not required for email access, computer logins, or access to sensitive data systems. This is a severe weakness, as compromised credentials can lead directly to unauthorized access to critical organizational assets without an additional layer of security.

  • Insufficient New Employee Security Training: While annual training is conducted, new employees do not receive immediate security awareness training. This leaves new hires vulnerable to social engineering attacks from day one, potentially exposing the organization to risks.

5. Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) Across All Critical Systems:

    • Action: Immediately deploy and enforce MFA for all email accounts.

    • Action: Implement MFA for all user logins to organizational computers and networks.

    • Action: Mandate MFA for access to all systems containing sensitive data (e.g., HR, finance, customer data).

    • Rationale: MFA significantly reduces the risk of unauthorized access even if primary credentials are compromised.

  2. Establish Comprehensive New Employee Security Awareness Training:

    • Action: Develop and implement a mandatory security awareness training program for all new employees as part of their onboarding process.

    • Rationale: Educating new hires early helps them understand security policies, recognize threats like phishing, and practice secure computing habits from day one.

  3. Conduct a Full Network and Vulnerability Scan:

    • Action: Engage in regular, comprehensive network vulnerability scanning of all internal and external-facing systems.

    • Rationale: Proactively identifies open ports, outdated software, misconfigurations, and known vulnerabilities that could be exploited by attackers. This report was unable to provide technical findings due to the absence of scan data.

  4. Regular Review of Security Policies and Controls:

    • Action: Periodically review and update security policies, including the acceptable use policy, to ensure they remain relevant and effective against evolving threats.

    • Rationale: Cybersecurity is a dynamic field; policies must adapt to new technologies and threat landscapes.
