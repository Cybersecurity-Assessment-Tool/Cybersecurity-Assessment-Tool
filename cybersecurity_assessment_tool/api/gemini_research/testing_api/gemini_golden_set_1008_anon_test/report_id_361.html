Analysis of Technical Findings

The scan identified an Nginx web server running on port 443 with version 1.18.0. Nginx 1.18.0 was released in April 2020. As of the current date, this version is considered outdated, with multiple stable and mainline releases (e.g., 1.24.x, 1.25.x) having addressed various bugs and security vulnerabilities. Running outdated software increases the risk of exploitation by attackers leveraging known vulnerabilities.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following actionable recommendations are provided:

  1. Implement MFA for Computer Logins:

    • Mandate Multi-Factor Authentication for all user accounts accessing organizational computers and workstations. This should apply to both local and domain logins where applicable.

    • Explore solutions such as hardware tokens, software authenticators, or biometric methods to supplement password-based authentication.

  2. Implement MFA for Sensitive Data Systems:

    • Enforce MFA for all access points to systems that store or process sensitive organizational data, including databases, file shares, and critical applications.

    • Conduct an inventory of all sensitive data systems to ensure comprehensive MFA coverage.

  3. Establish Security Awareness Training for New Employees:

    • Develop and implement a mandatory security awareness training program for all new hires as part of their onboarding process.

    • The training should cover topics such as phishing, social engineering, password hygiene, data handling best practices, and incident reporting procedures.

  4. Upgrade and Patch Nginx Web Server:

    • Plan and execute an upgrade of the Nginx web server on port 443 from version 1.18.0 to the latest stable release (e.g., Nginx 1.24.x or newer).

    • Implement a robust patch management strategy for all web servers and other internet-facing infrastructure to ensure that software is kept up-to-date with security patches in a timely manner.

    • Regularly review Nginx configuration for security best practices (e.g., disabling unnecessary modules, enforcing strong TLS protocols and ciphers).
