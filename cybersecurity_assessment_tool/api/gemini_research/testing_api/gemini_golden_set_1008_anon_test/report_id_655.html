5. Risk Assessment

This section outlines the identified risks, categorized by their source and potential impact. Risks are derived from the security control review and the technical scan results. No pre-existing vulnerabilities were provided in the input data.

Risk Name Severity Overview
Lack of MFA for Computer Login High Without Multi-Factor Authentication (MFA) for computer logins, compromised credentials (e.g., via phishing or weak passwords) can lead to unauthorized access to internal workstations and potentially the network, increasing the risk of data breaches and malware propagation.
Infrequent Security Awareness Training Medium While new employees receive training, the lack of annual security awareness training for all employees means that the workforce may not be up-to-date on evolving threats (e.g., new phishing tactics, social engineering techniques) and best practices, increasing human-factor risk and susceptibility to attacks.
Open SSH Port Medium Port 22 (SSH) is open on the target. SSH is a critical service for remote administration, but if not securely configured (e.g., weak passwords, root login enabled, outdated software, lack of rate limiting), it can be a significant attack vector for brute-force attacks or exploitation of software vulnerabilities. 6. Recommendations

Based on the assessment, the following recommendations are provided to mitigate identified risks and enhance the organization’s overall cybersecurity posture.

  1. Implement Multi-Factor Authentication (MFA) for All Computer Logins:

    • Action: Mandate MFA for all user logins to company workstations and servers.

    • Benefit: Significantly reduces the risk of unauthorized access even if primary credentials are compromised, forming a crucial layer of defense against credential theft.

    • Priority: High

  2. Establish Annual Security Awareness Training Program:

    • Action: Develop and implement a mandatory annual security awareness training program for all employees, covering current threat landscapes, phishing, social engineering, data handling, and organizational policies.

    • Benefit: Keeps employees informed and vigilant against evolving threats, strengthening the human firewall and reducing susceptibility to social engineering attacks.

    • Priority: High

  3. Secure SSH Service Configuration:

    • Action: Conduct a thorough review of the SSH service on the open port (22). Ensure it is configured with best practices, including:

      • Enforcing strong, key-based authentication (disable password authentication where possible).

      • Disabling root login directly via SSH.

      • Implementing rate limiting or an intrusion prevention system (IPS) to detect and block brute-force attempts.

      • Keeping the SSH server software updated to the latest stable version to patch known vulnerabilities.

      • Limiting SSH access to trusted source IP addresses or using a jump box/bastion host.

    • Benefit: Reduces the attack surface and prevents unauthorized remote access to critical systems, protecting against brute-force attacks and exploits.

    • Priority: Medium (becomes High if misconfigured or outdated)
