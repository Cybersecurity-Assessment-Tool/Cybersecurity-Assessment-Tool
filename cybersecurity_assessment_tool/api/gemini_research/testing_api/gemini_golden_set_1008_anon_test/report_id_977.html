Recommendations

To mitigate the identified risks and improve the overall cybersecurity posture, the following recommendations are provided.

Multi-Factor Authentication (MFA)

  • Implement MFA for Email: Mandate MFA for all email accounts to significantly reduce the risk of phishing and credential theft. This is a fundamental control for protecting user identities.

  • Implement MFA for Sensitive Data Systems: Ensure all access to systems containing sensitive data requires MFA. This adds an essential layer of security to prevent unauthorized data access, even if primary credentials are compromised.

Security Awareness Training

  • Mandatory New Employee Training: Implement compulsory security awareness training for all new hires during onboarding. This ensures that new employees understand their role in maintaining security from day one.

  • Annual Security Awareness Training: Conduct annual, mandatory security awareness training for all employees. This training should be reinforced with regular phishing simulations and updated content to address evolving threats.

Network Security (RDP Exposure)

  • Restrict Direct RDP Access: The RDP port (3389/tcp for ms-wbt-server) should not be directly exposed to the internet. Consider implementing a firewall rule to block external access to this port. (As per existing recommendation: Close Port)

  • Implement Secure Remote Access: For legitimate remote access needs, mandate the use of a Virtual Private Network (VPN). Ensure MFA is enforced for VPN access. (As per existing recommendation: VPN)

  • RDP Hardening: If RDP must be used internally, ensure strong, unique passwords for RDP accounts, enforce account lockout policies, and apply all available security patches promptly. Consider enabling Network Level Authentication (NLA) if not already active.

General Security Posture

  • Regular Vulnerability Scanning & Patch Management: Implement a routine schedule for vulnerability scanning to proactively identify new exposures and ensure prompt application of security patches across all systems and applications.

  • Incident Response Plan: Develop and regularly test a comprehensive incident response plan to ensure the organization is prepared to effectively detect, contain, eradicate, recover from, and learn from security incidents.
