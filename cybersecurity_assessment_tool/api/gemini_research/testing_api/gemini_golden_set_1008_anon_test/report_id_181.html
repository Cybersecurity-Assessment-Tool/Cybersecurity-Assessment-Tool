2.1 Analysis of Security Control Gaps

The review highlights several critical gaps in foundational security practices:

  • Lack of MFA Implementation: MFA is not required for email, computer login, or sensitive data systems. This significantly increases the risk of unauthorized access due to compromised credentials.

  • Absence of Acceptable Use Policy (AUP): Without an AUP, employees lack clear guidelines on the appropriate use of organizational IT resources, leading to potential misuse or security incidents.

  • Inadequate Annual Security Awareness Training: While new employees receive training, the lack of recurring annual training leaves the entire workforce vulnerable to evolving threats and social engineering tactics over time.

3.2 Technical Analysis

The primary technical finding is the detection of Nginx 1.18.0 running on port 443 (HTTPS). Nginx 1.18.0 was released in April 2020 and is now considered significantly outdated. Modern stable versions (e.g., 1.24.x, 1.26.x) have addressed numerous security vulnerabilities found in older releases. Running an outdated web server exposes the organization to known vulnerabilities, potentially allowing for remote code execution, denial-of-service attacks, or information disclosure. This constitutes a High Severity technical risk.

5. Recommendations

Based on the assessment, the following recommendations are provided to enhance the organization’s cybersecurity posture:

  1. Implement Multi-Factor Authentication (MFA):

    • Deploy and enforce MFA for all user accounts accessing email, logging into corporate computers, and accessing sensitive data systems. Prioritize high-privilege accounts.

    • Severity: High

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining the proper use of all organizational IT assets, data, and internet access.

    • Disseminate the AUP to all employees and obtain signed acknowledgments.

    • Severity: High

  3. Establish Annual Security Awareness Training:

    • Implement a mandatory annual security awareness training program for all employees, covering topics such as phishing, social engineering, password hygiene, and incident reporting.

    • Severity: High

  4. Upgrade Nginx Web Server:

    • Plan and execute an upgrade of the Nginx web server from version 1.18.0 to the latest stable release (e.g., 1.26.x or the latest version in the mainline branch).

    • Ensure proper testing is conducted in a staging environment before deploying to production.

    • Implement a patching strategy to keep all software, especially public-facing services, up-to-date.

    • Severity: High

6. Conclusion

The current cybersecurity posture of requires urgent attention, particularly in the areas of access control (MFA), policy enforcement (AUP), employee education, and software patch management. Addressing the identified high-severity risks will significantly improve the organization’s defense against common cyber threats and enhance its overall resilience.
