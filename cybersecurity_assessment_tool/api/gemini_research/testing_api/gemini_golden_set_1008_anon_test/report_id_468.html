4. Recommendations

To address the identified gaps and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Email Access Immediately:

    • Deploy MFA for all user accounts accessing organizational email. Prioritize solutions that are user-friendly but robust (e.g., authenticator apps, FIDO2 security keys).

    • Conduct immediate awareness campaigns to educate users on the importance and process of MFA.

  2. Implement MFA for Computer Login:

    • Integrate MFA for all computer logins, especially for administrative accounts and machines with access to sensitive data. Consider solutions compatible with existing directory services (e.g., Active Directory, Azure AD).

    • Explore single sign-on (SSO) solutions that enforce MFA across multiple systems for a streamlined yet secure user experience.

  3. Provide Complete and Valid Data for Re-assessment:

    • Ensure future network scan data is provided in a valid, parsable format to allow for a comprehensive technical vulnerability assessment.

    • Supply accurate and complete details of existing organizational vulnerabilities and risk registers for a more thorough risk correlation and management analysis.

  4. Maintain and Enhance Security Awareness Training:

    • Continue with annual security awareness training for all employees and specific training for new hires.

    • Incorporate real-world examples of phishing and social engineering attacks, emphasizing the role of MFA in mitigating these threats.

These recommendations are crucial steps towards mitigating immediate high-priority risks and establishing a more robust security foundation. A follow-up assessment is advised once the recommended MFA controls are implemented and comprehensive technical and existing risk data becomes available.
