6. Recommendations

The following recommendations are provided to address the identified risks and significantly enhance the organization’s cybersecurity posture.

6.1. Immediate Actions (Critical/High Severity)

  • Mitigate RDP Exposure (Port 3389):

    • Restrict Access: Immediately implement firewall rules to block direct external access to Port 3389. Access should only be permitted from trusted source IPs (e.g., office network) or via a secure Virtual Private Network (VPN) connection.

    • Enforce MFA for RDP: If external RDP access is absolutely necessary (even via VPN), ensure Multi-Factor Authentication (MFA) is strictly enforced for all RDP connections.

    • Regular Patching: Ensure all systems accessible via RDP are fully patched and running the latest security updates.

  • Implement Multi-Factor Authentication (MFA):

    • MFA for Computer Logins: Deploy and enforce MFA for all employee computer logins, especially for administrative accounts and systems processing sensitive information.

    • MFA for Sensitive Data Systems: Extend MFA enforcement to all systems that store, process, or transmit sensitive organizational data.

6.2. Mid-Term Actions (Medium Severity)

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly defines acceptable and unacceptable use of organizational IT resources, data, and internet access.

    • Communicate the AUP to all employees, ensure understanding, and obtain signed acknowledgments.

    • Integrate AUP review into employee onboarding and annual security awareness training.

6.3. Long-Term Strategy

  • Regular Security Audits and Assessments: Establish a schedule for periodic network vulnerability scans, penetration testing, and security posture assessments to proactively identify and address new risks.

  • Enhanced Security Awareness Training: Continue and expand security awareness training programs, incorporating specific modules on secure remote access practices, phishing, social engineering, and the importance of MFA.

  • Incident Response Planning: Develop, document, and regularly test a comprehensive incident response plan to ensure the organization can effectively detect, respond to, and recover from security incidents.
