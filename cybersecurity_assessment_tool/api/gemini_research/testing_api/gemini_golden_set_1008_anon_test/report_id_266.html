6. Recommendations

To mitigate the identified risks and enhance the organization’s cybersecurity posture, the following recommendations are provided. These are prioritized based on severity and potential impact.

  1. Disable Anonymous FTP Access Immediately: Reconfigure the FTP service to disallow anonymous logins. Implement strong authentication mechanisms and restrict access to authorized users only. Consider replacing FTP with a more secure protocol like SFTP or SCP.

  2. Upgrade/Replace Outdated FTP Service: Migrate from vsftpd 2.3.4 to the latest stable and supported version of vsftpd or a more secure file transfer solution. Ensure all services are regularly patched and monitored.

  3. Implement Multi-Factor Authentication (MFA) for Email and Sensitive Data Systems: Enforce MFA for all email accounts and access to systems containing sensitive organizational data. This significantly reduces the risk of credential compromise.

  4. Develop and Implement an Employee Acceptable Use Policy (AUP): Create a comprehensive AUP outlining the proper and acceptable use of organizational IT resources. Ensure all employees acknowledge and understand this policy.

  5. Conduct Regular Security Awareness Training: Implement mandatory annual (or more frequent) security awareness training for all employees, covering topics such as phishing, social engineering, data handling, and policy adherence.

  6. Upgrade Outdated Operating Systems: Prioritize the upgrade of all systems running outdated operating systems (e.g., Windows 7 as identified in Input 3) to a currently supported version (e.g., Windows 10/11) to ensure they receive critical security patches and updates.
