Technical Analysis

The scan identified an Nginx web server version 1.18.0 running on port 443. This version of Nginx was released in April 2020 and reached its End-of-Life (EOL) in April 2021. Running EOL software is a critical security risk as it no longer receives security updates, leaving it vulnerable to known exploits and common attack vectors. This significantly increases the attack surface for the organization.

Recommendations

To address the identified risks and improve the overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Email Access: Immediately deploy and enforce Multi-Factor Authentication (MFA) for all email accounts to significantly reduce the risk of credential compromise.

  2. Implement MFA for Computer Login: Implement MFA for all corporate computer logins, especially for accounts with privileged access, to strengthen authentication mechanisms against unauthorized access.

  3. Develop and Implement an Acceptable Use Policy: Create a comprehensive Acceptable Use Policy (AUP) that clearly defines appropriate use of organizational IT assets, systems, and networks. Ensure all employees acknowledge and understand this policy.

  4. Mandate Annual Security Awareness Training: Establish a program for mandatory annual security awareness training for all employees. This training should cover current threat landscapes, phishing recognition, data handling best practices, and organizational security policies.

  5. Upgrade/Replace Outdated Nginx Server: Promptly upgrade the Nginx web server from version 1.18.0 to a currently supported and patched stable release. Alternatively, consider replacing it with a modern, actively maintained web server solution. Ensure regular patching and configuration reviews for any deployed web server.
