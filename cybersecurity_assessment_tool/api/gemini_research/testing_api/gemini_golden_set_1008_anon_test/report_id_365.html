Recommendations

To mitigate the identified risks and substantially enhance the overall cybersecurity posture, the following actionable recommendations are provided. These are prioritized based on their severity, potential impact, and interconnectedness.

Critical Recommendations

  • Immediately Secure Remote Desktop Protocol (RDP) Access:

    • Restrict Direct Internet Access: Implement a Virtual Private Network (VPN) solution, requiring all users to connect to the VPN before accessing any internal resources, including RDP. This completely removes RDP from direct internet exposure.

    • Firewall Restrictions: If a VPN cannot be immediately deployed, configure firewall rules to strictly limit RDP access (Port 3389) to only trusted, whitelisted IP addresses. Avoid ‘0.0.0.0/0‘ (any IP) access.

    • Implement Account Lockout Policies: Configure Group Policy or local security policies to lock out user accounts after a small number of failed login attempts to deter brute-force attacks.

    • Apply Security Patches: Ensure all systems running RDP are fully patched with the latest security updates to address known vulnerabilities.

  • Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Deploy MFA for all user logins to company computers, particularly for any systems accessible remotely (e.g., via RDP). This significantly raises the bar for attackers, as stolen credentials alone would not be sufficient for unauthorized access.

High Priority Recommendations

  • Develop and Implement a Formal Acceptable Use Policy (AUP):

    • Create a clear, comprehensive AUP that explicitly defines employee responsibilities and permissible behaviors when using company IT assets, networks, and data.

    • Require all employees to read and formally acknowledge their understanding and agreement to the AUP upon hiring and periodically thereafter (e.g., annually).

    • Integrate AUP principles into new employee onboarding and ongoing training.

  • Establish and Conduct Regular Security Awareness Training Programs:

    • New Employee Training: Implement mandatory security awareness training for all new employees during their onboarding process, covering foundational security concepts, company policies, and threat recognition.

    • Annual Training for All Employees: Conduct mandatory, engaging security awareness training for all existing employees at least once per year. This training should be updated regularly to reflect current threat landscapes (e.g., new phishing tactics, ransomware trends).

    • Phishing Simulations: Consider running controlled phishing simulation campaigns to test employee vigilance and reinforce the lessons learned from security awareness training in a practical context.
