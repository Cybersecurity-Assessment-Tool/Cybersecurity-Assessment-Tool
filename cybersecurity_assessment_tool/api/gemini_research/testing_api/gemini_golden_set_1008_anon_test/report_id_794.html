6. Recommendations

To address the identified risks and improve the organization’s cybersecurity posture, the following actionable recommendations are provided, categorized by priority.

6.1. Critical Recommendations (Immediate Action Required)

  • Disable Anonymous FTP Access Immediately: Reconfigure the FTP server to disallow anonymous logins. Ensure that all access requires strong, authenticated credentials.

  • Upgrade or Replace FTP Server: Migrate from to a current, supported version of an FTP server, or consider more secure alternatives like SFTP (SSH File Transfer Protocol), FTPS (FTP Secure), or modern cloud storage solutions with robust access controls. If FTP is essential, ensure it is patched, securely configured, and exposed only to necessary networks.

6.2. High Priority Recommendations

  • Implement MFA for Computer Logins: Deploy multi-factor authentication (MFA) for all computer logins across the organization. This should be a top priority to protect against credential-based attacks and significantly enhance access security.

  • Develop and Implement Employee Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines acceptable use of company IT resources, data handling procedures, and security expectations. All employees should be required to read, understand, and formally acknowledge the policy. Integrate this into onboarding and annual training.

6.3. Medium Priority Recommendations

  • Upgrade End-of-Life Operating Systems: Initiate a project to upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10/11 or a suitable, patched Linux distribution). Prioritize a phased rollout, ensuring data backup and application compatibility.

6.4. General Security Best Practices

  • Regular Vulnerability Management: Implement a program for regular vulnerability scanning and penetration testing of all external and internal systems to proactively identify and address new vulnerabilities and misconfigurations.

  • Principle of Least Privilege: Review and enforce the principle of least privilege for all user accounts and system services, ensuring they only have the minimum necessary permissions to perform their required functions.

  • Enhanced Security Awareness Training: Continue and expand security awareness training for all employees, emphasizing current threats such as phishing, ransomware, and the importance of strong, unique passwords.
