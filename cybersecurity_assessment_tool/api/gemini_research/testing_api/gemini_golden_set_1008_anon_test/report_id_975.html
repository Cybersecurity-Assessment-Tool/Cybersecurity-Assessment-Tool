Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA):

    • Enforce MFA for all email accounts, computer logins, and access to sensitive data systems. This is a foundational security control to significantly reduce the risk of credential compromise and unauthorized access.

  2. Upgrade or Isolate End-of-Life Software:

    • Immediately upgrade MySQL from version 5.7.33 to a currently supported version (e.g., MySQL 8.0) to ensure ongoing security updates and patches.

    • If an immediate upgrade is not feasible, isolate the MySQL instance behind a robust firewall, allowing access only from trusted internal IP addresses, and restrict access to the absolute minimum necessary.

  3. Restrict Database Access (Port 3306):

    • Implement strict firewall rules to limit access to port 3306 (MySQL) to only necessary internal hosts or via a secure Virtual Private Network (VPN) connection. Public exposure of this port, especially with EOL software, is a severe security vulnerability.

  4. Conduct Regular Security Awareness Training:

    • Establish and enforce a mandatory annual security awareness training program for all employees. This training should educate them on current threats (e.g., phishing, social engineering), best practices for data handling, and organizational security policies.

  5. Review and Enforce Acceptable Use Policy:

    • Although an acceptable use policy is stated to exist, ensure it is up-to-date, clearly communicated to all employees, and regularly enforced to maintain a baseline of secure user behavior.
