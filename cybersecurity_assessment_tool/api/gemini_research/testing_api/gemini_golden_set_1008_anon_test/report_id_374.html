Recommendations

Based on the identified risks, the following prioritized recommendations are provided to enhance the cybersecurity posture of .

Immediate Actions (Critical Priority)

  • Secure or Decommission FTP Service on :

    • Critical Action: Immediately disable anonymous FTP access. Ensure that only authenticated users with strong credentials can access the service, if at all.

    • Critical Action: If FTP is not absolutely essential for business operations, disable or uninstall the service entirely to remove the attack vector.

    • Critical Action: If FTP functionality is required, upgrade vsftpd to a current, patched, and supported version, or migrate to a more secure file transfer protocol such as SFTP (SSH File Transfer Protocol) or FTPS (FTP over SSL/TLS), ensuring strong authentication and encryption are utilized.

    • Review and enforce strict access control lists (ACLs) on the FTP server’s directories to adhere to the principle of least privilege, minimizing potential data exposure.

  • Implement Mandatory Security Awareness Training for New Employees:

    • Develop and integrate a comprehensive security awareness training program into the onboarding process for all new hires. This training should be mandatory before granting network or system access.

    • Ensure the training covers essential topics such as phishing detection, social engineering tactics, strong password hygiene, data handling policies, and proper incident reporting procedures.

Mid-Term Actions (High Priority)

  • Upgrade Outdated Operating Systems:

    • Develop and execute a phased plan to upgrade all workstations currently running Windows 7 to a modern, supported operating system (e.g., Windows 10 or 11).

    • Prioritize the upgrade of systems handling sensitive data, those with network exposure, or those used by key personnel.

    • For any Windows 7 machines that cannot be immediately upgraded, implement compensating controls such as network segmentation, enhanced endpoint detection and response (EDR) solutions, and strict firewall rules to isolate them from the rest of the network.

  • Establish Regular Vulnerability Scanning Program:

    • Implement a schedule for regular, automated vulnerability scans of all internal and external-facing assets.

    • Establish clear procedures for reviewing scan results, prioritizing findings based on severity, and promptly remediating identified vulnerabilities.

Long-Term Strategic Actions (Ongoing Priority)

  • Continuous Security Training and Reinforcement:

    • Beyond annual training, implement ongoing security awareness campaigns, regular phishing simulations, and frequent internal communications to reinforce secure behaviors and keep employees vigilant against evolving threats.

  • Robust Patch Management Program:

    • Establish and maintain a comprehensive patch management program for all operating systems, applications, and network infrastructure devices to ensure timely application of security updates and patches.

  • Incident Response Plan Enhancement and Testing:

    • Regularly review and update the organization’s incident response plan to address new and emerging threat vectors.

    • Conduct periodic tabletop exercises and simulations to test the effectiveness of the incident response plan and ensure all relevant personnel are familiar with their roles and responsibilities during a security incident.
