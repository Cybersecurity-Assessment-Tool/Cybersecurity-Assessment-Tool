6. Recommendations

To address the identified vulnerabilities and control gaps, the following recommendations are provided. These should be prioritized based on their severity and potential impact.

  1. Immediate Decommission or Secure FTP Service:

    • Decommission the FTP server on port 21 immediately if it is not business-critical.

    • If FTP is essential, disable anonymous access and enforce strong authentication and authorization.

    • Update vsftpd to the latest stable and secure version. Consider replacing FTP with more secure protocols like SFTP or SCP for file transfers.

  2. Implement Multi-Factor Authentication (MFA) Universally:

    • Mandate MFA for all user logins to computers and workstations.

    • Mandate MFA for accessing all sensitive data systems and applications. This is a critical control against credential compromise.

  3. Strengthen Security Awareness Training Program:

    • Implement mandatory security awareness training for all new employees during their onboarding process.

    • Conduct annual, mandatory security awareness training for all employees to keep them informed about current threats and best practices.

  4. Upgrade End-of-Life Operating Systems:

    • Prioritize and execute a plan to upgrade all computers running Windows 7 to a currently supported operating system (e.g., Windows 10/11) to ensure critical security patches and updates are received.

  5. Regular Vulnerability Management:

    • Establish a routine schedule for network and application vulnerability scanning.

    • Implement a robust patch management program to ensure all systems and software are kept up-to-date.

  6. Develop and Test an Incident Response Plan:

    • Ensure a comprehensive incident response plan is in place, regularly reviewed, and tested through tabletop exercises to prepare for potential security incidents.
