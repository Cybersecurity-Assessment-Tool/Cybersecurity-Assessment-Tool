6. Recommendations

To address the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided:

6.1. High Priority Recommendations

  • Implement MFA for Computer Logins: Immediately deploy and enforce Multi-Factor Authentication for all employee computer logins. This significantly reduces the risk of unauthorized access due to compromised credentials.

  • Develop and Implement an Acceptable Use Policy (AUP): Create and formally implement a comprehensive AUP. All employees should be required to read, understand, and sign the policy upon hiring and annually thereafter.

  • Mandatory Security Awareness Training for New Employees: Establish and implement mandatory cybersecurity awareness training as part of the onboarding process for all new employees. This should cover common threats, organizational policies, and best practices.

6.2. Medium Priority Recommendations

  • Harden SSH Service on Port 22:

    • Disable Password Authentication: Configure SSH to use strong, key-based authentication exclusively, disabling password-based logins.

    • Implement Rate Limiting and IPS/IDS: Deploy firewall rules or an Intrusion Prevention/Detection System (IPS/IDS) to monitor and block suspicious SSH login attempts.

    • Restrict Access by IP Address: Limit SSH access to only essential administrative IP addresses or networks.

    • Ensure Up-to-Date Software: Regularly patch and update the SSH daemon and underlying operating system to mitigate known vulnerabilities.

    • Change Default SSH Port: Consider changing the default SSH port (22) to a non-standard high port to deter automated scanning and basic attacks (though this is security through obscurity, it can reduce noise).

6.3. General Security Best Practices

  • Regular Vulnerability Scanning and Penetration Testing: Schedule regular vulnerability scans and periodic penetration tests to proactively identify and address security weaknesses.

  • Patch Management: Ensure a robust patch management program is in place for all operating systems, applications, and network devices.

  • Review and Update Security Policies: Regularly review and update all security policies and procedures (including the AUP and training programs) to adapt to evolving threat landscapes.

  • Incident Response Plan: Develop and maintain a comprehensive incident response plan to effectively prepare for, detect, respond to, and recover from cybersecurity incidents.

7. Conclusion
