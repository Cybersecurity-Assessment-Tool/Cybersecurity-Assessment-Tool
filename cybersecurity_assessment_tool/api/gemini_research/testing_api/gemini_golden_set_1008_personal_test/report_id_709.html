Executive Summary

This report details a comprehensive cybersecurity posture assessment for Nexus Dynamics, based on an organizational security questionnaire and a targeted network scan conducted on November 22, 2025. The assessment identifies several critical vulnerabilities and significant security gaps that require immediate attention.

Key findings include:

  • Critical Organisational Gaps: A pervasive lack of Multi-Factor Authentication (MFA) across email, computer login, and sensitive data systems, coupled with the absence of an acceptable use policy and regular security awareness training, exposes the organization to significant human-factor risks and credential compromise.

  • Critical Technical Vulnerability: An internet-facing Nginx web server (version 1.18.0) running on port 443 was identified. This version is End-of-Life (EOL) and highly vulnerable to known exploits, representing an immediate and severe risk.

  • Potential Certificate Misconfiguration: The SSL certificate presented by the web server uses a Common Name of www.acme-corp.com, which does not match the organization’s domain. This could indicate a misconfigured, outdated certificate, or potentially discloses information about past or associated entities.

Immediate remediation efforts should prioritize the implementation of MFA, upgrading the Nginx web server, and establishing foundational security policies and training programs to significantly enhance the organization’s security posture.

Analysis of Security Control Gaps

The questionnaire results reveal a complete absence of several fundamental security controls, collectively representing a high risk:

  • Lack of MFA: The non-implementation of Multi-Factor Authentication across all specified access points (email, computer login, sensitive data systems) creates a significant attack surface. Credential theft, a common vector for breaches, can lead to unauthorized access to critical systems with only a username and password.

  • Policy Deficiencies: The absence of an employee acceptable use policy leaves the organization vulnerable to misuse of IT resources, data leakage, and undefined security expectations for employees.

  • Training Deficiencies: Without regular security awareness training for both new and existing employees, the workforce remains susceptible to social engineering attacks, phishing, and malware, turning them into potential weak points in the organization’s defenses.

Detailed Technical Findings

  • Port 443 (HTTPS) - Nginx 1.18.0:

    • An Nginx web server is running on the standard HTTPS port (443), indicating an internet-facing web service.

    • The identified version, Nginx 1.18.0, was released in April 2020 and reached its End-of-Life (EOL) in April 2022. This means the version is no longer supported by the vendor and does not receive security updates, making it highly vulnerable to publicly known exploits and zero-day attacks. This is a critical security flaw.

    • The SSL certificate presented by this service has a Common Name (CN) of www.acme-corp.com. This is a clear mismatch with the organization’s provided domain (NexusDynamics.net) and could indicate a misconfigured, outdated, or generic certificate. This also potentially discloses information about previous associations or other services, and could lead to browser warnings for users.

Recommendations

To mitigate the identified risks and significantly enhance the organization’s cybersecurity posture, the following actionable recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA):

    • Immediately deploy MFA for all critical access points, including email systems, computer logins, and sensitive data applications.

    • Prioritize robust MFA methods such as hardware tokens or authenticator apps over SMS-based MFA.

    • Develop a clear rollout plan, provide comprehensive user training, and establish robust recovery procedures for MFA.

  2. Upgrade/Replace Outdated Nginx Web Server:

    • Urgently plan and execute an upgrade of the Nginx web server to the latest stable and supported version (e.g., Nginx 1.26.x or newer).

    • If an upgrade is not feasible, migrate the web service to a currently supported web server solution.

    • Perform thorough testing in a staging environment before deploying to production to ensure application compatibility and stability.

    • Implement a robust patch management process for all public-facing services.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines acceptable use of company IT resources, data handling procedures, internet usage guidelines, and employee security responsibilities.

    • Distribute the AUP to all employees, obtain signed acknowledgments, and review it periodically (e.g., annually or upon significant changes).

    • Integrate AUP review and acknowledgment into the new employee onboarding process.

  4. Establish and Maintain Security Awareness Training Program:

    • Implement mandatory security awareness training for all new employees during onboarding, covering foundational cybersecurity concepts.

    • Conduct annual refresher security awareness training for all employees, using engaging content and covering current threats like phishing, social engineering, ransomware, and data protection best practices.

    • Consider incorporating simulated phishing campaigns to reinforce training and assess employee vigilance in a controlled manner.

  5. Review and Correct SSL Certificate Configuration:

    • Immediately investigate the SSL certificate on 192.168.10.5 to understand why www.acme-corp.com is being used as the Common Name instead of a domain related to Nexus Dynamics.

    • Replace the existing certificate with a valid, trusted SSL/TLS certificate that accurately reflects the organization’s domain (NexusDynamics.net) and ensures proper encryption.

    • Establish a certificate management process to track expiration dates and ensure timely renewals to avoid service interruptions and trust warnings.
