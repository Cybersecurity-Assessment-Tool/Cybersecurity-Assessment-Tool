Executive Summary

This report details the cybersecurity posture of Digital Drift, based on a combination of organizational self-assessment data, an external network scan, and an review of existing risks. While the organization demonstrates good practices in employee acceptable use policies and security awareness training, several critical gaps have been identified. Key findings include a significant lack of Multi-Factor Authentication (MFA) for computer logins and sensitive data systems, and the presence of an outdated and End-of-Life (EOL) Nginx web server exposed to the internet. These issues collectively represent a high to critical risk to the organization’s security and data integrity, requiring immediate attention and remediation.

Key Findings

  • State: Open

  • Service Name: https

  • Version: 1.18.0

  • Observation: The Nginx web server version 1.18.0 was released in April 2020 and reached its End-of-Life (EOL) in April 2023. This means it no longer receives security updates or bug fixes, making it highly vulnerable to known and newly discovered exploits. Exposing an EOL web server is a critical security risk.

Recommendations

To mitigate the identified risks and enhance Digital Drift’s cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Computer Logins: Mandate Multi-Factor Authentication (MFA) for all user logins to computers and workstations. This adds a crucial layer of security, significantly reducing the risk of unauthorized access even if credentials are stolen.

  2. Implement MFA for Sensitive Data Systems: Deploy and enforce MFA for all systems, applications, and databases that store or process sensitive organizational or customer data. This is paramount to protecting critical assets from unauthorized access.

  3. Upgrade/Patch Nginx Web Server: Immediately upgrade the Nginx web server to a currently supported and patched version (e.g., the latest stable mainline or long-term support version). Alternatively, replace it with a fully supported web server solution. Ensure regular patching and updates are part of the maintenance cycle for all public-facing services.

  4. Regular Software Audits: Establish a process for regular auditing of all installed software, especially on internet-facing systems, to identify and update or decommission End-of-Life (EOL) or vulnerable versions.

  5. Continuous Security Awareness Training: While good training is in place, reinforce the importance of MFA and secure access practices through ongoing security awareness programs for all employees.

Conclusion

Digital Drift has established foundational security practices in policy and training. However, the absence of MFA for critical internal access points and the exposure of an End-of-Life web server represent severe vulnerabilities that could lead to significant security incidents. Addressing these critical risks through the recommended actions should be prioritized to strengthen the organization’s overall cybersecurity resilience.
