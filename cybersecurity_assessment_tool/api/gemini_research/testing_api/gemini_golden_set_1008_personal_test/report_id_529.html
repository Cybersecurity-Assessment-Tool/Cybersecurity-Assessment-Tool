Executive Summary

This report details the findings of a cybersecurity posture assessment for Moxie Marketing, conducted on 2026-01-27. The assessment involved an analysis of organizational security controls, a review of network scan data, and correlation with any pre-existing known risks.

Key findings indicate several critical gaps in security controls, particularly concerning Multi-Factor Authentication (MFA) implementation for computer logins and sensitive data systems, and the absence of an Employee Acceptable Use Policy. Technically, an outdated web server version (Nginx 1.18.0) was identified, along with a potential SSL certificate common name mismatch. While no pre-existing vulnerabilities were reported in the provided data, these identified areas present significant risks to the organization’s overall security posture. Actionable recommendations are provided to mitigate these risks and enhance security.

Analysis of Security Control Gaps

The questionnaire highlights critical gaps in the organization’s security posture:

  • Lack of MFA for Computer Logins (SC-02): Without Multi-Factor Authentication for computer access, a single compromised credential can lead to unauthorized network access, data breaches, and system compromise. This represents a significant vulnerability.

  • Lack of MFA for Sensitive Data Systems (SC-03): Similarly, the absence of MFA for systems holding sensitive data drastically increases the risk of data exfiltration and compliance violations should credentials for these systems be compromised.

  • Absence of an Employee Acceptable Use Policy (SC-04): An Acceptable Use Policy (AUP) sets clear guidelines for employee behavior regarding company IT assets and data. Its absence can lead to inconsistent security practices, increased risk of misuse, and difficulties in enforcing security policies.

Technical Findings Analysis

  • Outdated Nginx Version: Nginx version 1.18.0 was released in April 2020. As of the time of this report, significantly newer stable versions (e.g., Nginx 1.24.x or 1.26.x) are available. Running outdated software increases the risk of known vulnerabilities being exploited. For instance, CVEs discovered after 1.18.0 may not be patched, leaving the service vulnerable to various attacks including denial-of-service, information disclosure, or remote code execution.

  • SSL Certificate Common Name Mismatch/Third-Party Certificate: The SSL certificate’s common name () does not match the organization’s stated website domain (www.MoxieMarketing.com). This could indicate a misconfiguration, a shared hosting environment, or a potential security concern where the certificate does not legitimately belong to the observed service. This mismatch can lead to browser warnings for users, undermine trust, and may make it easier for phishing attacks to succeed.

Recommendations

To mitigate the identified risks and enhance the organization’s overall cybersecurity posture, the following recommendations are provided:

Critical Recommendations (High Severity Risks)

  1. Implement Multi-Factor Authentication (MFA) for Computer Logins: Immediately deploy and enforce MFA for all employee computer logins across the organization. This should apply to both local and network-based authentication mechanisms.

  2. Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems: Mandate MFA for access to all systems containing sensitive organizational or client data. Prioritize systems with the highest data classification levels.

  3. Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP outlining the proper use of company IT assets, networks, and data. Ensure all employees read, understand, and acknowledge this policy, with regular refreshers and mandatory training.

Important Recommendations (Medium Severity Risks)

  1. Upgrade Nginx Web Server: Plan and execute an upgrade of the Nginx web server from version 1.18.0 to the latest stable release (e.g., 1.26.x or newer). Ensure proper testing is conducted in a staging environment before deploying to production to minimize service disruption.

  2. Investigate and Correct SSL Certificate Configuration: Determine the reason for the SSL certificate common name mismatch. If it’s a dedicated server, acquire and install a certificate issued for www.MoxieMarketing.com or the appropriate domain. If it’s a shared hosting environment, confirm the certificate configuration is correct for the intended service and ensure it aligns with security best practices.
