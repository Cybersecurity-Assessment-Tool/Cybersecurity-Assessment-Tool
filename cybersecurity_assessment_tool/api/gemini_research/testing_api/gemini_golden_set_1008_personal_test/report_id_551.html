Executive Summary

Calyx Botany’s cybersecurity posture exhibits a mix of strengths and critical weaknesses. While the organization employs MFA for email and computer logins, its absence for sensitive data systems represents a significant vulnerability. Key foundational security elements, such as an Acceptable Use Policy and regular security awareness training, are currently lacking. Technically, an Nmap scan confirmed an open SSH port (22) on the loopback address 127.0.0.1, which correlates with a documented critical risk of "Localhost Exposed" (CVSS 10.0). This report details these findings and provides actionable recommendations to mitigate the identified risks and strengthen security controls.

Recommendations

To mitigate the identified risks and enhance Calyx Botany’s overall cybersecurity posture, the following recommendations are provided:

  1. Implement MFA for Sensitive Data Systems:

    • Action: Mandate and deploy Multi-Factor Authentication for all systems handling sensitive organizational data.

    • Benefit: Significantly reduces the risk of unauthorized access due to compromised credentials.

  2. Develop and Implement an Acceptable Use Policy (AUP):

    • Action: Draft, review, and formally implement a comprehensive AUP for all employees. Ensure all employees read and acknowledge the policy annually.

    • Benefit: Establishes clear expectations for employee conduct regarding IT resource usage, minimizing misuse and enhancing legal defensibility.

  3. Establish Security Awareness Training for New Employees:

    • Action: Develop and integrate mandatory security awareness training into the onboarding process for all new hires.

    • Benefit: Equips new employees with essential knowledge to identify and respond to common cyber threats from day one.

  4. Conduct Annual Security Awareness Training for All Employees:

    • Action: Implement a program for mandatory annual security awareness training for all employees, covering current threats like phishing, ransomware, and social engineering.

    • Benefit: Reinforces security best practices, keeps employees updated on evolving threats, and fosters a security-conscious culture.

  5. Investigate and Remediate "Localhost Exposed" Vulnerability:

    • Action: Urgently investigate how services on 127.0.0.1 (e.g., SSH on port 22) are exposed or externally accessible, contrary to the typical loopback interface isolation. Determine if this indicates an accidental external exposure, misconfiguration, or a component of a larger system.

    • Action: Based on the investigation, implement appropriate remediation steps, such as:

      • Restricting SSH access to necessary internal IP addresses only.

      • Disabling SSH if not actively required, especially if not externally facing.

      • Ensuring proper firewall rules are in place to prevent external access to internal services.

      • Reviewing network architecture to prevent unintended exposure of internal services.

    • Benefit: Eliminates a critical vulnerability that could lead to unauthorized access, data breaches, or full system compromise.
