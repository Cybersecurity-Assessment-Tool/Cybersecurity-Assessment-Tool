Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Stone Arch Masonry. The analysis integrates findings from a recent network scan, organizational security questionnaire responses, and pre-existing risk data.

Key findings include the direct exposure of a MySQL database service (port 3306) to the network, which poses a significant risk for data compromise. Furthermore, critical gaps were identified in the organization’s security controls, specifically the lack of Multi-Factor Authentication (MFA) for sensitive data systems and the absence of an employee acceptable use policy. These issues, combined with existing vulnerabilities, indicate areas requiring immediate attention to enhance the overall security posture and reduce the likelihood of a successful cyberattack.

Analysis of Security Controls

The organization demonstrates good practice in implementing MFA for email and computer logins, and in conducting regular security awareness training. However, two critical gaps were identified:

  • Lack of MFA for Sensitive Data Systems: Failure to implement MFA for sensitive data systems significantly increases the risk of unauthorized access should credentials be compromised. This is a severe vulnerability.

  • Absence of an Acceptable Use Policy (AUP): Without an AUP, employees may not be aware of their responsibilities regarding the proper use of organizational assets and data, leading to potential misuse or security incidents.

Analysis of Technical Findings

The scan revealed an open MySQL database service on port 3306 running version 5.7.33. Direct exposure of a database port to the network, especially if accessible from external or untrusted networks, is a critical security vulnerability. While MySQL 5.7.33 is still under extended support until October 2023, the primary risk lies in the accessibility of the service itself. An attacker gaining access to this service could potentially compromise sensitive data stored within the database.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture, the following recommendations are provided. These are prioritized based on severity and potential impact.

High Priority Recommendations

  1. Secure Database Access (Port 3306):

    • Immediate Fix: Implement strict firewall rules to restrict access to MySQL port 3306 to only necessary, trusted internal IPs. If external access is required, it should be severely limited and ideally facilitated through a secure VPN or an application-layer proxy, not direct exposure.

    • Long-Term Fix: Implement a Virtual Private Network (VPN) solution for all remote database access. Ensure the database server is not directly exposed to the internet. Consider database-level encryption for data at rest and in transit.

    • Review and update all database user accounts, ensuring strong, unique passwords and the principle of least privilege.

  2. Implement MFA for Sensitive Data Systems:

    • Mandate Multi-Factor Authentication (MFA) for all systems that store, process, or transmit sensitive organizational data. This adds a crucial layer of security, significantly reducing the risk of unauthorized access even if primary credentials are stolen.

    • Conduct an inventory of all systems containing sensitive data to ensure comprehensive MFA coverage.

Medium Priority Recommendations

  1. Develop and Enforce an Acceptable Use Policy (AUP):

    • Develop a comprehensive AUP that clearly outlines employee responsibilities, acceptable and unacceptable use of company IT resources, and data handling procedures.

    • Distribute the AUP to all employees, obtain signed acknowledgments, and integrate it into new employee onboarding and regular security awareness training.

General Best Practices

  1. Regular Software Updates and Patching: Ensure all operating systems, applications, and network devices, especially the MySQL server, are regularly updated and patched to protect against known vulnerabilities.

  2. Regular Vulnerability Scanning and Penetration Testing: Schedule periodic vulnerability scans and penetration tests to proactively identify and address security weaknesses.

  3. Data Backup and Recovery Plan: Implement and regularly test a robust data backup and recovery plan to ensure business continuity in case of data loss or ransomware attacks.

Conclusion

The assessment highlights that while Stone Arch Masonry has some foundational security practices in place, significant vulnerabilities related to database exposure and critical control gaps in MFA for sensitive data and AUP presence need immediate attention. Addressing these recommendations will substantially improve the organization’s defense against cyber threats and bolster its overall security posture.
