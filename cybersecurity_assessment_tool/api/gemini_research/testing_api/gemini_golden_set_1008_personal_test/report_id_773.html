Executive Summary

This report details the cybersecurity posture of Obsidian Operatives, based on an analysis of network scan data, organizational security questionnaire responses, and pre-existing risk assessments. The organization demonstrates a commendable adoption of multi-factor authentication (MFA) for critical access points. However, significant vulnerabilities were identified regarding employee security awareness training, which is currently lacking for both new and existing staff. Furthermore, network scans revealed multiple instances of exposed Remote Desktop Protocol (RDP) services, corroborating and expanding upon previously identified high-severity risks. These findings indicate a critical need for immediate intervention in security awareness programs and the secure configuration of network services to mitigate potential exploitation and enhance overall security resilience.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of Obsidian Operatives, the following actionable recommendations are provided:

  1. Implement Comprehensive Security Awareness Training Programs:

    • For New Hires: Establish and enforce mandatory security awareness training for all new employees during their onboarding process. This training should cover fundamental security concepts, common attack vectors (e.g., phishing, social engineering), strong password practices, and incident reporting procedures.

    • For All Employees (Annual): Institute a mandatory annual security awareness training program for all staff. Training content should be dynamic, updated regularly to reflect current threat landscapes, and delivered engagingly to reinforce secure behaviors and educate employees on new risks.

  2. Secure Exposed RDP Services:

    • Immediate Network Restrictions: For all identified RDP-exposed hosts (, , and any others), implement strict network firewall rules to limit RDP access to only authorized administrative IPs or mandate access exclusively through a secured Virtual Private Network (VPN) with MFA.

    • Enhanced Authentication: Where RDP remains necessary, ensure that multi-factor authentication (MFA) is enforced for all RDP logins. Implement strong password policies, account lockout thresholds, and monitor RDP login attempts for anomalies.

    • Vulnerability Management: Conduct regular vulnerability scanning and penetration testing to proactively identify and remediate all instances of exposed or insecure RDP services across the network. Prioritize patching RDP vulnerabilities promptly.

  3. Review and Update Incident Response Plan: Given the critical nature of the identified risks (especially RDP exposure and human error potential), it is crucial to review and update the organization’s incident response plan. Ensure the plan adequately addresses potential compromises originating from RDP exploitation, social engineering, or human error due to a lack of training, including clear steps for detection, containment, eradication, recovery, and post-incident analysis.

  4. Conduct Regular Security Audits and Assessments: Establish a schedule for periodic comprehensive security audits and penetration tests. These proactive assessments will help identify emerging vulnerabilities, evaluate the effectiveness of implemented controls, and ensure ongoing compliance with best practices.
