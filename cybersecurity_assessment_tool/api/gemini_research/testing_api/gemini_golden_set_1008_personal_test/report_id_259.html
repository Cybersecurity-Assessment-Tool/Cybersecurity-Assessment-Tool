Executive Summary

This report details the security posture assessment for Firebrand Media, conducted on 2026-01-26. The assessment involved a review of organizational security controls, an external network scan, and an analysis of identified risks.

Overall, Firebrand Media demonstrates a commendable commitment to multi-factor authentication (MFA) across critical systems and has an acceptable use policy in place. However, critical gaps were identified in security awareness training for both new and existing employees, representing a significant human-element vulnerability. A network scan revealed an open SSH port, for which further investigation is recommended due to a lack of version information. While no pre-existing vulnerabilities were reported, the identified gaps introduce considerable risk that requires immediate attention. Addressing these areas is crucial for enhancing the organization’s defense against modern cyber threats.

Key Findings from Control Review

  • Strengths: Firebrand Media has successfully implemented Multi-Factor Authentication (MFA) for email, computer logins, and sensitive data systems. An acceptable use policy is also in place, indicating foundational policy controls are established.

  • Gaps: There are significant deficiencies in security awareness training. Specifically, new employees do not receive security training upon onboarding, and existing employees do not undergo annual training. This leaves the organization highly vulnerable to human-centric attacks such as phishing, social engineering, and the exploitation of common user errors, as employees may lack the knowledge to identify and respond to threats.

Recommendations

To mitigate the identified risks and enhance Firebrand Media’s overall security posture, the following actionable recommendations are provided:

  1. Implement Mandatory New Employee Security Awareness Training: Develop and implement a comprehensive security awareness training program for all new employees. This training should be mandatory and completed before new hires gain access to company systems and data. Content should include:

    • Phishing and social engineering recognition.

    • Password best practices and MFA usage.

    • Acceptable Use Policy review.

    • Data handling and classification guidelines.

  2. Establish Annual Employee Security Awareness Training Program: Institute a mandatory annual (or more frequent) security awareness training program for all existing employees. This recurring training should:

    • Reinforce core security principles.

    • Educate on current and emerging threat vectors (e.g., ransomware, supply chain attacks).

    • Provide refresher on organizational policies and incident reporting procedures.

    • Utilize engaging formats (e.g., interactive modules, simulated phishing campaigns).

  3. Verify and Secure SSH Service on :

    • Conduct an authenticated scan or manual verification on the host at to identify the exact version of the SSH service running.

    • Ensure the SSH server is running a current, supported, and securely configured version. Apply all relevant security patches promptly.

    • Implement strong SSH security practices, including:

      • Disable root login via SSH.

      • Enforce key-based authentication exclusively, disabling password authentication.

      • Configure strong ciphers, Message Authentication Codes (MACs), and Key Exchange algorithms.

      • Limit SSH access to only necessary administrative IP addresses or subnets via firewall rules.

      • Implement a robust logging and monitoring solution for SSH access attempts.

Conclusion

Firebrand Media has a solid foundation regarding MFA implementation across critical access points. However, the identified critical gaps in security awareness training pose a significant and immediate risk to the organization, as the human element remains a primary target for attackers. Additionally, the open SSH service requires immediate attention to ensure it is securely configured and patched. Addressing these training deficiencies and thoroughly securing the identified open SSH service will substantially improve the organization’s defensive capabilities against a wide range of cyber threats. Proactive engagement with these recommendations is crucial for maintaining a robust and resilient security posture.
