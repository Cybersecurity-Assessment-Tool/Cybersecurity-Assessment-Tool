Executive Summary

This report details the cybersecurity posture of Foresight Strategies based on an organizational questionnaire, a technical network scan, and pre-existing risk data. The assessment highlights a strong foundation in some areas, such as an established acceptable use policy and regular security awareness training for all employees. However, critical gaps were identified, notably the lack of Multi-Factor Authentication (MFA) for computer logins and sensitive data system access, as well as a missed opportunity for security awareness training for new employees. The technical scan of an internal host (192.168.0.5) showed no open ports, indicating a good internal attack surface posture for that specific host, though one previously identified external risk related to an unencrypted web server requires further verification. Addressing the identified MFA and training deficiencies should be prioritized to enhance the overall security baseline.

Analysis of Controls

The review of security controls indicates both strengths and critical weaknesses:

  • Strengths: The organization has implemented MFA for email access, maintains an acceptable use policy, and conducts annual security awareness training for all employees. These are positive steps towards a robust security posture.

  • Critical Gaps: The absence of MFA for computer logins and sensitive data system access poses a significant risk, as compromised credentials could lead to unauthorized access to critical internal systems and data. Furthermore, neglecting security awareness training for new employees leaves a window of vulnerability during their onboarding period.

Recommendations

To strengthen Foresight Strategiesâ€™ cybersecurity posture, the following recommendations are provided:

  • Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Prioritize the deployment of MFA for all user accounts logging into organizational computers and network resources.

    • Evaluate solutions such as hardware tokens, authenticator apps, or biometrics.

  • Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems:

    • Extend MFA requirements to all systems that store, process, or transmit sensitive organizational or customer data.

    • Conduct an inventory of sensitive data systems to identify all systems requiring MFA enforcement.

  • Establish Security Awareness Training for New Employees:

    • Develop and implement mandatory security awareness training as part of the onboarding process for all new hires.

    • Cover topics such as phishing, password hygiene, social engineering, and acceptable use policies.

  • Verify and Mitigate Unencrypted Web Server Risk:

    • Investigate the previously identified "Unencrypted Web Server" risk to determine if it applies to any external-facing systems or other internal hosts.

    • Ensure all public-facing web services are secured with HTTPS/TLS. Redirect all HTTP traffic to HTTPS.

    • If the risk was indeed remediated on 192.168.0.5, document the remediation details.

  • Regular Network Scans and Vulnerability Assessments:

    • Conduct periodic internal and external network scans to identify any newly opened ports, services, or misconfigurations.

    • Perform regular vulnerability assessments and penetration tests to proactively identify and address weaknesses.
