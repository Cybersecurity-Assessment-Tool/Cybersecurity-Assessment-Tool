Analysis of Questionnaire Results

The organization demonstrates strong adherence to Multi-Factor Authentication (MFA) across critical access points (email, computers, sensitive data systems), which is a significant positive. However, two critical gaps were identified:

  • Lack of Employee Acceptable Use Policy (Q4): Without a formal policy, employees may be unaware of the proper and prohibited use of company IT resources, increasing risks of misuse, data leakage, and compliance issues.

  • No Security Awareness Training for New Employees (Q5): New hires represent a significant potential attack vector if not properly educated on security best practices from day one. This oversight can lead to increased susceptibility to social engineering, phishing attacks, and unintentional data breaches.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of Calyx Botany, the following recommendations are provided:

  1. Implement an Employee Acceptable Use Policy:

    • Develop and formally implement a comprehensive Acceptable Use Policy (AUP) for all IT assets and data.

    • Ensure all employees read, understand, and acknowledge the AUP, preferably annually.

    • The AUP should clearly define prohibited activities, data handling guidelines, and consequences for non-compliance.

  2. Establish Mandatory Security Awareness Training for New Employees:

    • Integrate security awareness training into the onboarding process for all new hires.

    • Cover essential topics such as phishing identification, password hygiene, data classification, physical security, and reporting suspicious activities.

    • Reinforce existing annual training programs for all employees.

  3. Harden and Monitor SSH Service:

    • Implement strong authentication for SSH, preferably using key-based authentication only, and disable password-based login if possible.

    • Ensure strong, unique passphrases for SSH keys.

    • Restrict SSH access to only necessary IP addresses or networks using firewall rules.

    • Disable root login directly via SSH.

    • Change the default SSH port (22) to a non-standard port (though this is often considered security by obscurity, it can deter automated scanning).

    • Configure SSH to use the latest secure protocols and ciphers.

    • Implement logging and monitoring for SSH access attempts to detect brute-force attacks or unauthorized access.

    • Conduct more in-depth scans using authenticated credentials or advanced Nmap scripts to identify service versions and potential vulnerabilities.

Conclusion

Calyx Botany has a good foundation with its widespread use of MFA. However, critical gaps in policy and initial employee training, coupled with an exposed SSH service, present tangible risks.
