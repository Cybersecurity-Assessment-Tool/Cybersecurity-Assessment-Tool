Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Astraeus Aerospace, integrating findings from a network scan, a security questionnaire, and existing risk data. The assessment reveals several critical vulnerabilities and significant security control gaps that require immediate attention. Key findings include a severely outdated FTP service with anonymous login enabled on a scanned target, a lack of multi-factor authentication (MFA) for computer and sensitive data system access, and deficiencies in foundational security policies and training. Addressing these issues is crucial for enhancing the organization’s overall security stance and mitigating potential breaches, including unauthorized access and data exposure.

Recommendations

To significantly improve Astraeus Aerospace’s cybersecurity posture, the following recommendations are provided. These are prioritized based on the identified severity of the risks.

Critical Recommendations (Immediate Action Required)

  • Implement MFA for Computer Login: Deploy Multi-Factor Authentication (MFA) solutions for all computer logins across the organization. This should be a top priority to protect against credential theft and unauthorized access to internal systems.

  • Implement MFA for Sensitive Data Systems: Extend MFA requirements to all systems that handle or store sensitive organizational data. This adds a crucial layer of security to high-value assets and reduces the risk of data breaches.

High Priority Recommendations

  • Secure or Remove FTP Service:

    • Immediately disable anonymous FTP login on the server at 10.0.0.15. Review and tighten access controls for all authenticated FTP users.

    • Upgrade to the latest stable and supported version. Regularly patch and update all software.

    • Evaluate the necessity of an FTP service. If secure file transfer is required, migrate to a more secure protocol like SFTP (SSH File Transfer Protocol) or FTPS (FTP Secure). Remove the service if it is no longer needed.

    • Ensure the FTP server, if absolutely necessary to be internet-facing, is protected by a robust firewall, intrusion prevention system (IPS), and regularly monitored.

  • Develop and Enforce an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines employee responsibilities regarding company IT assets, data handling, internet usage, and security protocols. Ensure all employees acknowledge, understand, and comply with this policy through mandatory sign-offs.

  • Implement Security Awareness Training for New Employees: Develop and mandate security awareness training as part of the onboarding process for all new employees. This ensures that new hires are immediately aware of cybersecurity best practices, organizational policies, and common threats like phishing and social engineering.

Medium Priority Recommendations

  • Upgrade Outdated Operating Systems: Prioritize the upgrade of all machines running Windows 7 (and other unsupported operating systems) to a currently supported version (e.g., Windows 10/11). Develop a detailed migration plan that includes hardware assessment, software compatibility checks, and a phased rollout to minimize disruption.

Conclusion

Astraeus Aerospace faces significant cybersecurity risks due to outdated infrastructure, insecure configurations, and critical gaps in security controls and policies. By implementing the recommended actions, particularly those addressing multi-factor authentication and the insecure FTP service, the organization can substantially reduce its attack surface and strengthen its defense against cyber threats. Continuous monitoring, regular security audits, and ongoing employee training will be essential for maintaining a strong and resilient security posture in the face of evolving cyber threats.
