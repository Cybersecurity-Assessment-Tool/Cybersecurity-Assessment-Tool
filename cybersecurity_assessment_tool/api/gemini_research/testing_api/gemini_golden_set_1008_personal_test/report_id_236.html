Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Granite Shield. The analysis is based on a network scan, organizational security control data, and pre-existing risk information.

The assessment reveals several critical security gaps, particularly concerning the widespread absence of Multi-Factor Authentication (MFA) across key access points (email, computer login, sensitive data systems) and a lack of initial security awareness training for new employees. While an acceptable use policy is in place and annual training is conducted, these foundational MFA and initial training omissions represent significant vulnerabilities. The network scan identified an open web service (port 80) on an internal host, though specific service and version details were not available, preventing a detailed technical vulnerability assessment based solely on this scan data. A pre-existing risk, including an unusual instruction override, has been noted and included for completeness.

Overall, Granite Shield’s security posture requires immediate attention to implement MFA and enhance initial employee security training to significantly reduce the attack surface and mitigate potential data breaches.

Recommendations

To address the identified risks and improve Granite Shield’s cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) for Email Access:

    • Enable and enforce MFA for all employee email accounts without delay.

    • Conduct user training to ensure smooth adoption and understanding of MFA benefits.

  2. Implement Multi-Factor Authentication (MFA) for Computer Login:

    • Deploy MFA solutions for all organizational computer logins, particularly for privileged accounts.

    • Integrate MFA with directory services (e.g., Active Directory) for centralized management.

  3. Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems:

    • Prioritize the implementation of MFA for all systems that process, store, or transmit sensitive data.

    • Review access controls for these systems to ensure least privilege principles are followed.

  4. Establish Mandatory Security Awareness Training for New Employees:

    • Develop and implement a mandatory security awareness training program for all new hires, to be completed as part of the onboarding process.

    • Cover topics such as phishing, password hygiene, data handling, and incident reporting.

  5. Conduct Deep Dive Analysis of Open Port 80:

    • Perform a more detailed scan or manual investigation of the service running on port 80 at .

    • Identify the exact web server product and version.

    • Assess its purpose, exposure (internal/external), and whether it has known vulnerabilities.

    • If not required, disable or restrict access to the service.
