Executive Summary

This report details the cybersecurity posture of Digital Drift, based on an analysis of provided organizational data, a network scan, and existing risk information. The assessment reveals several critical areas requiring immediate attention, particularly regarding Multi-Factor Authentication (MFA), an acceptable use policy, and the security of exposed services.

Key findings include:

  • Security Control Gaps: Three critical security control gaps were identified from the organizational questionnaire, primarily related to MFA implementation for email and computer access, and the absence of an employee acceptable use policy.

  • Technical Vulnerabilities: An open port (22) was identified on the target 127.0.0.1, presenting a potential attack vector if not properly secured.

  • Pre-existing Risks: One critical pre-existing risk, ’Localhost Exposed’, was noted with a severity of 10.0.

Immediate action is recommended to mitigate these identified risks and enhance the overall security posture.

Recommendations

Based on the identified gaps and vulnerabilities, the following recommendations are provided to enhance the security posture of Digital Drift:

  1. Address the ’Localhost Exposed’ risk by reviewing network configurations, access controls, and ensuring localhost services are not unnecessarily exposed or have unhardened configurations.

  2. Develop and implement an employee acceptable use policy.

  3. Implement Multi-Factor Authentication (MFA) for email.

  4. Implement Multi-Factor Authentication (MFA) for log into computers.

  5. Review and secure services running on open port 22 on 127.0.0.1. Implement firewall rules, strong authentication, and ensure services are patched and configured securely.
