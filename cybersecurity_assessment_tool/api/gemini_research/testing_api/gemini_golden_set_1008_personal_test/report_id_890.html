Executive Summary

This report details the cybersecurity posture of Atlas Mapping, synthesizing information from a network scan, organizational security questionnaire responses, and pre-existing risk documentation. The analysis reveals several critical vulnerabilities and significant security control gaps that require immediate attention.

Key findings include the presence of an anonymous and outdated FTP service on an internal network host, which presents a severe risk of unauthorized access and data compromise. Furthermore, the organization lacks Multi-Factor Authentication (MFA) for both computer logins and sensitive data systems, significantly increasing the risk of account compromise. The absence of an Employee Acceptable Use Policy and the continued use of outdated operating systems further weaken the overall security posture. Addressing these identified risks and implementing the recommended controls is crucial for enhancing Atlas Mapping’s resilience against cyber threats.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of Atlas Mapping, the following recommendations are provided. These actions are prioritized based on their potential impact and severity.

  1. Immediately Address Anonymous FTP Service:

    • Disable anonymous login for the FTP service on .

    • Review and restrict access to the FTP server to only necessary internal IP addresses or authenticated users.

    • Consider migrating from FTP to a more secure file transfer protocol (e.g., SFTP, SCP) if file transfer functionality is still required.

  2. Upgrade/Replace Outdated FTP Server Software:

    • Update vsftpd to the latest stable and supported version, or replace it with a modern, secure FTP/SFTP server solution.

    • Ensure all server software is regularly patched and maintained.

  3. Implement Multi-Factor Authentication (MFA) for All Logins:

    • Deploy MFA for all computer login processes across the organization.

    • Implement MFA for accessing all sensitive data systems and applications. This is a critical control against credential compromise.

  4. Develop and Enforce an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining acceptable and unacceptable use of company IT resources, data, and internet access.

    • Ensure all employees are trained on the AUP and acknowledge its terms.

    • Regularly review and update the AUP.

  5. Upgrade Outdated Operating Systems:

    • Prioritize the upgrade of all Windows 7 workstations to a currently supported operating system (e.g., Windows 10 or 11).

    • Implement a robust patch management program for all operating systems and applications.

  6. Regular Security Awareness Training:

    • Continue and expand existing security awareness training programs to cover topics related to MFA importance, phishing, social engineering, and safe computing practices.

    • Emphasize the risks associated with outdated systems and software.

  7. Conduct Regular Vulnerability Scanning and Penetration Testing:

    • Establish a schedule for regular internal and external vulnerability scanning to identify new weaknesses.

    • Periodically engage third-party security experts to perform penetration tests to simulate real-world attacks.
