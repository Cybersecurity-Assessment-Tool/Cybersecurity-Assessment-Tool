Risk Assessment

Based on the analysis of the questionnaire responses and technical scan results, the following risks have been identified:

Identified Risks
Risk Name Overview Severity
Lack of Multi-Factor Authentication (MFA) for Email Access The organization does not require MFA for accessing email accounts, significantly increasing the risk of account compromise via phishing or brute-force attacks. High
Lack of Multi-Factor Authentication (MFA) for Sensitive Data Systems MFA is not enforced for sensitive data systems, leaving critical information vulnerable to unauthorized access if credentials are compromised. High
Use of End-of-Life (EOL) Nginx Version 1.18.0 The Nginx web server running on port 443 (version 1.18.0) is an End-of-Life product. This version no longer receives security updates, leaving it vulnerable to known exploits and future disclosed vulnerabilities. Critical
SSL Certificate Common Name Mismatch The SSL certificate used by the Nginx server has a common name (www.acme-corp.com) that does not match the organization’s domain (www.StoneArchMasonry.com). This can lead to browser warnings, diminish user trust, and may indicate a misconfiguration or improper certificate use. Medium

Recommendations

To address the identified risks and enhance the overall cybersecurity posture of , the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) for Email Access:

    • Enforce MFA for all user accounts accessing email. This significantly reduces the risk of unauthorized access even if passwords are stolen.

  2. Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems:

    • Require MFA for login to any system or application that stores or processes sensitive organizational data.

  3. Upgrade End-of-Life (EOL) Nginx Server:

    • Immediately upgrade the Nginx web server (currently 1.18.0) to a current, supported stable version (e.g., 1.24.x or newer). This will ensure the server receives critical security patches and bug fixes.

    • Consider migrating to Nginx Plus or a similar enterprise-grade solution if advanced features and dedicated support are required.

  4. Rectify SSL Certificate Mismatch:

    • Obtain and install a valid SSL certificate for the Nginx server that correctly matches the domain name associated with the service (e.g., www.StoneArchMasonry.com or a suitable internal FQDN if it’s an internal-only service).

    • Ensure all certificates are properly managed, renewed, and configured to maintain trust and encryption.

Conclusion

While demonstrates good practices in security awareness training and acceptable use policies, critical vulnerabilities exist primarily around MFA implementation and the use of outdated, misconfigured software. Addressing these areas, particularly the EOL Nginx server and lack of MFA for key services, should be prioritized to significantly reduce the organization’s attack surface and protect against common cyber threats.
