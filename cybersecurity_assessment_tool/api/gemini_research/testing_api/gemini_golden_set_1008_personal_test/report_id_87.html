6. Recommendations

The following actionable recommendations are provided to address the identified risks and enhance the overall cybersecurity posture of Terraform Global.

6.1. Immediate Actions

  • Mitigate Database Exposure:

    • Implement strict firewall rules to restrict inbound access to port 3306 (172.16.50.20) to only necessary, trusted internal IP addresses or specific administration jump hosts.

    • Ensure strong, unique passwords are used for all database accounts, especially for administrative users.

    • Review and enforce the principle of least privilege for all database users.

  • Implement MFA for Critical Systems:

    • Immediately enable and enforce Multi-Factor Authentication (MFA) for all organizational email accounts.

    • Implement and enforce MFA for all access points to systems containing sensitive data.

6.2. Long-Term Strategies

  • Secure Database Architecture:

    • Re-architect network access to databases, ideally placing them behind application layers and accessing them via secure VPNs or SSH tunnels rather than direct network exposure.

    • Develop a patching strategy for database software to ensure timely updates to the latest secure versions of MySQL.

  • Enhance Security Awareness Training Program:

    • Establish a mandatory, annual security awareness training program for all employees, covering current threats such as sophisticated phishing techniques, ransomware, and social engineering.

    • Consider integrating simulated phishing campaigns to test and reinforce employee awareness.

  • Regular Security Assessments:

    • Schedule periodic internal and external vulnerability assessments and penetration tests to proactively identify and remediate security weaknesses before they can be exploited.
