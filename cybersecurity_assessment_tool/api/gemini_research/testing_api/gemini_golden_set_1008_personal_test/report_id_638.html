Executive Summary

This report details the findings from a cybersecurity posture assessment for Sovereign Trust. The assessment involved a review of organizational security controls through a questionnaire, a technical network scan, and an analysis of pre-existing known vulnerabilities.

Our findings indicate several critical security gaps and high-risk vulnerabilities. Key concerns include the pervasive lack of multi-factor authentication (MFA) for email and computer access, the absence of an employee acceptable use policy, and critical misconfigurations on internet-facing services, specifically an outdated FTP server allowing anonymous access. Furthermore, the continued use of end-of-life operating systems (Windows 7) presents a significant and immediate threat. Addressing these issues is paramount to strengthening Sovereign Trust’s security posture against prevalent cyber threats and reducing exposure to common attack vectors.

Recommendations

To mitigate the identified risks and significantly improve Sovereign Trust’s overall cybersecurity posture, the following actionable recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA):

    • Immediate Action: Enable and enforce MFA for all email accounts across the organization. This is a primary defense against credential theft.

    • Implement MFA for all computer logins, especially for administrative accounts and systems processing sensitive data. Explore solutions like Windows Hello for Business, YubiKeys, or similar.

  2. Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines acceptable and unacceptable use of all company IT resources, including internet, email, software, and hardware.

    • Ensure all employees review, understand, and acknowledge the AUP annually. Integrate this into onboarding and annual compliance.

  3. Enhance Security Awareness Training Program:

    • Establish and enforce a program for mandatory annual security awareness training for all employees, covering current threats like phishing, social engineering, password hygiene, and secure data handling.

    • Conduct regular phishing simulations to test employee vigilance and provide targeted retraining.

  4. Secure and Update FTP Service (or Decommission):

    • Critical Immediate Action: Disable anonymous FTP access on the server at 10.0.0.15. This is a severe misconfiguration that must be rectified immediately.

    • Evaluate the business need for FTP. If not essential, decommission the service and close Port 21 on the firewall.

    • If FTP functionality is required, update vsftpd to the latest stable, supported version, or replace it with a more secure file transfer solution such as SFTP (SSH File Transfer Protocol) or SCP (Secure Copy Protocol).

  5. Upgrade Outdated Operating Systems:

    • Critical Action: Develop and execute an urgent plan to migrate all Windows 7 workstations to a currently supported operating system (e.g., Windows 10/11) as soon as possible.

    • Ensure all systems are regularly patched and kept up-to-date with the latest security updates, and implement a robust endpoint detection and response (EDR) solution.
