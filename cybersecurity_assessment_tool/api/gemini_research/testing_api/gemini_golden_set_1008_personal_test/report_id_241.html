Executive Summary

This report details the findings of a cybersecurity posture assessment for Firebrand Media. The assessment involved a review of organizational security controls via a questionnaire and a targeted network scan.

Overall, the organization demonstrates some foundational security practices, such as requiring MFA for computer and sensitive data system logins, and annual security awareness training for all employees. However, several critical gaps and high-risk areas were identified, including the absence of MFA for email access, a lack of a formal employee acceptable use policy, and the omission of security awareness training for new employees. Technical findings revealed an outdated Nginx web server version and a potential SSL certificate common name mismatch, where the certificate used www.acme-corp.com instead of the organization’s domain www.FirebrandMedia.org. Addressing these findings is crucial to enhance the organization’s resilience against modern cyber threats.

Risk Assessment

Based on the analysis of the questionnaire and technical scan results, the following risks have been identified:

Severity Risk Name Overview
Critical Lack of MFA for Email Access The organization does not require Multi-Factor Authentication (MFA) for email access. This significantly increases the risk of account compromise through phishing or credential stuffing attacks, leading to potential data breaches and business email compromise.
High Absence of Employee Acceptable Use Policy (AUP) There is no formal Acceptable Use Policy (AUP) in place for employees. This lack of clear guidelines on acceptable technology use increases legal and security risks, as employees may unknowingly or knowingly engage in activities detrimental to the organization’s security posture.
High No Security Awareness Training for New Employees New employees do not receive mandatory security awareness training. This leaves them vulnerable to common social engineering attacks (e.g., phishing) and unaware of security best practices from the start, increasing the organization’s overall human risk factor.
High Outdated Web Server Software The Nginx web server on port 443 is running version 1.18.0. This version is outdated and may contain known vulnerabilities that could be exploited.
Medium SSL Certificate Common Name Mismatch The SSL certificate observed on port 443 has a common name of www.acme-corp.com, which does not match the organization’s stated website domain www.FirebrandMedia.org. This can lead to trust warnings for users and may indicate a misconfiguration or a certificate intended for a different entity.

Recommendations

To mitigate the identified risks and enhance the organization’s cybersecurity posture, the following recommendations are provided:

  1. Immediately implement Multi-Factor Authentication (MFA) for all email accounts to prevent unauthorized access, even if credentials are compromised. Evaluate suitable MFA solutions compatible with your existing email infrastructure.

  2. Develop, document, and formally implement an Employee Acceptable Use Policy (AUP). Ensure all employees acknowledge and understand the policy, outlining appropriate use of company assets, data, and networks.

  3. Establish a mandatory security awareness training program for all new employees during their onboarding process. This should cover common threats, organizational policies, and best practices from day one.

  4. Upgrade the Nginx web server on 192.168.10.5 (port 443) to the latest stable version. This will ensure that known vulnerabilities are patched and the server benefits from the latest security features. Perform this update in a controlled environment after thorough testing.

  5. Rectify the SSL certificate mismatch on 192.168.10.5 (port 443). Obtain and install a valid SSL/TLS certificate with a common name that precisely matches the organization’s website domain (www.FirebrandMedia.org). This will resolve browser warnings and build trust with users.
