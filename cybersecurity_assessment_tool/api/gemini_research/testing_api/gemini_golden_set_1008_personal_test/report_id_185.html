Executive Summary

This report details the cybersecurity posture of Great Lakes based on a review of organizational data, a technical network scan, and an assessment of current risks. The overall posture indicates significant areas for improvement, particularly concerning Multi-Factor Authentication (MFA) implementation and routine security awareness training. A critical finding from the technical scan is the exposure of Remote Desktop Protocol (RDP) on a network host, which, combined with the lack of MFA, presents a severe risk of unauthorized access. Several existing risks align with these findings, underscoring the urgency of the recommended remediations.

Detailed Risk Findings:

  1. Widespread Lack of Multi-Factor Authentication (MFA): The absence of MFA across email, computer logins, and sensitive data systems represents a foundational security weakness. This significantly increases the risk of successful phishing, credential stuffing, and brute-force attacks leading to unauthorized access. Any compromised credentials could provide attackers with unhindered access to critical assets.

  2. Insufficient Annual Security Awareness Training (SAT): While new employees receive training, the lack of annual refreshers for all staff leaves the organization vulnerable to evolving social engineering tactics, phishing attempts, and general human error. This increases the likelihood of an initial breach.

  3. Exposed Remote Desktop Protocol (RDP): The open RDP port on 10.10.10.51, combined with the overall lack of MFA, creates an easily exploitable entry point for attackers. RDP is a prime target for brute-force attacks and credential theft, which could lead to lateral movement within the network. The existing risk data also indicates a similar RDP exposure on 10.10.10.50, suggesting a pattern of insecure RDP configurations.

Recommendations

Based on the identified risks, the following recommendations are provided to enhance the cybersecurity posture of Great Lakes.

  1. Implement Multi-Factor Authentication (MFA) Universally:

    • Deploy MFA for all email access, internal computer logins, and access to any systems containing sensitive data. This is the single most impactful recommendation to prevent credential-based attacks.

    • Priority: Critical

  2. Address RDP Exposure on Exposed Hosts:

    • Immediately review and restrict access to RDP on 10.10.10.51 and 10.10.10.50.

    • Where RDP is necessary, implement strict network access controls (e.g., firewall rules to allow only specific source IPs).

    • For remote access, consider implementing a Virtual Private Network (VPN) solution that requires MFA, or a secure jump box, rather than directly exposing RDP to the internet or internal networks without sufficient controls.

    • Priority: Critical

  3. Establish Comprehensive Annual Security Awareness Training:

    • Develop and implement a mandatory annual security awareness training program for all employees to educate them on current threats (e.g., phishing, malware, social engineering) and best practices.

    • Conduct periodic simulated phishing campaigns to test employee vigilance.

    • Priority: High

  4. Regular Vulnerability Scanning and Patch Management:

    • Implement a routine vulnerability scanning program to proactively identify exposed services, outdated software, and misconfigurations.

    • Ensure a robust patch management process is in place for all operating systems and applications to address known vulnerabilities promptly.

    • Priority: Medium

  5. Review and Update Security Policies:

    • Periodically review and update the Acceptable Use Policy and other security policies to reflect current threats and organizational practices.

    • Priority: Medium

Implementing these recommendations will significantly reduce the attack surface and strengthen the overall security posture of Great Lakes.
