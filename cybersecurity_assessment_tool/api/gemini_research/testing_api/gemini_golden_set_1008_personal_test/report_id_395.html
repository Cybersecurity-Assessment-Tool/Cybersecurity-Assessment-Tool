Executive Summary

This report details the cybersecurity posture of Verve & Vigor, combining insights from organizational security controls, network scan results, and pre-existing risk data. Key findings indicate a mixed security posture with several critical vulnerabilities and policy gaps. While the organization currently employs Multi-Factor Authentication (MFA) for email and sensitive data systems and conducts annual security awareness training, significant weaknesses were identified. These include the lack of MFA for computer login, the absence of a formal acceptable use policy, and no dedicated security awareness training for new employees. Furthermore, a critical pre-existing vulnerability, ‘Localhost Exposed‘ (CVSS 10.0), was confirmed, directly correlating with an observed open SSH port (22) on the local host. Immediate attention to these identified risks and policy deficiencies is crucial to significantly enhance the overall security resilience of Verve & Vigor.

Key Gaps Identified from Questionnaire:

Based on the questionnaire responses, the following critical security control gaps were identified:

  • Lack of MFA for Computer Login: The absence of Multi-Factor Authentication for computer login significantly increases the risk of unauthorized access to systems if user credentials are compromised.

  • Absence of Acceptable Use Policy (AUP): Without an AUP, employees lack clear guidelines regarding the proper and secure use of organizational IT assets, potentially leading to misuse, data breaches, or compliance violations.

  • No New Employee Security Training: New hires are not receiving immediate security awareness training, leaving them vulnerable to common cyber threats and unaware of organizational security protocols from the start of their employment.

Recommendations

To address the identified risks and enhance the overall cybersecurity posture of Verve & Vigor, the following actionable recommendations are provided, prioritized by severity.

High Priority Recommendations

  1. Implement MFA for Computer Login: Immediately deploy and enforce Multi-Factor Authentication for all computer logins across the organization. This should be a mandatory control for all users to significantly reduce the risk of credential compromise leading to unauthorized access.

  2. Develop and Enforce Acceptable Use Policy (AUP): Draft, communicate, and enforce a comprehensive Employee Acceptable Use Policy. This policy should clearly define expectations for employee behavior regarding IT assets, data handling, and internet usage, and be acknowledged by all employees.

  3. Establish New Employee Security Awareness Training: Implement mandatory and comprehensive security awareness training for all new employees as part of their onboarding process. This training should cover essential security best practices, common threats (e.g., phishing, social engineering), and organizational security protocols from day one.

  4. Address ’Localhost Exposed’ Vulnerability: Conduct an immediate and thorough investigation into the ’Localhost Exposed’ vulnerability (CVSS 10.0). Determine the root cause of this critical exposure and implement mitigation strategies. This may involve closing unnecessary ports, hardening configurations, isolating affected services, or implementing host-based firewalls.

Medium Priority Recommendations

  1. Secure SSH Configuration on Localhost (Port 22): For the open SSH port (22) on , ensure it is securely configured. Key steps include:

    • Utilizing key-based authentication exclusively and disabling password authentication.

    • Disabling direct root login.

    • Restricting access to specific IP addresses if external access is absolutely required.

    • Ensuring the SSH server software is kept up-to-date with the latest security patches.

    • If the service is not strictly necessary, consider disabling or heavily restricting it.

  2. Conduct Detailed Service Scanning: Perform more in-depth network scans that actively probe for service banners, product names, and version numbers. This will allow for a more precise assessment of known software vulnerabilities and provide a clearer picture of the attack surface.
