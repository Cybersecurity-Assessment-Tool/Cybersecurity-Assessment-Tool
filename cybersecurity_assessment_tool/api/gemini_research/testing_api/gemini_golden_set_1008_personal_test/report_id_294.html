Executive Summary

This report details the cybersecurity posture of Blue Marble, based on a review of organizational security controls, a recent network scan, and existing risk data. The assessment identified a mixed security posture, with some commendable practices alongside significant critical gaps.

Positively, Blue Marble conducts regular security awareness training for all employees and mandates Multi-Factor Authentication (MFA) for computer logins. However, critical deficiencies were noted in the absence of MFA for email access and sensitive data systems, as well as a lack of a formal employee acceptable use policy.

The recent network scan of the target revealed no open ports, which is a positive finding. This finding also contradicts a previously reported existing risk concerning an "Unencrypted Web Server" on Port 80, suggesting potential remediation or a scope difference that warrants further investigation.

Overall, immediate attention should be directed towards implementing MFA broadly, establishing an AUP, and validating the current state of pre-existing risks to strengthen the organization’s defense against common attack vectors.

Recommendations

Based on the identified risks, the following actionable recommendations are provided to enhance Blue Marble’s cybersecurity posture.

General Recommendations

  • Implement MFA Across All Critical Systems:

    • Mandate Multi-Factor Authentication for all email accounts to significantly reduce the risk of unauthorized access.

    • Extend MFA requirements to all sensitive data systems and applications to protect critical information assets.

  • Develop and Enforce an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining employee responsibilities and appropriate use of all company IT resources (e.g., internet, email, hardware, software).

    • Ensure all employees receive training on the AUP and formally acknowledge their understanding and agreement.

  • Validate Existing Risks and Scan Scope:

    • Conduct an internal review to verify the current status of the "Unencrypted Web Server" risk. Confirm if Port 80 is indeed closed across all relevant external interfaces, or if there are other hosts/services where this risk might still be present.

    • If any public-facing web services exist and utilize HTTP (Port 80) without redirection to HTTPS, implement SSL/TLS encryption immediately.

  • Continuous Monitoring: Implement continuous monitoring solutions to detect and alert on any changes to network configurations, newly exposed services, or deviations from security baselines.
