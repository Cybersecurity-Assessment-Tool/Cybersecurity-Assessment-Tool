Executive Summary

This report details the cybersecurity posture of Catalyst Consulting, based on a review of organizational data, a recent network scan, and existing risk information. The assessment highlights several critical security gaps and areas for improvement, alongside positive security practices currently in place.

Key findings include:

  • Critical MFA Gaps: Multi-Factor Authentication (MFA) is not enforced for computer logins or access to sensitive data systems, significantly increasing the risk of unauthorized access from compromised credentials.

  • Missing Acceptable Use Policy: The absence of an Employee Acceptable Use Policy (AUP) leaves the organization vulnerable to misuse of IT resources and potential data leakage.

  • Outdated Web Server: An Nginx web server running version 1.18.0 was identified during the network scan. This version is outdated and may contain unpatched vulnerabilities.

  • Positive Practices: The organization currently utilizes MFA for email access and conducts regular security awareness training, which are commendable practices.

Immediate attention to the identified MFA gaps and the implementation of an AUP are strongly recommended to bolster the organization’s defense against common cyber threats.

Recommendations

The following recommendations are provided to address the identified risks and enhance Catalyst Consulting’s overall cybersecurity posture.

High Priority Recommendations

  • Implement Multi-Factor Authentication (MFA) for Computer Logins (R-001):

    • Action: Deploy and enforce MFA for all user logins to company computers, laptops, and virtual desktops.

    • Rationale: MFA adds an essential layer of security, making it significantly harder for attackers to gain access even if they obtain user passwords.

  • Implement Multi-Factor Authentication (MFA) for Sensitive Data Systems (R-002):

    • Action: Extend MFA enforcement to all systems and applications that store, process, or transmit sensitive organizational data.

    • Rationale: Protecting sensitive data systems with MFA is crucial to prevent data breaches and comply with data protection regulations.

  • Develop and Implement an Employee Acceptable Use Policy (AUP) (R-003):

    • Action: Draft a comprehensive AUP that outlines acceptable and unacceptable use of company IT resources, data, and internet access.

    • Action: Communicate the AUP to all employees, obtain signed acknowledgments, and integrate it into new employee onboarding and annual security awareness training.

    • Rationale: A clear AUP sets expectations, reduces the likelihood of internal security incidents, and provides a basis for disciplinary action if policies are violated.

Medium Priority Recommendations

  • Upgrade Nginx Web Server (R-004):

    • Action: Plan and execute an upgrade of the Nginx web server on to the latest stable version (e.g., Nginx 1.24.x or newer).

    • Action: Before deployment, thoroughly test the upgrade in a staging environment to ensure compatibility with existing web applications and services.

    • Rationale: Upgrading to the latest version provides critical security patches for known vulnerabilities, performance improvements, and access to new features.

Low/Ongoing Priority Recommendations

  • Regular Software Patching and Updates (General Best Practice):

    • Action: Establish and maintain a robust patch management program for all operating systems, applications, and network devices across the organization.

    • Rationale: Proactive and consistent patching minimizes the attack surface by addressing newly discovered vulnerabilities before they can be exploited.
