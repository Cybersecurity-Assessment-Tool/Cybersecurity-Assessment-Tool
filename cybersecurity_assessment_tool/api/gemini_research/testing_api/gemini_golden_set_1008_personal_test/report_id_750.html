Key recommendations include the immediate implementation of MFA across all critical access points and the establishment of comprehensive security awareness training for all new hires.

Risk Assessment

This section synthesizes findings from the security questionnaire, technical scan, and pre-existing risk register to provide a comprehensive view of Top Tier’s cybersecurity risks.

Identified Gaps from Questionnaire

Based on the security control review, the following critical gaps and high-risk areas were identified:

  • Lack of Multi-Factor Authentication (MFA): MFA is not required for email access, computer login, or access to sensitive data systems. This represents a critical vulnerability. Without MFA, compromised credentials (e.g., through phishing or brute-force attacks) could lead to unauthorized access to vital organizational assets, sensitive data, and internal systems, potentially resulting in data breaches, financial loss, or operational disruption.

  • Absence of New Employee Security Awareness Training: While annual security awareness training is conducted for all employees, new employees do not receive initial training upon onboarding. This leaves new hires potentially vulnerable to social engineering attacks, phishing campaigns, and unaware of critical organizational security policies and procedures during their initial employment period. This significantly increases the risk of human error leading to security incidents or accidental breaches.

Technical Vulnerabilities

The network scan of 192.168.0.5 did not identify any active open ports or running services that would suggest immediate technical vulnerabilities. This indicates a relatively secure perimeter posture for the scanned host at the time of the assessment.

Pre-existing Reported Risks

The existing risk register includes the following relevant vulnerability:

  • Risk Name: Unencrypted Web Server

  • Overview: Port 80 is open.

  • Severity (CVSS Score): 5.0 (Medium)

  • Affected Elements: Port 80

Correlation with Current Findings: The network scan performed on 192.168.0.5 found Port 80 to be closed. This directly contradicts the overview provided in the pre-existing "Unencrypted Web Server" risk. This discrepancy suggests that either the risk pertains to a different asset within the organization, the issue has been remediated on this specific host since the risk was first identified, or the initial assessment of port 80 being open for this target was incorrect. Further investigation is required to reconcile this finding and update the risk register accordingly.

Recommendations

To address the identified gaps and improve Top Tier’s overall cybersecurity posture, the following recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) Across Critical Access Points:

    • Action: Immediately implement MFA for all email access.

    • Action: Implement MFA for all computer logins across the organization.

    • Action: Extend MFA requirements to all sensitive data systems, ensuring that access to critical information is protected by an additional layer of security.

    • Priority: Critical

  2. Establish Comprehensive New Employee Security Awareness Training:

    • Action: Develop and implement a mandatory security awareness training program specifically for all new employees upon their onboarding.

    • Action: This training should cover essential topics such as common cyber threats (e.g., phishing, social engineering), organizational security policies, best practices for data handling, and incident reporting procedures.

    • Priority: High

  3. Re-evaluate "Unencrypted Web Server" Risk:

    • Action: Conduct a thorough investigation to reconcile the discrepancy between the pre-existing "Unencrypted Web Server" risk (indicating port 80 open) and the current network scan results (showing port 80 closed) for 192.168.0.5.

    • Action: Determine if the risk is associated with a different asset, if remediation has occurred, or if the initial risk assessment requires an update.

    • Action: Update the organization’s risk register to reflect the accurate status of this vulnerability.

    • Priority: Medium
