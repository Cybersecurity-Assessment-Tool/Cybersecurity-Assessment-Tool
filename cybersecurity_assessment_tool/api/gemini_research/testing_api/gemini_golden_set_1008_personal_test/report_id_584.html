1. Executive Summary

This report provides a comprehensive cybersecurity assessment for Vertex Solutions. The analysis is based on a review of organizational security controls, a targeted network scan, and an evaluation of existing risk documentation.

Overall, the organization demonstrates a proactive stance on multi-factor authentication (MFA) and employee security awareness training. However, a critical gap was identified in the absence of an employee acceptable use policy (AUP). The network scan revealed an open port 80, which warrants further investigation to ensure secure operation and prevent potential attack vectors. Several existing vulnerabilities, including one requiring attention due to its unusual nature, were also evaluated. Key recommendations include implementing an AUP and thoroughly investigating services exposed via port 80.

4.1.1. Port 80 (HTTP)

Port 80 (HTTP) was found to be open. Without further service details, it is difficult to ascertain the full risk. However, an open HTTP port can indicate an unencrypted web service, an improperly configured web server, or a vulnerable application. Further investigation is required to determine the nature of the service, whether it is intended to be public, and if it is adequately secured (e.g., protected by a Web Application Firewall, configured for HTTPS redirection).

6. Recommendations

Based on the assessment, the following recommendations are provided to enhance the organization’s cybersecurity posture:

6.1. Immediate Actions

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP document clearly defining employee responsibilities and acceptable behaviors when using company IT assets and networks.

    • Distribute the AUP to all employees and require acknowledgment of receipt and understanding.

    • Integrate AUP training into new employee onboarding and annual security awareness training.

  • Investigate Port 80 Service on :

    • Identify the specific service or application running on port 80.

    • Determine if the service is necessary and intended to be accessible from this segment of the network.

    • If public-facing or sensitive, ensure HTTPS is enforced (e.g., through redirects) and implement strong security controls (e.g., WAF, regular vulnerability scanning).

    • If not required, close the port or restrict access to only necessary internal hosts.

6.2. Long-Term Enhancements

  • Regular Vulnerability Scanning and Penetration Testing: Conduct periodic vulnerability scans and penetration tests on both internal and external assets to proactively identify and remediate weaknesses.

  • Service Hardening: Ensure all services, especially those exposed internally or externally, are hardened according to best practices, with unnecessary features disabled and default credentials changed.

  • Security Documentation and Policy Review: Regularly review and update all security policies and procedures, including the AUP, to adapt to evolving threats and organizational changes.
