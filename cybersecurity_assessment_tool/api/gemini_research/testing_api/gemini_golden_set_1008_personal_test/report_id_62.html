Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Solaris Energy, based on an analysis of organizational data, network scan results, and existing risk registers. The assessment reveals several critical vulnerabilities and significant security gaps that require immediate attention.

Key findings include:

  • Critical Authentication Gaps: Lack of Multi-Factor Authentication (MFA) for both email access and computer logins presents a severe risk of unauthorized access.

  • Critical Network Vulnerability: An accessible FTP service (vsftpd 2.3.4) on 10.0.0.15 allows anonymous login, enabling unrestricted access to the FTP server’s contents. The service version is also significantly outdated, posing additional known vulnerabilities.

  • Outdated Systems: The presence of Windows 7 workstations within the environment introduces known operating system vulnerabilities and potential compliance issues.

Immediate recommendations focus on implementing MFA, securing/disabling the anonymous FTP service, and upgrading outdated operating systems. Addressing these issues will significantly enhance Solaris Energy’s defensive capabilities against common cyber threats.

Technical Scan Results

A network scan was performed targeting 10.0.0.15. The scan revealed the following critical findings:

Nmap Scan Details

  • Target IP Address: 10.0.0.15

  • Status: Host is up

  • Open Ports and Services:

    • Port 21/tcp:

      • Service: FTP

      • Product: vsftpd

      • Version: 2.3.4

      • Nmap Script Output: ftp-anon: Anonymous FTP login allowed

Analysis of Technical Findings:

The presence of an open FTP service on port 21 is a significant concern. Specifically, the Nmap script ‘ftp-anon‘ confirmed that anonymous FTP login is allowed. This configuration grants unrestricted access to the FTP server’s contents to anyone who connects, without requiring authentication. This is a severe data exposure and integrity risk.

Furthermore, vsftpd 2.3.4 is an extremely outdated version of the Very Secure FTP Daemon, released in 2011. This version is known to have multiple vulnerabilities, including CVE-2011-2523, which is a backdoor vulnerability. Operating an outdated and insecure service like this significantly increases the attack surface and potential for compromise.

Detailed Risk Descriptions

Lack of Multi-Factor Authentication (MFA) for Email

  • Severity: High

  • Description: Without MFA, an attacker only needs a username and password to gain full access to an employee’s email. This significantly increases the risk of phishing, business email compromise (BEC), and unauthorized access to sensitive information often exchanged via email.

Lack of Multi-Factor Authentication (MFA) for Computer Login

  • Severity: High

  • Description: Similar to email, single-factor authentication for computer logins makes workstations susceptible to credential stuffing, brute-force attacks, and lateral movement once an initial compromise occurs. This can lead to system takeover and data exfiltration.

Anonymous FTP Access

  • Severity: Critical

  • Description: The FTP server on 10.0.0.15 is configured to allow anonymous login. This means anyone can connect to the server without credentials and access files, potentially allowing for data exposure, unauthorized uploads, or even the execution of malicious files if misconfigured.

Outdated vsftpd Server Version

  • Severity: Critical

  • Description: The vsftpd 2.3.4 version is over a decade old and has well-documented critical vulnerabilities, including a backdoor (CVE-2011-2523) that allows remote code execution. Continued use of this version presents an extreme risk of full system compromise.

Outdated Windows Policy (Windows 7)

  • Severity: Medium

  • Description: Operating systems like Windows 7 are no longer supported by Microsoft, meaning they do not receive security updates or patches for newly discovered vulnerabilities. This leaves workstations highly susceptible to malware, exploits, and compliance violations.

Recommendations

The following recommendations are provided to mitigate the identified risks and improve Solaris Energy’s overall cybersecurity posture. Recommendations are categorized by the urgency of implementation.

Immediate Actions (Critical Risks)

  • Disable Anonymous FTP: Immediately reconfigure the FTP server on 10.0.0.15 to disable anonymous access. If FTP access is required, implement strong authentication (e.g., SFTP/FTPS with strong passwords/keys) and restrict access to authorized users and IP addresses only.

  • Update/Replace FTP Server: Promptly update vsftpd to the latest stable and secure version, or migrate to a more secure file transfer protocol like SFTP or SCP if FTP functionality is essential. If FTP is not critical, the service should be disabled and removed.

  • Implement MFA for Email and Computer Login: Deploy and enforce Multi-Factor Authentication (MFA) for all user accounts accessing email systems and for logging into company computers. This is a foundational security control against credential theft.

Short-Term Actions (High to Medium Risks)

  • Upgrade Windows 7 Workstations: Plan and execute an upgrade path for all Windows 7 workstations to a currently supported operating system (e.g., Windows 10 or 11). Prioritize systems with access to sensitive data or network segments.

  • Review Network Segmentation: Evaluate current network segmentation practices to isolate critical assets and restrict communication paths, limiting the impact of any potential compromise.

Long-Term Actions (Continuous Improvement)

  • Regular Vulnerability Scanning: Implement a schedule for regular internal and external vulnerability scanning to proactively identify new exposures and misconfigurations.

  • Security Awareness Training Reinforcement: Continue and expand security awareness training, with specific modules on phishing, password hygiene, and the importance of MFA.

  • Patch Management Program: Establish and enforce a robust patch management program for all operating systems, applications, and network devices to ensure timely application of security updates.

  • Incident Response Plan Review: Develop or review an existing incident response plan to ensure the organization is prepared to detect, respond to, and recover from cybersecurity incidents effectively.

Addressing these recommendations will significantly bolster Solaris Energy’s defenses against current and emerging cyber threats.
