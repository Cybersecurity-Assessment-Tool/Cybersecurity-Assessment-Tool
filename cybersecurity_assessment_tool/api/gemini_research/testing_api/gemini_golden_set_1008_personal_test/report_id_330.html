Executive Summary

This report details the cybersecurity posture of Prism Logic based on a review of organizational security controls, an internal network scan, and an analysis of pre-existing risk data. The assessment reveals several critical gaps in foundational security practices, particularly concerning Multi-Factor Authentication (MFA) and Security Awareness Training (SAT). While the network scan of the specified target indicates a closed port 80, which is a positive finding, it contradicts a pre-existing risk entry, necessitating re-validation. Overall, the organization’s posture requires significant immediate attention to mitigate high-impact risks related to access control and human factors.

Analysis of Questionnaire Results

The questionnaire results highlight significant deficiencies in key security controls:

  • Multi-Factor Authentication (MFA): MFA is not required for email access, computer logins, or sensitive data systems. This represents a critical vulnerability, as compromised credentials could lead to unauthorized access across multiple critical systems.

  • Security Awareness Training (SAT): The absence of security awareness training for new employees and annual training for all employees leaves the organization highly susceptible to social engineering attacks, phishing, and other human-factor vulnerabilities.

  • Acceptable Use Policy (AUP): The presence of an AUP is a positive control, providing a baseline for employee conduct regarding technology use.

Risk Assessment

Identified Gaps and Risks

Based on the analysis of the provided data, the following risks and security gaps have been identified:

  • Lack of Multi-Factor Authentication (MFA): This is a critical gap across email, computer login, and sensitive data access. Without MFA, the organization is highly vulnerable to credential stuffing, brute-force attacks, and phishing that could compromise user accounts.

  • Insufficient Security Awareness Training (SAT): The absence of SAT for new and existing employees significantly increases the risk of successful phishing, malware, and social engineering attacks.

  • Unencrypted Web Server (Requires Re-validation): A pre-existing risk indicated an "Unencrypted Web Server" with "Port 80 is open." However, the current network scan shows Port 80 as closed on the target 192.168.0.5. This discrepancy suggests that either the pre-existing risk is outdated, refers to a different system/scope not covered by the scan, or the issue has been remediated. This risk requires immediate re-validation against the intended target and scope.

Recommendations

The following recommendations are provided to address the identified risks and improve the organization’s cybersecurity posture:

  1. Implement Multi-Factor Authentication (MFA) Across All Critical Systems:

    • Deploy MFA for all email accounts (e.g., using Conditional Access Policies).

    • Implement MFA for all computer logins, especially for administrative accounts.

    • Enforce MFA for access to all sensitive data systems and applications.

  2. Establish and Maintain a Comprehensive Security Awareness Training Program:

    • Develop and mandatory security awareness training for all new employees during their onboarding process.

    • Conduct annual, mandatory security awareness training for all existing employees, covering current threats like phishing, social engineering, and safe browsing habits.

  3. Re-validate "Unencrypted Web Server" Risk:

    • Investigate the discrepancy between the pre-existing risk assessment and the recent network scan results.

    • Determine the true scope and target for the "Unencrypted Web Server" risk. If it refers to a different server, scan that server. If the issue has been remediated, update the risk register accordingly.

    • Ensure all public-facing web services are secured with HTTPS and valid SSL/TLS certificates.

  4. Regular Vulnerability Scanning:

    • Implement a schedule for regular internal and external vulnerability scanning to proactively identify and remediate security weaknesses.

  5. Patch Management:

    • Establish a robust patch management process to ensure all operating systems, applications, and network devices are kept up-to-date with the latest security patches.
