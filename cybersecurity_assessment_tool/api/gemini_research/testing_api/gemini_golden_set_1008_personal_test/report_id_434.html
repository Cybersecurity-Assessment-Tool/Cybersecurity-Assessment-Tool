Analysis of Security Controls

Apex Legends Group demonstrates strong adherence to best practices concerning Multi-Factor Authentication (MFA) across key systems and consistent security awareness training for employees. This proactive approach forms a solid foundation for identity and access management. However, a critical gap exists with the absence of an employee acceptable use policy. This lack of formal guidelines for employee conduct regarding organizational IT assets and data use exposes the organization to various risks, including misuse of company resources, potential data breaches, compliance failures, and legal liabilities.

Technical Scan Results

A network scan was performed to identify open ports and services on a target host. The following critical finding was identified:

Host:

  • Open Port: 21/tcp

  • Service: FTP

  • Product/Version: vsftpd 2.3.4

  • Vulnerability: Anonymous FTP login allowed.

  • Impact: This configuration allows unauthorized users to connect to the FTP server without providing credentials, potentially granting access to sensitive files, enabling unauthorized file uploads, or serving as a staging point for further attacks. Furthermore, the vsftpd 2.3.4 version is significantly outdated (released 2011) and known to have critical vulnerabilities, including a backdoor (CVE-2011-2523), which severely exacerbates the risk of compromise and increases the likelihood of a successful attack.

Recommendations

To mitigate the identified risks and significantly enhance the overall cybersecurity posture of Apex Legends Group, the following actionable recommendations are provided:

Immediate Actions (Critical & High Priority)

  • R1: Secure or Decommission FTP Server:

    • Disable Anonymous Access: Immediately configure the FTP server at to disable anonymous login functionality. This is the most critical first step.

    • Upgrade/Replace Service: If FTP functionality is absolutely required, upgrade vsftpd to the latest stable and secure version. Alternatively, migrate to a more secure file transfer protocol such as SFTP or FTPS, ensuring strong authentication (e.g., SSH keys, strong passwords) and encryption are enforced.

    • Network Isolation: Implement strict network segmentation and firewall rules to restrict access to the FTP server only from authorized internal hosts and services, if it must remain active.

    • Vulnerability Scan: Conduct a comprehensive vulnerability assessment on the server to identify and remediate any other potential security weaknesses.

  • R2: Develop and Implement an Acceptable Use Policy (AUP):

    • Policy Development: Draft a comprehensive AUP that clearly outlines acceptable and unacceptable use of company IT resources, data, and internet access. The policy should cover topics such as data handling, privacy, software installation, and appropriate online behavior.

    • Communication & Acknowledgment: Ensure the AUP is clearly communicated to all employees (both new and existing). Obtain signed acknowledgments from all staff members to confirm their understanding and agreement to abide by the policy.

    • Integration: Integrate AUP review and refresher training into annual security awareness programs.

  • R3: Upgrade Workstation Operating Systems:

    • Phased Upgrade: Prioritize and execute a phased plan to upgrade all Windows 7 workstations to a currently supported operating system (e.g., Windows 10 or 11).

    • Isolation for Legacy Systems: For any systems that cannot be immediately upgraded due to business-critical legacy applications, implement strict network segmentation and enhanced monitoring to isolate them from critical assets and the broader corporate network. Restrict their internet access to only essential services.

    • Hardware Refresh: Develop a hardware refresh plan to replace outdated machines that cannot support modern operating systems.

General Best Practices

  • Policy Review: Regularly review and update all security policies and procedures to ensure they remain current and effective against evolving cyber threats.

  • Periodic Assessments: Conduct periodic internal and external vulnerability assessments and penetration tests to proactively identify and address security weaknesses across the IT infrastructure.

  • Patch Management: Maintain a robust and automated patch management program for all operating systems, applications, and network devices to ensure all systems are up-to-date with the latest security fixes.

  • Security Monitoring: Implement and continuously monitor network traffic, system logs, and security events for suspicious activity, intrusions, and anomalies.
