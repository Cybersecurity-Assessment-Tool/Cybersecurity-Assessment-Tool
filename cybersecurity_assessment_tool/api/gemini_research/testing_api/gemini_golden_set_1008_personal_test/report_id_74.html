Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Stone Arch Masonry, synthesizing findings from network scans, organizational security questionnaires, and existing risk registers. The assessment reveals several critical vulnerabilities and significant security control gaps that require immediate attention. Key findings include the presence of an anonymous FTP server running an extremely outdated and vulnerable software version, a lack of Multi-Factor Authentication (MFA) for sensitive data systems, and the absence of a formal Employee Acceptable Use Policy. Addressing these issues is paramount to mitigating potential data breaches, unauthorized access, and operational disruptions.

Detailed Findings

  • Port 21 (FTP):

    • Service: FTP (File Transfer Protocol)

    • Product: vsftpd

    • Version: 2.3.4

    • Critical Finding: The FTP server allows Anonymous FTP login. This configuration allows anyone to connect to the FTP server without credentials, potentially exposing sensitive files or allowing unauthorized uploads, leading to data breaches or system compromise.

    • Vulnerability: vsftpd version 2.3.4 is extremely outdated (released in 2011) and known to contain critical vulnerabilities, including a backdoor (CVE-2011-2523) which could allow remote code execution. Running this version poses an immediate and severe risk.

Recommendations

Based on the assessment, the following recommendations are provided to improve the cybersecurity posture of Stone Arch Masonry. These are categorized by urgency and scope.

Immediate Actions (Critical)

  1. Disable Anonymous FTP Access: Immediately configure the FTP server on 10.0.0.15 to disallow anonymous logins. Implement strong authentication mechanisms for all necessary FTP users.

  2. Upgrade/Replace Outdated FTP Software: Urgent upgrade of vsftpd to the latest stable and supported version. Alternatively, consider replacing FTP with a more secure file transfer protocol (e.g., SFTP, SCP) or a secure managed file transfer solution. Decommission the service if not absolutely critical.

  3. Implement MFA for Sensitive Data Systems: Deploy and enforce Multi-Factor Authentication for all systems that handle sensitive organizational data. This is a crucial step to prevent unauthorized access even if primary credentials are compromised.

Short-Term Actions (High Priority)

  1. Upgrade End-of-Life Operating Systems: Prioritize and execute the upgrade of all Windows 7 machines to a currently supported operating system (e.g., Windows 10 or 11). If hardware limitations exist, plan for hardware replacement alongside software upgrades.

  2. Develop Employee Acceptable Use Policy (AUP): Create a comprehensive AUP that outlines acceptable and unacceptable use of company IT resources, data, and internet access. Ensure all employees review, understand, and formally acknowledge this policy. Integrate it into onboarding and annual training.

Long-Term Strategy

  1. Regular Vulnerability Scanning and Penetration Testing: Establish a routine schedule for internal and external vulnerability scans and periodic penetration tests to proactively identify and remediate security weaknesses.

  2. Security Awareness Training Enhancement: Continue and expand security awareness training beyond just new employees and annual refreshers. Incorporate practical exercises, phishing simulations, and modules specific to identified risks (e.g., proper handling of sensitive data, AUP adherence).

  3. Incident Response Plan: Develop or review and update a formal Incident Response Plan to effectively detect, contain, eradicate, and recover from cybersecurity incidents. Conduct tabletop exercises to test its efficacy.

  4. Asset Inventory and Patch Management: Maintain an up-to-date inventory of all IT assets and implement a robust patch management program to ensure all systems and software are kept current with the latest security updates.

By systematically addressing these recommendations, Stone Arch Masonry can significantly enhance its cybersecurity posture, reduce its attack surface, and build a more resilient defense against evolving cyber threats.
