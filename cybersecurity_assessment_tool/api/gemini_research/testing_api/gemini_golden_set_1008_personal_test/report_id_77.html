Analysis of Security Controls

The questionnaire responses highlight critical gaps in foundational security controls. The absence of Multi-Factor Authentication (MFA) for email and computer logins represents a significant vulnerability, as it makes these access points susceptible to credential-based attacks. While MFA is used for sensitive data systems, its absence for broader access significantly increases risk. Furthermore, the lack of annual security awareness training for all employees suggests a potential weakness in the human element of security, leaving the organization vulnerable to phishing, social engineering, and other user-targeted attacks.

The identified risks demonstrate a clear pattern of insufficient controls around remote access and user authentication. The technical finding of exposed RDP on is critically compounded by the organizational policy of not requiring MFA for computer logins. This dramatically lowers the barrier for attackers to gain unauthorized access. The lack of comprehensive security awareness training exacerbates all these issues, as employees may not be equipped to identify or report suspicious activities that could lead to credential compromise.

Recommendations

To mitigate the identified risks and enhance Vivid Vision’s cybersecurity posture, the following actionable recommendations are provided:

  1. Implement Multi-Factor Authentication (MFA) for Email Access:

    • Mandate MFA for all email accounts to prevent unauthorized access, even if credentials are compromised.

    • Review and enforce strong password policies in conjunction with MFA.

  2. Implement Multi-Factor Authentication (MFA) for Computer Logins:

    • Deploy MFA for all computer logins, especially for systems accessible via RDP or other remote access protocols.

    • Prioritize implementation for administrative accounts and critical systems.

  3. Conduct Mandatory Annual Security Awareness Training:

    • Develop and implement a comprehensive security awareness training program for all employees, conducted at least once annually.

    • Include topics such as phishing, social engineering, password hygiene, and incident reporting.

  4. Secure Remote Desktop Protocol (RDP) Access:

    • Immediate Action (for and ): Restrict RDP access to a trusted internal network or specific whitelisted IP addresses using firewall rules.

    • Long-Term Solution: Implement a Virtual Private Network (VPN) solution for all remote access, ensuring users must first connect to the VPN before accessing RDP services.

    • Ensure NLA (Network Level Authentication) is enabled for all RDP connections.

    • Regularly patch and update systems running RDP to protect against known vulnerabilities.

  5. Conduct Regular Vulnerability Scans and Penetration Tests:

    • Establish a routine schedule for internal and external vulnerability scanning to proactively identify new exposures.

    • Consider engaging a third-party for periodic penetration testing to simulate real-world attacks.

Conclusion

Vivid Vision faces significant cybersecurity risks primarily due to exposed RDP services, a lack of comprehensive Multi-Factor Authentication, and gaps in employee security awareness training. Addressing these critical areas will substantially improve the organization’s defensive posture against common cyber threats. Implementing the recommended controls will build a stronger foundation for a secure operational environment.
