Key Observations from Security Control Review

  • Evergreen Alliance has implemented Multi-Factor Authentication (MFA) for email and computer login, which is a strong security practice that significantly reduces the risk of credential compromise.

  • Critical Gap: MFA is not required for accessing sensitive data systems. This leaves critical data vulnerable to unauthorized access if a single set of credentials is stolen or compromised.

  • Critical Gap: There is no documented employee acceptable use policy. The absence of such a policy can lead to inconsistent employee behavior, misuse of company resources, potential data breaches, and difficulties in enforcing cybersecurity best practices.

  • The practice of conducting regular security awareness training for both new and existing employees is highly commendable and helps foster a security-conscious culture.

Recommendations

The following recommendations are provided to address the identified risks and enhance the overall cybersecurity posture of Evergreen Alliance:

Immediate Actions

  • Implement MFA for Sensitive Data Systems: Immediately enforce Multi-Factor Authentication (MFA) for all systems, applications, and databases that store, process, or transmit sensitive organizational data. This should be a top priority to mitigate unauthorized access risks. Conduct an inventory of all sensitive data systems to ensure comprehensive MFA coverage.

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP that clearly outlines employee responsibilities, permitted and prohibited activities concerning company IT resources, data handling, and internet usage. The policy should address areas such as password management, data classification, remote access, software installation, and use of personal devices. Once developed, communicate it to all employees and require mandatory acknowledgment and regular review.

General Security Enhancements

  • Regular Vulnerability Scanning: While the current network scan showed no open ports on the specified target, regular internal and external vulnerability scans of all network assets, including applications and systems, should be conducted to identify configuration weaknesses, outdated software, and known vulnerabilities.

  • Penetration Testing: Consider periodic penetration tests to simulate real-world attacks and identify exploitable vulnerabilities that automated scans might miss, particularly for critical applications and infrastructure.

  • Incident Response Plan Review and Testing: Ensure an up-to-date and well-tested incident response plan is in place to effectively detect, contain, and recover from security incidents. Regular tabletop exercises should be conducted.

  • Principle of Least Privilege: Implement the principle of least privilege across all user accounts and systems, ensuring users and applications only have access to resources absolutely necessary for their job functions. Regularly review and revoke unnecessary privileges.

Conclusion

Evergreen Alliance demonstrates a commendable commitment to security awareness training and MFA for basic access. However, critical vulnerabilities exist in the lack of MFA for sensitive data systems and the absence of an acceptable use policy.
