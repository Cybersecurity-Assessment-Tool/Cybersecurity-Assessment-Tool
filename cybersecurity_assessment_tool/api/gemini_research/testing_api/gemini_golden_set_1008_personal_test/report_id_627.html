Executive Summary

This report details the findings from a cybersecurity posture assessment for Maple Leaf Logistics. The assessment involved a review of organizational security controls via questionnaire, analysis of network scan data, and correlation with pre-existing risk information.

Key findings indicate a concerning security posture, primarily driven by a critical lack of Multi-Factor Authentication (MFA) across all assessed access points (email, computer logins, sensitive data systems). Additionally, the absence of an Employee Acceptable Use Policy represents a significant policy gap. Technically, an End-of-Life (EOL) MySQL database (version 5.7.33) is externally exposed, posing a high risk. These critical gaps and high-severity risks require immediate attention and remediation to enhance the organization’s overall security.

Analysis of Technical Findings

The scan revealed an open MySQL port (3306) on the target system. The identified version, MySQL 5.7.33, is End-of-Life (EOL) as of October 2023. Running EOL software, especially a database service, poses a significant security risk due to the lack of ongoing security patches and vendor support. This vulnerability is compounded by the port being openly accessible, which allows for potential remote exploitation if the database is not properly secured or contains known vulnerabilities for this EOL version.

Recommendations

To address the identified risks and improve Maple Leaf Logistics’ cybersecurity posture, the following recommendations are provided:

Immediate Actions (Critical & High Risks)

  1. Implement Multi-Factor Authentication (MFA):

    • Prioritize implementing MFA for all email accounts.

    • Implement MFA for all user logins to company computers and network resources.

    • Extend MFA implementation to all sensitive data systems and applications.

  2. Database Security Enhancements:

    • Immediate Firewall Restriction: Apply strict firewall rules to restrict access to MySQL port 3306 () to only trusted internal IP addresses or administrative workstations. Remove external exposure.

    • Upgrade/Migrate Database: Plan and execute an upgrade of the MySQL database from version 5.7.33 to a currently supported version (e.g., MySQL 8.0 or newer). Alternatively, consider migrating to a fully managed database service in a secure cloud environment.

    • Secure Access Methods: Implement secure access methods for database administration, such as VPNs or bastion hosts, rather than direct access.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly outlines employee responsibilities for using organizational IT resources, including internet, email, and software.

    • Ensure the AUP covers proper handling of sensitive information, prohibitions on unauthorized software, and guidelines for personal device usage (if applicable).

    • Distribute the AUP to all employees and require acknowledgment of understanding and compliance.

Ongoing Best Practices

  1. Regular Vulnerability Scanning & Patch Management: Implement a routine schedule for vulnerability scanning and ensure all systems, applications, and network devices are regularly patched and updated.

  2. Security Awareness Training Reinforcement: Continue existing security awareness training programs and periodically update content to reflect new threats and best practices. Emphasize the importance of MFA and phishing prevention.

  3. Incident Response Plan: Develop and regularly test an incident response plan to ensure the organization can effectively detect, respond to, and recover from cybersecurity incidents.

Addressing these recommendations systematically will significantly improve Maple Leaf Logistics’ security posture and reduce its exposure to common cyber threats.
