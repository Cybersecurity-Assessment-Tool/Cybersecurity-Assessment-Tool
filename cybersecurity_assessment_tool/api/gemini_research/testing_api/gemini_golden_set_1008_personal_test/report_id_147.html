6. Recommendations

The following recommendations are provided to address the identified security gaps and risks:

  • Implement Mandatory Multi-Factor Authentication (MFA):

    • Email Access: Immediately implement mandatory MFA for all email accounts across the organization. This significantly reduces the risk of account compromise through phishing or credential stuffing.

    • Computer Logins: Roll out mandatory MFA for all computer logins, including workstations and servers. Consider solutions like FIDO2 keys, smart cards, or authenticator apps.

  • Enhance Security Awareness Training:

    • New Employees: Introduce mandatory security awareness training as a core component of the onboarding process for all new employees. This ensures they are educated on best practices and threat identification from their first day.

    • Ongoing Training: Continue and enhance annual security awareness training for all employees, covering current threats like phishing, ransomware, and social engineering.

  • Address Outdated Software:

    • MySQL Upgrade: Prioritize the upgrade of MySQL 5.7.33 to a currently supported version (e.g., MySQL 8.0 or newer). Develop a comprehensive plan for migration, including testing, backup, and rollback procedures.

  • Secure Database Access:

    • Firewall Rules: Implement strict firewall rules to restrict access to port 3306 (MySQL) to only authorized internal IP addresses or specific application servers. External access should be completely blocked unless absolutely necessary and, if so, only allowed through a secure VPN or bastion host.

    • Principle of Least Privilege: Ensure that database user accounts have only the minimum necessary permissions required for their function.

  • Regular Vulnerability Management:

    • Establish a routine vulnerability scanning and patch management program to identify and address security weaknesses proactively.
