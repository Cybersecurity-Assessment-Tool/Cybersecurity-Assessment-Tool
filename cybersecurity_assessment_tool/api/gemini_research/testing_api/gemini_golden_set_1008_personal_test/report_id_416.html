3.1. Analysis of Security Controls

The organization demonstrates good practices in requiring MFA for email access and implementing an acceptable use policy and initial security awareness training for new hires. However, critical gaps exist concerning:

  • Lack of MFA for Computer Login: Without MFA for computer login, an attacker who obtains a user’s password could gain unauthorized access to internal systems, potentially leading to data breaches or system compromise. This represents a significant risk to internal network security.

  • Lack of MFA for Sensitive Data Systems: Similar to computer login, the absence of MFA for sensitive data systems significantly increases the risk of unauthorized access to critical information, even if credentials are stolen. This directly impacts data confidentiality and integrity.

  • Inconsistent Annual Security Awareness Training: While new employees receive training, the lack of regular, annual training for all employees means that staff may not be updated on the latest threats, phishing techniques, or organizational security policies. This increases the organization’s susceptibility to social engineering attacks and human-error related incidents.

6. Recommendations

To enhance the cybersecurity posture of Symmetry Architecture, the following actionable recommendations are provided:

6.1. Security Control Enhancements

  • Implement MFA for Computer Login: Deploy and rigorously enforce Multi-Factor Authentication for all computer logins across the organization. This significantly reduces the risk of unauthorized access due to compromised user credentials and improves overall identity security.

  • Implement MFA for Sensitive Data Systems: Mandate MFA for accessing all systems and applications that contain or process sensitive organizational or client data. This provides an essential additional layer of security for critical information assets.

  • Establish Annual Security Awareness Training: Develop and implement a mandatory, comprehensive annual security awareness training program for all employees. Training should be updated regularly to cover current threat landscapes, sophisticated phishing tactics, secure data handling best practices, and organizational security policies.

6.2. Technical Security Enhancements

  • Investigate Open Port 80: Immediately investigate the service running on port 80 on the target IP address .

    • Identify the specific application or service, its business purpose, and necessity.

    • If the service is no longer needed, it should be promptly disabled, removed, or the port closed.

    • If the service is necessary, ensure it is running the latest patched version, apply least privilege principles, and restrict access through appropriate firewall rules or access control lists to only necessary users, systems, or network segments.

    • Conduct thorough vulnerability scanning and penetration testing on the identified service.
