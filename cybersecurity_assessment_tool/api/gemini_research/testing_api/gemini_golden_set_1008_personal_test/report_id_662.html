Executive Summary

This report details the cybersecurity posture assessment for Falcon Heavy, synthesized from organizational data, network scan results, and an existing risk inventory. The analysis reveals several critical vulnerabilities and control gaps that significantly elevate the organization’s risk profile. Key findings include an extremely outdated and insecure FTP service with anonymous access enabled on an internal host, a pervasive lack of Multi-Factor Authentication (MFA) across critical access points (email, computer login, sensitive data systems), and the absence of a fundamental Acceptable Use Policy. Furthermore, the reliance on End-of-Life (EOL) operating systems (Windows 7) for workstations poses a severe and ongoing threat. Immediate and decisive action is required to address these deficiencies and mitigate the high potential for critical breaches, unauthorized access, and operational disruption.

Network Scan Findings

Host 10.0.0.15

The internal scan of host 10.0.0.15 revealed the following critical findings:

  • Port 21/tcp (FTP) - Open

    • Service: FTP

    • Product: vsftpd

    • Version: 2.3.4

    • Vulnerability Details: vsftpd version 2.3.4, released in 2011, is an End-of-Life (EOL) product with numerous well-documented critical vulnerabilities. These include severe flaws like remote code execution (e.g., CVE-2011-2523, CVE-2011-0761), which could allow an attacker to gain complete control over the host.

    • Configuration Issue: The service explicitly allows Anonymous FTP login. This severe misconfiguration enables unauthorized users to access, potentially upload, or delete files without any authentication. This presents a critical risk for data exposure, data integrity compromise, and serves as a potential staging ground for further attacks on the internal network.

Recommendations

To significantly improve Falcon Heavy’s cybersecurity posture and mitigate the identified risks, the following actionable recommendations are provided, categorized by urgency and impact.

Immediate Actions (Critical Priority)

These actions address the most severe vulnerabilities and should be prioritized for immediate implementation.

  • Disable or Secure FTP Service:

    • Immediately disable the FTP service on 10.0.0.15 if it is not absolutely essential for business operations.

    • If FTP functionality is required, replace vsftpd 2.3.4 with a modern, securely configured SFTP (SSH File Transfer Protocol) or FTPS (FTP over SSL/TLS) solution. Ensure strong authentication and encryption are in place.

  • Eliminate Anonymous FTP Access: If the FTP service cannot be immediately replaced, reconfigure it to strictly disallow anonymous login. All access must require strong, unique credentials.

  • Implement MFA Across All Critical Systems:

    • Email Access: Deploy and enforce Multi-Factor Authentication for all email accounts across the organization.

    • Computer Login: Deploy and enforce Multi-Factor Authentication for all computer logins, especially for administrative and privileged accounts.

    • Sensitive Data Systems: Deploy and enforce Multi-Factor Authentication for all access to sensitive data systems, applications, and databases.

Strategic Improvements (High Priority)

These actions address significant risks and contribute to a stronger, more resilient security posture.

  • Develop and Implement an Acceptable Use Policy (AUP): Create a comprehensive AUP for all employees, clearly defining acceptable and unacceptable use of company IT resources, internet, email, and data. Ensure mandatory sign-off from all employees and conduct regular reviews and updates.

  • Upgrade End-of-Life Operating Systems: Develop and execute a phased plan to immediately upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or Windows 11). This ensures receipt of critical security updates, patches, and access to modern security features.

  • Implement Regular Vulnerability Scanning and Penetration Testing: Establish a schedule for regular internal and external vulnerability scanning to proactively identify and remediate new security weaknesses. Consider periodic penetration testing to validate the effectiveness of implemented controls.

  • Reinforce Security Awareness Training: While existing training is noted, expand and reinforce security awareness training, emphasizing the importance of secure configurations, strong password practices, phishing prevention (especially given the current lack of MFA), and incident reporting procedures.

  • Regular Security Audits: Conduct regular internal and external security audits to ensure compliance with best practices and internal policies.
