Executive Summary

This assessment provides a snapshot of the cybersecurity posture of Midnight Oil Studios. The organization has demonstrated a strong commitment to foundational security controls by implementing Multi-Factor Authentication (MFA) across email, computer logins, and sensitive data systems, alongside maintaining an acceptable use policy for employees. These are commendable practices that significantly reduce common attack vectors.

However, two critical areas of concern have been identified: the complete absence of regular security awareness training for both new and existing employees. This represents a significant human-factor vulnerability, as even the most robust technical controls can be circumvented by untrained personnel. Additionally, a network scan revealed an open SSH (port 22) service on a target system. While SSH is a legitimate administrative tool, its exposure requires strict hardening and monitoring to prevent unauthorized access.

The report provides detailed findings and actionable recommendations to mitigate these identified risks and enhance the overall security resilience of Midnight Oil Studios.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of Midnight Oil Studios, the following recommendations are provided:

  1. Implement Comprehensive Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new hires before they gain access to company systems.

    • Establish and enforce an annual, organization-wide security awareness training program for all employees. This training should cover topics such as phishing, social engineering, password hygiene, data handling, and incident reporting.

    • Utilize engaging and relevant content, possibly including simulated phishing exercises, to ensure effectiveness.

  2. Harden and Monitor SSH Service (Port 22):

    • Review Necessity: Confirm if SSH access to is strictly necessary for operational requirements. If not, disable the service.

    • Access Control: Restrict SSH access to only authorized personnel and specific source IP addresses/networks using firewall rules.

    • Authentication: Disable password-based authentication and enforce the use of strong SSH keys. Ensure all keys are properly secured and rotated periodically.

    • Software Updates: Ensure the SSH server software is kept up-to-date with the latest security patches to protect against known vulnerabilities.

    • Logging and Monitoring: Implement robust logging for all SSH connection attempts and activity. Monitor these logs for suspicious patterns, failed login attempts, and unauthorized access.

    • Change Default Port: Consider changing the default SSH port (22) to a non-standard port to reduce automated scanning attempts.

  3. Regular Vulnerability Scanning and Penetration Testing:

    • Conduct regular authenticated vulnerability scans of internal and external systems to identify misconfigurations and outdated software versions.

    • Consider engaging a third party for periodic penetration testing to simulate real-world attacks and uncover exploitable weaknesses.

  4. Maintain Inventory and Patch Management:

    • Establish and maintain an accurate inventory of all network devices, software, and services.

    • Implement a robust patch management program to ensure all operating systems and applications are regularly updated.
