Executive Summary

This report provides a comprehensive cybersecurity posture assessment for Swift Current Labs, based on a review of organizational data, existing risk register, and a targeted network scan. The assessment highlights key strengths, identifies critical gaps in security controls, and uncovers potential vulnerabilities.

Overall, Swift Current Labs demonstrates a foundational commitment to security, with Multi-Factor Authentication (MFA) in place for computer logins and sensitive data systems, and an acceptable use policy. However, critical vulnerabilities and gaps were identified, notably the exposure of Remote Desktop Protocol (RDP) services on internal hosts, the absence of MFA for email access, and a lack of consistent annual security awareness training for all employees. These areas present significant risks that require immediate attention to strengthen the organization’s defensive posture against common attack vectors.

Recommendations

To mitigate the identified risks and enhance the overall cybersecurity posture of Swift Current Labs, the following recommendations are provided:

Critical Recommendations

  • Implement MFA for Email Access: Immediately implement and enforce Multi-Factor Authentication (MFA) for all email accounts. This is a fundamental control to protect against credential theft and significantly reduces the risk of email compromise.

  • Secure Exposed RDP Services:

    • Review and restrict access to all RDP services (e.g., on and ).

    • Implement strong network-level access controls, such as firewalls, to limit RDP access only to trusted administrative hosts or VPN connections.

    • Enforce MFA for all RDP connections where feasible.

    • Ensure RDP servers are fully patched and configured with strong password policies.

High Priority Recommendations

  • Establish Regular Security Awareness Training: Implement a mandatory, annual (or more frequent) security awareness training program for all employees. Training should cover topics such as phishing, social engineering, password hygiene, and incident reporting.

  • Regular Vulnerability Scanning: Conduct regular vulnerability scans of both internal and external networks to proactively identify and address exposed services, misconfigurations, and outdated software versions.

  • Patch Management: Ensure a robust patch management process is in place to keep all operating systems, applications, and network devices updated with the latest security patches.

General Best Practices

  • Principle of Least Privilege: Review user permissions across all systems and applications, ensuring users only have the minimum necessary access to perform their job functions.

  • Incident Response Plan: Develop and regularly test a comprehensive incident response plan to ensure the organization can effectively detect, respond to, and recover from security incidents.

  • Data Backup and Recovery: Implement and regularly test a robust data backup and recovery strategy to protect against data loss from ransomware or other destructive attacks.

  • Endpoint Detection and Response (EDR): Consider deploying EDR solutions on all endpoints to provide advanced threat detection, investigation, and response capabilities.
