Executive Summary

This report details the cybersecurity posture of Skyward Bound based on a recent network scan, organizational security questionnaire responses, and a review of existing vulnerabilities. The assessment reveals several critical findings that require immediate attention. Notably, the organization lacks Multi-Factor Authentication (MFA) for access to sensitive data systems, presenting a significant authentication gap. Furthermore, Remote Desktop Protocol (RDP) services are exposed on internal hosts, posing a severe risk of unauthorized access and exploitation, a finding corroborated by both the network scan and pre-existing risk documentation. While many foundational security practices are in place, the identified gaps in MFA and RDP exposure elevate the overall risk profile to High. Comprehensive recommendations are provided to address these vulnerabilities and enhance the organization’s security posture.

The presence of an open RDP (Remote Desktop Protocol) service on port 3389 without specific version information is a significant security concern. RDP is a frequent target for attackers seeking to gain unauthorized access to internal systems through brute-force attacks, credential stuffing, or exploiting vulnerabilities in the RDP service itself. While the service is common, its exposure, especially on an internal network, warrants careful review and restriction.

Recommendations

Based on the assessment findings, the following recommendations are provided to improve the security posture of Skyward Bound.

Immediate Actions

  • Implement MFA for Sensitive Data Systems: Immediately enforce Multi-Factor Authentication for all access attempts to systems containing sensitive data. This is a critical control to prevent unauthorized access even if primary credentials are stolen.

  • Restrict RDP Access (10.10.10.51 and 10.10.10.50):

    • Review and implement strict firewall rules to limit RDP access on 10.10.10.51 and 10.10.10.50 to only authorized administrative jump boxes or a secure VPN solution.

    • Disable RDP if it is not absolutely essential for the system’s function.

    • For critical systems requiring RDP, ensure Network Level Authentication (NLA) is enabled and strong, unique passwords are used.

Short-Term Actions

  • Comprehensive Vulnerability Scanning: Conduct a full, authenticated vulnerability scan across all internal network segments to identify other open services, outdated software, and misconfigurations.

  • Network Segmentation Review: Evaluate and enhance network segmentation to isolate critical systems and sensitive data from less secure parts of the network, limiting the lateral movement of attackers.

  • Patch Management Review: Ensure a robust patch management process is in place to keep all operating systems and applications, especially those exposing services like RDP, up-to-date with the latest security patches.

Long-Term Actions

  • Develop an Identity and Access Management (IAM) Strategy: Formalize an IAM policy that includes principles of least privilege, regular access reviews, and robust authentication mechanisms across the entire organization.

  • Advanced Security Awareness Training: Expand security awareness training to include specific modules on secure remote access practices, identifying social engineering attempts aimed at credential theft, and the importance of MFA.

  • Security Information and Event Management (SIEM): Consider implementing a SIEM solution to centralize log collection and enable real-time monitoring and alerting for suspicious activities, such as repeated failed RDP login attempts.
