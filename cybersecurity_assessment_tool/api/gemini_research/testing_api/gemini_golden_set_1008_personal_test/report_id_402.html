1. Executive Summary

This report details the cybersecurity posture of Vertex Solutions, based on an analysis of provided organizational data, network scan findings, and pre-existing risk information.

Key Strengths:

  • Strong implementation of Multi-Factor Authentication (MFA) across email, computer logins, and sensitive data systems significantly enhances identity and access management.

  • Security awareness training is conducted for new employees.

Key Findings & Gaps:

  • Policy Gap: Absence of a formal employee acceptable use policy (AUP), which is crucial for defining expected behavior and protecting organizational assets.

  • Training Gap: Lack of annual security awareness training for all employees, increasing susceptibility to evolving cyber threats.

  • Technical Scan: No open ports were detected on the scanned target, which is a positive security indicator.

  • Risk Discrepancy: A previously identified risk, "Unencrypted Web Server" (severity: Medium), cited an open Port 80. However, the current network scan shows Port 80 as closed, suggesting this risk may be mitigated or requires re-evaluation based on the latest findings.

Recommendations: Immediate actions should focus on establishing an AUP, implementing annual security awareness training, and validating the status of previously identified risks against current technical findings.

6. Recommendations

Based on the assessment, the following recommendations are provided to enhance Vertex Solutions’ cybersecurity posture.

6.1. Policy and Governance

  • Implement an Acceptable Use Policy (AUP): Develop, communicate, and enforce a clear AUP for all employees. This policy should define appropriate use of company assets, data, and internet access, outlining consequences for non-compliance. Regular reviews and updates are essential.

  • Mandate Annual Security Awareness Training: Establish a program for mandatory annual security awareness training for all employees. Training should cover current threats (e.g., phishing, ransomware), password best practices, data handling, and the importance of the AUP.

6.2. Technical Controls

  • Verify Previously Identified Risks: Conduct a thorough review of the "Unencrypted Web Server" risk. Given the recent scan shows Port 80 is closed, investigate whether this port was intentionally closed as a mitigation measure. If the service is no longer needed or has been moved, formally close the risk. If the service is still required, ensure it is properly secured (e.g., using HTTPS on Port 443 instead of HTTP on Port 80).

  • Regular Vulnerability Scanning: Continue with regular network and vulnerability scanning across all external and internal assets to proactively identify and address potential weaknesses.

6.3. General Security Practices

  • Maintain Strong MFA Implementation: Continue the current practice of requiring MFA for email, computer logins, and sensitive data systems. Explore expanding MFA to other critical applications or services where applicable.

  • Documentation and Review: Ensure all security policies, procedures, and risk assessments are thoroughly documented and reviewed regularly (at least annually) to adapt to evolving threats and organizational changes.
