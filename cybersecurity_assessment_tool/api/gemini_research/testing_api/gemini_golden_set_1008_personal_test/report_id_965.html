Analysis of Security Control Gaps

The review reveals significant gaps in critical areas:

  • Multi-Factor Authentication (MFA): The absence of MFA for email and computer logins represents a severe vulnerability, making the organization highly susceptible to credential stuffing, phishing, and brute-force attacks.

  • Security Awareness Training (SAT): Lack of structured SAT for new hires and annual refreshers for all employees leaves the workforce unprepared against social engineering tactics, phishing, and other human-centric attacks.

Recommendations

Based on the analysis, the following recommendations are provided to mitigate the identified risks and enhance the overall cybersecurity posture of Obsidian Operatives.

Immediate Actions

  1. Secure RDP Access on 10.10.10.51:

    • Immediately restrict RDP access on 10.10.10.51 (port 3389) to only authorized personnel and secure it behind a Virtual Private Network (VPN) or a dedicated bastion host.

    • If RDP is not critical for day-to-day operations on this system, disable the service entirely.

    • Implement strong password policies for all accounts with RDP access.

    • Address Existing RDP Exposure on 10.10.10.50:

    • Expedite the remediation of the "RDP Exposure" on 10.10.10.50 (CVSS 9.0) as per the existing risk documentation. Ensure a consistent security strategy for all RDP instances.

Strategic Enhancements

  1. Implement Multi-Factor Authentication (MFA) Widely:

    • Email Access: Implement mandatory MFA for all email accounts across the organization.

    • Computer Logins: Enforce mandatory MFA for all computer logins, prioritizing administrative accounts and critical systems.

    • Review Existing MFA: Periodically review and audit MFA implementations for sensitive data systems to ensure consistent and robust application.

  2. Establish Comprehensive Security Awareness Training (SAT) Program:

    • New Employees: Develop and deliver mandatory security awareness training as part of the onboarding process for all new employees.

    • Annual Training: Implement a mandatory annual security awareness training program for all existing employees, covering current threats like phishing, ransomware, and social engineering.

    • Phishing Simulations: Consider integrating regular phishing simulation exercises to test employee vigilance and reinforce training.

  3. Review and Update Security Policies:

    • Although an Acceptable Use Policy (AUP) is in place, review it to ensure it explicitly addresses the new MFA requirements, secure RDP usage guidelines, and employee responsibilities regarding security awareness.
