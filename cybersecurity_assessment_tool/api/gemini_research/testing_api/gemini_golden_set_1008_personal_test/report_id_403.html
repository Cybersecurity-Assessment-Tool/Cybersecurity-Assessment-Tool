1. Executive Summary

This report summarizes the cybersecurity posture of Nomad Gear Co., based on a recent network scan, an organizational security questionnaire, and a review of pre-existing risk data.

The assessment reveals a mixed security posture. On the positive side, Nomad Gear Co. has implemented Multi-Factor Authentication (MFA) for email, computer login, and sensitive data systems, along with an employee acceptable use policy. These controls demonstrate a foundational commitment to identity and access management and governance.

However, critical gaps were identified concerning security awareness training, with no current programs for new hires or annual refreshers for all employees. This oversight significantly elevates the risk of human-error-based security incidents such as phishing, social engineering, or accidental data exposure. Furthermore, an open SSH port was detected on an internal host, presenting a potential attack surface that requires immediate attention.

Addressing these training deficiencies and securing exposed services are crucial next steps to fortify the organization’s overall cybersecurity resilience.

6. Recommendations

The following recommendations are provided to mitigate the identified risks and enhance the overall cybersecurity posture of Nomad Gear Co..

6.1. General Security Awareness Training (R01, R02)

  • Implement Mandatory Onboarding Training: Develop and implement a mandatory cybersecurity awareness training program for all new employees during their onboarding process. This training should cover fundamental security principles, organizational policies, and common threat vectors.

  • Establish Annual Refreshers: Institute an annual, mandatory security awareness training program for all employees. This training should be regularly updated to address the latest threat landscape, including advanced phishing techniques, social engineering tactics, secure remote work practices, and data protection best practices.

  • Regular Phishing Simulations: Conduct regular, simulated phishing campaigns to test employee vigilance and reinforce training concepts. Provide immediate feedback and additional training for those who fall victim to simulations.

  • Policy Review and Communication: Ensure that security policies (e.g., acceptable use, password policy, data handling) are clearly communicated, easily accessible, and regularly reviewed with all employees.

6.2. Secure Open SSH Port (R03)

  • Restrict Access by IP: Implement firewall rules to restrict SSH access (port 22) to specific trusted IP addresses or networks only. Avoid exposing SSH to the public internet unless absolutely necessary, and if so, use a VPN for access.

  • Disable Password Authentication: Configure the SSH server to disable password-based authentication, enforcing the exclusive use of strong SSH keys. Ensure all keys are managed securely.

  • Implement Rate Limiting: Utilize tools like ‘fail2ban‘ or firewall rules to automatically block IP addresses that show signs of brute-force attempts on the SSH port.

  • Keep Software Updated: Ensure the SSH server software (e.g., OpenSSH) is always running the latest stable version and is patched against all known vulnerabilities.

  • Consider Non-Standard Port (Optional): While not a strong security control on its own, changing the default SSH port (22) to a non-standard port can deter opportunistic scanning and automated attacks. This should be combined with the stronger controls listed above.

  • Enable Logging and Monitoring: Configure robust logging for all SSH connection attempts (successful and failed) and implement monitoring to alert on suspicious activity.
