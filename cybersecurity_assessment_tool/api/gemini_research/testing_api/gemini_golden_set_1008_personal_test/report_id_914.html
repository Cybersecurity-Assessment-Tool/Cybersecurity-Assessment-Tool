Executive Summary

This report details the cybersecurity posture of Nebula Creative based on a network scan, organizational security questionnaire responses, and a review of existing vulnerabilities. The assessment identifies several critical and high-severity risks that require immediate attention to safeguard organizational assets and data.

Key findings include the presence of an outdated and highly vulnerable FTP service allowing anonymous access, a complete lack of Multi-Factor Authentication (MFA) across critical access points (email, computers, sensitive data), and significant gaps in security policies and training. Addressing these issues proactively will significantly enhance the organization’s defensive capabilities against common cyber threats.

Technical Scan Results

A network scan was conducted on the target 10.0.0.15. The scan revealed the following critical findings:

  • Target IP Address: 10.0.0.15

  • Open Port: 21/tcp (FTP - File Transfer Protocol)

  • Service: vsftpd 2.3.4

  • Script Findings:

    • ftp-anon: Anonymous FTP login allowed.

Analysis of Technical Findings: The presence of an open FTP service running vsftpd version 2.3.4 is a critical vulnerability. This specific version is known to contain a backdoor (CVE-2011-2523) allowing remote code execution, which can lead to complete system compromise. Furthermore, the allowance of anonymous FTP login exacerbates this risk by providing an unauthenticated entry point, enabling unauthorized file transfers and potentially sensitive data exposure.

Recommendations

The following recommendations are provided to mitigate the identified risks and improve Nebula Creative’s cybersecurity posture. They are prioritized by severity.

Critical Recommendations

  • Immediately address the vulnerable FTP service:

    • Disable the FTP service on 10.0.0.15 if it is not business-critical.

    • If FTP is required, upgrade vsftpd to the latest stable version or replace it with a more secure file transfer protocol (e.g., SFTP, SCP) and ensure it is properly configured with strong authentication.

    • Implement host-based firewalls to restrict access to the FTP port to only necessary source IPs.

  • Disable anonymous FTP access:

    • Reconfigure the FTP server to disallow anonymous logins. All access should require strong, unique credentials.

High Recommendations

  • Implement Multi-Factor Authentication (MFA) across all critical access points:

    • Enable and enforce MFA for all employee email accounts.

    • Implement MFA for logging into all company computers and workstations.

    • Deploy MFA for access to all systems containing sensitive organizational or client data.

  • Address outdated operating systems:

    • Immediately plan and execute an upgrade of all Windows 7 machines to a supported operating system (e.g., Windows 10/11).

    • Consider a hardware refresh if older machines cannot support newer operating systems.

Medium Recommendations

  • Develop and enforce an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP outlining the acceptable use of company IT resources, data handling, and security expectations.

    • Ensure all employees review and acknowledge the AUP annually.

  • Establish annual security awareness training for all employees:

    • Conduct mandatory security awareness training for all employees at least once per year. This training should cover topics such as phishing, social engineering, password hygiene, and data protection.

    • Implement regular phishing simulations to test employee awareness and reinforce training.
