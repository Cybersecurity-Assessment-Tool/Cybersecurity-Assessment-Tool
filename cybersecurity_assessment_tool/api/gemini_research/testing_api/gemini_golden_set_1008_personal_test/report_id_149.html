1. Executive Summary

This report provides a comprehensive assessment of the current cybersecurity posture for Firebrand Media. The analysis is based on a review of organizational security controls, an external network scan targeting 10.10.10.51, and an existing risk register. Key findings indicate a mixed security landscape with strong adherence to some best practices, such as MFA for email and computer logins, alongside significant gaps in sensitive data access controls and employee security awareness training. Notably, Remote Desktop Protocol (RDP) exposure was identified on multiple internal hosts, presenting a critical vulnerability. Recommendations are provided to address these identified risks and enhance the overall security posture.

6. Recommendations

To address the identified risks and enhance the cybersecurity posture of Firebrand Media, the following recommendations are provided:

6.1. RDP Exposure Mitigation

  • Restrict Access: Implement strict access controls for RDP on both 10.10.10.51 and 10.10.10.50. Only allow connections from trusted internal IP addresses or require access through a secured Virtual Private Network (VPN).

  • Strong Authentication: Ensure all RDP accounts utilize strong, unique passwords and consider implementing Multi-Factor Authentication (MFA) for RDP access where feasible.

  • Patching and Updates: Regularly patch and update operating systems and RDP services to mitigate known vulnerabilities.

  • Network Segmentation: Implement network segmentation to isolate critical assets, limiting the blast radius should an RDP service be compromised.

6.2. Enhance Multi-Factor Authentication

  • Implement MFA for Sensitive Data Systems: Immediately implement and enforce multi-factor authentication for all systems containing or providing access to sensitive organizational data. Conduct an inventory of all such systems to ensure comprehensive coverage.

6.3. Improve Security Awareness Training

  • Mandatory New Employee Training: Develop and implement a mandatory security awareness training program for all new hires. This training should be completed before new employees gain access to internal systems and sensitive data.

  • Regular Ongoing Training: Establish an annual or bi-annual mandatory security awareness training program for all employees. This program should cover current threats, phishing identification, social engineering tactics, password best practices, and secure data handling procedures.

  • Phishing Simulations: Implement regular simulated phishing campaigns to test employee vigilance and reinforce the effectiveness of security awareness training. Provide immediate feedback and additional training for those who fall victim.
