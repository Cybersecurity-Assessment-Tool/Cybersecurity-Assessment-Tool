Executive Summary

This report details the findings from a cybersecurity posture assessment for Top Tier. The assessment combined network scan data, organizational security questionnaire responses, and a review of pre-existing vulnerabilities. Key findings indicate a proactive approach to some security controls, such as security awareness training and Multi-Factor Authentication (MFA) for email and sensitive data systems. However, critical gaps were identified, notably the exposure of Remote Desktop Protocol (RDP) on an internal host (), a lack of MFA for computer logins, and the absence of an Employee Acceptable Use Policy (AUP). These areas represent significant risks that require immediate attention to strengthen the organization’s overall security posture.

Analysis of Control Gaps

The questionnaire results reveal two significant control gaps:

  • Lack of MFA for Computer Login: This is a critical vulnerability. Without MFA, a compromised username and password for a computer account could grant an attacker full access to a workstation or server, potentially leading to lateral movement and data exfiltration.

  • Absence of Employee Acceptable Use Policy (AUP): An AUP is fundamental for defining acceptable behavior, system usage, and security expectations for employees. Its absence can lead to inconsistent security practices, increased risk of insider threats, and legal liabilities.

Recommendations

Based on the findings, the following recommendations are provided to enhance the cybersecurity posture of Top Tier:

  1. Secure Exposed RDP Services:

    • Immediately review and restrict access to RDP on both and .

    • Implement strong firewall rules to limit RDP access to only trusted administrative workstations or through a secure VPN tunnel.

    • Ensure RDP is patched to the latest version and configured with Network Level Authentication (NLA).

    • Consider disabling RDP entirely on systems where it is not strictly necessary.

  2. Implement Multi-Factor Authentication (MFA) for All Computer Logins:

    • Deploy MFA solutions for all user logins to company computers, including workstations and servers. This significantly mitigates the risk of credential theft and unauthorized access.

    • Evaluate solutions such as Windows Hello for Business, third-party MFA providers, or conditional access policies for robust implementation.

  3. Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly defines acceptable and unacceptable use of company IT assets, data, and networks.

    • Ensure the AUP covers topics such as password hygiene, data handling, internet usage, software installation, and reporting security incidents.

    • Communicate the AUP to all employees and obtain their acknowledgment. Integrate it into new employee onboarding and regular security awareness training.

  4. Regularly Review and Patch Systems:

    • Establish a robust patch management program to ensure all operating systems and applications are kept up-to-date, reducing the attack surface.

    • Conduct regular vulnerability scans and penetration tests to proactively identify and address security weaknesses across the entire IT infrastructure.

Conclusion

While Top Tier demonstrates a commitment to security awareness training and MFA for key systems, the identified RDP exposures, lack of MFA for computer logins, and the absence of an AUP present significant areas for improvement. Addressing these issues with the recommended actions will substantially enhance the organization’s defense against common cyber threats and strengthen its overall security posture.
