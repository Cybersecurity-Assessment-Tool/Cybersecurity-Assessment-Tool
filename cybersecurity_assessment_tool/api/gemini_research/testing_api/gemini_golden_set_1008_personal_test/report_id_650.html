Executive Summary

This report presents a cybersecurity posture assessment for Falcon Heavy. While the organization demonstrates commendable practices in some areas, such as Multi-Factor Authentication (MFA) for email and sensitive data, and regular security awareness training, several critical vulnerabilities and high-risk gaps were identified. Key findings include an extremely outdated and misconfigured FTP server allowing anonymous access, a significant lack of MFA for computer logins, and the absence of an Employee Acceptable Use Policy. Additionally, the continued use of End-of-Life (EOL) Windows 7 workstations poses an inherent security risk. Immediate remediation of the FTP server and implementation of MFA for computer access are paramount to enhancing the organization’s security posture and reducing its attack surface.

Technical Scan Results

A network scan targeting revealed the following critical findings. (Note: Scan date information was not provided in the scan metadata.)

Target:

  • Port 21/TCP: Open - FTP Service

    • Service/Product: vsftpd 2.3.4

    • Finding 1: Outdated and Vulnerable FTP Software

      • Description: The vsftpd service running on port 21 is identified as version 2.3.4. This version is severely outdated and End-of-Life (EOL), having reached its end of support many years ago. It is known to contain critical vulnerabilities, including a widely publicized backdoor vulnerability (CVE-2011-2523) which allows unauthorized remote access. Running EOL software exposes the system to known exploits for which no official patches are available.

      • Severity: Critical

    • Finding 2: Anonymous FTP Login Allowed

      • Description: The FTP server is configured to allow anonymous login. This permits any user to connect to the FTP server without providing credentials, potentially granting them access to sensitive files, the ability to upload malicious content, or use the server for illicit activities. This represents a severe misconfiguration and a major security loophole.

      • Severity: Critical

Recommendations

Based on the assessment, the following recommendations are provided to enhance the organization’s cybersecurity posture:

Immediate Actions (Critical & High Priority)

  • FTP Server Remediation:

    • Disable Anonymous Access: Immediately disable anonymous login capabilities on the FTP server. Enforce strong authentication for all legitimate users.

    • Upgrade/Replace FTP Service: Urgently upgrade vsftpd to a current, supported, and patched version, or migrate to a more secure file transfer protocol such as SFTP (SSH File Transfer Protocol) or SCP (Secure Copy Protocol).

    • Content Review: Conduct a thorough audit of all files and directories on the FTP server to identify and remove any unauthorized content, malware, or sensitive data.

  • Implement MFA for Computer Logins:

    • Phased Deployment: Implement and mandate Multi-Factor Authentication for all computer login processes. Prioritize critical workstations and sensitive systems.

    • User Training: Provide comprehensive training to employees on the importance and usage of MFA.

  • Develop and Implement an Employee Acceptable Use Policy (AUP):

    • Policy Creation: Draft and formalize a clear, comprehensive AUP outlining acceptable and unacceptable use of all organizational IT resources, including networks, systems, software, and data.

    • Dissemination & Acknowledgment: Ensure all employees read, understand, and formally acknowledge the AUP. Integrate it into onboarding and annual security awareness training.

Mid-Term Actions (Medium Priority)

  • Upgrade End-of-Life Operating Systems:

    • OS Migration: Develop and execute a plan to upgrade all Windows 7 workstations to a currently supported operating system (e.g., Windows 10/11).

    • Hardware Refresh: If existing hardware is incompatible with newer OS versions, budget for and implement a hardware refresh program.
