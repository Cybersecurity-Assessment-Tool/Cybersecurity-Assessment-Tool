Executive Summary

This report details the cybersecurity posture of Oasis Wellness based on a review of organizational data, a security questionnaire, and technical network scan results. Our assessment identified several critical security gaps, primarily concerning the absence of multi-factor authentication (MFA) for email and computer access, an outdated web server, and a lack of an acceptable use policy. Additionally, an SSL certificate mismatch was noted. Addressing these items is crucial for enhancing the organization’s overall security resilience and mitigating potential threats.

Analysis of Questionnaire Gaps

The "No" responses in the questionnaire indicate significant gaps in the organization’s security posture and expose potential attack vectors:

  • Lack of MFA for Email Access: Email is a primary vector for phishing and account compromise. Without MFA, compromised credentials can grant attackers full access to an employee’s inbox, potentially leading to business email compromise, data exfiltration, or further internal network access.

  • Lack of MFA for Computer Login: Similar to email, access to user workstations or servers without MFA significantly increases the risk of unauthorized access if credentials are stolen or guessed. This can be a gateway to internal network lateral movement.

  • Absence of an Employee Acceptable Use Policy (AUP): An AUP sets clear expectations for employee behavior regarding the use of company IT resources. Without one, employees may unknowingly engage in risky activities, and the organization lacks a clear framework for addressing misuse or security incidents stemming from user actions. This weakens the overall security culture.

Technical Analysis

The network scan revealed critical technical vulnerabilities and misconfigurations:

  • Outdated Nginx Version (1.18.0): Nginx version 1.18.0 was released in April 2020. This version is significantly outdated, with numerous security patches and feature updates having been released in subsequent stable (e.g., 1.26.x) and mainline (e.g., 1.27.x) versions. Running outdated software increases the risk of known vulnerabilities being exploited, which could potentially lead to denial-of-service, information disclosure, or even remote code execution.

  • SSL Certificate Common Name Mismatch: The SSL certificate presented by the server at 192.168.10.5 has a common name of www.acme-corp.com, which does not match the organization’s stated website domain of www.OasisWellness.net. While this specific scan was on an internal IP address, such a mismatch indicates a potential misconfiguration, the use of an incorrect certificate, or a server repurposed without proper certificate updates. If this server were publicly accessible and intended for www.OasisWellness.net, this mismatch would cause browser warnings and distrust.

Recommendations

To mitigate the identified risks and significantly improve the overall cybersecurity posture of Oasis Wellness, the following actionable recommendations are provided:

  • Implement Multi-Factor Authentication (MFA) for Email and Computer Access:

    • Action: Immediately deploy and enforce MFA for all employee email accounts and for logging into all company computers, servers, and endpoints.

    • Benefit: Drastically reduces the risk of credential-based attacks, even if passwords are stolen or phished, serving as a critical layer of defense.

    • Priority: Critical

  • Update Web Server Software (Nginx):

    • Action: Plan and execute an upgrade of the Nginx web server on 192.168.10.5 (and any other instances) to the latest stable version (e.g., 1.26.x or newer). Ensure a proper testing phase in a non-production environment before deployment to production.

    • Benefit: Patches known vulnerabilities, improves performance, and ensures the server benefits from the latest security features.

    • Priority: High

  • Develop and Implement an Acceptable Use Policy (AUP):

    • Action: Draft, thoroughly review (involving legal and HR), and formally implement an AUP that clearly outlines employee responsibilities and acceptable conduct when using company IT resources. Ensure all employees read, understand, and formally acknowledge the policy upon hire and annually thereafter.

    • Benefit: Establishes clear expectations, reduces insider threats, provides a legal and ethical framework for IT resource usage, and strengthens the overall security culture.

    • Priority: High

  • Review and Correct SSL Certificate Configuration:

    • Action: Investigate the SSL certificate on the server at 192.168.10.5. If this server is intended for www.OasisWellness.net, replace the certificate with one issued for the correct common name. If it’s an internal-only server, ensure the certificate is appropriate for its intended use (e.g., a wildcard certificate for an internal domain) and trusted by internal clients.

    • Benefit: Ensures proper trust, avoids browser warnings (if externally facing), and maintains data integrity and confidentiality for communications with the server.

    • Priority: Medium

Conclusion

The assessment reveals that while Oasis Wellness has commendable security awareness training programs for new and existing employees and has implemented MFA for sensitive data systems, there are critical foundational security controls missing. Specifically, the absence of MFA for general email and computer access, coupled with an outdated Nginx web server and a lack of a formal acceptable use policy, present significant vulnerabilities. Addressing these areas with the provided recommendations will substantially enhance the organization’s defensive capabilities against common cyber threats and foster a more robust security posture.
