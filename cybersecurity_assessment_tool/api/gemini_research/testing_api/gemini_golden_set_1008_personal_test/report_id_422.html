Executive Summary

This report details the findings of a cybersecurity assessment conducted for Evergreen Alliance. The assessment involved a network scan, a review of organizational security controls via a questionnaire, and an analysis of pre-existing reported risks. The overall security posture reveals several critical vulnerabilities and significant gaps in security controls, necessitating immediate attention. Key findings include an outdated FTP server with anonymous access enabled, a lack of multi-factor authentication for sensitive data systems, insufficient security awareness training, and widespread use of end-of-life operating systems. Recommendations are provided to address these issues.

Technical Scan Results

A network scan was performed on the internal target 10.0.0.15. The scan revealed the following critical findings:

Target: 10.0.0.15

  • Open Port: 21/tcp

  • State: open

  • Service: ftp

  • Product: vsftpd

  • Configuration Issue: Anonymous FTP login allowed

Analysis of Technical Findings

The presence of an FTP server (vsftpd 2.3.4) on an internal network segment is a significant concern. vsftpd version 2.3.4 was released in 2011 and is severely outdated, containing numerous known vulnerabilities, including backdoors and other critical exploits. Furthermore, the configuration allowing anonymous FTP login poses an immediate and severe risk, potentially allowing unauthorized access to files and data stored on the system, leading to data breaches or system compromise.

Recommendations

To mitigate the identified risks and improve the overall security posture, the following recommendations are provided:

Immediate Actions (Critical Risks)

  • FTP Server Decommissioning or Secure Configuration:

    • Immediately disable anonymous FTP access on 10.0.0.15.

    • Evaluate the necessity of the FTP service. If not essential, decommission it.

    • If FTP is required, upgrade to a modern, supported version of a secure file transfer protocol (e.g., SFTP, FTPS) and ensure it is properly configured with strong authentication and access controls.

    • Restrict access to this server only to necessary internal IP addresses.

  • Operating System Upgrade:

    • Prioritize the upgrade of all Windows 7 machines to a currently supported operating system (e.g., Windows 10 or 11). This is a critical step to ensure ongoing security updates and patches.

    • For systems that cannot be upgraded, implement strict network segmentation and compensatory controls.

Mid-Term Actions (High Risks)

  • Implement MFA for Sensitive Data Systems:

    • Deploy and enforce multi-factor authentication for all systems that handle or store sensitive organizational data. This adds a crucial layer of security against credential compromise.

  • Establish Comprehensive Security Awareness Training:

    • Develop and implement a mandatory security awareness training program for all new employees during their onboarding process.

    • Institute mandatory annual security awareness training for all existing employees to keep them informed about the latest threats and best practices.

Long-Term Security Enhancements

  • Conduct regular vulnerability assessments and penetration testing.

  • Implement a robust patch management program for all software and operating systems.

  • Review and update security policies and procedures regularly.

  • Consider implementing an Intrusion Detection/Prevention System (IDS/IPS) and Security Information and Event Management (SIEM) solution.

Conclusion

The assessment highlights significant areas for improvement in Evergreen Alliance’s cybersecurity posture. By addressing the critical and high-priority recommendations outlined in this report, Evergreen Alliance can significantly reduce its attack surface and enhance its resilience against cyber threats. Continuous monitoring, regular assessments, and ongoing employee education will be vital for maintaining a strong security posture.
