Report Overview

This report details the cybersecurity posture of Midnight Oil Studios, derived from a network scan, organizational security questionnaire responses, and a review of existing risks. The assessment aims to identify vulnerabilities, policy gaps, and provide actionable recommendations to enhance the overall security landscape.

While strong Multi-Factor Authentication (MFA) controls are laudably in place for key access points such as email, computer logins, and sensitive data systems, significant vulnerabilities and policy gaps were identified. These include a critical issue with anonymous FTP access on an internet-facing system, the use of outdated software, and fundamental deficiencies in security policies and employee training. Addressing these identified areas will be crucial for Midnight Oil Studios to mitigate potential threats and improve its resilience against cyberattacks.

4.1. Open Ports and Services

  • Port 21 (FTP)

    • State: Open

    • Service: FTP (ftp)

    • Product: vsftpd

    • Version: 2.3.4

    • Details: A File Transfer Protocol (FTP) service is running on this port. The identified version vsftpd 2.3.4 is significantly outdated, having been released in 2011, and contains multiple known vulnerabilities, including a severe backdoor vulnerability (CVE-2011-2523).

    • Finding: The scan explicitly revealed that Anonymous FTP login is allowed. This critical configuration permits unauthorized users to connect to the FTP server without credentials, potentially granting read and/or write access to server files, leading to data exposure, integrity issues, or even remote code execution if misconfigured further.

Recommendations

The following recommendations are provided to address the identified risks and enhance the cybersecurity posture of Midnight Oil Studios. Prioritization should be given to Critical and High severity risks.

  1. Secure FTP Service or Decommission:

    • Immediately disable anonymous FTP access on the server at .

    • If FTP is essential, reconfigure it to require strong authentication and use SFTP or FTPS for encrypted transfers.

    • If FTP is not critical for business operations, decommission the service entirely to remove this attack surface.

  2. Update/Upgrade vsftpd:

    • Upgrade the vsftpd service to the latest stable, supported version. Ensure regular patching and updates are part of system maintenance.

    • Alternatively, replace vsftpd with a more modern and secure file transfer solution if a full upgrade path is not feasible.

  3. Develop and Implement an Acceptable Use Policy (AUP):

    • Create a comprehensive AUP that clearly defines appropriate use of company IT assets, data handling, internet usage, and security responsibilities.

    • Ensure all employees read, understand, and formally acknowledge the AUP, ideally upon hiring and annually thereafter.

  4. Establish Mandatory New Employee Security Awareness Training:

    • Implement a mandatory security awareness training program for all new hires. This training should cover fundamental cybersecurity principles, phishing recognition, password best practices, and incident reporting procedures.

  5. Implement Annual Security Awareness Training:

    • Establish a recurring, mandatory security awareness training program for all existing employees, conducted at least once per year. This training should be updated regularly to address emerging threats and reinforce secure behaviors.

  6. Upgrade Outdated Operating Systems:

    • Plan and execute a project to upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or 11).

    • Implement a proactive OS lifecycle management policy to prevent similar issues in the future.
