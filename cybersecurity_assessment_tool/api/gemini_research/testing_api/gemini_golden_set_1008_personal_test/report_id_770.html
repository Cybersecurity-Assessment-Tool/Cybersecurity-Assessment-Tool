5. Risk Assessment

This section consolidates all identified risks from existing organizational data, the security control review, and the technical network scan. Risks are categorized by severity.

Consolidated Risk Register
Risk Name Overview Severity
Outdated Windows Policy Computers are running Windows 7, which is End-of-Life and no longer receives security updates, increasing vulnerability to known exploits.
Outdated vsftpd Version The FTP server is running vsftpd 2.3.4, an End-of-Life version with known critical vulnerabilities, including a backdoor (CVE-2011-2523).
Anonymous FTP Access The FTP server allows anonymous logins, potentially exposing sensitive data or enabling unauthorized uploads/modifications by any user with network access.
Lack of MFA for Computer Logins Multi-Factor Authentication (MFA) is not required for computer logins, significantly increasing the risk of credential compromise and unauthorized access due to weak passwords or phishing.
Insufficient Annual Security Awareness Training Employees do not receive mandatory annual security awareness training, leading to potential lapses in security best practices and increased susceptibility to phishing/social engineering attacks.

6. Recommendations

The following recommendations are provided to mitigate the identified risks and improve the overall cybersecurity posture of Quantum Reach. Prioritization should be given to Critical and High severity risks.

6.1. Immediate Actions (Critical/High Risks)

  • Address Outdated FTP Server and Anonymous Access:

    • Immediately disable anonymous FTP access on 10.0.0.15. Review configurations to ensure only authorized users can access the service with strong authentication.

    • Decommission the vsftpd 2.3.4 server if its functionality is not strictly required.

    • If FTP functionality is necessary, upgrade to a currently supported and patched version of a secure FTP server.

    • Consider migrating to a more secure file transfer protocol such as SFTP (SSH File Transfer Protocol) or FTPS (FTP over SSL/TLS) for data in transit encryption and stronger authentication.

  • Implement MFA for Computer Logins:

    • Evaluate and deploy an MFA solution for all computer logins across the organization. This significantly reduces the risk of unauthorized access due to compromised credentials, even if passwords are stolen.

6.2. Mid-Term Actions (Medium Risks)

  • Upgrade Outdated Operating Systems:

    • Develop and execute a comprehensive plan to upgrade all workstations running Windows 7 to a currently supported operating system (e.g., Windows 10 or 11). This ensures continued access to critical security updates and vendor support.

    • Implement a robust patch management system to keep all operating systems and applications up-to-date.

  • Enhance Security Awareness Training Program:

    • Establish a mandatory annual security awareness training program for all employees, supplementing the new hire training. This should cover up-to-date topics such as phishing, social engineering, password hygiene, identifying malicious links, and proper data handling practices.

    • Conduct simulated phishing exercises to gauge employee awareness and reinforce training.

7. Conclusion

This assessment highlights several critical and high-priority areas requiring immediate attention to bolster Quantum Reach’s cybersecurity defenses. By implementing the recommended actions, particularly those addressing the vulnerable FTP service and MFA gaps, the organization can significantly reduce its attack surface and enhance its resilience against cyber threats. Continuous monitoring, regular security assessments, and an evolving security awareness program are also recommended to maintain a strong and adaptive security posture.
