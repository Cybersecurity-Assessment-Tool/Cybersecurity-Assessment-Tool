1. Executive Summary

This report provides a comprehensive cybersecurity assessment for Great Lakes, based on network scans, organizational data, and a review of existing risks. The analysis reveals a high-risk security posture demanding immediate attention.

A critical vulnerability was identified on an internal host (10.0.0.15). The system is running a dangerously outdated version of vsftpd (2.3.4), which contains a known public backdoor (CVE-2011-2523). This vulnerability is exacerbated by a misconfiguration allowing anonymous FTP access, presenting a direct and severe threat of unauthorized system compromise.

Furthermore, the assessment identified significant gaps in fundamental security controls. The lack of Multi-Factor Authentication (MFA) for sensitive data systems, coupled with the absence of an employee acceptable use policy and a formal security awareness training program, indicates a weak security culture. These procedural failings significantly increase the organization’s susceptibility to both external attacks and insider threats.

Immediate remediation of the critical FTP vulnerability is paramount. Following this, we strongly recommend a strategic initiative to implement the missing security controls to build a more resilient and defensible security posture.

4. Technical Scan Results

An Nmap scan was performed on the target host 10.0.0.15. The results indicate a critical vulnerability that requires immediate remediation.

Host: 10.0.0.15

Port State Service Product Version Notes
21/tcp open ftp vsftpd 2.3.4 Anonymous FTP login allowed

Analysis of Findings:

The scan identified an open FTP port running vsftpd version 2.3.4. This specific version is widely known to contain a critical backdoor vulnerability (CVE-2011-2523), which was intentionally added to the source code. An attacker can connect to the FTP port and enter a specific string in the username field to gain a root-level command shell on the server. The additional finding that "Anonymous FTP login allowed" confirms a severe misconfiguration that lowers the barrier for an attacker to discover and exploit this service.

6. Recommendations

The following actions are recommended to mitigate the identified risks. They are prioritized based on the severity and exploitability of the findings.

  1. [Critical] Remediate Vulnerable FTP Server:

    • Immediate Action: Take the FTP service on host 10.0.0.15 offline immediately to prevent exploitation.

    • Short-Term: If the service is required, upgrade vsftpd to the latest stable version and reconfigure it to disable anonymous access. Enforce strong authentication.

    • Long-Term: Evaluate the business need for FTP. If possible, replace it with a more secure file transfer protocol like SFTP (SSH File Transfer Protocol).

  2. [High] Implement MFA for Sensitive Systems:

    • Prioritize and enforce MFA on all systems that store, process, or access sensitive or regulated data. This is a critical compensating control for credential theft.

  3. [High] Develop and Implement Security Policies:

    • Create a formal Employee Acceptable Use Policy (AUP) that clearly defines rules for using company assets, data handling, and internet usage.

    • Require all employees to read and acknowledge the policy upon hire and annually thereafter.

  4. [High] Establish a Security Awareness Program:

    • Implement mandatory security awareness training for all new hires.

    • Conduct annual refresher training for all employees covering topics such as phishing, password security, and social engineering.

  5. [Medium] Address Outdated Windows Policy:

    • Continue with the existing recommendation to upgrade all Windows 7 workstations to a modern, supported operating system such as Windows 10 or 11.
